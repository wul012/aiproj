# v526 运行截图和解释

## 本版目标

v526 把 v524/v525 已验证有效的 newline suppression 从实验脚本推进到 core generator 的可选能力。新增字段是 `GenerationRequest.blocked_token_texts`，默认值为空，因此旧生成行为不变。

本版不把 newline suppression 设为默认策略，也不改变服务端已有请求的输出。只有显式传入 blocked token texts 时，生成时才会屏蔽匹配 token。

## 真实运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_newline_suppression_probe.py `
  e\523\解释\model-capability-required-term-pair-loss-alias-focus-newline-cleanup-metrics `
  --out-dir e\526\解释\model-capability-required-term-pair-loss-alias-generator-blocked-token-profile `
  --force `
  --require-pass
```
