# v570 required-term pair route held-out expanded suite 运行说明

## 本版目标

v569 在三种 held-out prompt surface 上得到 `3/3` pair-full。v570 不继续改训练，而是把 held-out suite 扩大到七种 prompt surface，检查 v562 seed `1535` 的 pair-full 信号是否只是刚好适配了三种常见格式。

本版不新增模型、不新增 corpus mode，不把结果解释为多 seed 稳定。它是对 v569 targeted route held-out evidence 的扩大复核。

## 代码改动

- `scripts/run_model_capability_required_term_pair_route_heldout_replay.py`
  - 新增 `--prompt-spec SPEC_ID FIXED_PROMPT LOSS_PROMPT`。
  - 支持重复传入 prompt spec，覆盖默认三 spec。
- `tests/test_model_capability_required_term_pair_route_heldout_replay.py`
  - 新增 CLI prompt spec 解析测试，确保后续可以复现更大 held-out suite。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_route_heldout_replay.py `
  e\568\解释\model-capability-required-term-pair-first-token-route-decision `
  --out-dir e\570\解释\model-capability-required-term-pair-route-heldout-expanded-suite `
  --prompt-spec colon-spaced "fixed: " "loss: " `
  --prompt-spec equals fixed= loss= `
  --prompt-spec arrow "fixed -> " "loss -> " `
  --prompt-spec colon-tight fixed: loss: `
  --prompt-spec equals-spaced "fixed = " "loss = " `
  --prompt-spec value-label "fixed value: " "loss value: " `
  --prompt-spec item-label "fixed item: " "loss item: " `
  --force `
  --require-pass
```
