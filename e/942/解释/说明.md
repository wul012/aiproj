# v942 randomized holdout publication registry downstream consumer packet check 运行说明

## 本版目标

v942 消费 v941 downstream consumer packet，做 contract check：

```text
v941 downstream consumer packet
  -> v942 downstream consumer packet check
```

它解决的问题是：consumer packet 已经可以给下游读取，但在继续索引之前，需要证明它可以从 v940 receipt review 重新推导出来，防止 packet JSON 被手动篡改成更宽用途、错误 lookup key 或 promotion-ready 状态。

本版明确不做：

- 不重新训练模型。
- 不重新 replay randomized holdout。
- 不批准 production promotion。
- 不改写 v941 packet，只做复核。
