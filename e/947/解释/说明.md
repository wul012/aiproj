# v947 运行截图与解释

## 本版目标

v947 新增 `randomized_holdout_publication_registry_downstream_consumer_ack_bundle`，把 v945 downstream consumer ack 与 v946 ack contract check 打包成一个可审核 bundle。

这版解决的问题是：ack 和 ack check 分开存在时，下游 review 需要分别找两个文件。v947 把两份证据收束为 bundle，并为每份证据记录 `path`、`sha256`、`status`、`decision` 和 `failed_count`。

## 输入与输出

- 输入 1：`e/945/解释/randomized-holdout-publication-registry-downstream-consumer-ack/randomized_holdout_publication_registry_downstream_consumer_ack.json`
- 输入 2：`e/946/解释/randomized-holdout-publication-registry-downstream-consumer-ack-check/randomized_holdout_publication_registry_downstream_consumer_ack_check.json`
- 输出目录：`e/947/解释/randomized-holdout-publication-registry-downstream-consumer-ack-bundle`
- 截图目录：`e/947/图片`

生成命令：

```powershell
python scripts\build_randomized_holdout_publication_registry_downstream_consumer_ack_bundle.py --consumer-ack e\945\解释\randomized-holdout-publication-registry-downstream-consumer-ack --consumer-ack-check e\946\解释\randomized-holdout-publication-registry-downstream-consumer-ack-check --out-dir e\947\解释\randomized-holdout-publication-registry-downstream-consumer-ack-bundle --require-bundle-ready --require-lookup-ready --force
```

## 边界说明

v947 bundle 不是生产发布包。它只打包 downstream consumer ack 与其 contract check，且继续要求 acked use 为 `downstream_governance_lookup_only`，promotion 为 `False`。若 ack check 失败、acked use 漂移、证据文件缺失或源 next step 不匹配，bundle 会失败。

一句话总结：v947 把 downstream consumer ack 和 contract check 收束成带 digest 的审核包。
