# v488 运行截图和解释：required-term balanced training

## 本版目标

v488 承接 v487 的 balanced corpus candidate，真正训练一个 tiny checkpoint，并用 9 个 required-term scaffold prompt 做 continuation probe。目标不是宣称模型能力提升，而是确认“更均衡、更少重复的语料”是否足以让 tiny model 稳定续写 required term。

## 输入和输出

输入：

```text
e/487/解释/model-capability-required-term-balanced-corpus/
```

输出：

```text
e/488/解释/model-capability-required-term-balanced-training/
e/488/解释/model-capability-required-term-balanced-training-cli.txt
e/488/解释/model-capability-required-term-balanced-training-snapshot.md
e/488/图片/01-model-capability-required-term-balanced-training.png
```

## 关键结果

```text
status=pass
decision=required_term_balanced_training_completed_without_uptake
balanced_training_decision=balanced_training_completed_without_uptake
term_count=9
generation_count=9
continuation_hit_count=0
generated_hit_count=9
continuation_hit_rate=0.0
training_status=pass
checkpoint_exists=True
prompt_alignment_ready=False
prompt_leading_line_count=0
model_quality_claim=not_claimed
```

解释：

- 训练成功，checkpoint/tokenizer/metrics/train config 都存在。
- `generated_hit_count=9` 主要来自 prompt 本身含 required term，不代表模型续写学会了 term。
- `continuation_hit_count=0` 说明短 continuation 里没有 required term。
- `prompt_alignment_ready=False` 和 `prompt_leading_line_count=0` 是本版最重要的诊断：v487 语料虽然平衡、去重，但行首是 `cycle=...|pattern=...`，不是 `data:`、`because:` 这样的 scaffold prompt。
- 真实生成预览中出现了类似 `|pate=07|pat` 的片段，说明模型更像在学习 pattern 前缀结构，而不是 prompt-to-term continuation。

## 截图说明

`01-model-capability-required-term-balanced-training.png` 是 Playwright MCP 对 HTML 报告的全页截图，证明报告可以正常渲染，并能看到 status、decision、continuation hits、prompt alignment 和 generation rows。

`model-capability-required-term-balanced-training-snapshot.md` 是同一页面的 accessibility snapshot，用于补充文本证据。

## 边界和下一步

本版结论不是“balanced corpus 没用”，而是更具体：当前 balanced corpus 缺少 prompt-leading scaffold-to-term rows。下一步应该重建语料，让每个 required term 至少有若干行以 `scaffold_prompt` 开头，同时保留 v487 的 term 平衡和低重复率，再重新训练。

一句话总结：v488 证明 v487 语料可以训练出 checkpoint，但也定位了 continuation 不提升的原因是 prompt alignment 失败。
