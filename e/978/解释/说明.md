# v978 randomized holdout publication receipt packet index review

## 本版目标

v978 审核 v977 生成的 lookup-only receipt packet index。它解决的问题是：在进入 publication 前，不直接信任 index 文件本身，而是再次确认 index、source packet、source contract check 和 source evidence 都仍然存在，并且没有把 `downstream_governance_lookup_only` 误升级为 promotion。

本版不做模型训练，不接受候选模型推广，也不扩大模型质量声明。

## 路径长度修正

真实运行时发现完整长链 artifact 文件名会触碰 Windows 路径长度边界。本版保留可追踪的模块名，但把输出文件名收短为 `randomized_holdout_publication_receipt_packet_index_review_v978.*`。这属于维护性修正，不改变 JSON schema，也不改变 CLI 输入方式。

## 截图说明

HTML 页面顶部展示 `status=pass`、`review ready=True`、`downstream=True`、`publish=True`、`evidence=2`、`failed=0`。Review Boundary 区域列出 source index、source packet、source packet check 和下一步 publication。Checks 表显示 23 条检查均为 pass。

## 边界说明

`publish_ready=True` 只表示可进入 lookup-only publication；它不等于模型推广。`promotion_ready=False` 和 `approved_for_promotion=False` 仍然是本版的核心保护。

一句话总结：v978 在 publication 前复核 v977 index 的下游消费边界，并用短 artifact 文件名修掉了长路径维护风险。
