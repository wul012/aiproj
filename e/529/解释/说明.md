# v529 运行截图和解释

## 本版目标

v529 把 v528 证明有价值的 blocked-token 解码从实验报告参数推进成显式 generation profile。用户不需要手动写 `blocked_token_texts=["\n","\r"]`，可以在 API、playground 和 CLI 中使用 profile 名称：

```text
suppress_newline_tokens
```

本版仍然不改变默认解码。默认 profile 仍是 `default`。

## 真实运行命令

使用 v528 中 seed `528` 的 checkpoint，它是多 seed sweep 中默认解码只达到 `2/4`、blocked-token 达到 `4/4` 的 seed。

```powershell
python -B scripts\generate.py `
  --checkpoint e\528\解释\model-capability-required-term-pair-loss-alias-fresh-seed-sweep\fresh-compare-run\fresh-focus-run\seed-runs\seed-528\loss-alias-focus-run\checkpoint.pt `
  --prompt 'omega:' `
  --max-new-tokens 12 `
  --temperature 0.2 `
  --top-k 1 `
  --seed 528 `
  --out e\529\解释\generation-profile-playground\default-omega.txt

python -B scripts\generate.py `
  --checkpoint e\528\解释\model-capability-required-term-pair-loss-alias-fresh-seed-sweep\fresh-compare-run\fresh-focus-run\seed-runs\seed-528\loss-alias-focus-run\checkpoint.pt `
  --prompt 'omega:' `
  --max-new-tokens 12 `
  --temperature 0.2 `
  --top-k 1 `
  --seed 528 `
  --generation-profile suppress_newline_tokens `
  --out e\529\解释\generation-profile-playground\suppress-newline-omega.txt
```

## 关键结果

默认解码：

```text
omega: los
sssssss
```

显式 profile 解码：

```text
omega: losssssssss
```

API 真实响应保存在：

```text
e/529/解释/generation-profile-playground/api-generate-suppress-newline.json
```

关键字段：

```text
generation_profile=suppress_newline_tokens
blocked_token_texts=["\n","\r"]
blocked_token_count=1
```

解释：v529 不是重新证明模型能力，而是把 v528 的稳定 profile 结论变成用户可调用、可审计的接口能力。
