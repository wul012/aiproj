# v794 运行截图和解释：model capability route promotion gate

## 本版目标

v794 是功能版本，接在 v792 portfolio snapshot 和 v793 regression monitor 后面。它新增 route promotion gate：只有 portfolio ready、regression monitor pass、active route 未丢失、boundary 稳定、claim 没有变宽，才允许进入下游 model capability review。

本版明确不做：

- 不重新训练。
- 不重新 replay。
- 不新增 route。
- 不把 tiny pair-probe gate pass 解释为生产模型质量。

## 运行命令和结果

```powershell
python -B scripts\check_model_capability_route_promotion_gate.py --portfolio e\792\解释\model-capability-route-promotion-portfolio --regression-monitor e\793\解释\model-capability-route-promotion-regression-monitor --out-dir e\794\解释\model-capability-route-promotion-gate --require-pass --force
```

输出摘要：

```text
status=pass
decision=model_capability_route_promotion_gate_passed
exit_code=0
failed_count=0
route_promotion_gate_ready=True
gate_decision=allow_downstream_model_capability_review
active_route_count=1
boundary=tiny_required_term_pair_probe_only
model_quality_claim=seed_stable_pair_probe_route_accepted
allowed_next_steps=route_promotion_release_packet,model_capability_route_review
```

## 关键观察

- gate 消费真实 v792 portfolio 和真实 v793 regression monitor。
- `route_promotion_gate_ready=True`，说明当前 active route 可以进入下游能力审阅。
- `allowed_next_steps` 明确为 `route_promotion_release_packet` 和 `model_capability_route_review`。
- `boundary=tiny_required_term_pair_probe_only`，边界没有扩大。
- `model_quality_claim=seed_stable_pair_probe_route_accepted`，仍然是受限 pair-probe claim。
