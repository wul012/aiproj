# v490 运行截图和解释

## 本版目标

v490 的目标是接住 v489 的 prompt-leading corpus：真正训练一个 tiny checkpoint，并把 required-term continuation 结果和 v488 的 balanced-training baseline 做对比。

本版不声称模型能力提升，也不把 prompt/full-text 命中当作有效能力证据。有效观察口径仍然是 `continuation_hit_count`，也就是模型在 prompt 后续写出的新文本是否包含 required term。

## 真实运行

运行命令：

```powershell
python -B scripts\run_model_capability_required_term_prompt_leading_training.py e\489\解释\model-capability-required-term-prompt-leading-corpus --out-dir e\490\解释\model-capability-required-term-prompt-leading-training --max-iters 600 --eval-iters 2 --batch-size 16 --block-size 8 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --max-new-tokens 12 --temperature 0.2 --top-k 1 --generation-seed 490 --device cpu --require-pass --force
```

关键结果：

```text
status=pass
decision=required_term_prompt_leading_training_completed_without_uptake
prompt_leading_training_decision=prompt_leading_training_completed_without_uptake
term_count=9
generation_count=9
continuation_hit_count=0
previous_continuation_hit_count=0
continuation_hit_delta=0
checkpoint_exists=True
prompt_alignment_ready=True
prompt_leading_line_count=360
model_quality_claim=not_claimed
```

解释：v489 已经把 `prompt_alignment_ready` 修到 `True`，但 v490 训练后仍然没有产生 required-term continuation hit。这说明前一版修复了输入形状问题，却不足以让 tiny 模型在短续写中稳定输出目标词。

## 下一步

下一版不应继续只换报告壳，而应针对 v490 暴露的问题做更具体的实验：增加训练预算、减少 prompt-leading pattern 变体，或构造更短的 `prompt -> term` 语料，观察 continuation 是否能从 0 变为稳定命中。

一句话总结：v490 证明 prompt-leading 对齐是必要修复，但还不是足够条件，当前 tiny 模型 required-term 续写能力仍没有提升。
