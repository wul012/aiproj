# v514 运行截图与解释

## 本版主题

v514 接着 v513 的 alias matrix 做真实训练补口。v513 已经证明 `fixed` 的 alias 泛化存在，但 `loss:`、`beta:`、`theta:`、`omega:` 全部没有输出 `loss`。本版只围绕这个缺口建立 `loss-alias objective`，训练一个 tiny checkpoint，再用同一批 loss prompts 回测。

边界也很明确：v514 不是大模型扩容，也不宣称通用语言能力提升。它只验证“把缺失 alias 明确写入小语料后，tiny GPT 能不能恢复 loss continuation”。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_objective.py `
  e\513\解释\model-capability-required-term-pair-continuation-span-alias-matrix `
  --out-dir e\514\解释\model-capability-required-term-pair-loss-alias-objective `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_loss_alias_continuation_full_hit`
- `loss_alias_decision=loss_alias_heldout_full_hit`
- `training_status=pass`
- `checkpoint_exists=True`
- `case_count=4`
- `generation_count=4`
- `generation_hit_case_count=4`
- `source_loss_hit=True`
- `heldout_loss_alias_hit_case_count=3`
- `heldout_loss_alias_full_coverage=True`

case 级别结果：

| case | prompt | source hit count | candidate hit count |
| --- | --- | ---: | ---: |
| source-loss | `loss:` | 0 | 1 |
| heldout-beta-loss | `beta:` | 0 | 1 |
| heldout-omega-loss | `omega:` | 0 | 1 |
| heldout-theta-loss | `theta:` | 0 | 1 |

结论：v514 的 tiny loss-alias 训练在单 seed 上把 v513 暴露的四个 `loss` 缺口全部补上了；但这仍需下一版跨 seed 复验，不能直接提升为稳定能力声明。
