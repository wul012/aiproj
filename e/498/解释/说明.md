# v498 运行截图和解释

## 本版主题

v498 继续 v497 的保守路线。v497 已经确认：简单增加训练步数、embedding 宽度或语料密度，没有让 `fixed/loss` 恢复 full-hit。v498 因此不再训练新 checkpoint，而是复用 v497 中两个 partial checkpoint，扫描不同解码配置，判断缺失项是否只是短生成或采样策略导致。

本版不做模型扩容，不改 corpus，也不声明模型质量提升。它只检验一个更窄的问题：同一 checkpoint 在更长生成或采样下，能不能同时保住 `fixed` 和 `loss`。

真实观察：

- `baseline-repeat` 的所有解码 profile 仍只命中 `loss`，没有命中 `fixed`。
- `longer-iters` 的 deterministic profile 仍只命中 `loss`。
- `longer-iters` 的 sampling profile 能让 `fixed` 出现，但对应 `loss:` prompt 又丢掉 `loss`。
- 因此采样可以改变偏向，但不能同时稳定保住两个目标词。

## 一句话总结

v498 证明 `fixed/loss` 的 full-hit 缺失不是简单解码长度或采样参数能修复的问题，下一步应回到 prompt target separation 和 pair corpus row design，而不是继续盲目加训练容量。
