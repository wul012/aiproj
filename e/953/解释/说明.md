# v953 运行截图与解释

## 本版目标

v953 新增 `randomized_holdout_publication_registry_downstream_consumer_ack_bundle_publication_receipt`，记录下游对 v952 lookup-only publication index review 的接收回执。

这版解决的问题是：v952 已经审核通过 publication index，但下游消费需要一个 receipt，证明消费方只获得 `downstream_governance_lookup_only` 权限，并继续保留 production promotion、model quality expansion、training data claim expansion 三类阻断用途。

## 边界说明

v953 是 receipt，不是 promotion。它只记录下游消费方接收了 lookup-only publication index review，并把三类 blocked uses 保留下来。

如果 requested use 改成 production promotion、publication row 缺失、source publication/check 缺失、index review 未通过、或 no-promotion 字段被打开，receipt 会失败。

一句话总结：v953 把 v952 lookup-only review 变成下游接收回执，同时保留完整阻断用途和 no-promotion 边界。
