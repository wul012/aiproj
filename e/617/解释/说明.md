# v617 required-term pair refresh forced-choice diagnostic 运行说明

## 本版目标

v617 用 v616 新增的 forced-choice diagnostic 跑 v611、v612、v613 的真实 checkpoint。目标是判断采样失败背后，模型内部是否至少在 teacher-forced 下偏好正确候选 term。

本版不训练新 checkpoint，只做真实 checkpoint 诊断。

## 输入证据

```text
v611 contrast-free checkpoint
v612 delimiter-span checkpoint
v613 context-switch checkpoint
```

## 结果

```text
status=pass
decision=refresh_forced_choice_partial_internal_match
expected_best_prompt_count=3
forced_choice_full_match_source_count=0
model_quality_claim=partial_internal_signal_only
```

明细：

```text
v611 fixed= best fixed, loss= best fixed
v612 fixed= best fixed, loss= best fixed
v613 fixed= best fixed, loss= best fixed
```

这说明三个 checkpoint 内部都偏向 fixed branch；`loss=` 在 teacher-forced 候选比较里也输给 fixed。

## 结论

v617 证明问题不是单纯采样随机性：内部打分也没有让 `loss=` 优先选择 loss。下一步应停止本批 objective，并改设计 loss prompt 的内部偏好。
