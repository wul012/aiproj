# v542 运行截图和解释：top-k2 stability

## 本版目标

v541 在不重训的 decode boundary check 中发现 `wider-k2-t020-n12` 可以把 v540 的 seed `535` 从非 pair-full 恢复为 pair-full。v542 把这个发现放回正式 stability runner：保持 v540 的高训练预算和同一组 seed，只把 replay 的 `top_k` 从 `1` 改成 `2`，验证这个 decode spec 是否能在 stability 报告中复现。

本版不新增代码，不改变 corpus 和模型结构。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_colon_immediate_stability.py --out-dir e\542\解释\model-capability-required-term-pair-topk2-stability --seeds 535 1535 2535 --corpus-mode colon_immediate --repeat 420 --bridge-repeat 40 --max-iters 2200 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --max-new-tokens 12 --temperature 0.2 --top-k 2 --force --require-pass

python -B scripts\run_model_capability_required_term_pair_colon_immediate_missed_seed_diagnostic.py e\542\解释\model-capability-required-term-pair-topk2-stability --out-dir e\542\解释\model-capability-required-term-pair-topk2-missed-seed-diagnostic --top-k 8 --force --require-pass
```

## 结论

v542 复验了 v541 的 decode-boundary 收益：`top_k=2` 对 seed `535` 有效，但不能让三 seed 稳定。后续不应把 `top_k=2` 当作最终修复，而应把它作为评估配置的一部分，同时继续针对 first-token preference 改训练数据。

一句话总结：v542 把 v541 的离线发现推进为正式 stability 证据，确认 decode width 有收益但不足以稳定 fixed/loss pair。
