# v578 required-term pair route closeout summary 运行说明

## 本版目标

v578 是 v569-v577 这轮 required-term pair route 的收口版。它把 held-out 正结果、fresh-seed 负结果、变量 comparison 和 route decision 放进同一份 closeout summary，明确下一步不能继续做 first-token rows 或 width scaling 小变体。

本版不训练、不新增 corpus mode，也不把 held-out pass 误写成泛化能力提升。它的边界是：路线证据收口，准备切换到新的 branch-binding objective。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_route_closeout_summary.py `
  --heldout e\570\解释\model-capability-required-term-pair-route-heldout-expanded-suite `
  --fresh-seed v571-loss-balanced e\571\解释\model-capability-required-term-pair-route-fresh-seed-3535 `
  --fresh-seed v573-first-token e\573\解释\model-capability-required-term-pair-route-fresh-seed-3535-first-token-repair `
  --fresh-seed v575-wider-embd e\575\解释\model-capability-required-term-pair-route-fresh-seed-3535-wider-embd `
  --comparison e\576\解释\model-capability-required-term-pair-route-fresh-seed-3535-variable-comparison `
  --decision e\577\解释\model-capability-required-term-pair-fresh-seed-route-decision `
  --out-dir e\578\解释\model-capability-required-term-pair-route-closeout-summary `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
decision=close_required_term_pair_route_before_new_objective
failed_count=0
heldout_all_pair_full=True
fresh_seed_pair_full_count=0
route_decision=stop_first_token_and_width_for_fresh_seed

```

`bounded_transfer_not_generalized` 是本版最重要的边界：同一路线能通过 v570 的 held-out prompt surfaces，但没有在 seed `3535` 上形成稳定 pair-full。
