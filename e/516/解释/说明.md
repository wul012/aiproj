# v516 运行截图与解释

## 本版主题

v516 读取 v515 的 seed stability 报告，自动找出 seed `515` 没命中的 `loss` alias rows，并构建 focused corpus 进行补训。v515 的漏项是：

- `source-loss`: `loss:`
- `heldout-beta-loss`: `beta:`

本版目标不是继续加治理壳，而是验证“对 missed rows 加密训练”能否把 v515 的 partial signal 推到 full coverage。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_focus.py `
  e\515\解释\model-capability-required-term-pair-loss-alias-stability `
  --out-dir e\516\解释\model-capability-required-term-pair-loss-alias-focus `
  --seeds 515 `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_loss_alias_focus_no_repair`
- `loss_alias_focus_decision=loss_alias_focus_no_repair`
- `seed_count=1`
- `pass_count=1`
- `support_case_count=4`
- `focus_case_count=2`
- `focus_full_seed_count=0`
- `support_full_seed_count=0`
- `stable_focus_full_coverage=False`
- `stable_support_full_coverage=False`

focus case：

| case | prompt | missed seeds |
| --- | --- | --- |
| source-loss | `loss:` | 515 |
| heldout-beta-loss | `beta:` | 515 |

generation preview 显示模型输出接近 `loss`，但把字符拆成了换行形式，例如 `los\ns\ns\ns\ns`，因此严格 substring 命中仍为 false。这说明单纯提高 missed row 密度并没有解决输出形状问题，下一步更应该检查 normalization/path，而不是继续盲目加 repeat。
