# v557 required-term pair refresh forced-choice 运行说明

## 本版目标

v556 的 tied repair 仍然失败，且生成表现是 `fixed=` 继续偏向 loss-like continuation。v557 不再继续改语料，而是对 v556 checkpoint 做 teacher-forced forced-choice：在同一个 prompt 下分别计算候选 `fixed` 和 `loss` 的平均 NLL，判断模型内部评分是否已经偏向错误分支。

本版不训练、不改 checkpoint、不改 replay decode；它只读取 v556 的 refresh report、checkpoint 和 tokenizer 做评分诊断。

## 关键改动

- `src/minigpt/model_capability_required_term_pair_refresh_forced_choice.py`
  - 新增 refresh-level forced-choice builder。
  - 从 refresh report 抽取 checkpoint、tokenizer、prompt terms 和 candidate terms。
  - teacher-forced 计算候选 continuation 的 `total_nll`、`avg_nll` 和 first-token rank。
- `src/minigpt/model_capability_required_term_pair_refresh_forced_choice_artifacts.py`
  - 输出 JSON、CSV、text、Markdown 和 HTML。
- `scripts/run_model_capability_required_term_pair_refresh_forced_choice.py`
  - 提供 CLI 入口，支持 refresh JSON 或目录输入。
- `tests/test_model_capability_required_term_pair_refresh_forced_choice.py`
  - 覆盖 preference collapse、internal match、无效输入、locator 和 artifacts。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_refresh_forced_choice.py `
  e\556\解释\model-capability-required-term-pair-equals-surface-tied-repair\seed-reports\seed-1535 `
  --out-dir e\557\解释\model-capability-required-term-pair-refresh-forced-choice `
  --device cpu `
  --require-pass `
  --force
```

## 核心结果

```text
status=pass
decision=required_term_pair_refresh_forced_choice_preference_collapse
prompt_count=2
score_row_count=4
expected_best_count=1
forced_choice_full_match=False

```

prompt 级结果：

```text
fixed= -> best_candidate_term=loss, expected_rank=2, fixed_avg_nll=0.598999, loss_avg_nll=0.588642
loss=  -> best_candidate_term=loss, expected_rank=1, loss_avg_nll=0.603085
```

这说明 `fixed=` 的失败不是单纯采样阶段跑偏；在 teacher-forced 评分里，模型已经更偏向 `loss`。
