# v554 required-term pair equals-surface balanced repair 运行说明

## 本版目标

v552 的 fixed-biased equals repair 让 `fixed=` 在 suppression profile 下命中，但 `loss=` 没命中。v554 因此新增 `equals_surface_balanced_repair`，让 fixed/loss 在 equals surface 上对称出现，验证是否能同时保住两个分支。

本版继续只跑 seed `1535`，不扩大 seed sweep；目标是快速验证一个明确假设。

## 关键改动

- `src/minigpt/model_capability_required_term_pair_coexistence_corpus.py`
  - 新增 `equals_surface_balanced_repair` corpus mode。
  - `source_prompts()` 将该 mode 映射到 `fixed=` / `loss=` replay prompts。
- `tests/test_model_capability_required_term_pair_coexistence_refresh.py`
  - 新增逐行精确计数测试，确保 `fixed=fixed` 与 `loss=loss` 在 balanced mode 中对称。

## 核心结果

```text
status=pass
decision=required_term_pair_colon_immediate_not_stable
seed_count=1
pair_full_seed_count=0
pair_full_seed_rate=0.0
stable_pair_full=False
```

case 级观察：

- `loss=` 在 default 和 suppression profile 下都命中 loss。
- `fixed=` 在 default 和 suppression profile 下都没有命中 fixed。
- v552 是 fixed hit / loss miss；v554 变成 loss hit / fixed miss。

这说明 seed `1535` 在 equals surface 上存在分支偏置摆动，简单 fixed-biased 或 balanced corpus 都不足以稳定 pair-full。

## 验证说明

验证包括：

- balanced corpus mode 单测。
- source encoding 预检。
- 真实训练 seed `1535`，保存 checkpoint、tokenizer、metrics 和 replay sidecar。
- Playwright MCP HTML 截图和 snapshot。

一句话总结：v554 证明 balanced equals repair 仍不能恢复 seed `1535` 的 pair-full，问题更像小模型的分支竞争，而不是单纯样本比例不足。
