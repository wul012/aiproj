# v713 运行截图和解释：pair-readiness repair comparison

## 本版目标

v713 对比两次真实 pair-readiness 训练：

- v707：base split corpus。
- v712：loss-retention patched corpus。

目标是判断 loss-retention patch 是否改善模型能力。

## 结论

v713 关闭单边 prefix repair 路线：

- v707 default direct hit count = 1。
- v712 default direct hit count = 0。
- loss-retention patch 没有修复 loss，反而让 fixed 也丢失。
- 不应继续增加同类 `loss=lo/los` 前缀权重。

一句话总结：v713 用真实对比证明 loss-retention patch 回归，应停止单边 prefix weighting，转向更结构化的训练目标或模型容量/解码策略对比。
