# v531 运行截图和解释：generation profile contract check

## 本版目标

v531 在 v529 的 generation profile 表面和 v530 的 `/api/generation-profiles` 端点之后，补一层轻量 contract check。它不重新训练模型，也不改变默认生成行为，只检查这些已经归档的证据是否互相一致：

- v530 endpoint `generation-profiles.json`
- v530 health payload
- v529 suppression-profile API response
- 当前重新生成的 playground HTML
- v529 default/profile CLI 对照输出

## 运行命令

```powershell
python -B scripts\check_generation_profile_contract.py e\530\解释\generation-profiles-endpoint --health e\530\解释\generation-profiles-endpoint\health-with-generation-profiles.json --api-response e\529\解释\generation-profile-playground\api-generate-suppress-newline.json --playground-html e\531\解释\generation-profile-current-playground\playground.html --default-output e\529\解释\generation-profile-playground\default-omega.txt --profile-output e\529\解释\generation-profile-playground\suppress-newline-omega.txt --out-dir e\531\解释\generation-profile-contract-check --force --require-pass
```

补充生成当前 playground artifact：

```powershell
python -B scripts\build_playground.py --run-dir e\528\解释\model-capability-required-term-pair-loss-alias-fresh-seed-sweep\fresh-compare-run\fresh-focus-run\seed-runs\seed-528\loss-alias-focus-run --out e\531\解释\generation-profile-current-playground\playground.html --title "MiniGPT generation profile contract playground"
```

## 边界说明

本版不是新治理链，也不声称模型能力进一步提升。它只是把 v529/v530 的 profile contract 做成可复核入口。实际运行中，直接使用旧 v529 `playground.html` 会失败，因为它不包含 v530 新增的 runtime profile loader；因此本版重新生成当前 playground HTML 作为检查源，这也证明该 check 能识别过期 artifact。

一句话总结：v531 把 generation profile 从“已经能用”推进到“endpoint、health、API、CLI、playground 之间有一份可重复验证的契约”。
