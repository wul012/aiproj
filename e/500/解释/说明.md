# v500 运行截图和解释

## 本版主题

v500 接在 v499 后面。v499 找到 `fixed/loss` pair corpus 里的直接泄露行，例如 `fixed:fixed not loss` 和 `loss:loss not fixed`。v500 因此不再审计旧语料，而是实际构建 contrast-free pair corpus：同一个 scaffold prompt 后只放自己的目标词，不再放另一个 required term，也不再写 pair header 共享上下文。

本版是真实训练检查，但仍然只属于 tiny-model 能力实验，不声明生产级模型质量提升。

## 真实运行命令

```text
python -B scripts/run_model_capability_required_term_pair_contrast_free_training.py e/499/解释/model-capability-required-term-pair-prompt-separation-audit --out-dir e/500/解释/model-capability-required-term-pair-contrast-free-training --force --require-pass
```

输入源：

```text
e/499/解释/model-capability-required-term-pair-prompt-separation-audit/model_capability_required_term_pair_prompt_separation_audit.json
```

## 真实训练配置

本版选择 `01-fixed-loss` 这一组 pair，运行 3 个 contrast-free variants：

```text
contrast-baseline: repeat=240, max_iters=1600, n_embd=64
contrast-longer:   repeat=240, max_iters=2400, n_embd=64
contrast-denser:   repeat=360, max_iters=1600, n_embd=64
```

语料行只保留类似：

```text
fixed:fixed
fixed: fixed
comparison-baseline|fixed:fixed
loss:loss
loss: loss
factual-val-loss|loss:loss
```

不再保留：

```text
fixed:fixed not loss
loss:loss not fixed
pair fixed loss prompt fixed: target fixed
```

真实观察：

- 3 个 checkpoint 都训练成功。
- 3 个 variants 都命中 `fixed`，都漏掉 `loss`。
- 去掉 prompt-target 泄露后，输出偏向从 v497/v498 的 `loss` 变成了 `fixed`，但仍没有稳定做到 `fixed:` 命中 `fixed` 且 `loss:` 命中 `loss`。
- 这说明 v499 的语料修正是必要的，但不是充分条件；下一步更适合做 seed stability 或继续检查 `loss:` 这一路的 generation 形态，而不是直接扩展到三词组。

## 一句话总结

v500 证明 contrast-free corpus 可以去掉 v499 发现的明显语料泄露，但 tiny pair checkpoint 仍只得到 partial signal，下一步应围绕 seed 稳定性和 `loss:` 分支继续诊断。
