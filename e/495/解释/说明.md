# v495 运行截图和解释

## 本版目标

v495 接住 v494 的 pair-level partial-only 结果，不继续扩大到三词，而是先做 pair rebalance：仍然使用 v494 的 6 个 partial pair，改变两词语料组织方式，加入更明确的单词隔离行和 pair 对照行，再训练同样数量的 pair checkpoint。

本版不声称整体语言能力提升，也不把 `probe_hit_count` 下降包装成进步。它只回答一个更窄的问题：在同一 tiny checkpoint 里，是否能从 `0/6` full-hit pair 推进到至少一个 pair 同时保住两个 required term。

## 真实运行

运行命令：

```powershell
python -B scripts\run_model_capability_required_term_pair_rebalance.py e\494\解释\model-capability-required-term-pair-curriculum --out-dir e\495\解释\model-capability-required-term-pair-rebalance --force --require-pass
```

关键结果：

```text

```

解释：v495 出现了一个 full-hit pair，`fixed + loss` 从 v494 的只命中 `fixed` 变成同时命中 `fixed/loss`。这是一个真实的 pair capacity 信号；但总 probe hits 从 `6` 降到 `5`，另外两个 pair 退化为 zero-hit，所以本版只能说明“语料重平衡能打开一个 pair-level 窗口”，不能说明整体能力稳定提升。

## Pair 对比

```text
fixed + loss  -> before fixed, after fixed+loss, delta +1, full-hit
fixed + four  -> before four,  after four,       delta  0
fixed + chain -> before fixed, after chain,      delta  0
loss  + four  -> before loss,  after four,       delta  0
loss  + chain -> before chain, after none,       delta -1
four  + chain -> before chain, after none,       delta -1
```

## 下一步

下一步不宜立刻做三词。更合理的是对 v495 改善的 `fixed/loss` pair 做跨 seed 复测，确认这个 full-hit 是稳定能力信号还是单 seed 偶然；如果稳定，再考虑把该语料组织方式扩展到更多 pair。

一句话总结：v495 把 v494 的两词干扰边界推进了一小步，首次在 pair curriculum 中观察到一个 full-hit pair，但整体命中仍不稳定，需要 seed 复测后才能上升为稳态能力结论。
