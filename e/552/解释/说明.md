# v552 required-term pair equals-surface fixed repair 运行说明

## 本版目标

v551 已经把 v550 的唯一 held-out gap 定位为 `fixed=` surface 下的 fixed-term miss。v552 做一个定向 repair candidate：在已有 pair-coexistence refresh 里新增 `equals_surface_fixed_repair` corpus mode，并让该 mode 的 replay prompt 使用 `fixed=` / `loss=`。

本版不声称修复成功。它的价值是用真实训练排除“只要加入 fixed= 样本就能修好 seed 1535”的简单假设。

## 关键改动

- `PAIR_COEXISTENCE_CORPUS_MODES` 新增 `equals_surface_fixed_repair`。
- `build_pair_coexistence_refresh_corpus()` 新增 equals surface 样本。
- `_source_report()` 对该 mode 使用 `fixed=` / `loss=` 作为 replay scaffold prompt，默认旧模式仍使用 `fixed:` / `loss:`。

## 核心结果

```text
status=pass
decision=required_term_pair_colon_immediate_not_stable
seed_count=1
pair_full_seed_count=0
pair_full_seed_rate=0.0
stable_pair_full=False
```

case 级观察：

- `fixed=` 在 newline-suppression profile 下可以命中 fixed。
- `loss=` 在两个 profile 下都没有命中 loss。
- 因此本版没有实现 fixed/loss pair-full，只是把 failure 从 fixed miss 转移成 loss miss。
