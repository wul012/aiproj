# v1013 运行截图与解释

## 本版目标

v1013 把 v1011 的 lookup-only receipt 和 v1012 的 receipt contract check 收成一个新的 receipt index。它的作用是让后续 review 可以只读检索这组 receipt/check 证据，同时保留 SHA-256 摘要、源文件路径、contract check 状态和 no-promotion 边界。

本版不训练模型，不更新 benchmark，不提升模型质量声明，也不把 lookup-only 证据解释成 production promotion。

## 输入与输出

输入：

```text
e/1011/解释/receipt-v1011/...receipt_v1011.json
e/1012/解释/receipt-check-v1012/...receipt_check_v1012.json
```

输出：

```text
e/1013/解释/receipt-index-v1013/...v1013.json
e/1013/解释/receipt-index-v1013/...v1013.csv
e/1013/解释/receipt-index-v1013/...v1013.txt
e/1013/解释/receipt-index-v1013/...v1013.md
e/1013/解释/receipt-index-v1013/...v1013.html
e/1013/图片/v1013-receipt-index.png
```

JSON 是后续模块消费的机器证据；CSV 展开 receipt index row；HTML 和截图用于人工复核。

## 截图说明

`e/1013/图片/v1013-receipt-index.png` 来自 Playwright MCP 对 v1013 HTML report 的浏览器截图。截图用于确认报告页面可读，能够看到 `Status=pass`、`Index ready=True`、`Lookup keys=1`、`Contract=True` 和 `Failed=0` 等关键字段。

## 一句话总结

v1013 把 v1011/v1012 的 receipt/check 从“两个独立证据”收束成“一个可检索、可复核、仍禁止 promotion 的 receipt index”。
