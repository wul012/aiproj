# v744 运行截图和解释：pair prompt transfer training run

## 本版目标

v744 基于 v743 的 7680 行 pair prompt transfer corpus 进行真实 tiny 训练，并用 heldout direct probes 复核模型是否能同时输出 `fixed` 和 `loss`。

本版不新增 contract，也不改变数据；它只检验 v742-v743 的 surrogate pair-transfer rows 是否能让 v738 的 direct-completion 信号迁移到更接近 pair prompt 的训练目标。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_readiness_training_run.py e\743\解释\model-capability-required-term-pair-readiness-pair-prompt-transfer-corpus-materialization --out-dir e\744\解释\model-capability-required-term-pair-readiness-pair-prompt-transfer-training-run --seed 3535 --max-iters 1800 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 2 --n-head 2 --n-embd 96 --learning-rate 0.01 --max-new-tokens 12 --temperature 0.2 --top-k 1 --device cpu --require-pass --force
```

输出摘要：

```text

checkpoint_exists=True

```

## 关键观察

- 训练本身成功，checkpoint/tokenizer/metrics/train_config 都已生成。
- default replay 只命中 `loss`，漏掉 `fixed`。
- newline suppression replay 仍只命中 `loss`，说明不是换行 token 的简单解码问题。
- v744 相比 v738 出现回退：v738 direct hits 为 2，v744 direct hits 为 1。

一句话总结：v744 证明 pair-transfer surrogate rows 没有带来 pair-full 能力，反而把 direct-completion 信号从双命中压成了 loss-only。
