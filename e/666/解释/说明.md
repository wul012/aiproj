# v666 运行截图与解释

## 本版目标

v666 关闭 v659-v666 的 real checkpoint continuation 分支。它把 v664 矩阵和 v665 route decision 合成 closeout，明确 naive continuation 暂停，下一步转向 constrained decode 或更明确的 dual-objective boundary。

本版不训练新模型，也不把 continuation 失败包装成成功；它是分支收口。

## 关键代码优化

本版顺手泛化了 `generation_internal_batch_closeout`：

- `decision` 现在跟随自定义 `next_route`，不再只写死旧的 `joint_cycle_internal_repair`。
- `closeout_items` 不再包含历史批次专属的 balanced-anchor 文案。
- 当 source label 包含 `resume` 时，报告会增加 `resume_routes_rejected` 项。

## 关键结果

- `status=pass`
- `decision=close_batch_and_design_constrained_decode_or_explicit_dual_objective_boundary`
- `batch_version_count=8`
- `aligned_pair_full_count=0`
- `selected_generation_route=loss-internal-joint-cycle`
- `internal_anchor_route=joint-cycle-internal-repair`
- `next_action=design constrained_decode_or_explicit_dual_objective_boundary`

## 解释

v659 让 refresh wrapper 支持真实 checkpoint resume；v660/v662 运行了两条 continuation；v661/v663 用 forced-choice 证明它们没有恢复内部 pair-full；v664/v665 再把它们放回路线矩阵与路线决策。

v666 的结论是：resume branch 可以暂时停止。继续调小学习率、调 bridge-repeat 或换 light-merge 比例，不如换目标形态。
