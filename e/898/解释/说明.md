# v898 运行截图与解释

## 本版目标

v898 接在 v897 之后，构建一个 tokenizer-coverage-aware holdout suite。v897 已经证明 v803 中文 holdout prompt 与 v890 tokenizer 不对齐，因此 v898 不继续训练，也不直接 replay，而是先生成一套 v890 tokenizer 能完整表达的 ASCII holdout prompts。

本版保持 `fixed/loss` scoring contract 不变，保留 5 条 case 数量，只替换 prompt surface，使下一版可以做更公平的 dry-run/replay。

## 候选 prompts

- `answer with exactly two words: fixed loss\nanswer:`
- `return the two target words fixed loss\noutput:`
- `contrast route result fixed loss\nresult:`
- `jsonish answer_terms fixed loss\nanswer_terms:`
- `self check requires fixed and loss\nfinal answer:`

这些 prompts 只使用 v890 tokenizer 已覆盖的字符：小写英文字母、空格、冒号、下划线和换行。它们仍要求 generated continuation 命中 `fixed` 与 `loss`。

## 解释

v898 是评估口径修复，不是模型能力提升。它把 v897 发现的 tokenizer coverage gap 转换成一个可执行 suite 候选：

- 旧 suite 的中文 prompt surface 不再直接作为 v890 checkpoint 的公平 holdout。
- 新 suite 不引入中文字符，因此不会在 prompt 编码阶段制造 `<unk>`。
- 评分仍使用 `fixed/loss`，没有降低目标。
- 生成的 suite 还不能证明模型好，只能作为 v899 dry-run/replay 的输入。
