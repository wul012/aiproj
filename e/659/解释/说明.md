# v659 运行截图与解释

## 本版目标

v659 把 required-term pair refresh 入口接入真实 checkpoint continuation。`scripts/train.py` 原本已经支持 `--resume`，本版补齐的是模型能力实验 wrapper 的 `--resume-checkpoint` 参数、报告字段和测试。

## 关键结果

- refresh CLI 新增 `--resume-checkpoint`。
- refresh report 新增：
  - `settings.training_mode`
  - `settings.resume_checkpoint`
  - `settings.resume_checkpoint_exists`
  - `summary.training_mode`
  - `summary.resume_checkpoint_exists`
- contract：
  - `status=pass`
  - `decision=required_term_pair_refresh_resume_contract_ready`
  - `training_mode=checkpoint_continuation`

## 截图说明

- `e/659/图片/v659-refresh-resume-contract.png`：resume contract HTML 截图，显示 resume checkpoint 存在且 wrapper 能把 continuation 语义写进报告。

## 证据角色

本版不是训练结果，而是续训入口能力。它让 v660 可以从 v630 的 generation pair-full checkpoint 继续训练，而不是重新初始化模型。

一句话总结：v659 把“真正续训”接入 required-term pair refresh 链路，为下一版真实 continuation 实验铺路。
