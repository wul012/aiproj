# v525 运行截图和解释

## 本版目标

v525 验证 v524 的 newline-suppressed decoding recovery 是否只是 v523 checkpoint 的偶然结果。它不重新训练，而是把同一套 newline suppression probe 应用到两个已归档 focus reports：

- v518：首次把 strict/normalized metrics 接入 focus report 的 checkpoint。
- v523：把 newline cleanup metrics 接入主线 focus report 后重新生成的 checkpoint。

## 真实运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_newline_suppression_repeat.py `
  e\518\解释\model-capability-required-term-pair-loss-alias-focus-metrics `
  e\523\解释\model-capability-required-term-pair-loss-alias-focus-newline-cleanup-metrics `
  --out-dir e\525\解释\model-capability-required-term-pair-loss-alias-newline-suppression-repeat `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
```
