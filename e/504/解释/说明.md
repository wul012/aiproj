# v504 运行截图与解释

## 本版主题

v504 增加 `model_capability_required_term_pair_generation_gap`。它读取 v503 forced-choice diagnostic，并追溯 v503 中记录的 v502 branch-retention sweep，把 teacher-forced 内部偏好和 archived free-generation continuation 命中逐 prompt 对齐。

本版不训练新 checkpoint，也不重跑生成。它只回答一个问题：v503 看到的内部 `fixed/loss` full-match 信号，是否已经被 v502 的自由生成表达出来。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_generation_gap.py `
  e\503\解释\model-capability-required-term-pair-forced-choice-diagnostic `
  --out-dir e\504\解释\model-capability-required-term-pair-generation-gap `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_generation_gap_observed`
- `generation_gap_decision=generation_gap_internal_signal_not_expressed`
- `prompt_count=6`
- `variant_count=3`
- `forced_choice_full_match_variant_count=1`
- `generation_full_match_variant_count=0`
- `forced_generation_gap_variant_count=1`
- `aligned_hit_prompt_count=2`
- `internal_only_prompt_count=2`
- `generation_only_prompt_count=0`
- `aligned_miss_prompt_count=2`
- `best_gap_variant_id=symmetric-anchor`

`symmetric-anchor` 是本版核心证据：forced-choice 下 `fixed` 和 `loss` 都选对，但 v502 的自由生成里两个 continuation 都没有命中。因此它不是自由生成能力恢复，而是“内部偏好存在，但生成没有表达”的 gap。

## gap 分类含义

- `aligned_hit`：内部 forced-choice 选 expected，生成 continuation 也命中。
- `internal_only`：内部 forced-choice 选 expected，但生成 continuation 没命中。
- `generation_only`：内部 forced-choice 没选 expected，但生成 continuation 命中。
- `aligned_miss`：内部和生成都没有命中 expected。

真实结果里没有 `generation_only`，说明 v502 的自由生成不是“偶然表达了内部不支持的答案”；真正值得追的是 `symmetric-anchor` 的 `internal_only`。
