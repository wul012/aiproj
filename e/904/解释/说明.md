# v904 运行截图与解释

## 本版目标

v904 对 v902 target-hidden tokenizer-covered holdout suite 执行真实 checkpoint replay。

本版使用 v890 checkpoint/tokenizer，在不向 prompt 显式提供 `fixed` 或 `loss` 的情况下，让模型续写 5 个 target-hidden case，并用 v903 已验证过的 scoring contract 判断 continuation 是否同时包含 `fixed` 和 `loss`。

本版仍不直接做 promotion。即使真实 replay 通过，也需要后续 review 解释是否存在训练记忆、suite 过窄、prompt 仍过于暗示等风险。

## Continuation 摘要

5 个 target-hidden prompt 的 continuation 都命中 `fixed/loss`：

- `target-hidden-answer_learned_pair`: ` fixed loss\n\n\n\nfixed los`
- `target-hidden-return_target_pair`: ` fixed loss\n\n\n\n\n\nfixed l`
- `target-hidden-contrast_route_pair`: ` fixed loss\n\n\n\n\n\nfixed l`
- `target-hidden-jsonish_answer_terms`: ` fixed loss\n\n\n\n\n\n\n\n\n\nfix`
- `target-hidden-self_check_pair`: ` fixed loss\n\n\n\n\nfixed lo`

## 结果解释

v904 比 v900 更强，因为 v900 的 prompt 直接泄漏了 `fixed/loss`，而 v904 的 prompt 隐藏了目标词。

但 v904 仍然只是 bounded target-hidden suite 的 replay 证据。它不能单独证明广义语言能力，也不能直接变成 route promotion。下一版应做 review：

- suite 是否仍然过窄。
- target pair 是否来自训练记忆。
- prompt 中的 `learned pair`、`target pair` 是否仍存在任务暗示。
- 5/5 是否足以进入更宽 holdout 或 promotion gate。
