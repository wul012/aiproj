# v664 运行截图与解释

## 本版目标

v664 把两条真实 checkpoint continuation 路线纳入 generation/internal alignment matrix。它解决的问题是：v660 和 v662 已经证明 resume 能跑通，但还需要和旧的 v621、v630、v640 锚点放在同一张表里，判断 continuation 是否真的改变路线选择。

本版不声称模型能力提升，也不把失败路线当作坏输入；`expected_best_terms=[]` 在这里是有效的负样本。

## 输入与输出

输入包括五组 evidence：

- `loss-internal-first-token`: `e/621` + `e/624`
- `loss-internal-joint-cycle`: `e/630` + `e/631`
- `joint-cycle-internal-repair`: `e/640` + `e/641`
- `v630-internal-repair-resume`: `e/660` + `e/661`
- `v630-light-merge-resume`: `e/662` + `e/663`

输出位于：

- `e/664/解释/model-capability-required-term-pair-alignment-comparison-with-resume-routes/`
- `e/664/图片/v664-alignment-comparison-with-resume-routes.png`

## 关键结果

- `status=pass`
- `decision=keep_generation_pair_full_and_repair_internal_preference`
- `failed_count=0`
- `generation_pair_full_count=1`
- `internal_pair_full_count=2`
- `aligned_pair_full_count=0`
- `best_source_labels=loss-internal-first-token,loss-internal-joint-cycle`

## 解释

v664 的核心变化是把“内部完全没恢复”的 forced-choice 结果作为有效负样本纳入比较器，而不是把它误判为输入损坏。这样，light-merge resume 的失败可以被矩阵看见：它既没有生成 pair-full，也没有内部 expected-best match。

矩阵结论没有改变：`loss-internal-joint-cycle` 仍是 generation pair-full 锚点，`joint-cycle-internal-repair` 仍是内部 pair-full 锚点，两个 continuation 变体没有产生 aligned pair-full。

## 截图说明

- `e/664/图片/v664-alignment-comparison-with-resume-routes.png`: Playwright MCP 截取的 HTML 报告，显示 comparison 通过、下一步仍是 preserve generation pair-full while repairing internal preference。

一句话总结：v664 把真实 resume 失败纳入路线矩阵，确认 naive continuation 没有优于既有双锚点路线。
