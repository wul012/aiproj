# v572 required-term pair route fresh seed 3535 missed diagnostic 运行说明

## 本版目标

v571 的 fresh seed `3535` 没有 pair-full，但它有 `continuation_hit_count=1`。v572 继续只读诊断这次失败：检查 `fixed=` / `loss=` prompt 的 first-token 排名，判断是 continuation 训练不足，还是第一步 token 偏好就错了。

本版不重新训练、不改 corpus、不调 decode。它只解释 v571 为什么没有复现。

## 结论

v572 说明 v571 的 fresh seed 失败主要卡在 `loss=` 的首 token 偏好：模型把 `loss=` 带回 fixed-like 路径。下一步如果要继续提升 fresh-seed 稳定性，应修 first-token preference，而不是继续扩大 held-out prompt suite。
