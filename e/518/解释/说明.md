# v518 运行截图与解释

## 本版主题

v518 把 v517 的 strict-vs-normalized 发现正式并入 `loss-alias focus` 评估。v517 已经说明 v516 的 strict miss 背后有 normalized full signal；本版不再把这件事留在单独 audit 里，而是让 focused repair 报告自己同时输出 strict 与 normalized 指标。

边界：normalized hit 仍然不替代 strict hit。v518 的结论是“双指标并列”，不是把 v516 的 strict no-repair 改写成严格成功。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_focus.py `
  e\515\解释\model-capability-required-term-pair-loss-alias-stability `
  --out-dir e\518\解释\model-capability-required-term-pair-loss-alias-focus-metrics `
  --seeds 515 `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_loss_alias_focus_normalized_support_signal`
- `loss_alias_focus_decision=loss_alias_focus_no_repair`
- `loss_alias_focus_metric_decision=loss_alias_focus_strict_miss_normalized_support_full_signal`
- `focus_full_seed_count=0`
- `support_full_seed_count=0`
- `focus_normalized_full_seed_count=1`
- `support_normalized_full_seed_count=1`
- `normalization_gain_count=4`

seed 级别结果：

| seed | strict focus | strict support | normalized focus | normalized support | gains |
| ---: | ---: | ---: | ---: | ---: | ---: |
| 515 | 0/2 | 0/4 | 2/2 | 4/4 | 4 |

结论：strict 仍然失败，但 normalization 显示所有 support loss aliases 都包含目标字符序列。这把下一步从“继续加训练 repeat”改成“后续评估必须并列 strict 与 normalized 指标”。
