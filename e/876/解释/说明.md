# v876 运行截图和解释：bounded objective loss signal bridge target-only memory loss-suffix patch

## 本版目标

v876 接在 v875 后面。v875 已经把 v874 的 partial-hit 结果诊断为 `loss_suffix_uptake_gap`：模型稳定输出 `fixed l`，但不能完整输出 `loss`。v876 的目标是生成下一轮训练 corpus，专门补 `loss` 后缀和完整 `fixed loss` pair。

本版不训练，不 replay，不声明模型能力提升。它只产出 patch JSONL 和 patched corpus。

## 样本设计

v876 有四类样本：

```text
target_pair_memory       -> fixed loss
loss_suffix_memory       -> loss
loss_prefix_bridge       -> fixed l / loss / fixed loss
minimal_label_pair       -> answer: fixed loss 或 completion: fixed loss
```

它比 v872 更窄：v872 先把模型从 label echo 推到 target prefix，v876 则只补 `loss` 后缀和完整 pair。
