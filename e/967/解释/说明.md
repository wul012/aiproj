# v967 运行截图和解释

## 本版目标

v967 把 v965 publication receipt packet 和 v966 contract check 合成 lookup-only receipt packet index。它让下一步 review 不必分别追两个产物，而是读取一个带 source evidence 和 contract-ready 状态的 index。

本版不训练模型，不扩大模型质量声明，不批准 production promotion。

```powershell
python scripts\build_randomized_holdout_publication_registry_downstream_consumer_ack_bundle_publication_receipt_packet_index_publication_receipt_packet_index.py --receipt-packet e\965\解释\publication-receipt-packet --receipt-packet-check e\966\解释\publication-receipt-packet-check --out-dir e\967\解释\publication-receipt-packet-index --require-index-ready --require-lookup-ready --force
```

## 解释

v967 的 index row 记录 lookup key、packet id、packet status、consumer、granted use、source packet/check 路径、source evidence count、contract check ready 和 promotion_ready。它不是 promotion 入口，只是 downstream governance lookup 的索引。

一句话总结：v967 把 v965/v966 两个产物收束成一个可 review 的 lookup-only receipt packet index。
