# v543 运行截图和解释：temperature boundary check

## 本版目标

v542 证明 `top_k=2` 能把 high-budget colon-immediate stability 从 `0/3` 推到 `1/3`。v543 继续沿着解码边界排查，但不重训模型：先给 decode boundary CLI 增加自定义 `--decode-spec`，再在 v542 的真实 checkpoint 上测试温度矩阵，观察 rank-2 token 是否能通过更高 temperature 表达出来。

本版仍然是解码诊断，不把 `2/3` 解释为模型已稳定。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_decode_boundary_check.py e\542\解释\model-capability-required-term-pair-topk2-stability --out-dir e\543\解释\model-capability-required-term-pair-temperature-boundary-check --decode-spec greedy-k1-t020-n12:1:0.2:12 --decode-spec topk2-t040-n12:2:0.4:12 --decode-spec topk2-t080-n12:2:0.8:12 --decode-spec topk2-t120-n12:2:1.2:12 --decode-spec topk4-t080-n12:4:0.8:12 --force --require-pass
```

## 结论

v543 说明温度不是噪声参数，而是 fixed/loss pair 能否表达的重要边界。`top_k=2, temperature=0.8` 能把 `2535` 也恢复出来，但 `1535` 仍完全失败。下一步应把 `topk2-t080-n12` 放回正式 stability runner 复验，再决定是否继续改训练数据。

一句话总结：v543 将解码边界从 `top_k` 扩展到 temperature，并把可恢复 seed 从 `1/3` 推到 `2/3`。
