# v689 运行截图和解释：required-term pair surface variant selector

## 本版目标

v689 消费 v687 variant plan 和 v688 variant replay，在 5 个全部稳定的 contextual surface 变体中选择一个推荐默认项。

本版不重新跑模型，不训练 checkpoint，也不改变 promotion 边界。它的目标是从“所有变体都可用”收敛到“推荐哪个默认展示/后续对比模板”。

## 输入证据

- `e/687/解释/model-capability-required-term-pair-surface-variant-plan/`
- `e/688/解释/model-capability-required-term-pair-surface-variant-replay/`

v688 已证明 5 个变体全部稳定。v689 只做排序和选择。

## 实际运行

执行命令：

```powershell
python -B scripts\run_model_capability_required_term_pair_surface_variant_selector.py e\687\解释\model-capability-required-term-pair-surface-variant-plan e\688\解释\model-capability-required-term-pair-surface-variant-replay --out-dir e\689\解释\model-capability-required-term-pair-surface-variant-selector --require-pass --force
```

关键输出：

- `status=pass`
- `decision=required_term_pair_surface_variant_selected_for_contextual_demo`
- `selected_variant_id=space_context_control`
- `stable_variant_count=5`
- `promotion_allowed=False`
- `model_quality_claim=contextual_surface_variant_selected`

## 选择逻辑

因为 5 个变体全部稳定，所以选择标准不是命中率，而是：

- 是否稳定。
- 是否与 v686 execution profile 一致。
- 模板是否简洁且可读。
- 是否避免额外自然语言提示或标签融合。

`space_context_control` 的模板是：

```text
{other_term}={other_term} {term}=
```

它和 v686 profile 完全一致，既比 worded 版本少额外提示，也比 compact 版本更可读，因此适合作为默认展示/后续对比入口。

## 结论

v689 选择 `space_context_control` 作为推荐 contextual surface variant。它是稳定、可读、与原 execution profile 一致的默认项，但仍不允许 promotion。

一句话总结：v689 把 5 个全稳定 contextual 变体收敛为一个默认展示/对比模板，同时保留 `promotion_allowed=False` 的能力边界。
