# v528 运行截图和解释

## 本版目标

v528 顺着 v527 的结论继续做多 seed fresh sweep。v527 只证明 seed `527` 的 fresh checkpoint 默认解码已经 strict `4/4`，还不能判断这是稳定训练收益，还是单 seed 偶然。v528 同时训练 seeds `527,528,529`，并对每个 seed 比较默认解码与 blocked-token 解码。

本版不把 blocked-token 改成默认策略；它只验证这个 profile 在多 seed 上是否有稳定兜底价值。

## 真实运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_fresh_seed_sweep.py `
  e\515\解释\model-capability-required-term-pair-loss-alias-stability `
  --out-dir e\528\解释\model-capability-required-term-pair-loss-alias-fresh-seed-sweep `
  --seeds 527 528 529 `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
decision=required_term_pair_loss_alias_fresh_seed_sweep_blocked_token_stably_recovers
seed_count=3
baseline_strict_full_seed_count=2
blocked_token_strict_full_seed_count=3
blocked_token_gain_seed_count=1
total_blocked_token_strict_gain_count=2
```

逐 seed 结果：

```text
seed 527: baseline 4/4, blocked-token 4/4, gains 0
seed 528: baseline 2/4, blocked-token 4/4, gains 2
seed 529: baseline 4/4, blocked-token 4/4, gains 0
```

解释：fresh training 本身已经比旧 checkpoint 更强，但并非所有 seed 都默认 strict full。blocked-token profile 在 3 个 fresh seeds 上都达到 `4/4`，并对 seed `528` 提供 2 个 strict gains。因此它更适合作为显式可选 generation profile，而不是默认解码结论。
