# v724 运行截图和解释：capacity-probe training run

## 本版目标

v724 按 v723 plan 使用 v720 fixed-recovery corpus 跑稍大 tiny model。

目标是验证更大容量是否能同时保住 `fixed` 和 `loss`。结果是：训练成功，但仍是 fixed-only。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_readiness_training_run.py e\720\解释\model-capability-required-term-pair-readiness-fixed-recovery-corpus-materialization --out-dir e\724\解释\model-capability-required-term-pair-readiness-capacity-probe-training-run --seed 3535 --max-iters 1800 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 2 --n-head 2 --n-embd 96 --learning-rate 0.01 --max-new-tokens 12 --temperature 0.2 --top-k 1 --device cpu --require-pass --force
```

## 关键观察

- v724 的模型容量比 v721 大，但结果仍是 fixed-only。
- `loss=` 继续生成 fixed。
- 单纯提高到 2 层、2 头、96 维，没有解决当前 corpus 下的 fixed/loss 共存。

一句话总结：v724 证明这个轻量 capacity probe 仍未带来 pair-full，下一步应做 capacity comparison 后再决定是否扩大容量或改训练目标。
