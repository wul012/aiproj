# v523 运行截图和解释

## 本版目标

v523 把 v522 的 bounded decode cleanup 结论收进主线 `loss_alias_focus` 报告。它不重新定义 strict hit，也不把宽泛 normalized hit 当作模型严格成功，而是在同一份报告里同时展示：

- strict hit：原始 continuation 直接包含 `loss`。
- newline cleanup hit：只移除换行符后再匹配 `loss`。
- normalized hit：移除所有非字母数字后再匹配 `loss`。

## 真实运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_focus.py `
  e\515\解释\model-capability-required-term-pair-loss-alias-stability `
  --out-dir e\523\解释\model-capability-required-term-pair-loss-alias-focus-newline-cleanup-metrics `
  --seeds 515 `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
decision=required_term_pair_loss_alias_focus_newline_cleanup_support_signal
loss_alias_focus_decision=loss_alias_focus_no_repair
loss_alias_focus_surface_decision=loss_alias_focus_strict_miss_newline_cleanup_support_full_signal
loss_alias_focus_metric_decision=loss_alias_focus_strict_miss_normalized_support_full_signal
support_full_seed_count=0
support_newline_cleanup_full_seed_count=1
newline_cleanup_gain_count=4
support_normalized_full_seed_count=1
normalization_gain_count=4
```

解释：strict 仍然失败，说明不能宣称原始生成已经稳定输出 `loss`；但只去掉换行符后 4/4 支持样本都命中，说明这批失败主要是 line-broken surface 问题。
