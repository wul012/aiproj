# v481 运行截图和解释：required-term uptake audit

## 本版主题

v481 承接 v480 的结论：required terms 已经存在于 tiny corpus 和 suite，但没有稳定出现在生成结果中。本版继续只读，不重训模型，回放 v478 的 archived eval-suite 生成输出，检查这些词是否进入 `generated` 或 `continuation`。

## 运行输入

```text
source=e/480/解释/model-capability-required-term-coverage
resolved eval suites=e/478/解释/model-capability-token-budget-stability/seeds/*/token-cap-12/ladder/rungs/max-iters-*/run/eval_suite/eval_suite.json
```

CLI：

```text
python -B scripts/audit_model_capability_required_term_uptake.py e/480/解释/model-capability-required-term-coverage --out-dir e/481/解释/model-capability-required-term-uptake --require-pass --force
```


解释：v480 的 106 个 required-term 行，在两个 seed、两个 max-iters rung 中形成 212 个生成观察点。所有词都出现在 expected behavior 中，但没有任何一个词进入 generated/continuation。它进一步确认当前问题不是“词没进数据”，而是 tiny 模型在当前训练预算和解码条件下没有把这些 required terms 学到输出里。
