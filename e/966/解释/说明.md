# v966 运行截图和解释

## 本版目标

v966 给 v965 publication receipt packet 增加 contract check。它读取 v965 packet 里记录的 v964 source review path，重新调用 v965 builder 重建 packet，再比较稳定字段，证明 packet 不是孤立 JSON 拼接。

本版不训练模型，不扩大模型质量声明，不批准 production promotion。v966 的作用是可复核性，而不是模型能力提升。

## 解释

v966 的核心是“从源重新推导”。check 报告不直接相信 v965 packet，而是读取 packet 中的 `receipt_review_path`，重新加载 v964 review，再调用 v965 builder 生成 rebuilt packet。

比较范围包括 status、decision、failed count、summary、packet body、packet rows、source evidence rows 和 source check rows。只要 packet 中的 lookup scope、source path、evidence row、no-promotion 字段被手动改动，contract check 都会变成 fail。

一句话总结：v966 把 v965 packet 从“可读产物”推进到“可重建、可审计、可继续索引”的 contract-checked 产物。
