# v981 randomized holdout publication receipt packet index publication index

## 本版目标

v981 消费 v979 lookup-only publication 和 v980 publication contract check，生成一个短名 publication index。它解决的问题是：v980 证明 publication 可从源 review 重建，但下游仍需要一个稳定 lookup index 来定位 publication、contract check 和源证据。

本版不做模型训练，不改变 checkpoint，不做生产发布，也不打开 promotion。它只是把已经通过 contract check 的 publication 包装成 lookup-only index。

## 边界说明

v981 的 index 只支持 governance lookup。它不把 v979/v980 的治理证据转换为模型质量提升，也不允许生产 promotion。`promotion_ready=False` 是硬边界。

一句话总结：v981 把 v979 publication 与 v980 contract check 合成可查 index，并继续把新链路文件名收敛到短模块命名。
