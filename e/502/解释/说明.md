# v502 运行截图和解释

## 本版目标

v502 接在 v501 后面。v501 证明 `loss` 可以被 clean corpus 拉回，但一旦拉回 `loss`，`fixed` 就掉线。v502 因此不再做单侧 boost，而是训练更平衡的 branch-retention variants，测试能否在没有负对比泄露的前提下同时保住 `fixed` 和 `loss`。

本版不扩展到更多 required terms，也不把 partial 结果解释成模型能力提升。

## 输入来源

```text
e/501/解释/model-capability-required-term-pair-loss-branch-sweep/model_capability_required_term_pair_loss_branch_sweep.json
```

前置结论：

```text
loss_branch_sweep_decision=loss_branch_sweep_tradeoff_only
focus_term_hit_variant_count=3
pair_full_hit_variant_count=0
branch_tradeoff_variant_count=3
```

也就是说，v501 的三个 variants 都命中 `loss`，但都丢掉 `fixed`。

## 运行命令

```powershell
python -B scripts/run_model_capability_required_term_pair_branch_retention_sweep.py e/501/解释/model-capability-required-term-pair-loss-branch-sweep --out-dir e/502/解释/model-capability-required-term-pair-branch-retention-sweep --force --require-pass
```

## 实验设置

本版训练 3 个 balanced clean variants：

| Variant | 策略 | 目的 |
| --- | --- | --- |
| `alternating-balanced` | 每轮交替 `fixed/loss` 顺序 | 去除固定顺序偏置 |
| `symmetric-boost` | 两个分支都双份复制 | 避免只 boost `loss` |
| `symmetric-anchor` | 两个分支都有 terminal anchors | 检查对称 anchor 能否稳定保留两边 |

所有 corpus 仍然只写自己的 prompt 到自己的 term，不写 `fixed:fixed not loss` 或 `loss:loss not fixed`。

## 真实结果

```text
status=pass
decision=required_term_pair_branch_retention_partial
branch_retention_sweep_decision=branch_retention_sweep_tradeoff_remains
target_count=1
variant_count=3
training_run_count=3
training_pass_count=3
probe_count=6
probe_hit_count=2
focus_term_hit_variant_count=0
source_retained_variant_count=2
balanced_retention_variant_count=0
pair_full_hit_variant_count=0
retention_tradeoff_variant_count=2
best_variant_id=symmetric-boost
model_quality_claim=not_claimed
```

结果不是 full-hit 恢复：

| Variant | 命中 | 丢失 | 结论 |
| --- | --- | --- | --- |
| `alternating-balanced` | `fixed` | `loss` | 回到 v500 一侧 |
| `symmetric-boost` | `fixed` | `loss` | 回到 v500 一侧 |
| `symmetric-anchor` | 无 | `fixed/loss` | 过强 anchor 后两边都不稳 |

关键 continuation：

```text
alternating-balanced fixed -> " fixed\nfixed"
alternating-balanced loss  -> " fixed\nfixed"
symmetric-boost fixed      -> " fixed\nfixed"
symmetric-boost loss       -> " fixed\nfixed"
symmetric-anchor fixed     -> "didddf\nf\nfix"
symmetric-anchor loss      -> "ssssssssssss"
```

这说明靠 clean corpus 的顺序、对称复制和对称 anchor 继续微调，仍然没有解决 pair 条件选择问题。

## 一句话总结

v502 证明在 clean pair corpus 下，交替顺序、对称复制和对称 anchor 仍不能让 tiny checkpoint 同时保住 `fixed` 与 `loss`，下一步应停止继续堆 corpus 权重，转向解码和条件选择层面的诊断。
