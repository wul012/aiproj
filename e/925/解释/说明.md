# v925 randomized holdout acceptance publication packet 运行说明

## 本版目标

v925 把 v923 acceptance summary 和 v924 contract check 打包成 publication packet。

它不是新模型能力评估，也不是 production promotion。它的作用是给下游一个稳定入口：

```text
这次 bounded randomized holdout acceptance 能被引用；
引用时必须同时带上 summary 和 contract check；
允许用途仍然只有 bounded_model_capability_governance_only。
```

## 输入和输出

输入：

```text
e/923/解释/randomized-holdout-acceptance-summary
e/924/解释/randomized-holdout-acceptance-summary-check
```

输出：

```text
e/925/解释/randomized-holdout-acceptance-publication-packet
```

核心文件：

```text
randomized_holdout_acceptance_publication_packet.json
randomized_holdout_acceptance_publication_packet.csv
randomized_holdout_acceptance_publication_packet.txt
randomized_holdout_acceptance_publication_packet.md
randomized_holdout_acceptance_publication_packet.html
```

截图：

```text
e/925/图片/v925-randomized-holdout-acceptance-publication-packet.png
```

## 运行命令

```powershell
python scripts\build_randomized_holdout_acceptance_publication_packet.py `
  --acceptance-summary e\923\解释\randomized-holdout-acceptance-summary `
  --contract-check e\924\解释\randomized-holdout-acceptance-summary-check `
  --out-dir e\925\解释\randomized-holdout-acceptance-publication-packet `
  --require-packet-ready `
  --force
```


## 本版检查重点

v925 检查：

- v923 summary 必须 pass。
- v923 summary 必须 ready。
- v924 contract check 必须 pass。
- v924 contract check 必须 ready。
- v924 的 original/rebuilt decision 必须一致。
- bounded acceptance 必须为 true。
- accepted claim 至少 1 个。
- blocked claim 至少 3 个。
- `allowed_use` 必须保持 `bounded_model_capability_governance_only`。
- `promotion_ready` 必须保持 false。
- `approved_for_promotion` 必须保持 false。
- summary 和 contract check 两个证据文件必须存在。

v925 把 bounded randomized holdout acceptance 从“摘要 + 检查”推进成“可交接的 publication packet”，但仍然只允许 bounded governance 使用。
