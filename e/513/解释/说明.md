# v513 运行截图与解释

## 本版主题

v513 把 v512 的 held-out probe 扩展成 alias matrix。它仍然不训练新模型，只复用 v511 的两个 seed checkpoint，把 held-out prompt 从 `alpha/beta` 扩展到 `alpha/gamma/delta` 对 `fixed`，以及 `beta/theta/omega` 对 `loss`。

本版边界很重要：它不是新训练能力，只是更密的泛化检查，用来判断 v512 的 held-out 命中是不是单 alias 偶然。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_continuation_span_heldout.py `
  e\511\解释\model-capability-required-term-pair-continuation-span-stability `
  --out-dir e\513\解释\model-capability-required-term-pair-continuation-span-alias-matrix `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_continuation_span_heldout_signal`
- `generation_count=16`
- `source_hit_case_count=1`
- `heldout_hit_case_count=3`
- `heldout_hit_term_count=1`
- `heldout_full_term_coverage=False`

case 级别结果：

| case | expected | hit count |
| --- | --- | ---: |
| source-fixed | fixed | 2/2 |
| source-loss | loss | 0/2 |
| heldout-alpha-fixed | fixed | 2/2 |
| heldout-gamma-fixed | fixed | 2/2 |
| heldout-delta-fixed | fixed | 2/2 |
| heldout-beta-loss | loss | 0/2 |
| heldout-theta-loss | loss | 0/2 |
| heldout-omega-loss | loss | 0/2 |

结论：`fixed` 的 alias 信号不是单点偶然，`loss` 仍然没有 source 或 held-out 泛化。
