# v559 required-term pair equals-surface tied wider embedding 运行说明

## 本版目标

v559 接在 v556-v558 后面，验证一个很具体的假设：seed `1535` 的 `fixed=`/`loss=` equals-surface 失败，是否只是 tiny 模型嵌入维度太小导致。

本版不新增治理链，也不宣称模型质量提升。它只把 v556 的 tied repair 训练配置复制下来，把 `n_embd` 从 `64` 加到 `96`，然后观察同一 seed 是否能恢复 pair-full。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_colon_immediate_stability.py `
  --out-dir e\559\解释\model-capability-required-term-pair-equals-surface-tied-wider-embd `
  --seeds 1535 `
  --corpus-mode equals_surface_tied_repair `
  --repeat 260 `
  --bridge-repeat 20 `
  --max-iters 1400 `
  --eval-iters 2 `
  --batch-size 16 `
  --block-size 16 `
  --n-layer 1 `
  --n-head 1 `
  --n-embd 96 `
  --learning-rate 0.02 `
  --max-new-tokens 12 `
  --temperature 0.8 `
  --top-k 2 `
  --device cpu `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
decision=required_term_pair_colon_immediate_not_stable
pair_full_seed_count=0
seed_count=1
pair_full_seed_rate=0.0
stable_pair_full=False
```

replay 侧进一步显示：

```text
default_continuation_hit_count=0
suppression_continuation_hit_count=0
suppression_hit_delta=0
```

两个 replay prompt 都没有命中目标项：

```text
fixed= -> fixed=01=fixxed=01
loss=  -> loss=01s fixxed p
```

这比 v556 更弱。v556 至少还能保留 `loss=` 命中，而 v559 的加宽版本两个分支都没有命中。

## 结论

v559 削弱了“只要加宽 embedding 就能修复 equals surface”的判断。下一步更合理的是回到目标函数和语料结构，而不是继续单纯扩大这个 tiny 模型。
