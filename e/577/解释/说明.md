# v577 required-term pair fresh-seed route decision 运行说明

## 本版目标

v576 已经把 v571、v573、v575 三条 fresh-seed `3535` 路线放到同一个 comparison 中。v577 的目标是把这个人工可读结论固化成机器可读 route decision，防止后续继续在 first-token rows 或 width scaling 上重复消耗版本。

本版不训练、不扩大模型，也不宣称模型能力提升。它只把已有真实训练证据转成下一步路线约束。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_fresh_seed_route_decision.py `
  e\576\解释\model-capability-required-term-pair-route-fresh-seed-3535-variable-comparison `
  --out-dir e\577\解释\model-capability-required-term-pair-fresh-seed-route-decision `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
decision=stop_first_token_and_width_for_fresh_seed
failed_count=0
route_count=3
pair_full_route_count=0
best_residual_signal=v571-loss-balanced
model_quality_claim=route_decision_only
```

`best_residual_signal=v571-loss-balanced` 不是能力提升结论，只说明 v571 保留了 partial fixed hit；v573 和 v575 没有修复 loss branch。
