# v940 randomized holdout publication registry downstream receipt review 运行说明

## 本版目标

v940 消费 v939 downstream receipt，补一层 receipt review：

```text
v939 downstream receipt
  -> v940 downstream receipt review
```

它解决的问题是：receipt 已经记录下游消费者和授予用途，但在进入 consumer packet 之前，需要再次确认 receipt 没有被篡改成生产晋级、没有丢失 blocked uses，也没有指向错误的 source guard digest。

本版明确不做：

- 不重新训练模型。
- 不重新 replay randomized holdout。
- 不修改 v939 receipt。
- 不批准 production promotion。


输出：
