# v909 运行截图与解释

## 本版目标

v909 对 v908 semantic target-hidden real replay 做 review。

v908 已经证明真实 v890 checkpoint 在 v906 semantic prompt 上 5/5 命中 `fixed/loss`。v909 不再重复执行模型，而是复核这批 prompt 是否仍存在目标泄漏或已知 task hint，并决定下一步是否进入更宽的 prompt-mutation holdout。

本版明确不做：

- 不重新训练。
- 不重新 replay。
- 不批准 promotion。
- 不扩大到生产模型质量声明。

## 结果解释

v909 的核心结论是：

- v908 的真实 replay 本身通过。
- v906 suite summary 仍显示 `task_hint_case_count=0`。
- review 对 5 个 prompt 逐行检查，没有发现 `fixed/loss` 目标泄漏。
- review 对已知 pair/target hint 逐行检查，没有发现已知 task hint。
- 允许进入更宽的 prompt-mutation holdout。
- 不允许直接 promotion。

## 字段边界

- `approved_for_prompt_mutation_holdout=True`：允许构造下一层更宽、更扰动的 holdout。
- `approved_for_promotion=False`：仍不批准 promotion。
- `model_quality_claim=semantic_target_hidden_holdout_clean_signal_reviewed`：说明当前能力证据只覆盖 clean semantic holdout review。
- `next_step=build_prompt_mutation_target_hidden_holdout_suite`：下一版应扩展 prompt 变化，而不是继续重复 review。
