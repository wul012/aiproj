# v517 运行截图与解释

## 本版主题

v517 对 v516 的 no-repair 结果做 read-only normalization audit。v516 严格命中为 0，但 generation preview 出现 `los\ns` 这类被换行打散的文本。本版不重新训练，只把 continuation 做归一化：大小写折叠，并移除非字母数字字符，再检查是否包含 required term `loss`。

边界：normalized hit 不是替代严格命中，也不是直接提升模型能力等级；它用于解释 v516 的失败是“完全没生成目标词”，还是“生成了目标字符但格式不合格”。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_normalized_audit.py `
  e\516\解释\model-capability-required-term-pair-loss-alias-focus `
  --out-dir e\517\解释\model-capability-required-term-pair-loss-alias-normalized-audit `
  --force `
  --require-pass
```

行级结果：

| case | strict hit | normalized hit | normalized preview |
| --- | --- | --- | --- |
| source-loss | False | True | `losssss` |
| heldout-beta-loss | False | True | `losssss` |
| heldout-omega-loss | False | True | `losssss` |

结论：v516 的 focused corpus 确实让模型输出了 `loss` 的字符序列，但字符被换行/分隔打散，严格 substring 指标不通过。下一步应该把 strict 和 normalized 两套指标并列进能力报告，而不是继续盲目加训练 repeat。
