# v902 运行截图与解释

## 本版目标

v902 解决 v901 暴露的一个关键问题：v900 的 5/5 replay 不能直接说明模型泛化，因为 v898 的 5 个 prompt 全部显式包含 `fixed` 或 `loss`。

本版新增 target-hidden tokenizer-covered holdout suite：

- prompt 仍然被 v890 tokenizer 完整覆盖。
- prompt 不再出现 `fixed` 或 `loss`。
- scoring contract 仍要求生成 continuation 同时包含 `fixed` 和 `loss`。
- 本版只构造 suite，不运行模型，不声明 promotion。

## Prompt 边界

本版生成的 5 个 prompt 都不含目标词：

- `answer with the learned pair\nanswer:`
- `return the target pair\noutput:`
- `contrast route pair\nresult:`
- `jsonish answer_terms target pair\nanswer_terms:`
- `self check requires target pair\nfinal answer:`

它们仍然复用 `fixed/loss` scoring contract。也就是说，下一版如果 real replay 能通过，才更接近“模型从上下文任务理解出目标对”的证据。
