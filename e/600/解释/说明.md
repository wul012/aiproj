# v600 required-term pair fixed-retention balanced seed 3535 运行说明

## 本版目标

v600 使用 v599 的 `equals_surface_no_pair_id_fixed_retention_balanced_repair` 做真实 tiny training。目标是验证 fixed-retention balanced corpus 是否能在 seed `3535` 上恢复 `fixed` 与 `loss` 同时命中。

## 关键命令

```text
python -B scripts\run_model_capability_required_term_pair_coexistence_refresh.py --out-dir e\600\解释\model-capability-required-term-pair-fixed-retention-balanced-seed-3535 --seed 3535 --corpus-mode equals_surface_no_pair_id_fixed_retention_balanced_repair --repeat 260 --bridge-repeat 20 --max-iters 1800 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --max-new-tokens 12 --temperature 0.2 --top-k 1 --device cpu --require-pass --force
```

## 关键结果

```text
status=pass
decision=required_term_pair_coexistence_refresh_no_pair_full
training_status=pass
checkpoint_exists=True
pair_full_observed=False
```

## 观察

balanced fixed-retention 并没有恢复 pair-full：

```text
fixed= -> losssssssss
loss= -> fixed= losss
```

这说明单纯增加 balanced fixed-retention 行仍不足以稳定 `fixed=` 的首 token。

## 结论

v600 是有效负面结果：balanced fixed-retention 仍未解决 first-token drift，下一步应测试更直接的 first-token retention mode。
