# v503 运行截图与解释

## 本版主题

v503 增加 `model_capability_required_term_pair_forced_choice_diagnostic`。它不训练新模型，也不修改 v502 产物，而是读取 v502 branch-retention sweep 的真实 checkpoint，对同一个 `fixed/loss` pair 做 teacher-forced forced-choice 打分。

这个版本要回答的问题是：v502 的自由生成没有恢复 full-hit，到底是模型内部完全没有学到 `loss` 分支，还是学到了部分偏好但在生成/解码时塌缩。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_forced_choice_diagnostic.py `
  e\502\解释\model-capability-required-term-pair-branch-retention-sweep `
  --out-dir e\503\解释\model-capability-required-term-pair-forced-choice-diagnostic `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_forced_choice_internal_match`
- `forced_choice_diagnostic_decision=forced_choice_full_match_observed`
- `run_count=3`
- `score_row_count=12`
- `expected_best_count=4`
- `expected_best_rate=0.6667`
- `forced_choice_full_match_variant_count=1`
- `best_variant_id=symmetric-anchor`

逐 prompt 看，`alternating-balanced` 和 `symmetric-boost` 都在 `loss:` prompt 上仍偏向 `fixed`，但 `symmetric-anchor` 同时满足：

```text
fixed -> fixed
loss  -> loss
```

因此 v503 不能宣称自由生成能力已经恢复，但可以说明至少一个 v502 checkpoint 在内部打分上已经具备 pair-level forced-choice 信号。下一步应对比 forced-choice winner 与自由生成结果，优先排查解码、采样长度和 prompt continuation 形态，而不是继续盲目加权语料。
