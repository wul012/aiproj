# v533 运行截图和解释：required-term pair coexistence refresh

## 本版目标

v533 根据 v532 的结论，停止继续调 generation profile，改做一个真实 tiny 训练 refresh：构造 fixed/loss 直接共存 corpus，训练一个新 checkpoint，再用 v532 的 replay 逻辑检查 `fixed:` 和 `loss:` 是否都能在 continuation 中命中对应 term。

本版不声称模型能力提升，只有在 pair-full 出现后才会把它标为 targeted signal。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_coexistence_refresh.py --out-dir e\533\解释\model-capability-required-term-pair-coexistence-refresh --seed 533 --repeat 260 --bridge-repeat 20 --max-iters 1400 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --force --require-pass
```

## 结论

v533 是一次真实训练推进，但结果是负向边界：直接 fixed/loss 共存 corpus 没有让 tiny checkpoint 同时学会两个 prompt 分支。下一步不应机械增加重复次数，而应检查 first-token preference 或改 corpus 的答案形态，避免继续被 `answer`/`ans` 前缀吸走。

一句话总结：v533 证明当前直接共存 refresh 训练能跑通，但还没有形成 fixed/loss pair-full 能力。
