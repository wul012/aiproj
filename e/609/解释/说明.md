# v609 required-term pair first-token preference diagnostic 运行说明

## 本版目标

v609 读取 v600、v601、v602、v606、v607 的真实 refresh 报告，诊断 `fixed=` / `loss=` 在首字母和对侧 term 起始处的偏好冲突。

本版不训练新 checkpoint，也不宣称模型能力提升；它只把 v608 closeout 的“需要检查 first-token preference”落成可复核报告。

## 输入证据

```text
v600 balanced fixed-retention
v601 first-token fixed-retention
v602 prompt-guard fixed-retention
v606 loss-rebalance
v607 dual-cycle
```

## 结果

```text
status=pass
decision=first_token_preference_tradeoff_confirmed
source_report_count=5
first_token_conflict_confirmed=True
mixed_branch_tradeoff_confirmed=True
other_term_start_count=8
model_quality_claim=diagnostic_only
```

解释：

- 多个 `fixed=` prompt 的 continuation 以 `loss` 起始。
- 多个 `loss=` prompt 的 continuation 以 `fixed` 起始。
- 已有路线在 fixed-only 和 loss-only 之间切换，没有 pair-full。

v609 把 v608 的下一步落实为诊断证据：下一版 objective 应避免继续加同类权重，而要分离 first-token choice 和 repeated term loop。
