# v792 运行截图和解释：model capability route promotion portfolio

## 本版目标

v792 是功能版本，接在 v791 route promotion history ledger 后面。v791 把 v790 的 route promotion manifest 写入历史 ledger；v792 再把这个 ledger 整理成 portfolio snapshot，让后续 regression monitor 或 portfolio review 能消费一个稳定的路线组合视图。

本版明确不做：

- 不重新训练。
- 不重新 replay。
- 不修改 v791 history ledger。
- 不把 tiny pair-probe 组合视图解释为生产模型质量。

## 运行命令和结果

```powershell
python -B scripts\build_model_capability_route_promotion_portfolio.py --history e\791\解释\model-capability-route-promotion-history --out-dir e\792\解释\model-capability-route-promotion-portfolio --require-ready-portfolio --force
```

输出摘要：

```text
status=pass
decision=model_capability_route_promotion_portfolio_ready
failed_count=0
route_promotion_portfolio_ready=True
route_count=1
active_route_count=1
blocked_route_count=0

model_quality_claim=seed_stable_pair_probe_route_accepted
```

## 关键观察

- portfolio 当前只有一个 active route：`objective_level_contrast`。
- `blocked_route_count=0`，说明没有 blocked route 被混入组合视图。
- `boundary=tiny_required_term_pair_probe_only`，延续 v789-v791 的受限模型能力边界。
- route card 保留了 source history、source manifest、acceptance review、seed rollup 的路径。
- checks 覆盖 source history pass、readiness pass、ready route 数、blocked route、boundary mismatch 和 claim bounded。
