# v703 运行截图和解释：minimal prompt batch closeout

## 本版目标

v703 把 v696、v699、v702 三次真实 minimal prompt 训练结果合并成批次 closeout。它不再继续训练，而是判断这条同族 minimal prompt 修复路线是否还值得继续消耗版本。

## 输入证据

v703 读取三份真实训练报告：

```text
v696 fixed-dominant minimal prompt training
v699 loss-dominant loss-first-token repair training
v702 balanced first-token repair training
```

三份报告都满足：

- `status=pass`
- `checkpoint_exists=True`
- `pair_full_observed=False`

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_minimal_prompt_batch_closeout.py --report v696-fixed-dominant=e\696\解释\model-capability-required-term-pair-minimal-prompt-training --report v699-loss-dominant=e\699\解释\model-capability-required-term-pair-minimal-prompt-loss-first-token-training --report v702-balanced=e\702\解释\model-capability-required-term-pair-minimal-prompt-balanced-training --out-dir e\703\解释\model-capability-required-term-pair-minimal-prompt-batch-closeout --require-pass --force
```

输出：

```text
status=pass
decision=minimal_prompt_batch_closed_without_pair_full
report_count=3
pair_full_report_count=0
fixed_only_report_count=1
loss_only_report_count=2
model_quality_claim=not_claimed
```

## 结论

v703 认为 minimal prompt 同族修复路线应暂停：

- v696 能学到 fixed，但 loss 不稳定。
- v699 能学到 loss，但 fixed 被污染。
- v702 balanced repair 仍不能 pair-full。
- 继续增加同类 prompt 行的收益已经很低。

下一步应转向 pair-readiness split 或更强的训练/评估分离，而不是继续盲训同族 minimal prompt。

一句话总结：v703 把三次真实负结果合并成路线收口证据，防止项目继续在同类 minimal prompt 训练上空转。
