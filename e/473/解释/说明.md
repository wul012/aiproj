# v473 运行截图和解释

## 本版目标

v473 是模型能力阶段的起点：不再继续加纯治理层，而是让 baseline-candidate eval loop 记录真实 tiny 训练产生的能力指标变化。

本版不宣称模型已经有生产级语言能力，也不把 scorecard 持平解释为显著提升。它只回答一个更朴素的问题：同一 tiny 配置下，candidate 增加训练步数后，loss、scorecard 和 generation-quality flags 是否发生可复核变化。

## 真实训练配置

- Suite：`standard-zh`
- Case token cap：`4`
- Baseline：`max_iters=1`
- Candidate：`max_iters=4`
- Seed：baseline 和 candidate 都是 `1337`
- 模型：`n_layer=1`、`n_head=1`、`n_embd=8`、CPU
- Gate mode：exploratory，不因为 candidate 未被接受而失败

## 本次结果

- Baseline score：`81.67`
- Candidate score：`81.67`
- Overall score delta：`0.0`
- Baseline best val loss：`5.345132350921631`
- Candidate best val loss：`5.3419880867004395`
- Best val loss delta：`-0.0031`
- Generation flags delta：`0.0`
- Decision：`reject_candidate`

这说明 candidate 的 best validation loss 有非常轻微下降，但 scorecard 和 generation-quality flags 没有改善；因此本版证据只能支持“训练步数增加带来微弱 loss 改善”，不能支持“模型能力明显提升”。
