# v690 运行截图和解释：required-term pair surface baseline contrast

## 本版目标

v690 把 v689 推荐的 contextual variant 与 v681 的 non-leaking single-label baselines 放在同一张对照表里。

本版不训练、不生成、不选择新 variant。它的目标是防止 v688 “5 个 surface 变体全稳定”被误读成 minimal prompt baseline 也稳定。

## 输入证据

- `e/681/解释/model-capability-required-term-pair-surface-policy-replay/`
- `e/689/解释/model-capability-required-term-pair-surface-variant-selector/`

v681 包含 non-leaking baseline 的 replay 结果。v689 包含推荐 contextual variant：`space_context_control`。

## 实际运行

执行命令：

```powershell
python -B scripts\run_model_capability_required_term_pair_surface_baseline_contrast.py e\681\解释\model-capability-required-term-pair-surface-policy-replay e\689\解释\model-capability-required-term-pair-surface-variant-selector --out-dir e\690\解释\model-capability-required-term-pair-surface-baseline-contrast --require-pass --force
```

关键输出：

- `status=pass`
- `decision=required_term_pair_contextual_variant_needed_over_non_leaking_baselines`
- `contextual_stable=True`
- `non_leaking_baseline_count=2`
- `stable_non_leaking_baseline_count=0`
- `contextual_anchor_required=True`
- `model_quality_claim=contextual_anchor_required_for_surface_stability`

## 对照解释

对照表包含三类信息：

- 推荐 contextual variant：`space_context_control`，稳定。
- `single_label_default`：non-leaking baseline，不稳定。
- `single_label_suppress_newline`：non-leaking baseline，不稳定。

这说明当前可用方案依赖上下文锚点。v688 的多 surface 稳定是好消息，但它不能证明模型在无锚点提示下自然输出 fixed/loss pair-full。

## 结论

v690 明确了本分支的能力边界：contextual variant 稳定，但 non-leaking baseline 仍不稳定，因此后续应把该路线收口为 contextual decode aid，除非另起新 objective 专门解决 minimal prompt。

一句话总结：v690 用对照证据确认 contextual anchor 仍然必需，避免把 v688 的多 surface 稳定误读成无锚点模型能力。
