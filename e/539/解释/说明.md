# v539 运行截图和解释：isolated prompt stability

## 本版目标

v538 证明短前缀 boost 没有提升稳定率。v539 尝试更强的数据分离：新增 `colon_immediate_isolated_prompt` corpus mode，把 fixed 和 loss 放入各自独立的 objective block，减少同一局部窗口里的互相干扰。

本版仍保持同样 3 个 seed 和同样训练预算，只比较 corpus 形态，不扩大模型或训练步数。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_colon_immediate_stability.py --out-dir e\539\解释\model-capability-required-term-pair-colon-immediate-isolated-prompt-stability --seeds 535 1535 2535 --corpus-mode colon_immediate_isolated_prompt --repeat 260 --bridge-repeat 20 --max-iters 1400 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --force --require-pass

```

## 结论

v539 的 isolated prompt corpus 没有改善稳定性，反而从 v538 的 `1/3` 降到 `0/3`。这说明把 fixed/loss 拆进块状语境会削弱短 prompt `fixed:` / `loss:` 的直接映射。下一步应回到直接 `fixed:fixed` / `loss:loss` 形式，在训练参数或重复比例上做受控对比。

一句话总结：v539 证明块状隔离不是当前 pair objective 的好方向，后续应回到直接 prompt 映射。
