# v507 运行截图与解释

## 本版主题

v507 增加 `model_capability_required_term_pair_first_token_repair`。它读取 v506 decoding path trace，对第一步没有采到 expected first token 的 probe，强制插入 expected first token，再沿用原 profile 继续生成。

本版不训练模型，也不是推理服务策略。它只是用 constrained generation 验证 v506 的判断：如果第一 token 被修正，`fixed/loss` 是否能稳定恢复。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_first_token_repair.py `
  e\506\解释\model-capability-required-term-pair-decoding-path-trace `
  --out-dir e\507\解释\model-capability-required-term-pair-first-token-repair `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_first_token_repair_improved_partial`
- `first_token_repair_decision=first_token_repair_improved_partial_expression`
- `repair_count=6`
- `source_continuation_hit_count=2`
- `repaired_continuation_hit_count=3`
- `improved_prompt_count=2`
- `repaired_profile_full_hit_count=0`
- `best_variant_id=symmetric-anchor`
- `best_profile_id=top2-24`

强制第一 token 后，`loss` 的表达明显改善，但没有任何 profile 同时命中 `fixed/loss`。这说明问题不止是第一 token 采样，后续 continuation modeling 仍然不稳。

## 一句话总结

v507 证明 first-token repair 有帮助但不足以恢复 pair-level generation，下一步应转向 continuation span 或目标词完整前缀的诊断。
