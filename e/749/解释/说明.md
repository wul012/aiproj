# v749 运行截图和解释：fixed-preserving transfer training run

## 本版目标

v749 使用 v748 物化出的 6400 行 fixed-preserving transfer corpus 训练 tiny 模型，并检查 heldout direct probes 是否同时命中。

本版可以观察模型行为，但仍不直接 promotion；因为下一步还需要独立 pair-probe replay 验证，确认 checkpoint 不是只在训练脚本内的即时采样里表现良好。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_readiness_training_run.py e\748\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-corpus-materialization --out-dir e\749\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-training-run --seed 3535 --max-iters 1800 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 2 --n-head 2 --n-embd 96 --learning-rate 0.01 --max-new-tokens 12 --temperature 0.2 --top-k 1 --device cpu --require-pass --force
```

```text

failed_count=0

checkpoint_exists=True

```

## 关键观察

- v749 恢复了 v738 的 pair-full 观察结果，并优于 v744 的 loss-only 回退。
- 训练配置与 v744 保持一致，差异主要来自 v748 的 fixed-preserving corpus。
- 输出保留 checkpoint、tokenizer、metrics、train config 和 HTML/Markdown/CSV/JSON 报告。
- `model_quality_claim=direct_pair_probe_hit` 只说明训练脚本内观察到 direct pair probe hit；还不能替代下一版独立 replay。

一句话总结：v749 证明 fixed-preserving transfer route 比 broad pair-transfer route 更有希望，但 promotion 前仍需要独立 heldout replay。
