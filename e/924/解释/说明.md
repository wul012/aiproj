# v924 randomized holdout acceptance summary check 运行说明

## 本版目标

v924 给 v923 acceptance summary 增加 contract check。

它读取 v923 summary，找到其中记录的 `source_decision_index`，再从 v922 index 重新构建一份 acceptance summary，并比较关键字段。

本版不重新训练、不重新 replay，也不重新解释模型能力。它只回答一个问题：

```text
v923 summary 是否仍能从 v922 decision index 重建出来？
```

## 输入和输出

输入：

```text
e/923/解释/randomized-holdout-acceptance-summary
```

输出：

```text
e/924/解释/randomized-holdout-acceptance-summary-check
```

核心输出：

```text
randomized_holdout_acceptance_summary_check.json
randomized_holdout_acceptance_summary_check.csv
randomized_holdout_acceptance_summary_check.txt
randomized_holdout_acceptance_summary_check.md
randomized_holdout_acceptance_summary_check.html
```

截图：

```text
e/924/图片/v924-randomized-holdout-acceptance-summary-check.png
```

## 运行命令

```powershell
python scripts\check_randomized_holdout_acceptance_summary.py `
  e\923\解释\randomized-holdout-acceptance-summary `
  --out-dir e\924\解释\randomized-holdout-acceptance-summary-check `
  --require-pass `
  --force
```

## 结果摘要

```text
status=pass

failed_count=0
contract_check_ready=True
original_decision=randomized_holdout_acceptance_summary_ready
rebuilt_decision=randomized_holdout_acceptance_summary_ready
original_accepted_claim_count=1
rebuilt_accepted_claim_count=1
original_blocked_claim_count=3
rebuilt_blocked_claim_count=3
```

## 检查范围

v924 对比：

- `status`
- `decision`
- `failed_count`
- summary 里的 ready、accepted、blocked、case、seed、pass rate、allowed use、next step
- `accepted_claims`
- `blocked_claims`
- `source_rows`

如果 v923 summary 被手动改成生产推广、改掉 blocked claim、或者丢失 source decision index，contract check 会失败。

## 截图验证


## 一句话总结

v924 把 v923 的人读 acceptance summary 变成可重建、可校验的 contract artifact，防止 bounded claim 在摘要层被悄悄扩大。
