# v512 运行截图与解释

## 本版主题

v512 使用 v511 的两个 continuation-span seed checkpoint 做 held-out prompt probe。它不再训练新模型，而是用 source prompt 和 alias prompt 检查 v511 的稳定 prefix gain 是否开始离开原始 `fixed:` / `loss:` scaffold。

本版不宣称完整泛化。它只记录 held-out alias prompt 是否能触发目标词。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_continuation_span_heldout.py `
  e\511\解释\model-capability-required-term-pair-continuation-span-stability `
  --out-dir e\512\解释\model-capability-required-term-pair-continuation-span-heldout `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_continuation_span_heldout_signal`
- `heldout_decision=heldout_prompt_generalization_observed`
- `seed_count=2`
- `generation_count=8`
- `source_hit_case_count=1`
- `heldout_hit_case_count=1`
- `heldout_generalization_observed=True`

按 case 看：

| case | type | expected | hit count |
| --- | --- | --- | ---: |
| source-fixed | source | fixed | 2/2 |
| source-loss | source | loss | 0/2 |
| heldout-alpha-fixed | heldout | fixed | 2/2 |
| heldout-beta-loss | heldout | loss | 0/2 |

所以本版只能说明：`fixed` 在 `alpha:` 这个 alias prompt 上出现了跨 seed held-out 信号；`loss` 仍没有泛化，不能说 fixed/loss pair 已整体恢复。
