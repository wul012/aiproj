# v750 运行截图和解释：fixed-preserving transfer pair-probe replay

## 本版目标

v750 对 v749 checkpoint 做独立 heldout pair-probe replay，确认训练脚本内观察到的 pair-full 是否能在重新加载 checkpoint 后复现。

本版不 promotion。它的目标是把 v749 的正向训练观察放到更严格的 replay 入口里复核。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_readiness_fixed_preserving_transfer_pair_probe_replay.py e\749\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-training-run --out-dir e\750\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-pair-probe-replay --require-pass --force
```

- `e/750/图片/v750-fixed-preserving-transfer-pair-probe-replay.png`

截图显示 exact heldout pair 没有 pair-full，required all 为 False，但 arrow-heldout-pair surface 出现 pair-full。

## 关键观察

- v749 的训练脚本内 pair-full 不能直接升级为 promotion。
- exact prompt `fixed=|loss=` 只命中 1 个 term。
- spaced prompt `fixed= | loss=` 也只命中 1 个 term。
- arrow prompt `fixed -> | loss ->` 命中 2 个 term，是当前唯一 pair-full surface。
- 结论应收敛到 prompt-surface sensitivity，而不是继续扩写训练 corpus。

一句话总结：v750 证明 fixed-preserving checkpoint 有局部 pair surface 能力，但 exact heldout pair 仍未通过，下一版应诊断 surface 敏感性。
