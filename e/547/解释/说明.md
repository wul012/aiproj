# v547 required-term pair seed coverage tradeoff 运行说明

## 本版目标

v547 不再继续盲目增加 corpus 或训练预算，而是读取 v544 与 v546 的真实稳定性报告，检查两个配置在 seed 层面是否互补。

本版要回答的问题是：v544 已恢复 seed `535`、`2535`，v546 恢复 seed `1535`，这是否构成可复核的 union coverage 证据。

## 输入证据

- `e/544/解释/model-capability-required-term-pair-topk2-t080-stability/model_capability_required_term_pair_colon_immediate_stability.json`
- `e/546/解释/model-capability-required-term-pair-loss-calibrated-topk2-t080-stability/model_capability_required_term_pair_colon_immediate_stability.json`

这两个输入都是已经提交过的 stability report；v547 不重新训练，只做只读对账。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_seed_coverage_tradeoff.py `
  e\544\解释\model-capability-required-term-pair-topk2-t080-stability `
  e\546\解释\model-capability-required-term-pair-loss-calibrated-topk2-t080-stability `
  --label v544-topk2-t080 `
  --label v546-loss-calibrated-topk2-t080 `
  --out-dir e\547\解释\model-capability-required-term-pair-seed-coverage-tradeoff `
  --force `
  --require-pass
```

## 核心结果

```text
status=pass
decision=required_term_pair_seed_coverage_tradeoff_complementary_full_union
config_count=2
seed_count=3
union_pair_full_seed_count=3
best_single_config_id=v544-topk2-t080
best_single_pair_full_seed_count=2
tradeoff_detected=True
```

seed 覆盖关系：

| Seed | 覆盖配置 |
| ---: | --- |
| 535 | v544-topk2-t080 |
| 1535 | v546-loss-calibrated-topk2-t080 |
| 2535 | v544-topk2-t080 |

解释：两个配置的 union 覆盖达到 `3/3`，但任何单一配置最高只有 `2/3`。这证明了 seed-level tradeoff，而不是证明单一 tiny GPT 已稳定掌握 fixed/loss pair 能力。
