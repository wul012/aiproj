# v751 运行截图和解释：prompt-surface sensitivity diagnostic

## 本版目标

v751 汇总 v749 training run 和 v750 independent pair-probe replay，判断 fixed-preserving transfer checkpoint 的问题是否来自 prompt surface sensitivity。

本版不训练模型、不修改 corpus、不 promotion；它只把 v750 的 partial replay 结果转成下一步 repair plan 的依据。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_readiness_fixed_preserving_transfer_prompt_surface_sensitivity_diagnostic.py e\749\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-training-run e\750\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-pair-probe-replay --out-dir e\751\解释\model-capability-required-term-pair-readiness-fixed-preserving-transfer-prompt-surface-sensitivity-diagnostic --require-pass --force
```

输出摘要：

```text
status=pass
decision=pair_readiness_fixed_preserving_transfer_prompt_surface_sensitivity_found
surface_sensitivity_observed=True
promotion_blocked=True
required_missed_surface_ids=['exact-heldout-pair']
optional_pair_full_surface_ids=['arrow-heldout-pair']
model_quality_claim=prompt_surface_sensitive_pair_probe
```

## 截图说明

- `e/751/图片/v751-prompt-surface-sensitivity-diagnostic.png`

截图显示 sensitive 和 promotion blocked 均为 True，required missed 是 exact-heldout-pair，optional passed 是 arrow-heldout-pair。

## 关键观察

- v749 的训练内 pair-full 是真实信号，但不能直接 promotion。
- v750 说明 exact prompt `fixed=|loss=` 未通过，arrow prompt `fixed -> | loss ->` 通过。
- v751 将问题定位为 prompt-surface sensitivity，而不是 checkpoint 文件缺失、replay 执行失败或训练报告无效。
- 下一步应做 minimal exact-surface repair plan，目标是补 exact surface，而不是继续扩大 broad transfer rows。

一句话总结：v751 把 v750 的 partial replay 收敛为明确诊断，后续应围绕 exact heldout surface 做最小修复。
