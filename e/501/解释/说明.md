# v501 运行截图和解释

## 本版目标

v501 接在 v500 后面。v500 已经移除了 `fixed/loss` pair corpus 里的 prompt-target 泄露，但三个 contrast-free variants 都只命中 `fixed`，稳定漏掉 `loss`。本版不扩展到三词组，也不把结果包装成模型质量提升，而是专门追问一个问题：

`loss` 分支到底是学不会，还是只是在当前 pair 训练里被 `fixed` 压住了？

## 输入来源

```text
e/500/解释/model-capability-required-term-pair-contrast-free-training/model_capability_required_term_pair_contrast_free_training.json
```

该报告的关键前置结论是：

```text
contrast_free_training_decision=contrast_free_training_partial_only
variant_pair_full_hit_count=0
variant_pair_partial_hit_count=3
best_variant_id=contrast-longer
```

v501 从这个报告中自动选择 `01-fixed-loss`，识别到：

- source hit term: `fixed`
- focus missed term: `loss`
- source variants: `contrast-baseline`、`contrast-longer`、`contrast-denser`

## 运行命令

```powershell
python -B scripts/run_model_capability_required_term_pair_loss_branch_sweep.py e/500/解释/model-capability-required-term-pair-contrast-free-training --out-dir e/501/解释/model-capability-required-term-pair-loss-branch-sweep --force --require-pass
```

## 实验设置

本版训练 3 个新的 clean corpus variants：

| Variant | 目的 | 是否恢复负对比 |
| --- | --- | --- |
| `missed-first-order` | 每轮先写 `loss`，检查顺序偏置 | 否 |
| `missed-boosted` | 给 `loss` 多一份 clean row copy | 否 |
| `missed-anchored` | 在 loss-first 和 boost 之外追加 `loss` anchor | 否 |

所有 corpus 都继续遵守 v500 的边界：同一个 scaffold prompt 后只放自己的目标词，不写 `fixed:fixed not loss` 或 `loss:loss not fixed`。

## 真实结果

```text
status=pass
decision=required_term_pair_loss_branch_tradeoff
loss_branch_sweep_decision=loss_branch_sweep_tradeoff_only
target_count=1
variant_count=3
training_run_count=3
training_pass_count=3
probe_count=6
probe_hit_count=3
focus_term_hit_variant_count=3
pair_full_hit_variant_count=0
branch_tradeoff_variant_count=3
best_variant_id=missed-first-order
model_quality_claim=loss_branch_tradeoff_signal_only
```

三个 variants 都成功把 `loss` 拉回来了，但也都把 v500 原本稳定命中的 `fixed` 丢掉了：

| Variant | 命中 | 丢失 | 结论 |
| --- | --- | --- | --- |
| `missed-first-order` | `loss` | `fixed` | tradeoff |
| `missed-boosted` | `loss` | `fixed` | tradeoff |
| `missed-anchored` | `loss` | `fixed` | tradeoff |

关键 continuation 形态：

```text
missed-first-order fixed -> " losssssssss"
missed-first-order loss  -> " losssssssss"
missed-boosted fixed     -> "lossssssssss"
missed-boosted loss      -> "lossssssssss"
missed-anchored fixed    -> "lossssssssss"
missed-anchored loss     -> "lossssssssss"
```

这说明 `loss` 不是完全无法被 tiny checkpoint 记住；问题更像是 pair 内 prompt-conditioned selection 不稳定，模型会倒向当前语料强调的一侧。

## 一句话总结

v501 证明了 `loss` 分支可以被 clean corpus 拉回，但会牺牲 `fixed`，因此当前 MiniGPT tiny pair 能力的主要瓶颈是两分支条件选择，而不是单个 required term 的孤立记忆能力。
