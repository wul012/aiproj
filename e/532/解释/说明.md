# v532 运行截图和解释：required-term pair generation profile replay

## 本版目标

v532 在 v531 的 generation profile contract 之后，把 `suppress_newline_tokens` profile 回放到早期 v502 fixed/loss pair branch-retention checkpoint 上。目标是判断这个 profile 是否只是修复 loss-alias 的 newline 表面，还是能帮助真正的 fixed/loss pair 共存。

本版不重新训练模型，不修改 generation profile 默认行为，也不把 profile 效果包装成模型能力提升。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_generation_profile_replay.py e\502\解释\model-capability-required-term-pair-branch-retention-sweep\model_capability_required_term_pair_branch_retention_sweep.json --out-dir e\532\解释\model-capability-required-term-pair-generation-profile-replay --variant-limit 3 --force --require-pass
```

## 结果摘要

```text

```

## 截图说明

- `e/532/图片/01-model-capability-required-term-pair-generation-profile-replay.png`：Playwright MCP 打开的 v532 HTML 报告截图，展示 default 与 suppression profile 在 3 个 fixed/loss 变体上的 pair-full 结果都为 0。

## 结论

v532 把 v529-v531 的 profile 能力放回 fixed/loss pair 历史问题里复验，结果说明 newline suppression 对 v502 pair branch-retention checkpoint 没有形成 pair coexistence gain。它仍然适合作为 loss-alias decode-surface cleanup，但下一步不能继续只调解码，应该回到训练目标设计。

一句话总结：v532 把 generation profile 的能力边界从“修复 loss newline 表面”限定到“不能替代 fixed/loss pair 共存训练”。
