# v1016 运行截图和解释

## 本版目标

v1016 给 v1015 的短名 receipt 补 contract check：从 v1015 receipt 里找到 v1014 source review，重新调用 v1015 builder 生成 rebuilt receipt，然后比较稳定字段。

这版不改变模型质量结论，不放开 promotion，也不扩展新的治理链；它只是验证 v1015 receipt 没有被篡改、没有丢 source review、没有把 lookup-only 边界改宽。

## 输入来源

- 输入目录：`e/1015/解释/receipt-v1015`
- 输入 JSON：`randomized_holdout_publication_receipt_index_receipt_v1015.json`
- source review：`e/1014/解释/review-v1014/...review_v1014.json`

## 测试意义

focused 测试覆盖：

- 可重建 receipt 通过；
- `granted_use` 被篡改时失败；
- source review 丢失时失败；
- source review digest 被篡改时失败；
- CLI 在 `--require-pass` 下对失败 check 返回 `1`；
- JSON/CSV/Text/Markdown/HTML 输出和目录定位可用。

全量测试结果：

```text

```

## 一句话总结

v1016 把 v1015 的短名 receipt 变成可重建、可复核的 contract artifact，为后续短名 index 继续推进打下证据基础。
