# v923 randomized holdout acceptance summary 运行说明

## 本版目标

v923 消费 v922 的 `randomized_holdout_decision_index`，把 bounded acceptance 写成更适合人工审阅和下游治理消费的 acceptance summary。

本版不重新训练、不重新 replay，也不重新读取 v918-v921 四层源产物。它信任 v922 index 已完成四层一致性检查，然后把“接受什么”和“不接受什么”拆成两组 claim。

## 输入和输出

输入：

```text
e/922/解释/randomized-holdout-decision-index
```

输出：

```text
e/923/解释/randomized-holdout-acceptance-summary
```

核心文件：

```text

randomized_holdout_acceptance_summary.csv
```

## 运行命令

```powershell
python scripts\build_randomized_holdout_acceptance_summary.py `
  --decision-index e\922\解释\randomized-holdout-decision-index `
  --out-dir e\923\解释\randomized-holdout-acceptance-summary `
  --require-summary-ready `
  --require-bounded-acceptance `
  --force
```

## accepted claims

本版只接受一个 claim：

```text
bounded_randomized_target_hidden_holdout_claim
```

边界：

```text
randomized_target_hidden_20_case_tiny_checkpoint_only
```

允许用途：

```text
bounded_model_capability_governance_only
```

## blocked claims

本版明确阻止三类扩大解释：

- `production_promotion`：因为 `promotion_ready=False`。
- `general_model_quality`：因为证据只覆盖 20-case tiny-checkpoint randomized holdout。
- `larger_corpus_transfer`：因为没有更大语料或外部 benchmark 证据。
