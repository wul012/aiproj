# v874 运行截图和解释：bounded objective loss signal bridge target-only memory replay comparison

## 本版目标

v874 接在 v873 后面。v873 已经把 v872 target-only memory patch 训练成真实 checkpoint。v874 的目标是用不变的 v836 bounded objective contract 回放这个 checkpoint，判断 target-only memory training 是否真正改善模型输出。

本版不训练新模型，不改 contract，不引入 holdout 结论。它只做同口径 replay。


和 v870 的 zero-hit 不同，v874 三条 case 都命中了 `fixed`，并且没有 zero-hit case。短板也很集中：`loss` 没有被完整生成，所以三个 case 都没有 pass。

## 解释

这版是实质信号改善，但不是能力恢复：

- 改善：从 `any_hit_case_count=0` 到 `any_hit_case_count=3`。
- 未恢复：`passed_case_count=0`，`objective_contract_recovered=False`。
- 下一步：不要继续盲目训练，应先诊断为什么输出停在 `fixed l`，再决定是增加 `loss` suffix memory、调整 decode budget，还是做更小的 completion-target patch。
