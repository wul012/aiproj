# v538 运行截图和解释：first-token boost stability

## 本版目标

v537 把 v536 的 missed seeds 定位为 first-token preference gap。v538 将这个诊断转成训练输入改动：新增 `colon_immediate_first_token_boost` corpus mode，加入 `fixed:f / loss:l / fixed:fi / loss:lo` 等短前缀目标，并让 stability runner 支持 `--corpus-mode`。

本版不是治理链扩展，而是一次真实 tiny 训练对比：同样 3 个 seed、同样训练预算，只换 corpus 形态。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_colon_immediate_stability.py --out-dir e\538\解释\model-capability-required-term-pair-colon-immediate-first-token-boost-stability --seeds 535 1535 2535 --corpus-mode colon_immediate_first_token_boost --repeat 260 --bridge-repeat 20 --max-iters 1400 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --force --require-pass

python -B scripts\run_model_capability_required_term_pair_colon_immediate_missed_seed_diagnostic.py e\538\解释\model-capability-required-term-pair-colon-immediate-first-token-boost-stability --out-dir e\538\解释\model-capability-required-term-pair-colon-immediate-first-token-boost-missed-seed-diagnostic --top-k 8 --force --require-pass
```

## 结论

v538 的短前缀增强没有提升稳定率，只是把 pair-full seed 从 `535` 转移到 `2535`。这说明简单增加 `fixed:f / loss:l` 这类短片段不够，下一步应尝试更强的分离式 corpus：减少 fixed/loss 同行混排，并让每个 prompt 的局部上下文更单纯。

一句话总结：v538 将 v537 的 first-token 诊断转成真实训练尝试，但结果证明短前缀 boost 还不足以稳定 pair-full。
