# v555 required-term pair equals-surface repair comparison 运行说明

## 本版目标

v552 的 fixed-biased repair 可以让 `fixed=` 在 suppression profile 下命中，但 `loss=` 失败；v554 的 balanced repair 则让 `loss=` 命中、`fixed=` 失败。v555 不再追加训练，而是把这两个真实结果放在同一张比较表里，判断 seed `1535` 是否存在“跨运行互补、单运行不共存”的分支竞争。

本版不改 checkpoint，不重新训练，也不扩大 seed sweep；它只复核已有 v552/v554 证据。

## 关键改动

- `src/minigpt/model_capability_required_term_pair_equals_surface_repair_comparison.py`
  - 读取多个 colon-immediate stability JSON。
  - 抽取 source report、term case rows 和 seed-level branch rows。
  - 判断 fixed/loss 是否跨 report 都能命中，以及是否没有任何单一 profile pair-full。
- `src/minigpt/model_capability_required_term_pair_equals_surface_repair_comparison_artifacts.py`
  - 输出 JSON、CSV、text、Markdown 和 HTML。
- `scripts/run_model_capability_required_term_pair_equals_surface_repair_comparison.py`
  - 提供命令行入口，支持 `--labels`、`--out-dir`、`--force` 和 `--require-pass`。
- `tests/test_model_capability_required_term_pair_equals_surface_repair_comparison.py`
  - 用小 fixture 覆盖 branch competition、pair-full found、无效输入和多格式输出。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_equals_surface_repair_comparison.py `
  e\552\解释\model-capability-required-term-pair-equals-surface-fixed-repair `
  e\554\解释\model-capability-required-term-pair-equals-surface-balanced-repair `
  --labels v552-fixed-repair v554-balanced-repair `
  --out-dir e\555\解释\model-capability-required-term-pair-equals-surface-repair-comparison `
  --require-pass `
  --force
```
