# v515 运行截图与解释

## 本版主题

v515 对 v514 的 `loss-alias objective` 做跨 seed 复验。v514 单 seed 全命中很有价值，但单次训练不能说明稳定能力。本版使用同一份 v513 alias matrix，分别训练 seed `514` 和 `515`，再汇总每个 seed 的 source/held-out loss alias 命中情况。

边界：v515 仍然只检查 `loss` alias，不把 `fixed` branch 加回来，也不扩大模型规模。它的目标是判断 v514 的恢复是不是跨 seed 稳定。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_stability.py `
  e\513\解释\model-capability-required-term-pair-continuation-span-alias-matrix `
  --out-dir e\515\解释\model-capability-required-term-pair-loss-alias-stability `
  --seeds 514 515 `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_loss_alias_stable_partial_hit`
- `loss_alias_stability_decision=loss_alias_stable_partial_hit`
- `seed_count=2`
- `pass_count=2`
- `checkpoint_seed_count=2`
- `heldout_loss_alias_partial_seed_count=2`
- `heldout_loss_alias_full_seed_count=1`
- `stable_loss_alias_full_coverage=False`

seed 级别结果：

| seed | hit cases | heldout hits | full coverage |
| ---: | ---: | ---: | --- |
| 514 | 4 | 3 | True |
| 515 | 2 | 2 | False |

seed `515` 仍然命中了两个 held-out loss aliases，但没有命中 `source-loss` 和 `heldout-beta-loss`。所以 v515 的判断是：`loss` alias 训练有稳定部分信号，但还不是全覆盖稳定能力。
