# v908 运行截图与解释

## 本版目标

v908 对 v906 semantic target-hidden holdout suite 执行真实 checkpoint replay。

它接在 v907 dry-run 之后，用真实 v890 checkpoint/tokenizer 验证：当 prompt 不再直接暗示 pair/target 任务，模型是否仍能在 continuation 中输出 `fixed` 和 `loss`。

本版明确不做：

- 不重新训练。
- 不改写 v906 suite。
- 不把 replay 通过直接升级为 promotion。
- 不扩大到生产模型质量声明。

## Replay 结果解释

5 个 semantic target-hidden prompt 全部命中 `fixed/loss`：

```text
semantic-hidden-memory_answer      -> fixed/loss hit
semantic-hidden-stored_result      -> fixed/loss hit
semantic-hidden-learned_route      -> fixed/loss hit
semantic-hidden-final_words        -> fixed/loss hit
semantic-hidden-memory_self_check  -> fixed/loss hit
```

这比 v905 的 review 结果更强：v905 确认 v904 的 target-hidden replay 通过，但也发现 prompt 仍带有任务提示；v908 则在 v906 去掉这些提示后仍然 5/5 通过。

## 字段边界

- `status=pass`：表示输入、执行和检查项全部通过。
- `holdout_model_quality_ready=True`：表示本 replay suite 的 5 个 case 全部通过。
- `semantic_holdout_model_quality_ready=True`：显式标记这是 semantic target-hidden holdout 范围内的通过。
- `promotion_ready=False`：表示本版仍不批准路线 promotion。
- `model_quality_claim=semantic_target_hidden_holdout_replay_only`：声明范围只到语义 target-hidden replay。
