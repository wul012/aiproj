# v494 运行截图和解释

## 本版目标

v494 接住 v493 的跨 seed one-term 稳定信号，开始测试两词小组 curriculum：只选择 v493 中跨 3 个 seed 全稳定的 `fixed/loss/four/chain`，两两组合成 6 个 pair，每个 pair 训练一个 checkpoint，再分别 probe 两个 prompt。

本版不做三词或四词训练，也不扩大模型规模。它只回答一个窄问题：单目标稳定词放进同一个训练分布后，是否还能同时保住两个 required term。

## 真实运行

运行命令：

```powershell
python -B scripts\run_model_capability_required_term_pair_curriculum.py e\493\解释\model-capability-required-term-one-term-seed-stability --out-dir e\494\解释\model-capability-required-term-pair-curriculum --force --require-pass
```

解释：v493 已经证明 4 个词在单目标条件下跨 seed 稳定；v494 把它们两两放在一起后，6 个 pair 全部只能命中其中一个词，没有任何 pair 同时命中两个词。这说明当前 tiny 设置下，多目标干扰仍然明显。

## Pair 明细

```text
fixed + loss  -> hit fixed, miss loss
fixed + four  -> hit four,  miss fixed
fixed + chain -> hit fixed, miss chain
loss  + four  -> hit loss,  miss four
loss  + chain -> hit chain, miss loss
four  + chain -> hit chain, miss four
```

## 下一步

下一步不建议直接扩大到三词。更合理的是做 pair rebalance：针对 6 个 partial pair，改变语料顺序、repeat 比例或训练步数，观察是否能出现至少一个 full-hit pair；如果仍然没有，再回头看 prompt 分隔或模型容量。

一句话总结：v494 把 v493 的“单目标稳定能力”推进到“两目标干扰边界”验证，结果显示当前 tiny GPT 还不能在同一 checkpoint 内稳定同时保住两个 required term。
