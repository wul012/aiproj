# v691 运行截图和解释：required-term pair surface route decision

## 本版目标

v691 基于 v690 baseline contrast 和 v689 variant selector 做路线决策：当前 contextual surface policy 分支应该收口为可用的 decode aid，而不是继续扩展 surface 变体。

本版不训练、不生成、不新增 variant。它把下一步方向写成机器可读 route。

## 输入证据

- `e/690/解释/model-capability-required-term-pair-surface-baseline-contrast/`
- `e/689/解释/model-capability-required-term-pair-surface-variant-selector/`

v690 已证明 contextual anchor 仍然必需。v689 已选出默认 contextual variant：`space_context_control`。

## 实际运行

执行命令：

```powershell
python -B scripts\run_model_capability_required_term_pair_surface_route_decision.py e\690\解释\model-capability-required-term-pair-surface-baseline-contrast e\689\解释\model-capability-required-term-pair-surface-variant-selector --out-dir e\691\解释\model-capability-required-term-pair-surface-route-decision --require-pass --force
```

关键输出：

- `status=pass`
- `decision=close_contextual_decode_branch_and_design_minimal_prompt_objective`
- `selected_variant_id=space_context_control`
- `current_route=contextual_decode_aid_closeout`
- `recommended_next_route=minimal_prompt_surface_objective`
- `promotion_allowed=False`
- `model_quality_claim=contextual_decode_aid_not_baseline`

## 路线解释

本版将当前路线分成两层：

- 当前可收口能力：`contextual_decode_aid_closeout`
- 如果继续追求模型能力：`minimal_prompt_surface_objective`

这意味着 `space_context_control` 可以作为 demo/diagnostic 的 contextual decode aid，但不能作为 baseline promotion 或 minimal prompt claim。

## 结论

v691 关闭了“继续堆 contextual surface 变体”的必要性。当前分支可以作为 contextual decode aid 收口；真正要提升模型能力，需要另起 minimal-prompt objective。

一句话总结：v691 把本批次路线定为“contextual decode aid 可用但不 promotion，下一条能力路线应是 minimal prompt objective”。
