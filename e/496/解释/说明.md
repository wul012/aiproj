# v496 运行截图和解释

## 本版目标

v496 接住 v495 的局部 pair-capacity 信号：`fixed/loss` 在 v495 中从 partial 变成 full-hit。为了避免把单 seed 偶然误认为稳定能力，本版只复测这个改善 pair，跨 3 个 seed 重新训练同一 rebalance corpus。

本版不扩大到三词，不调整模型规模，也不声称整体模型质量提升。它只回答一个窄问题：v495 的 `fixed/loss` full-hit 是否能跨 seed 复现。

## 真实运行

运行命令：

```powershell
python -B scripts\run_model_capability_required_term_pair_rebalance_seed_stability.py e\495\解释\model-capability-required-term-pair-rebalance --out-dir e\496\解释\model-capability-required-term-pair-rebalance-seed-stability --force --require-pass
```


解释：3 个 seed 都训练成功，也都命中了 `loss`，但全部漏掉 `fixed`。这说明 v495 的 `fixed/loss` full-hit 没有跨 seed 复现，不能作为稳定 pair-level 能力结论。

## 下一步

下一步不建议继续三词 curriculum。更合理的是回到 pair 能力本身：要么提高训练预算或模型宽度，观察 `fixed/loss` 是否能稳定 full-hit；要么调整语料，让两个 prompt 的输出边界更清晰。

一句话总结：v496 把 v495 的局部成功放到跨 seed 复核下，结果显示该 full-hit 是脆弱信号，不足以支撑扩大 curriculum。
