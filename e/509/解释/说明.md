# v509 运行截图与解释

## 本版主题

v509 增加 `model_capability_required_term_pair_diagnostic_rollup`。它把 v503-v508 六个 required-term pair 诊断串成一个阶段结论，并明确下一步不再继续堆 decoding tweak，而是做最小 continuation-span objective。

本版不训练模型，不改 checkpoint，也不把 forced-prefix 证据包装成真实模型能力提升。它只是一个收口判断：现有证据是否足够支持进入下一类实验。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_diagnostic_rollup.py `
  e `
  --out-dir e\509\解释\model-capability-required-term-pair-diagnostic-rollup `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_next_span_objective`
- `rollup_decision=rollup_continue_with_span_objective`
- `next_plan_id=continuation_span_objective_fixed_loss`
- `stage_count=6`
- `passing_stage_count=6`
- `decoding_profile_full_hit_count=0`
- `span_completion_gap_probe_count=3`

这说明 v503-v508 的结论是一致的：

- v503 证明内部 forced-choice 有完整配对信号。
- v504 证明这个信号没有稳定表达到自由生成。
- v505-v506 证明小 decoding profile 和 first-token 排名不足以恢复 full-hit。
- v507 证明强制 first token 只能局部改善。
- v508 证明 `fixed` 的完整 span completion 明显弱于 `loss`。

因此 v509 的下一步建议是做最小 continuation-span 训练目标，而不是继续增加 decoding profile。
