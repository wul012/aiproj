# v705 运行截图和解释：pair-readiness split contract

## 本版目标

v705 把 v704 的 pair-readiness split plan 具体化为可检查 contract。它明确训练行、评估 probe、heldout pair probe 和泄漏检查。

本版仍不训练模型；它是训练前的输入/评估分离契约。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_readiness_split_contract.py e\704\解释\model-capability-required-term-pair-readiness-split-plan --out-dir e\705\解释\model-capability-required-term-pair-readiness-split-contract --require-pass --force
```

输出：

```text
status=pass
decision=pair_readiness_split_contract_ready
contract_ready=True
training_row_count=12
evaluation_probe_count=3
heldout_pair_probe=fixed=|loss=
model_quality_claim=contract_only
```

## Contract 内容

training rows 包含 direct completion 与 anti-contamination：

```text
fixed=f
fixed=fi
fixed=fix
fixed=fixed
loss=l
loss=lo
loss=los
loss=loss
fixed branch does not start loss
loss branch does not start fixed
```

evaluation probes：

```text
fixed=
loss=
fixed=|loss=
```

其中 `fixed=|loss=` 是 heldout pair probe，不能进入 training rows。

## 结论

v705 把下一轮训练前最重要的边界写清楚：

- 训练行可以包含 direct completion。
- 评估 probe 可以用 prompt prefix。
- heldout pair probe 不能作为训练行出现。
- 只有 heldout fixed/loss 都命中时，才允许 claim pair capability。

一句话总结：v705 把 v704 的计划变成可机器检查的 train/eval contract，为后续 materialize corpus 做准备。
