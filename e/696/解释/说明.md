# v696 运行截图和解释：minimal prompt training

## 本版目标

v696 是 minimal-prompt 路线的第一次真实训练。它使用 v695 注册并验证过的 `minimal_prompt_equals_surface_objective` 语料模式，训练 seed `3535` 的 tiny checkpoint，然后用现有 generation profile replay 检查 `fixed=` 和 `loss=` 两个最小提示词是否都能生成对应 term。

## 运行命令和结果

```powershell
python -B scripts\run_model_capability_required_term_pair_coexistence_refresh.py --out-dir e\696\解释\model-capability-required-term-pair-minimal-prompt-training --seed 3535 --corpus-mode minimal_prompt_equals_surface_objective --repeat 260 --bridge-repeat 20 --max-iters 1400 --eval-iters 2 --batch-size 16 --block-size 16 --n-layer 1 --n-head 1 --n-embd 64 --learning-rate 0.02 --max-new-tokens 12 --temperature 0.2 --top-k 1 --device cpu --require-pass --force
```

结果：

```text
status=pass
decision=required_term_pair_coexistence_refresh_no_pair_full
training_status=pass
checkpoint_exists=True
default_pair_full_variant_count=0
suppression_pair_full_variant_count=0
pair_full_observed=False
model_quality_claim=not_claimed
```

## 关键观察

训练成功，checkpoint、tokenizer、metrics、train config 都存在。但 replay 没有 pair-full：

- `fixed=` 生成 `fixed=fixed=fixed=`，命中 fixed。
- `loss=` 生成 `loss=fixed=fixed=`，没有命中 loss。
- newline suppression 没有改变这个结果。

所以 v696 的结论不是“模型没训练起来”，而是“loss 分支在 minimal prompt 下被 fixed 分支吸走”。

## 链路结论

v696 是有效负证据：minimal prompt 语料可以训练出 checkpoint，但 seed `3535` 在 `loss=` 上仍向 `fixed` 塌缩。下一步不应该宣称提升，而应该做 first-token / branch-bias diagnostic。

一句话总结：v696 把 minimal-prompt 路线从契约推进到真实 tiny 训练，并确认第一轮失败点集中在 loss 分支被 fixed 吸走。
