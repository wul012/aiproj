# v793 运行截图和解释：model capability route promotion regression monitor

## 本版目标

v793 是功能版本，接在 v792 route promotion portfolio snapshot 后面。v792 给出了当前 active route 组合视图；v793 增加 regression monitor，用 baseline/current 两个 portfolio 检查 active route 是否丢失、boundary 是否变化、model quality claim 是否变宽。

本版真实归档使用 v792 自身同时作为 baseline 和 current，用来建立“无回归”的初始监控证据。

本版明确不做：

- 不重新训练。
- 不重新 replay。
- 不新增 route。
- 不把 tiny pair-probe 结果解释成生产模型质量。

## 运行命令和结果

```powershell
python -B scripts\check_model_capability_route_promotion_regression.py --current e\792\解释\model-capability-route-promotion-portfolio --baseline e\792\解释\model-capability-route-promotion-portfolio --out-dir e\793\解释\model-capability-route-promotion-regression-monitor --require-pass --force
```

输出摘要：

```text
status=pass
decision=model_capability_route_promotion_regression_monitor_passed
failed_count=0
regression_monitor_passed=True
lost_active_route_count=0
gained_active_route_count=0
boundary_changed=False
model_quality_claim_changed=False
current_model_quality_claim=seed_stable_pair_probe_route_accepted
```

## 关键观察

- v793 建立了 route portfolio 的回归保护入口。
- 当前 baseline 和 current 相同，因此应当是无回归通过。
- `lost_active_route_count=0`，说明 active route 没有丢失。
- `boundary_changed=False`，说明 tiny pair-probe 边界没有变化。
- `model_quality_claim_changed=False`，说明模型质量声明没有被放宽。
- 后续新 portfolio 可以直接与 v792 baseline 或上一版 portfolio 对比。
