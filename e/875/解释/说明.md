# v875 运行截图和解释：bounded objective loss signal bridge target-only memory partial-hit diagnostic

## 本版目标

v875 接在 v874 后面。v874 已经证明 target-only memory checkpoint 不再 zero-hit：三条 contract case 都命中了 `fixed`，但都没有完整命中 `loss`。v875 的目标是把这个 partial-hit 结果进一步诊断成可执行的下一步修补方向。

本版不训练，不 replay，不修改 contract。它只解释 v874 为什么还没有 contract recovered。

## 关键解释

v875 将 v874 的三条 continuation 归类为同一个模式：

```text
fixed l
```

这不是原来的 label echo，也不是完全不会生成目标词。它说明模型已经稳定进入 `fixed` 和 `loss` 的前缀附近，但还不能把 `loss` 完整输出。

## Root causes

- `loss_suffix_uptake_gap`
- `fixed_dominates_required_pair`
- `no_anchor_partial_signal`
- `partial_signal_without_contract_pass`

下一版不应该继续做大而散的训练，而应该构造更窄的 `loss` suffix / `fixed loss` pair completion patch。
