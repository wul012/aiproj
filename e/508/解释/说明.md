# v508 运行截图与解释

## 本版主题

v508 增加 `model_capability_required_term_pair_prefix_completion_sweep`。它读取 v506 decoding path trace，对每个 probe 逐步强制 expected term 的 1、2、...、完整 token 前缀，然后沿用原 profile 继续生成。

本版不训练模型，不扩展语料，也不把 forced-prefix generation 当作真实自由生成能力。它只是回答 v507 后的问题：问题是否只是第一 token，还是完整 required term span 都不稳。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_prefix_completion_sweep.py `
  e\506\解释\model-capability-required-term-pair-decoding-path-trace `
  --out-dir e\508\解释\model-capability-required-term-pair-prefix-completion-sweep `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_prefix_completion_long_prefix`
- `prefix_completion_sweep_decision=prefix_completion_requires_longer_prefix`
- `prefix_row_count=27`
- `probe_summary_count=6`
- `any_prefix_hit_probe_count=6`
- `one_token_prefix_hit_probe_count=3`
- `full_prefix_hit_probe_count=6`
- `span_completion_gap_probe_count=3`

结论很明确：

- `loss` 在三个 profile 下都是 `minimum_hit_prefix_token_count=1`。
- `fixed` 在三个 profile 下都需要 `minimum_hit_prefix_token_count=4`，而完整 token 数是 `5`。

所以 v507 的问题不只是第一 token 采样，`fixed` 的完整词 span completion 明显弱于 `loss`。

## 截图说明

`01-model-capability-required-term-pair-prefix-completion-sweep.png` 展示 v508 HTML 报告。截图重点看三处：

- 顶部 summary 显示 one-token hits 为 `3`、full-prefix hits 为 `6`。

v508 把 v507 的局部修复进一步定位为 span-completion 不对称：`loss` 容易补全，`fixed` 需要接近完整词前缀。
