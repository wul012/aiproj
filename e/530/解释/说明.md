# v530 运行截图和解释

## 本版目标

v530 在 v529 的显式 generation profile 基础上，增加运行时 profile registry endpoint：

```text
/api/generation-profiles
```

这样 playground 不只依赖静态 HTML 内置的 profile 列表，而是启动后可以从当前 server 读取 profile registry，降低前端与后端 profile 定义漂移的风险。

本版不新增新的解码策略，只把已有 `default` 与 `suppress_newline_tokens` 做成可查询契约。

## 真实运行命令

启动 playground server：

```powershell
python -B scripts\serve_playground.py `
  --run-dir e\528\解释\model-capability-required-term-pair-loss-alias-fresh-seed-sweep\fresh-compare-run\fresh-focus-run\seed-runs\seed-528\loss-alias-focus-run `
  --port 8530 `
  --device cpu `
  --request-log e\530\解释\generation-profiles-endpoint\inference_requests.jsonl
```

查询 profile endpoint：

```powershell
Invoke-RestMethod -Method Get -Uri http://127.0.0.1:8530/api/generation-profiles
```

## 关键结果

`generation-profiles.json` 中：

```text
status=ok
default_generation_profile_id=default
profile_count=2
profiles=default,suppress_newline_tokens
```

`health-with-generation-profiles.json` 中也暴露：

```text
generation_profiles_endpoint=/api/generation-profiles
generation_profiles=[default,suppress_newline_tokens]
```

Playwright 还验证了 playground runtime data：

```text
profiles=default,suppress_newline_tokens
command contains --generation-profile 'suppress_newline_tokens'
```

## 一句话总结

v530 把 generation profile 从静态 UI 选项升级为 server 可查询契约，使 CLI/API/playground 的 profile 列表能围绕同一 registry 收束。
