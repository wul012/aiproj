# v522 运行截图与解释

## 本版主题

v522 基于 v521 的 newline segmentation 结论，做一个只读 decode cleanup audit。它比较多种清理策略，确认是否需要改训练语料，还是先做有边界的 newline cleanup evaluation。

边界：本版不修改生成器、不改变模型输出、不把 cleanup 后的结果冒充原始 strict hit。它只回答一个问题：如果仅清理 newline，v518 的 strict miss 能否恢复成包含 `loss` 的 surface string。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_loss_alias_decode_cleanup.py `
  e\518\解释\model-capability-required-term-pair-loss-alias-focus-metrics `
  --out-dir e\522\解释\model-capability-required-term-pair-loss-alias-decode-cleanup `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_loss_alias_remove_newlines_cleanup_recovers_full`
- `decode_cleanup_decision=loss_alias_decode_cleanup_remove_newlines_full`
- `case_count=4`
- `raw_hit_count=0`
- `remove_newlines_hit_count=4`
- `collapse_whitespace_hit_count=0`
- `remove_all_whitespace_hit_count=4`
- `alnum_only_hit_count=4`
- `minimal_recovery_strategy=remove_newlines`
- `model_quality_claim=tiny_loss_alias_newline_cleanup_recovers_strict_surface`

case 级别结果：

| case | raw | remove newlines | collapse whitespace | all whitespace | minimal |
| --- | --- | --- | --- | --- | --- |
| source-loss | False | True | False | True | remove_newlines |
| heldout-beta-loss | False | True | False | True | remove_newlines |
| heldout-omega-loss | False | True | False | True | remove_newlines |
| heldout-theta-loss | False | True | False | True | remove_newlines |

结论：v518 strict miss 不需要宽泛的 alnum-only normalization 才能恢复；仅移除 newline 就能让 4/4 continuation 包含 `loss`。下一步应测试一个有边界的 newline cleanup evaluation，而不是直接改变训练目标。
