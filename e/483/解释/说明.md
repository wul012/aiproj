# v483 运行截图和解释：required-term micro-training

## 本版主题

v483 承接 v482 的结论：短 scaffold prompt 形如 `data:`、`text:` 时，归档 tiny checkpoint 仍没有把 required term 写进 continuation。本版做一个真正的小训练实验：用 v482 选出的 20 个 required-term scaffold 样本构造微型语料，训练一个新的 tiny checkpoint，再用同一批 prompt 做生成检查。

本版不宣称模型整体能力提升，不扩大 benchmark，也不把 prompt 中已有的词算作续写能力。关键指标只看 continuation 是否包含 required term。

## 运行输入

```text
source=e/482/解释/model-capability-required-term-scaffold-probe
corpus=e/483/解释/model-capability-required-term-micro-training/required_term_micro_corpus.txt
checkpoint=e/483/解释/model-capability-required-term-micro-training/micro-run/checkpoint.pt
```

CLI：

```text
python -B scripts/run_model_capability_required_term_micro_training.py e/482/解释/model-capability-required-term-scaffold-probe --out-dir e/483/解释/model-capability-required-term-micro-training --max-iters 300 --eval-iters 2 --batch-size 16 --block-size 8 --n-layer 1 --n-head 1 --n-embd 32 --learning-rate 0.02 --term-repeat 24 --max-new-tokens 8 --temperature 0.2 --top-k 1 --generation-seed 483 --device cpu --require-pass --force
```

解释：`generated_hit_count=20` 仍然包含 prompt 自身，不代表模型续写能力；真正新增的是 `continuation_hit_count=4`。这说明 tiny 模型在 300 步定向训练后开始学到一部分 scaffold-to-term 的复制模式，尤其是 `data:` 相关样本，但还没有学会所有 required terms。

## 结论

v483 证明：当 required term 被做成显式短 scaffold 并进入微型训练语料后，tiny checkpoint 可以出现少量 continuation uptake。这个结果只是 targeted micro-training signal，不是通用模型质量提升；下一步应做 held-out required-term slice，检查模型是否只是复制训练样本。
