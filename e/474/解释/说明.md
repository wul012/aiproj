# v474 运行截图和解释

## 本版目标

v474 继续模型能力阶段，但不再只看一个 baseline/candidate pair。本版新增 model capability ladder，把同一 tiny 配置在多个训练步数下跑完真实训练、eval suite、generation quality、pair baseline 和 benchmark scorecard，再把趋势汇总成一份独立报告。

本版不宣称模型已经有稳定语言能力，也不把 tiny CPU smoke 的轻微 loss 下降解释为生产级提升。它只回答：在同 seed、同模型、同数据和同评估套件下，增加训练步数是否带来可复核的训练信号变化。

## 真实训练配置

- Suite：`standard-zh`
- Case token cap：`4`
- Max iters ladder：`1,2,4`
- Seed：`1337`
- 模型：`n_layer=1`、`n_head=1`、`n_embd=8`、CPU
- 每个 rung 都运行完整链路：train -> eval suite -> generation quality -> same-checkpoint pair batch -> benchmark scorecard

## 本次结果

- Rung count：`3`
- Successful rung count：`3`
- Best loss max iters：`4`
- Best score max iters：`1`
- Best val loss delta：`-0.0031`
- Score delta：`0.0`
- Generation flags delta：`0.0`
- Trend decision：`loss_improved_without_eval_improvement`

逐项结果：

| Max iters | Best val loss | Scorecard | Generation flags |
| ---: | ---: | ---: | ---: |
| 1 | 5.345132350921631 | 81.67 | 0 |
| 2 | 5.345132350921631 | 81.67 | 0 |
| 4 | 5.3419880867004395 | 81.67 | 0 |

这说明训练步数增加到 4 时出现了非常轻微的 best validation loss 下降，但 scorecard 和 generation-quality flags 没有改善；因此本版证据仍然只能支持“训练信号略有变化”，不能支持“模型能力明显提升”。
