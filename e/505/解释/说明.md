# v505 运行截图与解释

## 本版主题

v505 增加 `model_capability_required_term_pair_decoding_gap_probe`。它读取 v504 generation gap report，选择其中的 best gap variant `symmetric-anchor`，再追溯 v503 checkpoint metadata，用少量 decoding profile 做生成侧复核。

本版不训练新 checkpoint，也不修改 v502-v504 证据。它只验证 v504 的结论：内部 forced-choice 偏好是否能在不同生成 profile 下被表达出来。

## 运行命令

```powershell
python -B scripts\run_model_capability_required_term_pair_decoding_gap_probe.py `
  e\504\解释\model-capability-required-term-pair-generation-gap `
  --out-dir e\505\解释\model-capability-required-term-pair-decoding-gap-probe `
  --force `
  --require-pass
```

## 关键结果

- `status=pass`
- `decision=required_term_pair_decoding_gap_partial_only`
- `decoding_gap_probe_decision=decoding_gap_probe_partial_expression_only`
- `target_count=1`
- `profile_count=3`
- `probe_count=6`
- `continuation_hit_count=2`
- `profile_full_hit_count=0`
- `best_variant_id=symmetric-anchor`
- `best_profile_id=top2-24`
- `best_profile_hit_count=1`

真实 profile 结果：

- `greedy-12`：`fixed/loss` 都未命中。
- `greedy-24`：`fixed` 命中，`loss` 未命中。
- `top2-24`：`loss` 命中，`fixed` 未命中。

这说明 v504 的内部偏好不是完全无法表达，但目前没有一个稳定 decoding profile 能同时表达 `fixed` 和 `loss` 两端。
