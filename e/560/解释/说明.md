# v560 required-term pair equals-surface no-pair-id repair 运行说明

## 本版目标

v559 的加宽实验出现了 `fixed=01=...` 这类输出，说明 `equals_surface_tied_repair` 里的 `pair=01` 可能在 equals 后续位置形成了强竞争目标。v560 因此新增一个轻量语料模式：保留 fixed/loss 同记录绑定，但去掉 `pair=01` 数字 ID。

本版不扩大模型、不新增治理链，也不宣称模型能力已提升。它只验证“去掉 numeric pair id 后，seed `1535` 是否能恢复 pair-full”。

## 代码改动

- `src/minigpt/model_capability_required_term_pair_coexistence_corpus.py`
  - 新增 `equals_surface_no_pair_id_repair`。
  - 新增 `_extend_equals_surface_no_pair_id_repair()`。
  - `source_prompts()` 继续让该模式使用 `fixed=` / `loss=` replay surface。
- `tests/test_model_capability_required_term_pair_coexistence_refresh.py`
  - 覆盖新 corpus 是否包含 paired records。
  - 覆盖 corpus 是否删除 `pair=01`。
  - 覆盖 replay prompt 是否仍为 equals surface。

## 结论

v560 说明 `pair=01` 的 equals 竞争确实值得移除，但去掉它会把问题收缩为 loss 分支不足。下一步更合理的是做 loss-balanced no-pair-id 版本，而不是回到模型加宽。
