# v565 required-term pair no-pair-id first-token migration comparison 运行说明

## 本版目标

v564 加入 first-token prefix rows 后，三 seed pair-full 仍是 `1/3`。v565 不训练新模型，而是把 v562 和 v564 放在同一张 comparison 里，看 first-token rows 到底让 coverage 如何迁移。

本版不新增代码、不改 corpus、不做 held-out。它只做 read-only comparison。

## 运行命令

```powershell
python scripts\run_model_capability_required_term_pair_equals_surface_repair_comparison.py `
  e\562\解释\model-capability-required-term-pair-equals-surface-no-pair-id-loss-balanced-stability `
  e\564\解释\model-capability-required-term-pair-no-pair-id-loss-balanced-first-token-stability `
  --labels v562-loss-balanced v564-first-token `
  --out-dir e\565\解释\model-capability-required-term-pair-no-pair-id-first-token-migration-comparison `
  --force `
  --require-pass
```

## 关键结果

```text
status=pass
compared_report_count=2
term_case_row_count=24
pair_full_profile_seed_count=1
union_hit_terms=fixed,loss
```

seed-level migration：

```text
1535 -> v562/v564 都 pair-full
2535 -> v562 loss-only, v564 all-miss
535  -> v562 all-miss, v564 fixed-only
```

这说明 first-token rows 没有把稳定性从 `1/3` 推高，只是把局部命中从一个 seed/term 转移到了另一个 seed/term。

## 结论

v565 说明 v564 的 first-token rows 不是稳定增益，而是 coverage 迁移。下一步应设计更可控的 objective mix 或 seed/config policy，而不是继续单方向堆 prefix rows。
