# v297 运行截图说明

## 本版主题

v297 把 CI workflow order evidence 从 project audit 继续传入 release bundle 和 release readiness。目标是让 promoted seed handoff assurance smoke 是否早于 coverage gate 的证据，在发布总包和发布就绪判断阶段也能直接看到。

本版不改变 GitHub Actions workflow、不新增发布阻断规则，也不调整 coverage 阈值。它只做证据传播和展示增强。

## tag 说明

`v297.0.0` 的含义是：MiniGPT v297 release readiness CI order evidence。

这一版把 v296 的 audit-level order context 继续传到发布总包和发布就绪阶段，让最终 ship/review 视图也能解释 CI workflow 顺序证据。

一句话总结：v297 让 CI workflow order evidence 进入 release bundle 和 release readiness，发布阶段不再只看到 CI pass/fail，还能看到 required order 和 order violation 计数。
