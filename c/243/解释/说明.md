# v243 release gate coverage audit policy 运行截图说明

## 本版主题

v243 把 coverage gate 证据接入 release gate policy。

v241 让 project audit 能读 coverage report，v242 让 release bundle 携带 coverage context，v243 则让 standard/review/strict release gate profile 要求 `test_coverage_report` audit check 通过。`legacy` profile 仍允许历史 bundle 缺失该检查。

## 截图清单

该截图来自：

```text
python -B -m unittest tests.test_release_gate tests.test_release_gate_comparison -q
```

它证明 release gate 和 profile comparison 都理解新的 coverage policy 字段，覆盖了 pass、missing、warn、legacy compatibility 和 profile comparison 输出。

## 证据边界

v243 证明的是 coverage gate 已经进入 release gate policy。

它不证明模型质量变好，也不替代 generation quality、benchmark scorecard 或真实 checkpoint 对比。覆盖率门禁只说明测试治理门槛达标。

## 一句话总结

v243 让 release gate 从“展示 coverage context”推进到“要求 coverage audit evidence”，并保留 legacy profile 兼容旧发布包。
