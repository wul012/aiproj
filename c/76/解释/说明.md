# v76 运行截图和解释

## 本版主题

v76 新增 controlled training scale handoff。它读取 v75 `training_scale_workflow.json` 和其中的 decision 产物，默认只验证 selected `--execute` command；只有显式传入 `--execute` 时，才真正运行命令并记录返回码、耗时、stdout/stderr tail 和预期训练产物是否存在。

这一版不新增报表变体，也不自动跑大训练。它把 v75 的“建议执行命令”推进成“受控执行证据”，但仍保留人工确认边界。

## 证据含义

validation smoke 证明 v76 可以读取 v75 workflow 的 decision，并把 command 作为受控 handoff 记录下来。execute smoke 使用 tiny sample corpus 和单个 variant，真实执行 selected command，产物仍留在临时目录，最终按 cleanup gate 删除。

## Tiny corpus execution note

The first real v76 execute smoke exposed an important planner boundary: the sample corpus only had 507 characters, so the original `block_size=64` made the validation split too short for `get_batch`.

This version fixes the source planner with `safe_block_size_for_char_count`. In the verified smoke, `scale-smoke` is reduced from `block_size=64` to `block_size=48`, then the controlled handoff completes with `returncode=0` and all 6 expected artifact checks available.
