# v150 运行截图说明

## 本版主题

v150 把 v149 的 CI workflow readiness regression 继续带入 `registry`。

本版让 registry run row、CSV、HTML readiness cell、Release Readiness Deltas leader table 和 release readiness delta summary 都能显示 CI workflow regression。它仍然是 registry tracking evidence，不改变 release gate 的硬阻断规则。

## 关键改动

- `src/minigpt/registry_data.py`
  - `RegisteredRun` 新增 `release_readiness_ci_workflow_regression_count`。
  - `summarize_registered_run()` 从 comparison summary 读取 `ci_workflow_regression_count`。
  - `_release_readiness_comparison_status()` 新增 `ci-regressed`。
- `src/minigpt/registry_rankings.py`
  - release readiness delta row 新增 CI workflow 状态和 failed-check delta。
  - release readiness delta summary 新增 CI workflow regression count、status changed count 和 max failed-check delta。
  - leaderboard 排序把 CI workflow 变化纳入优先级。
- `src/minigpt/registry_render.py`
  - Release Readiness cell 新增 `ci regressions=<n>`。
  - Release Readiness Deltas 表格新增 CI workflow 列。
- `src/minigpt/registry_artifacts.py`
  - registry CSV 新增 `release_readiness_ci_workflow_regression_count`。

## 证据边界

registry 现在能追踪 CI workflow readiness regression，但这不是模型能力证据，也不是 release gate hard block。

它证明的是：多个 run 的总览页面可以发现 release readiness comparison 中的 CI workflow hygiene 漂移。

## tag 含义

`v150.0.0` 标记的是 “registry CI workflow readiness regression tracking”：registry 可以汇总并展示 release readiness comparison 中的 CI workflow regression。

一句话总结：v150 把 CI workflow hygiene 退化信号从 comparison 推进到 registry，让多 run 总览也能看到 CI 策略漂移。
