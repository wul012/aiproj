# v197 benchmark scorecard artifact helper consolidation 运行说明

## 本版目标

v197 继续 v196 的 benchmark scorecard helper 收束，但这次只处理 artifact writer：`src/minigpt/benchmark_scorecard_artifacts.py`。

它解决的问题是 artifact 层继续保留本地 `_dict()` / `_list_of_dicts()`，而且旧 `_list_of_dicts()` 是 all-or-nothing：列表里只要混入一个非 dict，整段有效行也会被丢掉。本版改为复用 `report_utils.as_dict` 和 `report_utils.list_of_dicts`，让 artifact 导出跳过坏行、保留好行。

## 关键改动

- `src/minigpt/benchmark_scorecard_artifacts.py`
  - 新增 `from .report_utils import as_dict as _dict`。
  - 新增 `from .report_utils import list_of_dicts as _list_of_dicts`。
  - 删除本地 `_dict()` 和 `_list_of_dicts()`。
  - CSV、Markdown、HTML 渲染入口继续使用原来的 `_dict` / `_list_of_dicts` 别名，避免扩大改动面。
- `tests/test_benchmark_scorecard_artifacts.py`
  - 新增 mixed-row artifact 测试：component、drilldown、rubric 列表中混入字符串时，CSV/Markdown 仍保留有效 dict 行，并不输出无效字符串。

## 证据边界

本版证明的是 benchmark scorecard artifact 层的 helper 复用与 mixed-row 容错行为，不证明模型质量提升，也不改变 scorecard 评分策略。它属于维护质量版本，目的是让评估报告的输出层更稳定、更少复制小工具。
