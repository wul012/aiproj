# v181 CI workflow hygiene artifact split 运行说明

## 本版目标

v181 把 `ci_workflow_hygiene.py` 里的 JSON、CSV、Markdown 和 HTML 输出函数拆到 `ci_workflow_hygiene_artifacts.py`。上一版 v180 收束了 training scale workflow 的 artifact 层，本版继续处理当前 module pressure 暴露出的 CI workflow hygiene 模块体量问题。

本版明确不改 `build_ci_workflow_hygiene_report()` 的参数、TypedDict report schema、action 解析、Node 24 native action 策略、required command 检查、CLI 用法和旧的 `minigpt.ci_workflow_hygiene` 导入路径。

## 关键改动

- `src/minigpt/ci_workflow_hygiene_artifacts.py`：新增 artifact 模块，负责 `ci_workflow_hygiene.json`、CSV、Markdown 和 HTML 的写出与渲染。
- `src/minigpt/ci_workflow_hygiene.py`：保留 workflow 读取、action 收集、policy checks、summary、recommendations 和旧 artifact writer re-export。
- `tests/test_ci_workflow.py`：新增 facade identity 断言，确认旧入口导出的 artifact 函数就是新模块里的实现。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/README.md`：更新 v181 版本说明、证据索引和讲解索引。

## 运行验证

```powershell
python -B -m unittest tests.test_ci_workflow -v
python -B scripts/check_maintenance_batching.py --out-dir runs/maintenance-v181
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v181
python -B -m unittest discover -s tests -v
```

## tag 含义

`v181.0.0` 表示 MiniGPT 的 CI workflow hygiene artifact split 版本。这个 tag 对应的能力是：CI workflow hygiene 继续保持原有 policy 检查、TypedDict schema 和 CLI 契约，但 JSON/CSV/Markdown/HTML 输出层被拆到更清晰的 artifact 边界。

一句话总结：v181 把 CI workflow hygiene 的“策略检查”和“报告输出”分开，让 CI 质量门禁主模块继续可维护，同时保持所有旧导入和产物格式不变。
