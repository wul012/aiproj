# v179 maturity narrative summary section split 运行说明

## 本版目标

v179 把 `maturity_narrative.py` 里的 portfolio summary、recommendations、warnings、narrative sections 和 evidence matrix helper 拆到两个专用模块。上一版 v178 收束了本地推理 server，这一版回到当前最大模块 `maturity_narrative.py`，继续做低风险、契约保持的体量治理。

本版明确不改 `build_maturity_narrative()` 的输入参数、返回 JSON schema、recommendations/warnings 语义、Markdown/HTML artifact writers、CLI 用法和旧的 maturity narrative 导入路径。

## 关键改动

- `src/minigpt/maturity_narrative_summary.py`：新增 summary helper 模块，负责 portfolio status、benchmark/dataset/decision summary、recommendations 和 warnings。
- `src/minigpt/maturity_narrative_sections.py`：新增 section/evidence helper 模块，负责 human-readable sections、claim 文案、status-from-counts 和 evidence matrix 行。
- `src/minigpt/maturity_narrative.py`：保留输入发现、JSON 读取、summary/section/evidence 组装和旧 artifact writer 导出。
- `tests/test_maturity_narrative.py`：新增断言，确认 narrative 入口确实使用 split summary/section helper。

## 运行验证

```powershell
python -B -m unittest tests.test_maturity_narrative -v
python -B scripts/check_maintenance_batching.py --out-dir runs/maintenance-v179
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v179
python -B -m unittest discover -s tests -v
```

## tag 含义

`v179.0.0` 表示 MiniGPT 的 maturity narrative summary/section split 版本。这个 tag 对应的能力是：成熟度叙事入口继续保持输出契约稳定，但 summary 计算和面向人的 section/evidence 构造被拆到更清晰的维护边界。

一句话总结：v179 把 maturity narrative 的“证据计算”和“叙事区块/evidence matrix 构造”从主入口拆开，让项目成熟度叙事继续可维护，同时保持 portfolio narrative schema 和 artifact 输出不变。
