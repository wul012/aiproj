# v167 playground request history script split 运行说明

## 本版目标

v167 做的是一次轻量、定向的 UI 脚本收口：把 `playground_script.py` 中稳定的 request-history JavaScript 片段拆到 `playground_request_history_script.py`。

它解决的是 playground 主脚本继续膨胀的问题。v164 已经把服务端 `/api/request-history` 与 `/api/request-history-detail` 处理拆到 `server_request_history.py`，v167 则把浏览器端消费这两个 endpoint 的表格、过滤、CSV 链接、详情 JSON 和加载逻辑独立出来。

本版不改变 HTML 入口、不改变 endpoint、不改变 CSV/detail URL，也不改变 `playground_script(page_data)` 的调用方式。

## 关键改动

- `src/minigpt/playground_request_history_script.py`：新增 request-history JavaScript fragment，包含 `requestHistoryQuery()`、`requestHistoryDetailUrl()`、`renderRequestHistory()`、`showRequestHistoryDetail()` 和 `loadRequestHistory()`。
- `src/minigpt/playground_script.py`：通过 `playground_request_history_script()` 注入片段，继续返回完整 `<script>...</script>`。
- `tests/test_playground_request_history_script.py`：直接验证 fragment 的 endpoint、JSON 下载名、detail 链接和单花括号 JavaScript 形态。
- `tests/test_playground_asset_modules.py`：确认完整 playground script 仍包含拆出的 request-history fragment。
