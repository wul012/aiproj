# v159 server HTTP helper split 运行说明

## 本版主题

v159 把 `server.py` 里的 JSON/text response、SSE header/event、静态文件发送、request body parsing 和 run file serving helper 拆到 `src/minigpt/server_http.py`。

这是一版 contract-preserving refactor。`create_handler()`、`run_server()`、本地推理 API 路由、streaming 事件 schema 和旧 `minigpt.server` facade 导出保持不变。

## 关键改动

- 新增 `src/minigpt/server_http.py`，集中保存 HTTP response、SSE、文件发送和 body 解析 helper。
- 更新 `src/minigpt/server.py`，保留 route handler 和薄代理方法，把实际 HTTP helper 行为委托给 `server_http.py`。
- 新增 `tests/test_server_http.py`，直接验证 JSON body parsing、JSON/text/SSE/file response 和目录逃逸保护。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引同步到 v159。

## Tag 含义

`v159.0.0` 表示 MiniGPT 本地推理服务完成 HTTP helper 拆分：response/file/body/SSE helper 进入独立模块，同时 route handler、API 行为和旧导出保持兼容。

## 一句话总结

v159 让 `server.py` 从“路由 + 推理 + 响应字节层”继续收束为“路由 + 推理编排”，把稳定的 HTTP helper 行为交给可独立测试的 `server_http.py`。
