# v248 maturity narrative coverage regression context 运行截图说明

## 本版主题

v248 把 maturity summary 里的 coverage regression context 接入 maturity narrative。

v247 已经让 maturity summary 因 coverage regression 降为 `warn`。v248 继续把这个信号放进人读叙事报告：portfolio summary、Release Quality Trend claim、Markdown/HTML stats、CLI 输出和 review recommendation 都能解释为什么 `coverage-regressed` 需要 review。

## 截图清单

### 01-maturity-narrative-coverage-tests.png

该截图来自：

```text
python -B -m unittest tests.test_maturity_narrative -q
```

结果：

```text
Ran 9 tests
OK
```

它证明 narrative summary、section、artifact 输出都能承接 coverage regression 字段。

### 02-maturity-narrative-coverage-smoke.png

该截图来自：

```text
python -B scripts/build_maturity_narrative.py --project-root sample --out-dir narrative
```

关键结果：

```text
portfolio_status=review
maturity_status=warn
release_readiness_trend_status=coverage-regressed
release_readiness_test_coverage_regression_count=1
release_readiness_max_test_coverage_gap_delta=3
warnings=[]
```

它证明即使 request history、benchmark scorecard、benchmark decision 和 dataset card 都是 clean，coverage regression 仍会让 narrative 停在 review。

### 03-maturity-narrative-output-check.png

该截图来自对 narrative JSON、Markdown 和 HTML 的结构检查。

关键结果：

```text
portfolio_status=review

coverage_regressions=1
coverage_gap_delta=3

test coverage regressions=1

```

它证明 coverage regression 已进入：

- 机器读 summary：`maturity_narrative.json`
- Release Quality Trend claim
- Markdown portfolio summary
- HTML stat cards
- review recommendation

### 04-full-unittest.png

该截图来自：

```text
python -B -m unittest discover -s tests -q
```

结果：


它证明 v248 的 narrative 改动没有破坏全项目测试集合。

## 证据边界

v248 证明的是 narrative 能解释 coverage regression。

它不重新计算 coverage，不改变 maturity summary 的风险判断，也不把 coverage regression 说成模型质量退步。它只是把测试治理退步从机器字段翻译成人读组合叙事。

## 一句话总结

v248 让 maturity narrative 能讲清楚覆盖率退步：`coverage-regressed` 不再只是 JSON 字段，而会成为组合报告里的 release-quality review 原因。
