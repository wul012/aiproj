# v175 promoted training scale comparison artifact split 运行说明

## 本版目标

v175 把 `promoted_training_scale_comparison.py` 里的 JSON、CSV、Markdown 和 HTML 产物写出拆到 `promoted_training_scale_comparison_artifacts.py`。这一版解决的是 promoted comparison 模块同时承担 promoted-run 比较语义和证据渲染的问题。

本版明确不改 promotion index 加载、promoted-run 筛选、comparison input 解析、baseline 选择、`build_training_scale_run_comparison()` 复用、blockers、recommendations、`scripts/compare_promoted_training_scale_runs.py` 用法和旧的 `minigpt.promoted_training_scale_comparison` facade 导出。

## 关键改动

- `src/minigpt/promoted_training_scale_comparison_artifacts.py`：新增 artifact 模块，负责 `write_promoted_training_scale_comparison_json()`、`write_promoted_training_scale_comparison_csv()`、`render_promoted_training_scale_comparison_markdown()`、`render_promoted_training_scale_comparison_html()` 和 `write_promoted_training_scale_comparison_outputs()`。
- `src/minigpt/promoted_training_scale_comparison.py`：保留 promotion index 读取、promoted rows 解析、comparison inputs 构造、comparison report 合并、summary、blockers 和 recommendations，并从新 artifact 模块 re-export 旧 writer/render 入口。
- `tests/test_promoted_training_scale_comparison.py`：新增 facade identity 断言，确认旧模块和新 artifact 模块指向同一组函数对象。
- `README.md`、`c/README.md` 和项目成熟度阶段讲解索引：记录 v175 的版本能力、证据位置和讲解入口。

## 运行验证

```powershell
python -B -m unittest tests.test_promoted_training_scale_comparison -v
python -B scripts/check_maintenance_batching.py --out-dir runs/maintenance-v175
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v175
python -B -m unittest discover -s tests -v
```

定向测试覆盖 promoted-only comparison、promoted input 不足时 blocked、非法 baseline blocked、HTML escaping、产物写出和 facade identity。全量 unittest 确认该拆分没有破坏 promoted decision、promoted seed、training-scale workflow 等下游链路。

## tag 含义

`v175.0.0` 表示 MiniGPT 的 promoted training scale comparison artifact split 版本。这个 tag 对应的能力是：promoted-run 比较语义仍由主模块生成，而多格式发布产物由独立 artifact 模块负责。

一句话总结：v175 把 promoted training scale comparison 的“比较语义”和“证据发布”分开，让后续调整报告样式不再挤占 promoted-run 比较逻辑。
