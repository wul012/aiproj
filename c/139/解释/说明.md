# v139 benchmark scorecard comparison flag taxonomy 运行说明

## 本版主题

v139 把 v138 已经进入 benchmark scorecard 的 generation-quality flag taxonomy 继续接入跨 scorecard comparison。

本版解决的问题是：比较两个 benchmark scorecard 时，不只看 overall/rubric 分数和 case delta，也能看到生成质量问题类型是否变多、主导 flag 是否变化、最差生成 case 是否迁移。

本版不证明模型质量提升；它证明 scorecard comparison 已经能消费更细的生成质量诊断。

## 关键改动

- `src/minigpt/benchmark_scorecard_comparison.py`：run summary 新增 generation-quality total flags、dominant flag、worst generation case；baseline delta 新增 flag delta、flag relation、dominant flag changed、worst case changed。
- `src/minigpt/benchmark_scorecard_comparison_artifacts.py`：CSV、Markdown、HTML 展示 `Gen Flags`、dominant flag 和 flag regression/improvement 统计。
- `scripts/compare_benchmark_scorecards.py`：CLI 输出 generation-quality flag regression/improvement count 和 baseline dominant flag，便于 smoke 截图直接证明链路。
- `tests/test_benchmark_scorecard_comparison.py` 和 `tests/test_benchmark_scorecard_comparison_artifacts.py`：覆盖 flag delta、taxonomy shift、CSV header 和渲染输出。

## 证据边界

`c/139` 证明的是跨 scorecard comparison 能识别 generation-quality taxonomy 的变化。

如果 candidate 的 total flags 增加、dominant flag 变化或 worst generation case 变化，comparison 会把它作为 promotion review 风险暴露出来。

这仍然不是模型能力提升证据。模型能力提升需要真实 checkpoint、固定 prompt suite、多轮 scorecard comparison 和训练记录共同支撑。
