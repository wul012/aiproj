# v147 运行截图说明

## 本版主题

v147 把 v146 已经进入 `project_audit` 的 CI workflow hygiene 上下文继续带入 `release_bundle`。v145 让 CI workflow hygiene 能独立运行，v146 让 project audit 能审计它，v147 让 release bundle 能打包它。

本版不改变 release gate 的硬规则。CI workflow hygiene 进入 release bundle 是证据携带，不是直接升级为阻断项。

## 关键改动

- `src/minigpt/release_bundle.py`
  - 新增 `ci_workflow_hygiene_path` 可选输入。
  - 支持从 `project_audit.json` 的 `ci_workflow_hygiene_path` 自动解析 CI hygiene 报告。
  - release bundle summary 新增 `ci_workflow_status`、`ci_workflow_failed_checks`、`ci_workflow_node24_actions`。
  - 新增 `ci_workflow_context`。
  - Evidence artifacts 新增 `ci_workflow_hygiene_json`、`ci_workflow_hygiene_md`、`ci_workflow_hygiene_html`。
- `scripts/build_release_bundle.py`
  - 新增 `--ci-workflow-hygiene` 参数。
  - CLI 输出新增 CI workflow 状态和 failed check 数。
- `tests/test_release_bundle.py`
  - fixture 新增 CI workflow hygiene report 和 audit context。
  - 覆盖 release bundle summary、context、artifact rows 和显式 path 输入。

## 证据边界

release bundle 现在可以携带 CI workflow hygiene 证据，但 release gate 暂时没有把它作为必须通过的硬门槛。原因是 v147 仍处在“证据链接入”阶段，先让发布总包稳定携带这份证据，再决定是否需要 profile 级策略。

CI workflow hygiene 证明的是 workflow 没有退回旧 action/runtime 策略、没有漏掉质量步骤。它不证明模型能力、数据质量或 checkpoint 成熟度。

## tag 含义

`v147.0.0` 标记的是 “release bundle CI workflow hygiene evidence”：发布证据包可以携带 CI workflow hygiene 报告及其状态摘要。

一句话总结：v147 把 CI workflow hygiene 从项目审计继续推进到 release bundle，让发布总包能携带 CI 策略治理证据。
