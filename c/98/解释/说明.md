# v98 运行截图与解释

## 本版主题

v98 是 README 成熟度摘要收口版。

它解决的问题是：项目推进到 v97 后，README 开头已经积累了过长的功能流水账，读者很难在第一屏判断项目成熟度、证据位置和下一步短板。

本版改成：

- 成熟度矩阵。
- maturity snapshot。
- capability map。
- v98 focus。

本版不修改模型、训练、推理、评估、release gate、release bundle 或 report-utils 代码。

## 证据边界

这些截图不是模型质量证据，也不是训练结果证据。

它们证明的是：

- README 入口已经更适合快速评估项目成熟度。
- 版本记录、讲解记录和截图归档仍然互相对齐。
- 本版没有把文档清理伪装成模型能力提升。

## 后续使用

后续版本如果继续新增能力，README 开头优先更新成熟度矩阵和压力点，不再恢复超长功能流水账。

详细功能变化继续放在：

- 版本标签。
- 分阶段代码讲解。
- `c/<version>` 截图归档。

## 一句话总结

v98 的截图和说明证明：MiniGPT 项目开始把 README 第一屏从“历史功能堆叠”收束为“成熟度、证据和下一步方向”的工程入口。
