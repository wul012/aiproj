# v240 conservative coverage fail-under gate 运行说明

## 本版主题

v240 把 v239 的 coverage baseline 推进成保守 CI 门禁。

现在 `scripts/run_test_coverage.py` 支持 `--fail-under`，CI 使用 `--fail-under 80`。脚本会先写出 coverage JSON/CSV/Markdown/HTML，再根据覆盖率是否达标决定退出码。

## 关键改动

- `src/minigpt/test_coverage_report.py`：新增 `fail_under`、`threshold_enabled`、`coverage_gap` 和 pass/fail decision。
- `scripts/run_test_coverage.py`：新增 `--fail-under`，覆盖率不达标时返回退出码 2。
- `.github/workflows/ci.yml`：Unit tests 步骤增加 `--fail-under 80`。
- `src/minigpt/ci_workflow_hygiene.py`：新增 coverage fail-under gate 片段检查。
- `tests/test_test_coverage_report.py`：补充阈值通过、阈值失败、非法阈值测试。
- `tests/test_ci_workflow.py`：同步保护 workflow 中的 coverage gate 命令。

## 本版边界

`--fail-under 80` 是测试治理门禁，不是模型质量门禁。

80 的阈值比当前本地 baseline 约 90% 低很多，用来阻止明显倒退，而不是追求短期内把覆盖率推到极限。
