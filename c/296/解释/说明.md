# v296 运行截图说明

## 本版主题

v296 把 CI workflow order evidence 从 hygiene JSON 扩展到 CLI stdout 和 project audit context。核心目标是让 promoted seed handoff assurance smoke 必须早于 coverage gate 的证据，在 CI 日志和下游审计链中都能直接读取。

本版不改变 GitHub Actions 的执行顺序，不新增训练或推理功能，也不调整 coverage 阈值。它是一次证据传播增强。

## tag 说明

`v296.0.0` 的含义是：MiniGPT v296 CI workflow order evidence audit context。

这一版把 v295 的行号 order evidence 继续向外传递，让 CI 日志、project audit check、audit evidence 和 audit context 都能看到同一组 order 计数。

一句话总结：v296 让 CI workflow 顺序证据从单一 hygiene artifact 变成可被 stdout 与 project audit 共同消费的治理上下文。
