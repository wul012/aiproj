# v178 server POST route split 运行说明

## 本版目标

v178 把 `server.py` 里的 POST route handling 拆到 `server_post_routes.py`。这一版解决的是 HTTP handler 同时承担 Handler 装配、POST 路由分发、普通生成、流式生成、pair 生成和 pair artifact 写出的职责过宽问题。

本版明确不改 `/api/generate`、`/api/generate-stream`、`/api/generate-pair`、`/api/generate-pair-artifact` 的请求格式、返回格式、错误状态码、checkpoint 选择逻辑、SSE timeout/cancel 语义、pair artifact schema、request log 字段、`create_handler()`、`run_server()` 和旧的 `minigpt.server` facade 导出。

## 关键改动

- `src/minigpt/server_post_routes.py`：新增 POST route helper 模块，负责四条 POST 路由的分发和执行。
- `src/minigpt/server.py`：`do_POST()` 改为调用 `handle_post_request()`；继续保留 handler assembly、GET 委托、OPTIONS、日志写入 wrapper、HTTP helper wrapper 和 server startup。
- `tests/test_server_post_routes.py`：新增 facade identity、普通生成 helper、pair artifact helper 测试。
- `README.md`、`c/README.md` 和项目成熟度阶段讲解索引：记录 v178 的版本能力、证据位置和讲解入口。

## tag 含义

`v178.0.0` 表示 MiniGPT 的 server POST route split 版本。这个 tag 对应的能力是：本地推理 server 的 POST 生成路由从 HTTP handler 主体中独立出来，但外部 API 和旧导出保持稳定。

一句话总结：v178 把 MiniGPT 本地推理 server 的四条 POST 生成路由抽成 `server_post_routes.py`，让 `server.py` 回到 Handler 装配和启动入口，同时保留生成、流式、pair artifact 和日志契约不变。
