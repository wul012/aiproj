# v127 source encoding hygiene gate 运行截图说明

## 本版主题

v127 把一次 UTF-8 BOM 引起的 CI 失败，收口成了一个独立的 source encoding hygiene gate。

这版只做一件事：扫描 `src`、`scripts`、`tests` 里的 Python 源文件，检查 BOM 和语法错误，生成可归档的只读证据，并让 CI 在单测之前先卡住这类问题。

它不改模型，不改训练，不改推理接口，只做工程卫生治理。

## tag 含义

`v127.0.0` 表示这次发布不是模型能力版本，而是一次工程卫生门禁版本。它的意义是把 BOM 这类低级但高破坏力的问题前移到 CI 里，避免解析失败在更晚的阶段才暴露。

## 一句话总结

v127 让 MiniGPT 的工程治理多了一个“源文件编码卫生”层，CI 现在会先挡住 BOM 和语法异常，再进入更重的单测和发布证据链。
