# v212 运行截图说明

本版目标是把 v211 已经带到 promoted seed handoff 的 suite guard 字段，进一步整理成一个明确的 suite alignment verdict。也就是说，seed handoff 不只展示 `selected_handoff_selected_suite_path`、`seed_suite_path` 和 `plan_suite_path`，还会判断它们当前是 `pending-plan`、`consistent`、`mismatch` 还是 `missing`。

本版不重新计算训练质量，不改变 seed handoff 的 blocker 规则，不改变 planned-mode 审阅逻辑，也不改变 `--execute` 的命令运行语义。alignment verdict 只是审阅证据，`mismatch` 和 `missing` 不会自动阻止命令。

关键文件：

- `src/minigpt/promoted_training_scale_seed_handoff.py`
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
- `scripts/execute_promoted_training_scale_seed.py`
- `tests/test_promoted_training_scale_seed_handoff.py`
- `README.md`

这版截图要证明的事情：

1. 聚焦测试能覆盖 `pending-plan`、`consistent`、`mismatch` 和 `missing` 四种 alignment 状态。
2. CLI smoke 能看到 planned seed handoff 的 `pending-plan`，以及 execute 后生成 plan artifact 的 `consistent`。
3. JSON/CSV/Markdown/HTML 都能输出 `seed_handoff_suite_alignment_status` 和 mismatch/missing 计数。
4. README、`c/README.md` 和代码讲解记录已经把 v212 的位置、归档位置和链路角色对齐。
5. 全量 unittest 与 source encoding smoke 仍然干净，说明 alignment verdict 没有破坏原有 handoff 行为。
