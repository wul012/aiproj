# v134 maturity narrative artifact split 运行说明

## 本版主题

v134 把 `maturity_narrative.py` 中的 JSON、Markdown、HTML 输出写入逻辑拆到 `maturity_narrative_artifacts.py`，让原模块继续负责读取 maturity summary、registry、request history summary、benchmark scorecard 和 dataset card，并生成 portfolio summary、narrative sections、evidence matrix、recommendations 与 warnings。

这是一次轻量收口型优化，不改变 `maturity_narrative.json` schema，不改变输出文件名，不改变 `scripts/build_maturity_narrative.py` 的 CLI 行为，也不改变旧的 `minigpt.maturity_narrative` 导入方式。

## 证据边界

`c/134` 证明的是 maturity narrative 的证据发布层拆分后仍能跑通：旧入口仍可导入，新 artifact 模块能写出同名产物，CLI 仍能消费同一套 narrative dict，HTML 仍有浏览器渲染证据。它不证明模型质量提升，也不代表完成了生产级训练服务；模型能力仍需要更大数据、真实 checkpoint 和固定 benchmark 对比来证明。

本轮临时日志、临时 fixture 和渲染脚本都放在 `tmp/`，完成验证后删除；仓库只保留 `c/134/图片` 中的最终截图和本说明。
