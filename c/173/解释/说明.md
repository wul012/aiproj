# v173 training scale run decision artifact split 运行说明

## 本版目标

v173 把 `training_scale_run_decision.py` 里的 JSON、CSV、Markdown 和 HTML 产物写出拆到 `training_scale_run_decision_artifacts.py`。这一版解决的是训练规模运行决策模块职责偏宽的问题：原模块同时负责候选运行筛选、拒绝原因、推荐状态、执行命令构造和多格式报告输出。

本版明确不改候选选择策略、拒绝原因、readiness 阈值、gate pass 政策、batch started 政策、execute command 格式、`scripts/decide_training_scale_run.py` 用法和旧的 `minigpt.training_scale_run_decision` facade 导出。

## 关键改动

- `src/minigpt/training_scale_run_decision_artifacts.py`：新增 artifact 模块，负责 `write_training_scale_run_decision_json()`、`write_training_scale_run_decision_csv()`、`render_training_scale_run_decision_markdown()`、`render_training_scale_run_decision_html()` 和 `write_training_scale_run_decision_outputs()`。
- `src/minigpt/training_scale_run_decision.py`：保留 `build_training_scale_run_decision()`、`_rejection_reasons()`、`_select_candidate()`、`_build_execute_command()` 等决策逻辑，并从新 artifact 模块 re-export 旧 writer/render 入口。
- `tests/test_training_scale_run_decision.py`：新增 facade identity 断言，确认旧模块和新 artifact 模块指向同一组函数对象。
- `README.md`、`c/README.md` 和项目成熟度阶段讲解索引：记录 v173 的版本能力、证据位置和讲解入口。

## 运行验证

本版使用了以下命令验证收口：

```powershell
python -B -m unittest tests.test_training_scale_run_decision -v
python -B scripts/check_maintenance_batching.py --out-dir runs/maintenance-v173
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v173
python -B -m unittest discover -s tests -v
```

定向测试覆盖运行决策 schema、候选选择、拒绝原因、输出产物和 facade identity。全量 unittest 确认该拆分没有破坏其他训练规模、发布治理、registry、server 和 benchmark 链路。

## tag 含义

`v173.0.0` 表示 MiniGPT 的 training scale run decision artifact split 版本。这个 tag 对应的能力是：训练规模运行决策仍由主模块生成，而多格式发布产物由独立 artifact 模块负责。

一句话总结：v173 把训练规模运行决策的“决策生成”和“证据发布”分开，让后续调整报告样式不再挤占候选运行选择逻辑。
