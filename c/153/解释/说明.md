# v153 运行截图说明

## 本版主题

v153 把 release bundle 的 JSON/Markdown/HTML 渲染写出层从 `release_bundle.py` 拆到 `release_bundle_artifacts.py`。
这版是针对 module pressure 的收口型重构：保留旧 facade 导入路径，不改变 release bundle 数据结构和发布规则。

## 关键改动

- `src/minigpt/release_bundle_artifacts.py`
  - 新增 release bundle JSON/Markdown/HTML 写出函数。
  - 新增 Markdown 表格、HTML sections、artifact links、style 等展示辅助函数。
- `src/minigpt/release_bundle.py`
  - 保留 `build_release_bundle()`、输入解析、summary、context、recommendations。
  - 从 artifact 模块重新导出 `render_release_bundle_*` 和 `write_release_bundle_*`。
  - 行数从 709 降到约 409。
- `tests/test_release_bundle.py`
  - 新增 facade identity 测试，确保旧导入路径仍然等于 artifact 模块函数。

## 证据边界

`c/153` 证明 release bundle 的构建层和 artifact 输出层已经拆分，维护压力降低。
它不证明模型能力提升，也不表示 release gate 或 readiness 规则变化。

## tag 含义

`v153.0.0` 标记的是 “release bundle artifact split”：release bundle 的展示输出层独立成 artifact 模块，同时保留旧 facade。

一句话总结：v153 让 release bundle 从超大混合模块回到清晰边界，并把 module pressure 从 watch 拉回 pass。
