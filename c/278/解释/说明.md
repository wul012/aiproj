# v278 promoted seed handoff automation gate

## 本版目标

v278 把 promoted seed handoff 的两个 opt-in strict requirement 汇总成一个 `automation_gate`。

v216-v219 已经建立了 clean-evidence requirement，v277 又建立了 clean batch-review requirement。此前 CLI 在遇到第一个 fail 时会直接退出，严格运行时可能看不到另一个 requirement 的诊断。v278 的目标是让严格自动化先打印并持久化所有 requirement 的 pass/fail，再由 `automation_gate` 决定最终退出码。

本版不新增第三个用户开关，也不改变默认行为。`automation_gate` 只汇总已经请求的 `--require-clean-evidence` 和 `--require-clean-batch-review`。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_review.py`
  - 新增 `SeedHandoffAutomationGate`。
  - 新增 `build_seed_handoff_automation_gate()`。
  - 新增 `SEED_HANDOFF_AUTOMATION_GATE_STATUSES`。
- `src/minigpt/promoted_training_scale_seed_handoff.py`
  - report 新增 `automation_gate`。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - CSV/Markdown/HTML 展示 automation gate status、failed requirements、passed requirements 和 status domain。
- `scripts/execute_promoted_training_scale_seed.py`
  - strict 模式先打印 clean-evidence 和 clean batch-review 两类诊断，再打印 `automation_gate_*` 并根据总状态退出。
- `tests/test_promoted_training_scale_seed_handoff.py`
  - 覆盖 gate aggregation helper、单 requirement pass/fail、两个 requirement 同时 fail 时 CLI 必须完整打印失败列表。

## tag 含义

`v278.0.0` 表示：promoted seed handoff automation gate。它说明 seed handoff 层已经能把多个 strict requirement 聚合为一个自动化总门禁。

## 一句话总结

v278 让 promoted seed handoff 的严格自动化从“遇到第一个失败就退出”升级为“完整报告所有 requirement，再给出一个总 gate 结论”。
