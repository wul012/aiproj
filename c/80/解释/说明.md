# v80 promoted training scale baseline decision 运行说明

## 本版目标

v80 在 v79 promoted-only comparison 之后继续收口：读取已经完成的 promoted comparison，选择下一阶段可作为训练规模规划输入的稳定 baseline。

本版不重新训练模型，不重新生成 promotion index，也不从 blocked/review 证据里强行挑选 baseline。它只消费上游比较结果，并把 baseline 选择写成可复查的 JSON、CSV、Markdown 和 HTML 证据。

## 运行链路

```text
v75 consolidated training scale workflow
 -> v76 controlled training scale handoff
 -> v77 training scale promotion acceptance
 -> v78 training scale promotion index
 -> v79 promoted training scale comparison
 -> v80 promoted training scale baseline decision
```

v80 的 CLI 是：

```powershell
python scripts/decide_promoted_training_scale_baseline.py runs/training-scale-workflow/promoted-comparison --out-dir runs/training-scale-workflow/promoted-decision
```

输出包括：

```text
promoted_training_scale_decision.json
promoted_training_scale_decision.csv
promoted_training_scale_decision.md
promoted_training_scale_decision.html
```

## 验收标准

- 新单测覆盖 accepted、blocked、上游未 compared、HTML 转义与输出落盘。
- 全量 unittest 回归通过。
- `src`、`tests`、`scripts` 能通过 `compileall`。
- accepted smoke 与 blocked smoke 都能通过 CLI 生成可复查产物。
- Playwright/Chrome 能打开 HTML 报告并生成非空截图。
- 提交前删除临时 `tmp/`、`__pycache__/` 和 `.pytest_cache/`。

## 一句话总结

v80 把 promoted training scale 链路从“能比较 promoted runs”推进到“能选择下一阶段稳定 baseline，并在证据不足时明确阻断”。
