# v86 promoted decision report utility migration 运行截图说明

## 本版主题

v86 继续 `report_utils` 收口路线，把 v80 的 promoted training scale baseline decision 迁移到公共报告工具上。迁移后，baseline decision、seed generation、seed handoff、controlled execution handoff 都开始复用同一套 JSON/CSV 写出、Markdown/HTML 转义和 list/dict 归一化工具。

本版不改变 baseline 选择语义：candidate/rejected 判断、accepted/review/blocked 状态和 selected baseline 选择规则保持不变。

## 关键代码

```text
src/minigpt/promoted_training_scale_decision.py
src/minigpt/report_utils.py
tests/test_promoted_training_scale_decision.py
tests/test_report_utils.py
```

`promoted_training_scale_decision.py` 继续负责读取 promoted comparison、筛选 candidates、记录 rejected runs、选择 baseline 和输出 decision report。变化在于输出和渲染层复用 `report_utils.py`。

## 验证标准

v86 的成功标准是：

- 迁移后 promoted decision 的 accepted/blocked 行为不变。
- report utils 拥有第四个真实消费者。
- focused tests 和 full regression 都通过。
- accepted 与 blocked 两条 smoke 都能产出证据。
- HTML 报告能用 Chrome 打开。
- 文档和截图归档同步更新。

## 一句话总结

v86 把 baseline selector 接入公共报告工具，让 baseline decision 到 seed/handoff 的连续链路共享同一套基础设施。
