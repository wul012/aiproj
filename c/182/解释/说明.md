# v182 运行截图和解释

## 本版主题

v182 是 benchmark scorecard comparison section split。

本版把 cross-scorecard comparison 的 Markdown/HTML section 渲染拆到 `src/minigpt/benchmark_scorecard_comparison_sections.py`，让 `src/minigpt/benchmark_scorecard_comparison_artifacts.py` 回到 JSON/CSV 写入、Markdown/HTML 文件写入和统一输出编排。

## 关键改动

- 新增 `benchmark_scorecard_comparison_sections.py`，集中 `render_benchmark_scorecard_comparison_markdown()`、`render_benchmark_scorecard_comparison_html()` 和相关 section/helper。
- 保留 `benchmark_scorecard_comparison_artifacts.py` 的旧 render 导出，避免旧调用方改 import。
- 保留 `benchmark_scorecard_comparison.py` 的 facade 导出，comparison entrypoint 的外部 API 不变。
- 新增 facade identity 测试，明确证明 artifact module 与 section module 的 render 函数同源。
- 更新 README、`c/README.md` 和项目成熟度阶段代码讲解索引。

## tag 含义

`v182.0.0` 表示 MiniGPT 第 182 个可回看版本：benchmark scorecard comparison 的人类可读 section 渲染已经从 artifact writer 拆出，旧 API 保持兼容，证据归档在 `c/182`。

## 一句话总结

v182 用一次轻量定向拆分降低 benchmark comparison artifact 模块压力，让比较报告链路继续保持可维护、可验证、可回退。
