# v215 promoted seed handoff clean evidence status contract 运行说明

## 本版目标

v215 的目标不是新增一份报告，而是把 v214 的 clean-evidence readiness 状态域固化成稳定契约。

本版明确四个合法状态：

```text
ready
pending-plan
review
incomplete
```

并把它们通过 `seed_handoff_clean_evidence_status_domain` 输出到 summary、CSV、Markdown、HTML 和 CLI。

## 边界

本版不改变 `planned`、`completed`、`blocked`、`failed` 等 handoff 执行状态，不改变 `_handoff_allowed()`，不改变 `--execute` 行为，也不把 clean-evidence status domain 变成阻断策略。

status domain 是 schema evidence，用来帮助后续工具稳定消费 readiness。
