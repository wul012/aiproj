# v195 运行截图和解释

## 本版主题

v195 是 training evaluation timestamp helper consolidation。

v194 拆分了 `training_run_evidence.py`，本版继续做轻量、定向的质量优化：把训练评估链上重复的本地 `utc_now()` 收敛到 `report_utils.utc_now()`。这类重复 helper 和 mini-kv 的 trim/copy 类问题相似，单个函数很小，但散在多个模块里会让后续时间格式调整变得不一致。

## 关键改动

- 更新 `src/minigpt/benchmark_scorecard.py`，移除本地 `utc_now()` 和 `datetime/timezone` 导入，改用 `from .report_utils import utc_now`。
- 更新 `src/minigpt/training_portfolio.py`，移除本地 `utc_now()`，改用 `from minigpt.report_utils import utc_now`。
- 更新 `src/minigpt/training_portfolio_batch.py`，移除本地 `utc_now()`，改用 `from minigpt.report_utils import utc_now`。
- 调用点不变：`generated_at`、`started_at`、`finished_at` 仍然调用 `utc_now()`。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引更新到 v195。

## tag 含义

`v195.0.0` 表示 MiniGPT 第 195 个可回看版本：训练评估链上的三个重复 timestamp helper 已收敛到 `report_utils.utc_now()`，行为不变但工具函数来源更统一。运行截图与解释归档在 `c/195`。

## 一句话总结

v195 把训练评估链上的时间戳 helper 从“各模块自己定义”推进为“共享工具函数”，为后续继续收敛小型重复 helper 打了一个低风险样板。
