# v69 运行截图和解释

## 本版主题

v69 新增 training portfolio batch matrix。它把多个训练 variant 组织成一份 batch plan，每个 variant 仍然复用 v67 的 training portfolio pipeline，最后自动接入 v68 的 training portfolio comparison。

本版也开始执行新的归档规则：v69 及后续版本的运行截图和解释放在 `c/<version>`，`a/` 与 `b/` 作为历史阶段保留。

## 关键命令

目标单测：

```powershell
python -m unittest tests.test_training_portfolio_batch
```

全量回归：

```powershell
python -m unittest discover -s tests
```

编译检查：

```powershell
python -m compileall src tests scripts
```

默认 dry-run smoke：

```powershell
python scripts/run_training_portfolio_batch.py data --out-root tmp/v69-smoke/default-batch
```

自定义 variant smoke：

```powershell
python scripts/run_training_portfolio_batch.py data --variants tmp/v69-smoke/variants.json --baseline context --out-root tmp/v69-smoke/custom-batch
```

Playwright HTML 检查：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp/v69-smoke/default-batch/training_portfolio_batch.html -Out c/69/图片/05-playwright-training-portfolio-batch-html.png
```

## tag 含义

`v69.0.0` 表示项目已经具备训练组合实验矩阵能力。之后跑更大语料或不同模型配置时，可以先声明 variant matrix，再由 batch 自动生成每个 portfolio 和横向 comparison。

## 一句话总结

v69 的证据归档证明：MiniGPT 已经能把多组训练参数统一规划成 batch matrix，并自动产出 per-variant portfolio 与 baseline comparison。
