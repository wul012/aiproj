# v239 test coverage baseline report 运行说明

## 本版主题

v239 增加非阻塞测试覆盖率报告：CI 仍然跑全量 unittest，但入口改为 `scripts/run_test_coverage.py`，同时输出 coverage JSON/CSV/Markdown/HTML。

本版不启用 fail-under 门槛，只记录 baseline。这样后续要做 coverage gate 时，可以基于真实覆盖率走势设置阈值。

## 关键改动

- 新增 `src/minigpt/test_coverage_report.py`：读取 coverage.py JSON，生成 summary、file rows、recommendations 和四类报告。
- 新增 `scripts/run_test_coverage.py`：执行 coverage + unittest，并写出覆盖率证据。
- 更新 `.github/workflows/ci.yml`：Unit tests 步骤改为 coverage report 脚本。
- 更新 `src/minigpt/ci_workflow_hygiene.py` 和 `tests/test_ci_workflow.py`：CI hygiene 规则识别新的测试入口。
- 新增 `tests/test_test_coverage_report.py`：保护 summary、路径规范化、输出写入和 HTML escape。
- 更新 README、代码讲解索引和 `c/README.md`。

## 本版边界

coverage report 是测试治理证据，不等于模型质量证据。

90.14% 是本地 v239 smoke 的观察值。它可以作为后续 coverage gate 的参考 baseline，但本版不应该把它当成硬门槛。
