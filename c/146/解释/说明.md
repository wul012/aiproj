# v146 运行截图说明

## 本版主题

v146 把 v145 新增的 `ci_workflow_hygiene.json` 接入 `project_audit`。v145 解决的是 CI workflow 策略能被独立检查；v146 解决的是这份检查结果能不能进入项目审计证据链。

本版不改变 CI workflow 本身，不改变模型训练、推理、benchmark 或 release gate 硬规则。它只让项目审计可以读取 CI workflow hygiene 报告，并把缺失或失败的 CI hygiene 作为治理 review warning。

## 关键改动

- `src/minigpt/project_audit.py`
  - 新增 `ci_workflow_hygiene_path` 可选输入。
  - 新增 `ci_workflow_hygiene` check。
  - 新增 `ci_workflow_context` 和 summary 字段：`ci_workflow_status`、`ci_workflow_decision`、`ci_workflow_failed_checks`、`ci_workflow_node24_actions`。
- `src/minigpt/project_audit_artifacts.py`
  - Markdown/HTML 摘要展示 CI workflow hygiene 状态。
- `scripts/audit_project.py`
  - 新增 `--ci-workflow-hygiene` 参数，并在 CLI 输出里打印 CI workflow 状态。
- `tests/test_project_audit.py`
  - 新增 CI workflow hygiene pass/fail fixture。
  - 覆盖完整项目 pass、CI hygiene fail 触发 audit warn、输出文档包含 CI workflow hygiene。

## 证据边界

CI workflow hygiene 是项目治理证据，不是模型质量证据。它说明 workflow 的 action/runtime、Python 版本和必需质量步骤没有漂移；它不能证明训练数据更好、生成质量更强或 checkpoint 更成熟。

因此本版把 CI hygiene 失败处理为 audit `warn`，而不是直接判定 project audit `fail`。这和 request history summary 的处理方式一致：治理证据缺失需要 review，但不应混同于模型能力失败。

## tag 含义

`v146.0.0` 标记的是 “project audit CI workflow hygiene context”：CI workflow hygiene report 已经能进入 project audit，并在审计摘要、check 和上下文中体现。

一句话总结：v146 把 CI workflow hygiene 从独立门禁接入项目审计，让 CI 策略成为可审计的治理上下文。
