# v132 training portfolio artifact split 运行说明

## 本版主题

v132 把 `training_portfolio.py` 中的 JSON、Markdown、HTML 写出逻辑拆到 `training_portfolio_artifacts.py`，让原模块继续负责 pipeline plan、dry-run/execute 状态、artifact availability 和 recommendations。

这是一版轻量收口，不改变 training portfolio schema，不改变输出文件名，不改变 dry-run CLI 行为，也不改变 `scripts/run_training_portfolio.py` 的调用方式。

## 关键截图

- `01-unit-tests.png`: training portfolio 定向测试、compile check 和 full unittest discovery。
- `02-maintenance-smoke.png`: maintenance batching/module pressure smoke，确认拆分后模块压力仍为 pass。
- `03-training-portfolio-smoke.png`: 用 `scripts/run_training_portfolio.py` 生成 dry-run JSON/Markdown/HTML，并检查 facade identity、HTML escaping 和输出文件存在。
- `04-playwright-training-portfolio-html.png`: 用 Playwright 打开生成的 training portfolio HTML，确认浏览器页面可渲染。
- `05-docs-check.png`: README、`c/132`、项目成熟度讲解记录和 source encoding hygiene 检查。

## 证据边界

`c/132` 证明的是 training portfolio 的证据发布层拆分后仍能跑通：旧导入路径仍可用，新 artifact 模块能写出同名产物，CLI dry-run 仍产生可审阅的 portfolio 页面。它不证明真实训练质量提升，也不代表已完成生产级训练服务化；模型能力仍需要更大数据、真实 checkpoint 和固定 benchmark 对比来证明。

## 清理规则

本轮临时生成目录位于 `tmp/`，完成验证后删除，只保留 `c/132/图片` 中的最终截图和本说明。
