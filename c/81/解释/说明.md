# v81 promoted training scale next-cycle seed 运行说明

## 本版目标

v81 读取 v80 的 promoted baseline decision，把选出的稳定 baseline 转成下一轮 training scale plan 的启动证据。

本版不训练模型，不重新比较 promoted runs，也不执行 `plan_training_scale.py`。它只检查上游 baseline decision、selected run 和下一轮 corpus sources 是否完整，然后输出下一步命令或阻断原因。

## 运行链路

```text
v75 consolidated training scale workflow
 -> v76 controlled training scale handoff
 -> v77 training scale promotion acceptance
 -> v78 training scale promotion index
 -> v79 promoted training scale comparison
 -> v80 promoted training scale baseline decision
 -> v81 promoted training scale next-cycle seed
```

v81 的 CLI 是：

```powershell
python scripts/build_promoted_training_scale_seed.py runs/training-scale-workflow/promoted-decision data --out-dir runs/training-scale-workflow/promoted-seed --plan-out-dir runs/training-scale-workflow/next-plan
```

输出包括：

```text
promoted_training_scale_seed.json
promoted_training_scale_seed.csv
promoted_training_scale_seed.md
promoted_training_scale_seed.html
```

## 验收标准

- 新单测覆盖 ready、review、blocked、missing source、输出落盘和 HTML 转义。
- 全量 unittest 回归通过。
- `src`、`tests`、`scripts` 能通过 `compileall`。
- ready smoke 能打印 `plan_training_scale.py` 命令。
- blocked smoke 不输出可执行命令，并记录 missing source blocker。
- Playwright/Chrome 能打开 HTML 报告并生成非空截图。
- 提交前删除临时 `tmp/`、`__pycache__/` 和 `.pytest_cache/`。

## 一句话总结

v81 把 promoted baseline decision 继续向下一轮训练规模规划推进，让“选出 baseline”变成“能交接下一轮 corpus 和 plan 命令”的证据。
