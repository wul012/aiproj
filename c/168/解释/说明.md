# v168 promoted training scale seed artifact split 运行说明

## 本版目标

v168 做的是 promoted training scale seed 的 artifact 输出层拆分：把 `promoted_training_scale_seed.py` 中的 JSON、CSV、Markdown、HTML 写出和渲染逻辑移动到 `promoted_training_scale_seed_artifacts.py`。

它解决的是 seed builder 文件继续接近高压线的问题。主模块现在更聚焦 promoted baseline decision、selected run、corpus sources、next-plan command 和 seed status；artifact 模块负责把已经构建好的 seed report 写成可读、可归档的证据文件。

本版不改变 seed schema，不改变 `scripts/build_promoted_training_scale_seed.py`，不改变 next-plan command，也不改变旧的 `minigpt.promoted_training_scale_seed` 导出。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_artifacts.py`：新增 artifact 模块，负责 JSON/CSV/Markdown/HTML 输出。
- `src/minigpt/promoted_training_scale_seed.py`：保留 seed 构建逻辑，并从 artifact 模块 re-export 旧输出函数。
- `tests/test_promoted_training_scale_seed.py`：新增 facade identity 测试，确认旧模块导出的 render/write 函数就是新 artifact 模块函数。
- `README.md`、`c/README.md`、代码讲解索引：登记 v168 能力、tag 和证据位置。
