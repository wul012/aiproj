# v226 eval suite coverage readiness 运行说明

## 本版目标

v226 把固定 prompt eval suite 从“有多少 case”推进到“是否覆盖了足够任务、难度和标签”的 readiness 证据。

它解决的问题是：小型临时 suite 也能生成 eval report，但不应该直接被当成可比较 checkpoint 质量的强证据。

## 边界

本版不改变模型生成逻辑，不改变 prompt suite 加载方式，不改变 CSV/SVG/JSON 基础文件名。

本版新增的是 eval report 的 `coverage` 字段和 HTML 中的 Coverage Readiness 展示。

## 截图说明

- `图片/01-eval-suite-coverage-readiness.png`：使用 `builtin:standard-zh` 生成 eval suite 报告后，通过 Playwright 渲染 HTML，截图证明 coverage 和 comparison readiness 均为 `pass`，并展示 10 类 task、3 类 difficulty、21 个 tag 的覆盖情况。

## 运行产物角色

本版中间产物生成在 `runs/v226-eval-suite-coverage`，用于截图验证 HTML 渲染和 coverage 字段。提交前按清理门禁删除。

`c/226` 只保留最终截图和本说明，作为 v226 的运行证据。

## Tag 含义

`v226.0.0` 表示 eval suite 报告开始内置覆盖度 readiness 和 checkpoint comparison readiness，用来区分“能跑的 prompt 集”和“适合做模型质量对比的 benchmark 集”。

## 一句话总结

v226 把 MiniGPT 的固定 prompt 评估从 case 计数推进到覆盖度 readiness，让后续 checkpoint 质量对比有更清晰的证据边界。
