# v155 server logging split 运行说明

## 本版主题

v155 把 MiniGPT server 的 request-history 事件构造从 `server.py` 拆到 `server_logging.py`。

这是一版运行链路维护收口，不改变 HTTP API，也不改变 `inference_requests.jsonl` 的写入格式。

## 关键改动

- `src/minigpt/server_logging.py`
  - 新增 `build_generation_log_event`。
  - 新增 `build_pair_generation_log_event`。
  - 只构造 dict，不写文件、不操作 HTTP handler。
- `src/minigpt/server.py`
  - `_log_generation` 和 `_log_pair_generation` 改为调用 `server_logging.py`，再通过 `append_inference_log` 写 JSONL。
  - 保留旧 facade 导出，避免旧 import 断裂。
- `tests/test_server_logging.py`
  - 新增纯函数测试和 facade identity 测试。
