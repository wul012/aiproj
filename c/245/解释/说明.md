# v245 release readiness coverage comparison 运行截图说明

## 本版主题

v245 把 coverage gate 证据接入 release readiness comparison。

v244 已经让单个 readiness dashboard 能展示 coverage。v245 让多个 readiness 之间可以比较 coverage percent、coverage gap、coverage panel 状态，并统计 test coverage regression。

## 截图清单

### 01-readiness-comparison-tests.png

该截图来自：

```text
python -B -m unittest tests.test_release_readiness_comparison tests.test_release_readiness -q
```

结果：

```text
Ran 16 tests
OK
```

它证明 comparison 层能读取 coverage 字段、输出 coverage delta，并在 coverage 退步时给出 regression 计数和 recommendation。

### 02-readiness-comparison-coverage-smoke.png

该截图来自：

```text
python -B scripts/compare_release_readiness.py --readiness baseline --readiness current
```

关键结果：

```text
readiness=2
baseline_status=ready
regressed=1
changed_panel_deltas=1
ci_workflow_regressions=0
test_coverage_regressions=1
```

它证明 CLI 输出已经能直接暴露 test coverage regression。

### 03-readiness-comparison-markdown.png

该截图来自 comparison Markdown 的关键行。

关键结果：

```text
Test coverage regressions = 1
Coverage % delta = -14.17
Coverage gap delta = 4
test_coverage:pass->warn
```

它证明人读报告里能看到 coverage 的具体退步幅度，而不只是看到 readiness 从 ready 变 review。

### 04-full-unittest.png

该截图来自：

```text
python -B -m unittest discover -s tests -q
```

结果：

```text
Ran 482 tests
OK
```

它证明 v245 的 comparison 改动没有破坏全项目测试集合。

## 证据边界

v245 证明的是 readiness comparison 能识别 coverage regression。

它不证明模型能力变差，也不替代 benchmark scorecard 或 generation quality。coverage regression 只说明测试治理信号退步。

## 一句话总结

v245 让发布看板对比能捕捉 coverage 退步：覆盖率下降、gap 增大、coverage panel 变 warn 都会进入 comparison 的机器可读和人读证据。
