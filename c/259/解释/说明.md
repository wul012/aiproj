# v259 promoted decision batch review propagation 运行说明

## 本版主题

v259 把 v258 promoted comparison 中的 handoff selected batch review/blocker context 继续传到 promoted training scale baseline decision。这样 baseline selector 不只显示选中了哪个 promoted run，也会保留这个 selected baseline 是否仍带 selected batch review/blocker 风险。

本版不改变 baseline decision 的 `accepted`、`review`、`blocked` 判定：batch review/blocker context 是风险提示和 recommendation，不会自动让一个 otherwise accepted 的 baseline 变成 blocked。

## 关键改动

- `src/minigpt/promoted_training_scale_decision.py`：promoted row 接住 handoff batch review 字段，summary 写入 selected baseline 的 batch review/blocker 状态，并保留 comparison-ready review/blocker totals 和 blocker reasons。
- `scripts/decide_promoted_training_scale_baseline.py`：CLI 输出新增 selected handoff batch review/blocker 字段和 comparison-ready batch blocker reasons，方便 CI 日志直接判断风险是否传到 decision 层。
- `tests/test_promoted_training_scale_decision.py`：新增 promoted comparison -> decision 的 batch review propagation 测试，确认选中 `beta` 时风险字段仍进入 JSON/CSV/Markdown/HTML/CLI/recommendations。
- `README.md` 和 `代码讲解记录_项目成熟度阶段/273-v259-promoted-decision-batch-review-propagation.md`：同步说明本版目标、边界、输入输出和证据链角色。

## tag 含义

`v259.0.0` 表示 promoted baseline decision 已经接住 promoted-comparison handoff selected batch review/blocker context，并在选 baseline 阶段保持风险可见。

## 一句话总结

v259 让 promoted baseline decision 从“只选择下一轮 baseline”推进到“选择 baseline 时保留 selected handoff batch review/blocker 风险”。
