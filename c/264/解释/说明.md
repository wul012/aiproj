# v264 运行截图和解释

## 本版主题

v264 继续沿 promoted decision -> promoted seed -> seed handoff 这条链路做结构收束。v262、v263 已经分别拆出 seed handoff review helper 和 promoted seed review helper；本版把 promoted baseline decision 中的 selected handoff suite/batch-review summary projection 和 batch-review recommendations 拆到 `promoted_training_scale_decision_review.py`。

## 明确边界

本版不改变 promoted decision 的 baseline selection、`decision_status` 判定、CSV/Markdown/HTML/JSON 字段或 CLI 输出。它只是把 handoff review context 的汇总和建议生成从主 builder 中移走。`promoted_training_scale_decision.py` 仍然还有 600 行，主要因为 artifacts 渲染仍在同一文件里，后续如果继续收口，可以优先拆 artifact writer。

## 关键运行截图

- `01-focused-tests.png`：运行 promoted decision review helper 和 promoted decision 主流程测试。
- `02-contract-smoke.png`：构建带 handoff batch-review context 的 promoted decision report，检查 decision status、selected batch review、derived totals、recommendation 和文件行数。
- `03-full-unittest.png`：全量 unittest 结果。
- `04-source-encoding.png`：源码编码、BOM 和语法检查。
- `05-docs-check.png`：README、`c/264`、代码讲解索引、源码和测试关键词一致性检查。

## 本版结论

v264 把 promoted decision 的 selected handoff review helper 逻辑抽出，主文件从 716 行降到 600 行，输出契约不变；剩余压力主要来自 artifact rendering，可以留给后续更明确的一版。
