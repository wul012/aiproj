# v269 运行说明

## 本版主题

v269 把 v268 的 clean batch-review gate 从 standalone training-scale run decision 接入 consolidated training-scale workflow。默认 workflow 仍然保持兼容；只有显式传入 `decision_require_clean_batch_review=True` 或 CLI `--decision-require-clean-batch-review` 时，workflow 才会把 batch review actions、blocker actions 或 maturity coverage regressions 作为自动化阻断条件。

## 关键改动

- `run_training_scale_workflow()` 新增 `decision_require_clean_batch_review` 参数，并把它传入 nested run decision。
- `scripts/run_training_scale_workflow.py` 新增 `--decision-require-clean-batch-review`，并在命令输出里打印 `clean_batch_review_status`。
- workflow JSON/CSV/Markdown/HTML 都展示 `decision_require_clean_batch_review` 和 `clean_batch_review_status`。
- 单测覆盖默认 review 行为、strict workflow 阻断、nested decision artifact 字段和 artifact 渲染字段。

## Tag 含义

`v269.0.0` 标记 training scale workflow clean batch review gate。它表示 clean batch-review requirement 已经从 standalone decision 扩展到一键 workflow 入口，可以用于更严格的自动化执行前置检查。

## 一句话总结

v269 让一键训练规模治理 workflow 也能拒绝 review/blocker/coverage-regression execute evidence，把 v268 的决策层 gate 补成了入口层闭环。
