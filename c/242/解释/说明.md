# v242 release bundle coverage gate context 运行截图说明

## 本版主题

v242 把 v241 已经进入 project audit 的 coverage gate 结果继续带进 release bundle。

这版不改 coverage 阈值，不改变 release gate 语义，也不声称 coverage pass 等于模型质量 pass。它只让 release bundle 在发布交接时能直接展示 coverage gate 的状态、覆盖率、阈值、缺口和原始报告路径。

## 截图清单

### 01-release-bundle-tests.png

该截图来自：

```text
python -B -m unittest tests.test_release_bundle tests.test_project_audit -q
```

结果：

```text
Ran 16 tests
OK
```

它证明 v242 的 release bundle 改动没有破坏 v241 的 project audit coverage context，同时新增的 bundle summary、context、artifact key、Markdown 输出都被单测覆盖。

### 02-coverage-gate-smoke.png

该截图来自：

```text
python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-v242-smoke --fail-under 80
```

关键结果：

```text
Ran 476 tests
OK
status=pass
decision=continue_with_coverage_gate
line_coverage_percent=90.15
fail_under=80.0
coverage_gap=0.0
```

它证明 coverage report 是真实从当前测试套件生成的，不是手写 fixture。

### 03-release-bundle-coverage-smoke.png

该截图来自：

```text
python -B scripts/build_release_bundle.py ... --test-coverage-report runs/test-coverage-v242-smoke/test_coverage_report.json
```

关键结果：

```text

```

它证明 release bundle 能消费 coverage report，并把 coverage gate 字段输出到 CLI summary。

对应 `release_bundle.json` 中还包含：

```text
inputs.test_coverage_report_path
test_coverage_context
artifacts.test_coverage_report_json
artifacts.test_coverage_report_md
artifacts.test_coverage_report_html
```

这说明 coverage gate 不只是显示在终端里，也进入了机器可读 bundle 和人读 Markdown/HTML 报告。

### 04-full-unittest.png

该截图来自：

```text
python -B -m unittest discover -s tests -q
```

结果：

```text
Ran 476 tests
OK
```

它证明 v242 的改动没有破坏项目当前测试集合。

## 证据边界

v242 的 evidence 证明的是 release bundle 可以携带 coverage gate context。

它不证明模型能力提升，也不证明训练数据质量提升。模型质量仍然需要标准 benchmark、generation quality、scorecard 和真实 checkpoint 对比继续验证。

## 一句话总结

v242 让发布交接包从“能看到 audit 状态”推进到“能看到 audit 依赖的 coverage gate 状态和报告路径”，使覆盖率门禁成为 release bundle 里的可追溯证据。
