# v170 release readiness comparison artifact split 运行说明

## 本版主题

v170 做的是 release readiness comparison 的 artifact 输出层拆分：把 `release_readiness_comparison.py` 中的 JSON、CSV、delta CSV、Markdown、HTML 写出和渲染逻辑移动到 `release_readiness_comparison_artifacts.py`。

这版不改变 comparison schema、baseline override、status/panel/CI workflow delta logic、registry consumption，也不改变旧 `minigpt.release_readiness_comparison` 的导出入口。

## 关键改动

- `src/minigpt/release_readiness_comparison_artifacts.py`：新增 artifact 模块，负责 JSON/CSV/delta CSV/Markdown/HTML 输出。
- `src/minigpt/release_readiness_comparison.py`：保留 readiness report 读取、baseline 排序、row/delta/summary/recommendations 生成，并 re-export artifact 函数。
- `tests/test_release_readiness_comparison.py`：新增 facade identity 测试，证明旧模块导出仍指向新 artifact 模块函数。
- `README.md`、`c/README.md`、代码讲解索引：登记 v170 能力、tag 和证据位置。
