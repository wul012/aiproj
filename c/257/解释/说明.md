# v257 promotion index batch review propagation 运行说明

## 本版主题

v257 把 v256 training scale promotion 中的 handoff selected batch review/blocker context 继续传到 training scale promotion index。这样多个 promotion reports 汇总成 promoted-only compare inputs 时，index 不只告诉读者哪些 run 可以比较，也会保留这些 promoted run 是否仍带着 selected batch review/blocker 风险。

本版不改变 compare-ready 规则：promotion status 为 `promoted` 且 training scale run JSON 存在的报告仍进入 `comparison_inputs`。selected batch review/blocker 只是风险上下文和 recommendation，不自动降级 promotion status。

## 关键改动

- `src/minigpt/training_scale_promotion_index_helpers.py`：index row 接收 promotion summary 中的 handoff batch review 字段，并在 summary 中聚合 selected review/blocker counts、action totals 和 blocker reasons。
- `src/minigpt/training_scale_promotion_index.py`：CSV/Markdown/HTML 输出新增 batch review/blocker 字段和统计卡片。
- `scripts/index_training_scale_promotions.py`：CLI 输出新增 handoff batch review/blocker totals，方便 CI 日志直接检查。
- `tests/test_training_scale_promotion_index.py`：新增 promoted reports 带 blocker/review batch context 的索引测试，确认 compare-ready 语义不变但风险字段可见。
- `README.md` 和 `代码讲解记录_项目成熟度阶段/271-v257-promotion-index-batch-review-propagation.md`：同步说明本版边界、输入输出和证据链角色。

## tag 含义

`v257.0.0` 表示 promotion index 已经接住 promotion-level handoff selected batch review/blocker context，并在 promoted-only compare 选择前保持风险可见。

## 一句话总结

v257 让 promotion index 从“只聚合 promoted 输入”推进到“聚合 promoted 输入时保留上游 selected batch review/blocker 风险”。
