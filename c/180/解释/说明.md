# v180 training scale workflow artifact split 运行说明

## 本版目标

v180 把 `training_scale_workflow.py` 里的 JSON、CSV、Markdown 和 HTML 输出函数拆到 `training_scale_workflow_artifacts.py`。上一版 v179 已经把 maturity narrative 的 summary/section 边界收束，本版继续处理 module pressure 报告暴露出的 training-scale 工作流模块体量问题。

本版明确不改 `run_training_scale_workflow()` 的参数、返回 JSON schema、嵌套 `plan/`、`runs/<profile>/`、`comparison/`、`decision/` 产物结构、CLI 用法和旧的 `minigpt.training_scale_workflow` 导入路径。

## 关键改动

- `src/minigpt/training_scale_workflow_artifacts.py`：新增 artifact 模块，负责 `training_scale_workflow.json`、CSV、Markdown 和 HTML 的写出与渲染。
- `src/minigpt/training_scale_workflow.py`：保留计划生成、profile run 编排、comparison、decision、summary 和 recommendation 逻辑，并 re-export 旧 writer/renderer 名称。
- `tests/test_training_scale_workflow.py`：新增 facade identity 断言，确认旧入口导出的 artifact 函数就是新模块里的实现。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/README.md`：更新 v180 版本说明、证据索引和讲解索引。

## 运行验证

```powershell
python -B -m unittest tests.test_training_scale_workflow -v
python -B scripts/check_maintenance_batching.py --out-dir runs/maintenance-v180
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v180
python -B -m unittest discover -s tests -v
```

## tag 含义

`v180.0.0` 表示 MiniGPT 的 training scale workflow artifact split 版本。这个 tag 对应的能力是：训练规模工作流继续保持原有编排、产物 schema 和 CLI 契约，但 JSON/CSV/Markdown/HTML 输出层被拆到更清晰的 artifact 边界。

一句话总结：v180 把 training scale workflow 的“执行编排”和“报告输出”分开，让训练规模工作流主入口继续可维护，同时保持所有旧导入和产物格式不变。
