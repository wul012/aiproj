# v176 benchmark scorecard decision artifact split 运行说明

## 本版目标

v176 把 `benchmark_scorecard_decision.py` 里的 JSON、CSV、Markdown 和 HTML 产物写出拆到 `benchmark_scorecard_decision_artifacts.py`。这一版解决的是 benchmark scorecard promotion decision 模块同时承担候选选择语义和证据渲染的问题。

本版明确不改 scorecard comparison 加载、candidate evaluation、baseline 排除、rubric threshold、generation-quality review item、selected-run ranking、decision status、recommendations、`scripts/build_benchmark_scorecard_decision.py` 用法和旧的 `minigpt.benchmark_scorecard_decision` facade 导出。

## 关键改动

- `src/minigpt/benchmark_scorecard_decision_artifacts.py`：新增 artifact 模块，负责 `write_benchmark_scorecard_decision_json()`、`write_benchmark_scorecard_decision_csv()`、`render_benchmark_scorecard_decision_markdown()`、`render_benchmark_scorecard_decision_html()` 和 `write_benchmark_scorecard_decision_outputs()`。
- `src/minigpt/benchmark_scorecard_decision.py`：保留 comparison 读取、case delta 统计、candidate evaluation、candidate selection、decision status、summary 和 recommendations，并从新 artifact 模块 re-export 旧 writer/render 入口。
- `tests/test_benchmark_scorecard_decision.py`：新增 facade identity 断言，确认旧模块和新 artifact 模块指向同一组函数对象。
- `README.md`、`c/README.md` 和项目成熟度阶段讲解索引：记录 v176 的版本能力、证据位置和讲解入口。

## 运行验证


## tag 含义

`v176.0.0` 表示 MiniGPT 的 benchmark scorecard decision artifact split 版本。这个 tag 对应的能力是：scorecard promotion decision 语义仍由主模块生成，而多格式发布产物由独立 artifact 模块负责。

一句话总结：v176 把 benchmark scorecard promotion decision 的“候选决策”和“证据发布”分开，让后续调整报告样式不再挤占候选选择逻辑。
