# v152 运行截图说明

## 本版主题

v152 是 CI workflow hygiene 的轻量质量优化。
本版给内部 check/action/summary/report 数据结构补 `TypedDict`，并把 action major 解析从只支持 `v6` 扩展为支持 `v6`、`v6.0.0`、`6`。

这不是新的治理层，也不改变 release gate 或 maturity summary 的规则。

## 关键改动

- `src/minigpt/ci_workflow_hygiene.py`
  - 新增 `CheckStatus = Literal["pass", "fail"]`。
  - 新增 `CiWorkflowCheck`、`CiWorkflowAction`、`CiWorkflowSummary`、`CiWorkflowReport`。
  - `build_ci_workflow_hygiene_report()` 返回 `CiWorkflowReport`。
  - `_check()`、`_collect_actions()`、`_build_checks()` 使用更明确的内部类型。
  - `_major()` 改用 `_ACTION_MAJOR_RE`，支持 `v6`、`v6.0.0`、`6`。
- `tests/test_ci_workflow.py`
  - 新增 action major 解析测试。
  - 验证 semver/bare major 能被识别为 Node 24 native major。
  - 同时确认精确 action tag policy 仍然会提示修正。

## 证据边界

`c/152` 证明 CI workflow hygiene 的内部 schema 和 action version 解析更稳。
它不证明模型能力提升，也不表示新增了发布门禁。

## tag 含义

`v152.0.0` 标记的是 “CI workflow hygiene typed schema”：CI workflow hygiene 的固定报告结构获得局部类型约束，并加强 action major 解析。

一句话总结：v152 是一次低风险质量收口，让 CI workflow hygiene 源头更易维护、更少受 action tag 格式变化影响。
