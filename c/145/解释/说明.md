# v145 运行截图说明

## 本版主题

v145 把 v144 的 GitHub Actions Node 24 native action 策略做成可运行的 CI workflow hygiene gate。v144 已经把 `actions/checkout` 和 `actions/setup-python` 升到 v6，并删除 `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`；v145 的重点是防止以后 workflow 漂移，把这条策略写成可单独运行、可在 CI 中失败、可输出 JSON/CSV/Markdown/HTML 证据的检查器。

本版不改变模型、训练、推理服务或 benchmark 评分。它只加强 CI 维护证据链。

## 关键改动

- 新增 `src/minigpt/ci_workflow_hygiene.py`
  - 读取 `.github/workflows/ci.yml`。
  - 检查 `actions/checkout@v6`、`actions/setup-python@v6`、禁止 `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`、Python 3.11、source encoding gate、CI workflow hygiene gate、unittest discovery。
  - 输出 `summary`、`actions`、`checks`、`recommendations`。
- 新增 `scripts/check_ci_workflow_hygiene.py`
  - 写出 `ci_workflow_hygiene.json`、`.csv`、`.md`、`.html`。
  - 默认检查失败时返回非零退出码。
- 更新 `.github/workflows/ci.yml`
  - 在 source encoding gate 后新增 `CI workflow hygiene check`。
- 更新 `tests/test_ci_workflow.py`
  - 保留 v144 的 action major 守护。
  - 新增当前 workflow pass、旧 force-runtime workflow fail、输出文件渲染测试。

## 证据边界

这些截图证明 v145 的 CI workflow 策略已经从“测试里检查字符串”升级为“可运行门禁 + CI 步骤 + 报告证据”。它不证明模型能力提升，也不代表 GitHub Actions 以后不会发布新的 action major。未来如果 action metadata 再变化，应继续先核对 metadata，再调整 workflow 和 hygiene policy。

## tag 含义

`v145.0.0` 标记的是 “CI workflow hygiene report”：CI workflow 的 Node 24 native action 策略、Python 版本和必需质量步骤已经由独立 gate 守护。

一句话总结：v145 让 CI workflow 自己进入治理证据链，减少以后 action/runtime 策略悄悄漂移的风险。
