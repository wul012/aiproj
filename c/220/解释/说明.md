# v220 promoted seed handoff clean evidence recommendation actions 运行说明

## 本版目标

v220 把 `require_clean_evidence=True` 的 pass/fail 结果写入 seed handoff recommendations。

v219 已经让 `clean_evidence_requirement` 成为库级契约；v220 继续补一层人和自动化都容易读的 action：

```text
Clean-evidence requirement passed
Clean-evidence requirement failed
```

## 边界

本版不改变默认 `require_clean_evidence=False` 的 recommendation 输出。

本版不改变 `--require-clean-evidence` 的退出码，不改变 suite alignment、readiness 或 artifact schema 的核心字段。

它只在显式要求 clean evidence 时，把 gate 结果补进 `recommendations`。
