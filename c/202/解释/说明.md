# v202 training scale suite consistency evidence 运行说明

## 本版目标

v202 把 v198-v201 逐步传递的 benchmark suite 选择，补成训练规模比较和 promoted 比较里的显式一致性证据。

此前 `standard-zh` 已经能从 portfolio、batch、scale plan、workflow 进入 promoted seed。但当两个 training scale run 或 promoted run 被拿来比较时，报告只显示 readiness、gate、batch 等结果，没有直接说明这些 run 是否使用同一套 benchmark suite。这样容易把“评测基准不同造成的差异”误读成“模型质量干净提升”。

本版新增 `suite_consistency`、`suite_paths`、`suite_mismatch_count`、`suite_relation` 和 selected suite path，让 run comparison、run decision、promoted comparison、promoted decision 都能明确回答：这次比较是同套 suite、混合 suite，还是缺失 suite 信息。

## 关键改动

- `src/minigpt/training_scale_run_comparison.py`
  - run summary 读取 `scale_plan_summary` / `batch_summary` 的 `suite_mode`、`suite_name`、`suite_path`。
  - comparison summary 输出 `suite_consistency`、`baseline_suite_path`、`suite_paths`、`suite_mismatch_count`。
  - baseline delta 输出 `suite_relation`，suite 变化时 explanation 写出变化路径。
- `src/minigpt/training_scale_run_decision.py`
  - decision summary 继承 comparison summary 的 suite 一致性。
  - selected run 带出 `selected_suite_path`。
  - mixed suite 时给出“这只是执行 triage，不是干净模型质量 delta”的建议。
- `src/minigpt/training_scale_run_decision_artifacts.py`
  - CSV、Markdown、HTML 显示 selected suite 和 suite consistency。
- `src/minigpt/promoted_training_scale_comparison.py`
  - promoted rows 带出 run 的 `suite_path`。
  - promoted summary 继承内层 comparison 的 suite 一致性字段。
- `src/minigpt/promoted_training_scale_comparison_artifacts.py`
  - CSV、Markdown、HTML 显示 promoted comparison 的 suite path 和 consistency。
- `src/minigpt/promoted_training_scale_decision.py`
  - promoted baseline decision summary 输出 `selected_suite_path` 与 suite consistency。
  - accepted 决策建议把 selected suite 带入下一轮 seed。
- `tests/test_training_scale_run_comparison.py`
  - 新增 default suite 与 `builtin:standard-zh` 混合比较测试。
- `tests/test_training_scale_run_decision.py`
  - 覆盖 decision 不吞掉 mixed suite 风险提示。
- `tests/test_promoted_training_scale_comparison.py`
  - 覆盖 promoted comparison 把 mixed suite 传到 summary 和 recommendations。
- `tests/test_promoted_training_scale_decision.py`
  - 覆盖 promoted decision 记录 `selected_suite_path=builtin:standard-zh`。

## 证据边界

本版不新增 prompt，不训练模型，不改变 readiness 计算，不改变 run 或 baseline 的选择策略。

它只补足比较证据的公平性解释：当 suite 混用时，报告仍然可以服务于执行治理和人工 triage，但不能被当作干净的模型质量提升证明。

## tag 含义

`v202.0.0` 表示 MiniGPT v202 training scale suite consistency evidence：训练规模比较、决策和 promoted baseline 链路已经能显式记录 benchmark suite 一致性。

一句话总结：v202 让训练规模链路从“能携带标准 suite”进一步变成“比较时能说明 suite 是否一致”。
