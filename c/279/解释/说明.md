# v279 promoted seed handoff automation decision contract

## 本版主题

v279 在 v278 的 `automation_gate` 上继续推进，把 gate 从“状态摘要”升级成更适合自动化直接消费的决策契约。

v278 已经能汇总 clean-evidence 和 clean batch-review 两个 strict requirement，并在失败前打印完整诊断。v279 不新增用户开关，也不改变默认 handoff 行为；它只给现有 `automation_gate` 增加机器可读字段：

- `decision`
- `exit_code`
- `required_requirement_count`
- `passed_requirement_count`
- `failed_requirement_count`
- `blocking_requirement_count`
- `decision_domain`

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_review.py`
  - 新增 `SeedHandoffAutomationGateDecision`。
  - 新增 `SEED_HANDOFF_AUTOMATION_GATE_DECISIONS`。
  - `build_seed_handoff_automation_gate()` 现在同时输出 status 和 decision。

- `scripts/execute_promoted_training_scale_seed.py`
  - strict 模式打印 `automation_gate_decision`、`automation_gate_exit_code` 和 requirement 计数。
  - CLI 退出逻辑改为读取 `automation_gate.decision == "stop"` 和 `automation_gate.exit_code`。

- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - CSV、Markdown、HTML 都新增 automation decision 和计数字段。

- `tests/test_promoted_training_scale_seed_handoff.py`
  - 覆盖 decision domain、exit code、required/pass/fail/blocking count、CLI 输出和 artifact 字段。

## Tag 含义

`v279.0.0` 表示：promoted seed handoff automation decision contract。它说明 seed handoff 的自动化总 gate 不仅能判断 pass/fail，还能直接告诉脚本是否继续、用什么退出码退出、哪些 requirement 阻塞了流程。

## 一句话总结

v279 让 promoted seed handoff 的 `automation_gate` 从“可读状态”升级为“可执行决策契约”，更适合 CI、脚本和后续 orchestration 消费。
