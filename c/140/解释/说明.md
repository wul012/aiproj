# v140 benchmark scorecard promotion decision 运行说明

## 本版主题

v140 把 v139 的 benchmark scorecard comparison 继续推进到轻量决策层。

本版新增 `benchmark_scorecard_decision.py`：读取 `benchmark_scorecard_comparison.json`，检查非 baseline candidate 的 rubric/overall delta、case regression、generation-quality flag 增减、dominant flag 变化和 worst generation case 变化，然后输出 `promote`、`review` 或 `blocked`。

本版不证明模型质量提升，也不替代 release gate；它只是把 scorecard comparison 的证据变成更明确的 benchmark promotion review 入口。

## 关键改动

- `src/minigpt/benchmark_scorecard_decision.py`：新增决策模块，输出 JSON/CSV/Markdown/HTML。
- `scripts/build_benchmark_scorecard_decision.py`：新增 CLI，可从 comparison JSON 或目录生成 decision 产物。
- `tests/test_benchmark_scorecard_decision.py`：覆盖 blocked、promote、目录加载、渲染转义和空 comparison 拒绝。
- `src/minigpt/__init__.py`：导出 decision 构建与输出函数。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/README.md`：同步 v140 版本说明、证据入口和讲解索引。

## 证据边界

`c/140` 证明的是 scorecard comparison 的结果可以进入轻量 promotion decision。

决策输出里的 `promote` 只表示“相对 baseline 的 benchmark scorecard 证据可提升”，不表示模型已经生产可用，也不表示训练策略必然优于 baseline。
