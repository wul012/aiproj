# v130 experiment card artifact split 运行说明

## 本版主题

v130 把 `experiment_card.py` 中的 JSON、Markdown、HTML 写出逻辑拆到 `experiment_card_artifacts.py`，让原模块继续负责读取 run 目录、registry、manifest、dataset quality 和 eval suite，并组装 experiment card 数据。

这是一版轻量收口，不改变 experiment card schema，不改变输出文件名，也不改变 `scripts/build_experiment_card.py` 的调用方式。v129 已经把 `training_portfolio_batch.py` 的输出层拆到 `training_portfolio_batch_artifacts.py`，本轮说明把 v129/v130 作为相邻 artifact split 一起记录，避免为小拆分制造过多文档碎片。

## 关键截图

- `01-unit-tests.png`: experiment card 定向测试、训练组合批处理相关回归、全量 unittest discovery 结果。
- `02-maintenance-smoke.png`: maintenance batching/module pressure smoke，确认当前没有 warn/critical 模块。
- `03-experiment-card-smoke.png`: 用示例 run 生成 experiment card JSON/Markdown/HTML，并检查 facade identity、HTML escaping 和输出文件存在。
- `04-playwright-experiment-card-html.png`: 用 Playwright 打开生成的 experiment card HTML，确认浏览器页面可渲染。
- `05-docs-check.png`: README、`c/130`、代码讲解记录和 source encoding hygiene 检查。

## 证据边界

`c/130` 里的截图证明的是本地 artifact split 的运行闭环：旧导入路径仍可用，新 artifact 模块能写出同名产物，HTML 能被浏览器打开。它不证明模型质量提升，也不代表生产级训练能力；模型质量仍需要更大的数据、固定 benchmark 和真实 checkpoint 对比来证明。

## 清理规则

本轮临时生成目录位于 `tmp/`，完成验证后删除，只保留 `c/130/图片` 中的最终截图和本说明。
