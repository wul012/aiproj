# v95 promoted training scale comparison report utility migration 运行截图说明

## 本版主题

v95 继续 `report_utils` 收口路线，把 v79 的 `promoted_training_scale_comparison.py` 迁移到公共报告工具层。
本版只回收 promoted comparison 里的重复 JSON 写出、Markdown/HTML 转义和 list/dict 归一化 helper，不改变 promoted-only comparison 的筛选和阻断规则。

迁移后，training scale promotion acceptance、promotion index 和 promoted comparison 三层证据开始共享同一套报告基础设施。

## 关键代码

```text
src/minigpt/promoted_training_scale_comparison.py
src/minigpt/report_utils.py
src/minigpt/training_scale_run_comparison.py
tests/test_promoted_training_scale_comparison.py
tests/test_report_utils.py
scripts/compare_promoted_training_scale_runs.py
```

`promoted_training_scale_comparison.py` 仍然负责读取 promotion index，只把 promoted inputs 送入基础 training scale run comparison，并输出 JSON/CSV/Markdown/HTML 证据。

## 验证标准

v95 的成功标准是：

- promoted comparison 私有 `utc_now`、`_dict`、`_list_of_dicts`、`_string_list`、`_md`、`_e` 被公共 `report_utils` 替代。
- JSON writer 使用 `write_json_payload`。
- 两个 promoted inputs 仍然能进入比较。
- review/blocked inputs 仍然不能进入比较。
- promoted inputs 不足时仍然 blocked。
- HTML 报告能被真实 Chrome 打开。
- README、代码讲解和截图归档同步到 v95。
