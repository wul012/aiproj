# v162 model card artifact split 运行说明

## 本版主题

v162 把 model card 的 JSON、Markdown 和 HTML 写出/渲染逻辑，从 `src/minigpt/model_card.py` 拆到 `src/minigpt/model_card_artifacts.py`。

这是一版 contract-preserving refactor。`build_model_card()` 的 card schema 不变，`scripts/build_model_card.py` 的调用方式不变，旧模块里的 writer/renderer 导出也保持兼容。

## 关键改动

- 新增 `src/minigpt/model_card_artifacts.py`，集中保存 model card 的 artifact 写出、Markdown/HTML 渲染、run table、链接和页面样式。
- 更新 `src/minigpt/model_card.py`，保留 registry/experiment card 读取、summary、coverage、top runs、recommendation 和 warning 构建，导入并继续导出 artifact 函数。
- 更新 `tests/test_model_card.py`，增加 facade identity 测试，确认旧导出指向新 artifact 实现。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引同步到 v162。

v162 让 model card 的“项目级模型治理证据汇总”和“报告发布”分开，降低当前最大模块压力，同时保持旧入口稳定。
