# v203 strict suite consistency decision guard 运行说明

## 本版目标

v203 把 v202 的 suite-consistency evidence 变成可选的严格决策护栏。

v202 已经能说明两个 training-scale run 或 promoted run 是否来自同一套 benchmark suite；但它默认只做提示，不阻断 mixed suite。这个默认是合理的，因为 mixed suite 仍然能用于治理 triage：看 gate、batch、readiness、promotion 是否完整。但如果本次决策要服务“干净模型质量对比”，就需要一个明确开关让 mixed/missing suite 直接阻断。

本版新增 `--require-suite-consistency`。默认不改变兼容行为；显式启用后，suite consistency 不是 `consistent` 的 comparison 会拒绝候选，并在 rejected reasons 中写出 `benchmark suite consistency is mixed` 或 `missing`。

## 关键改动

- `src/minigpt/training_scale_run_decision.py`
  - 新增 `require_suite_consistency` 参数。
  - strict mode 下把 suite consistency 问题加入每个候选的 rejection reasons。
  - summary 输出 `require_suite_consistency`。
  - recommendations 增加 fix suite consistency 提示。
- `src/minigpt/training_scale_run_decision_artifacts.py`
  - CSV、Markdown、HTML 显示 strict guard 是否启用。
- `src/minigpt/promoted_training_scale_decision.py`
  - promoted baseline decision 同样支持 `require_suite_consistency`。
  - 默认 mixed-suite accepted/review 仍给治理 triage 提示。
  - strict mode 下阻断 promoted baseline 选择。
- `scripts/decide_training_scale_run.py`
  - 新增 `--require-suite-consistency`。
- `scripts/decide_promoted_training_scale_baseline.py`
  - 新增 `--require-suite-consistency`。
- `tests/test_training_scale_run_decision.py`
  - 覆盖 mixed suite 默认不阻断、strict mode 阻断。
- `tests/test_promoted_training_scale_decision.py`
  - 覆盖 promoted mixed suite 默认可 accepted、strict mode 阻断。

## 证据边界

本版不改变 comparison 的 suite 计算，不改变 readiness 排序，不改变默认 decision 兼容行为，也不训练模型。

它只提供一个严格开关：当用户或自动流程需要“同一套 benchmark suite 下的干净模型质量比较”时，可以用 strict guard 阻断 mixed/missing suite。普通治理复盘仍可继续读取 mixed-suite 报告。

## tag 含义

`v203.0.0` 表示 MiniGPT v203 strict suite consistency decision guard：训练规模和 promoted baseline 决策已经能按需阻断 mixed/missing suite comparison。
