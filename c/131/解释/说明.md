# v131 project audit artifact split 运行说明

## 本版主题

v131 把 `project_audit.py` 中的 JSON、Markdown、HTML 写出逻辑拆到 `project_audit_artifacts.py`，让原模块继续负责读取 registry、model card、request history summary，并计算项目审计 checks、summary 和 recommendations。

这是一版轻量收口，不改变 project audit schema，不改变 readiness scoring，不改变输出文件名，也不改变 `scripts/audit_project.py` 的调用方式。

## 关键截图

- `01-unit-tests.png`: project audit 定向测试、compile check 和全量 unittest discovery。
- `02-maintenance-smoke.png`: maintenance batching/module pressure smoke，确认拆分后模块压力仍为 pass。
- `03-project-audit-smoke.png`: 用示例 registry/model card/request history 生成 project audit JSON/Markdown/HTML，并检查 facade identity、HTML escaping 和输出文件存在。
- `04-playwright-project-audit-html.png`: 用 Playwright 打开生成的 project audit HTML，确认浏览器页面可渲染。
- `05-docs-check.png`: README、`c/131`、代码讲解记录和 source encoding hygiene 检查。

## 证据边界

`c/131` 证明 project audit 的输出层拆分后仍能跑通，并且旧导入路径仍指向新 artifact 实现。它不证明模型质量提升，也不代表 project audit 本身等同生产发布审计；它证明的是本地 AI 工程治理证据链继续保持可运行、可阅读、可回归。

## 清理规则

本轮临时生成目录位于 `tmp/`，完成验证后删除，只保留 `c/131/图片` 中的最终截图和本说明。
