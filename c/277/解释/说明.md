# v277 promoted seed handoff clean batch requirement

## 本版目标

v277 在 v276 的基础上，把 promoted seed handoff 的 clean batch-review evidence 从“可见”推进到“可选机器门禁”。

v276 已经让 seed handoff 报告显示 selected handoff 是否要求 clean batch review、状态是否为 `clean`、以及 clean/unclean promoted inputs 的数量。v277 增加 `clean_batch_review_requirement`，让自动化可以显式要求这条证据必须干净。

本版默认行为不变：不传 `--require-clean-batch-review` 时，seed handoff 仍然只做验证/执行和记录；只有显式传参时，dirty clean-required handoff 才会让 CLI 非零退出。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_review.py`
  - 新增 `SeedHandoffCleanBatchReviewRequirement` 和 `SEED_HANDOFF_CLEAN_BATCH_REVIEW_REQUIREMENT_STATUSES`。
  - 新增 `build_seed_handoff_clean_batch_review_requirement()`，输出 `not-required/pass/fail` 状态、selected status、detail 和 status domain。
  - 新增 requirement-aware recommendation。
- `src/minigpt/promoted_training_scale_seed_handoff.py`
  - `build_promoted_training_scale_seed_handoff()` 新增 `require_clean_batch_review` 参数。
  - 报告中写入 `clean_batch_review_requirement`。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - CSV/Markdown/HTML 展示 clean batch-review gate status、detail 和 status domain。
- `scripts/execute_promoted_training_scale_seed.py`
  - 新增 `--require-clean-batch-review`。
  - strict 模式下打印 `clean_batch_review_required_*` 诊断，并在 fail 时返回非零。
- `tests/test_promoted_training_scale_seed_handoff.py`
  - 覆盖 helper status domain、默认 not-required、clean pass、dirty fail、artifact 写入和 CLI 非零退出。

## tag 含义
