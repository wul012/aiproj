# v85 promoted seed report utility migration 运行截图说明

## 本版主题

v85 继续 `report_utils` 收口路线，把 v81 的 promoted training scale next-cycle seed builder 迁移到公共报告工具上。迁移后，seed generation、seed handoff、controlled execution handoff 都开始复用同一套 JSON/CSV 写出、命令展示、Markdown/HTML 转义和 list/dict 归一化工具。

本版不改变 seed 的业务语义：ready/review/blocked 的判断、source 检查和 next plan command 生成规则保持不变。

## 关键代码

```text
src/minigpt/promoted_training_scale_seed.py
src/minigpt/report_utils.py
tests/test_promoted_training_scale_seed.py
tests/test_report_utils.py
```

`promoted_training_scale_seed.py` 继续负责读取 promoted baseline decision、检查 next corpus sources、生成 `plan_training_scale.py` 命令和 seed 报告。变化在于输出和渲染层复用 `report_utils.py`。

## 验证标准

v85 的成功标准是：

- 迁移后 promoted seed 的 ready/review/blocked 行为不变。
- report utils 拥有第三个真实消费者。
- focused tests 和 full regression 都通过。
- ready 与 blocked 两条 smoke 都能产出证据。
- HTML 报告能用 Chrome 打开。
- 文档和截图归档同步更新。

## 一句话总结

v85 把 next-cycle seed generation 接入公共报告工具，让 seed 到 handoff 的连续链路开始共享同一套基础设施。
