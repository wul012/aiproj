# v219 promoted seed handoff clean evidence requirement contract 运行说明

## 本版目标

v219 把 v218 写入 artifacts 的 `clean_evidence_requirement` 从 CLI 私有计算提升为库级契约。

新增的核心边界是：

```text
build_seed_handoff_clean_evidence_requirement()
build_promoted_training_scale_seed_handoff(..., require_clean_evidence=True)
SEED_HANDOFF_CLEAN_EVIDENCE_REQUIREMENT_STATUSES
```

同时，本版顺手完成悬置的 `utc_now` 清理：删除只转调 `report_utils.utc_now()` 的本地 `_utc_now()` wrapper。

## 边界

本版不改变 seed handoff 的默认执行行为，不改变 `--require-clean-evidence` 的退出码语义，不新增独立报告。

它做的是契约收口：CLI、库调用方、JSON/CSV/Markdown/HTML artifacts 共享同一套 requirement status。
