# v280 promoted seed handoff automation summary

## 本版主题

v280 在 v279 的 `automation_gate` 决策契约之上新增顶层 `automation_summary`。

v279 已经让 strict gate 本身带有 `decision` 和 `exit_code`，但如果 gate 没有阻塞、而 handoff 执行本身失败，调用方仍然需要同时读取 `automation_gate` 和 `handoff_status`。v280 把这两个判断合成一个最终自动化摘要。

本版不新增用户开关，不改变默认 handoff 行为。它只明确最终自动化出口：

- gate 阻塞时，blocking source 是 `automation_gate`。
- handoff 执行失败/阻塞/超时时，blocking source 是 `handoff_execution`。
- 都没有阻塞时，decision 是 `continue`。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_review.py`
  - 新增 `SeedHandoffAutomationSummary`。
  - 新增 `build_seed_handoff_automation_summary()`。
  - 新增 `SEED_HANDOFF_AUTOMATION_SUMMARY_DECISIONS`。

- `src/minigpt/promoted_training_scale_seed_handoff.py`
  - report 顶层新增 `automation_summary`。

- `scripts/execute_promoted_training_scale_seed.py`
  - CLI 总是打印 summary decision、exit code、blocking source 和 failed requirements。
  - 最终退出逻辑改为读取 `automation_summary.decision`。

- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - CSV、Markdown、HTML 都记录 automation summary 字段。

## Tag 含义

`v280.0.0` 表示：promoted seed handoff automation summary。它说明 seed handoff 自动化现在有一个最终出口对象，调用方不必自己组合 gate 和 handoff execution 状态。

## 一句话总结

v280 把 promoted seed handoff 的自动化出口收束为 `automation_summary`，让 CI 和脚本能从一个字段组判断是否继续、为何停止、用什么退出码退出。
