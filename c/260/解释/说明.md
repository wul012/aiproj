# v260 promoted seed batch review propagation 运行说明

## 本版主题

v260 把 v259 promoted baseline decision 中的 handoff selected batch review/blocker context 继续传到 promoted training scale next-cycle seed。这样 seed report 不只生成下一轮 `plan_training_scale.py` 命令，也会保留这个命令来自的 selected baseline 是否仍带 selected batch review/blocker 风险。

本版不改变 seed 的 `ready`、`review`、`blocked` 判定：batch review/blocker context 是风险提示和 recommendation，不会自动阻断原本 ready 的 next-cycle seed。

## 关键改动

- `src/minigpt/promoted_training_scale_seed.py`：新增 `handoff_batch_review` 数据块，和 `handoff_suite_guard` 并列保存 selected batch review/blocker 状态、action counts、blocker reasons 和 comparison-ready totals。
- `src/minigpt/promoted_training_scale_seed_artifacts.py`：CSV/Markdown/HTML 输出新增 selected handoff batch review/blocker 字段和 comparison-ready batch review/blocker counts。
- `scripts/build_promoted_training_scale_seed.py`：CLI 输出新增 selected handoff batch review/blocker 字段和 comparison-ready blocker reasons。
- `tests/test_promoted_training_scale_seed.py`：新增 decision -> seed 的 batch review propagation 测试，确认 seed 仍 ready，但风险字段进入 JSON/CSV/Markdown/HTML/CLI/recommendations。
- `README.md` 和 `代码讲解记录_项目成熟度阶段/274-v260-promoted-seed-batch-review-propagation.md`：同步说明本版目标、边界、输入输出和证据链角色。

## tag 含义

`v260.0.0` 表示 promoted next-cycle seed 已经接住 promoted-decision handoff selected batch review/blocker context，并在生成下一轮 plan handoff 时保持风险可见。

## 一句话总结

v260 让 promoted next-cycle seed 从“生成下一轮 plan 命令”推进到“生成命令时继续保留 selected handoff batch review/blocker 风险”。
