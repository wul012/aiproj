# v194 运行截图和解释

## 本版主题

v194 是 training run evidence component split。

v193 把外部 baseline pair 能力扩展到了 batch variant 矩阵。v194 不继续扩功能，而是按前面质量评估建议做一次轻量、定向的收口：`training_run_evidence.py` 已经达到 580 行，超过本项目对大文件的维护压力阈值，所以本版把 checks、summary、section、recommendation 和 value helper 拆到独立组件模块。

## 关键改动

- 新增 `src/minigpt/training_run_evidence_components.py`，承载训练运行证据的检查、摘要、分区、建议和数值转换逻辑。
- 更新 `src/minigpt/training_run_evidence.py`，只保留 JSON/JSONL 读取、artifact row 收集和报告组装。
- `training_run_evidence.py` 从 580 行降到 151 行。
- 对外 API 不变：`build_training_run_evidence()` 和 artifact writer facade 仍从 `training_run_evidence.py` 暴露。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引更新到 v194。

## tag 含义

`v194.0.0` 表示 MiniGPT 第 194 个可回看版本：训练运行证据模块完成一次契约保持的组件拆分，降低文件膨胀和后续维护压力。运行截图与解释归档在 `c/194`。

## 一句话总结

v194 把训练运行证据从“单文件承载所有治理逻辑”推进为“编排层 + 组件层”的结构，让后续扩展 eval、quality、scorecard 证据时更容易控制复杂度。
