# v108 运行截图说明

## 本版主题

v108 是 batched release governance report-utils migration。

本版不再一模块一版本，而是把 `release_gate.py`、`release_gate_comparison.py` 和 `request_history_summary.py` 合并迁移到 `report_utils`。迁移内容限于 JSON 写出、UTC 时间、HTML 转义、dict/list 归一化和部分 CSV 标量处理；发布治理业务规则仍留在各自模块内部。

## 证据边界

这些截图证明的是 batched report-utils migration 没有破坏 release gate、profile comparison 和 request history summary 的核心输出和展示链路。

它们不证明模型能力提升，也不证明新的发布策略更严格。v108 是一次轻量、定向、合并式的代码质量收束。

## Tag 含义

`v108.0.0` 表示 release gate、release gate profile comparison 和 request history summary 已经作为同类批次接入公共报告基础设施，但发布治理业务判断仍然留在各自模块内部。

## 一句话总结

v108 让发布治理链路里的三个相邻模块共享同一套报告基础工具，同时保持 gate policy、profile delta 和 request-history summary 的原有语义。
