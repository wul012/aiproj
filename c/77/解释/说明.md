# v77 运行截图和解释

## 本版主题

v77 新增 training scale promotion acceptance。它读取 v76 `training_scale_handoff.json`，继续追踪背后的 `training_scale_run.json`、`training_portfolio_batch.json` 和每个 variant 的 `training_portfolio.json`，最后判断这次执行结果是 `promoted`、`review` 还是 `blocked`。

这一版不重新训练、不复制 checkpoint、不绕过 handoff。它解决的是“命令跑完以后，这次结果能不能成为后续对比和成熟度证据”的问题。

## 关键命令

```powershell
python -m unittest tests.test_training_scale_promotion
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts/run_training_scale_workflow.py data --out-root tmp\v77-smoke\workflow --profile review --baseline-profile review --dataset-name sample-zh --dataset-version-prefix v77 --max-variants 1
python scripts/execute_training_scale_handoff.py tmp\v77-smoke\workflow --out-dir tmp\v77-smoke\handoff-execute --execute --timeout-seconds 300
python scripts/build_training_scale_promotion.py tmp\v77-smoke\handoff-execute --out-dir tmp\v77-smoke\promotion
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp\v77-smoke\promotion\training_scale_promotion.html -Out c\77\图片\05-playwright-training-scale-promotion-html.png
```

## 证据含义

promotion smoke 证明 v77 可以从已完成 handoff 继续读取真实训练产物，并确认这些证据是否完整。被检查的核心 variant evidence 包括：

```text
checkpoint
run_manifest
eval_suite
generation_quality
benchmark_scorecard
dataset_card
registry
maturity_summary
maturity_narrative
```

真实 smoke 使用 tiny sample corpus 和单个 variant，产物留在临时目录中，最终按 cleanup gate 删除。仓库保留的是截图、说明、源码和测试。

## tag 含义

`v77.0.0` 表示项目从“受控执行训练建议”推进到“验收执行结果是否可以进入项目级证据链”。
