# v208 运行截图说明

本版目标是把 v207 promotion index 已经保留的 handoff suite boundary，再向下游接入 promoted training-scale comparison。这样 promoted-only comparison 不只比较真实 scale run，也能在输入表和 summary 里继续看到这些 promoted 输入来自哪条 handoff suite guard。

本版不重新计算 suite consistency，不改变 promoted-only 选择规则，不改变 baseline 选择和 readiness 评分。真实 suite 一致性仍由 `training_scale_run_comparison.py` 从 scale run 里计算；v208 只消费 promotion index 提供的 handoff suite guard 字段。

关键文件：

- `src/minigpt/promoted_training_scale_comparison.py`
- `src/minigpt/promoted_training_scale_comparison_artifacts.py`
- `scripts/compare_promoted_training_scale_runs.py`
- `tests/test_promoted_training_scale_comparison.py`
- `README.md`

这版截图要证明的事情：

1. 聚焦测试能通过，说明 promoted comparison row、summary、CLI 输出都接住了 index 层 suite guard 字段。
2. CLI smoke 能从 promotion index 读取两个 promoted 输入，完成 promoted-only comparison，并打印 handoff suite guard 统计。
3. JSON/CSV/Markdown/HTML 都能看到 `handoff_require_suite_consistency`、`handoff_suite_consistency`、`handoff_suite_mismatch_count`、`handoff_selected_suite_path`。
4. README、`c/README.md` 和代码讲解记录已经把 v208 的位置、归档位置和链路角色对齐。
