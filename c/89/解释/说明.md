# v89 gated run report utility migration 运行截图说明

## 本版主题

v89 继续 `report_utils` 收口路线，把 v72 的 `training_scale_run.py` 迁移到公共报告工具上。迁移后，gated training scale run、training scale run comparison、training scale run decision、promoted baseline decision、promoted seed generation、promoted seed handoff 和 controlled execution handoff 都开始复用同一套 JSON/CSV 写出、Markdown/HTML 转义和 list/dict 归一化工具。

本版不改变 gate-to-batch 的业务语义：gate 放行、warn 阻断、allow-fail 强制放行、batch dry-run 是否启动，以及 status/blocked reason 的规则保持不变。

## 关键代码

```text
src/minigpt/training_scale_run.py
src/minigpt/report_utils.py
tests/test_training_scale_run.py
tests/test_report_utils.py
```

`training_scale_run.py` 继续负责读取 training scale plan、运行 gate、判断是否 allowed、写出 variants、运行 batch runner，并汇总 gated run report。变化在于 JSON/CSV 写出和 Markdown/HTML 渲染基础层复用 `report_utils.py`。

## 验证标准

v89 的成功标准是：

- 迁移后 allowed/blocked 行为不变。
- report utils 拥有新的 gated run 层消费者。
- focused tests 和 full regression 都通过。
- allowed 与 blocked 两条 smoke 都能产出证据。
- HTML 报告能用 Chrome 打开。
- README、代码讲解和截图归档同步更新。
