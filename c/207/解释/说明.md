# v207 运行截图说明

本版目标是把 promotion index 再往上游一步，保留 promoted promotion 继承的 handoff suite boundary，让 compare-ready 选择不只是“promoted 列表”，也是“带 clean-comparison 证据的 promoted 列表”。

本版不重新计算 suite consistency，不改变 promotion 状态，也不新增阻断规则。它只把 promotion report 里已有的 handoff suite evidence 汇总到 index 的 JSON/CSV/Markdown/HTML 里，并让 CLI 输出能看到这些统计。

关键文件：

- `src/minigpt/training_scale_promotion_index_helpers.py`
- `src/minigpt/training_scale_promotion_index.py`
- `scripts/index_training_scale_promotions.py`
- `tests/test_training_scale_promotion_index.py`
- `README.md`

这版截图要证明的事情：

1. 测试和语法检查能通过，说明 index row、summary、CLI 输出都接住了 suite guard 字段。
2. CLI smoke 能把两个 promoted promotion 汇总成 compare-ready index。
3. JSON/CSV/Markdown/HTML 都能看到 `handoff_require_suite_consistency`、`handoff_suite_consistency`、`handoff_suite_mismatch_count`、`handoff_selected_suite_path`。
4. README 和 `c/README.md` 已经把 v207 的位置、归档位置和讲解目录对齐。
