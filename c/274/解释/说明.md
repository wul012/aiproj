# v274 运行说明

## 本版主题

v274 把 v273 promoted comparison 已经输出的 clean batch-review evidence 继续带到 promoted baseline decision。此前 comparison 层会过滤 dirty clean-required promoted input；本版让 decision 层也能看见 selected clean 状态、clean/unclean 计数，并在 dirty clean-required row 仍出现在 promotions 证据里时拒绝它成为 baseline candidate。

## 关键改动

- promoted decision row 新增 `handoff_require_clean_batch_review` 和 `handoff_clean_batch_review_status`。
- baseline rejection 新增 `clean batch-review requirement is not clean`，作为下游防漏网。
- decision summary 新增 selected clean status、handoff clean-required/clean/unclean counts、comparison-ready clean counts。
- CSV、Markdown、HTML 和 CLI 输出新增 clean batch-review 字段。
- recommendation helper 会说明 rejected promoted inputs 中的不干净 clean-required handoff 不能进入 baseline selection。

## Tag 含义

`v274.0.0` 标记 promoted training scale decision clean batch gate。它表示 clean batch-review requirement 已从 decision、workflow、handoff、promotion、promotion index、promoted comparison 继续推进到 promoted baseline decision。

## 一句话总结

v274 让 promoted baseline decision 自己消费 clean batch-review 证据，避免 dirty clean-required promoted row 被误选为下一轮训练 baseline。
