# v142 运行截图说明

## 本版主题

v142 把 `source_encoding_hygiene` 从 BOM/语法检查推进到目标 Python 兼容性门禁。它解决的是这次 GitHub CI 暴露出的具体问题：本地较新 Python 可以解析的 f-string，在 Python 3.11 CI 中可能直接失败。

本版不新增模型训练、benchmark 或 maturity narrative 功能，只增强源代码卫生门禁，让 Python 版本漂移问题能在本地、报告和 CI 日志中更早定位。

## 关键改动

- `src/minigpt/source_encoding_hygiene.py`：新增 `target_python`、compatibility summary/file 字段、Python 3.11 f-string 风险扫描，以及 Markdown/HTML/CSV 输出字段。
- `scripts/check_source_encoding.py`：新增 `--target-python` 参数，并打印 `compatibility_error_count` 与 `target_python`。
- `tests/test_source_encoding_hygiene.py`：新增目标兼容性 fixture，覆盖 Python 3.13 与 Python 3.11 下的不同表现，并断言 HTML 能展示 compatibility-only offending file。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/156-v142-source-encoding-target-compatibility.md`：记录本版目标、证据和边界。

## 证据边界

这些截图证明 v142 的源代码卫生门禁闭环，不证明模型能力提升，也不代表 benchmark 分数改善。它属于工程可靠性增强：把“远端 CI 才发现的解释器版本语法问题”前移到本地报告和测试。

## tag 含义

`v142.0.0` 标记的是 “source encoding target compatibility”：source encoding hygiene gate 开始显式记录目标 Python 版本，并把 Python 3.11 不兼容的 f-string 语法作为可报告、可测试、可审计的门禁项。

一句话总结：v142 让 MiniGPT 的源码卫生检查从“看当前解释器”升级为“看 CI 目标解释器”，减少本地通过但 GitHub Actions 失败的风险。
