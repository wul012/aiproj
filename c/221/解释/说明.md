# v221 numeric helper consolidation 运行说明

## 本版目标

v221 回应一次轻量质量优化建议：多个模块里存在 `_int()` / `_float()` 这类重复数值转换 helper。

本版把可复用转换语义收敛到 `report_utils`：

```text
number_or_none()
number_or_default()
```

然后先迁移训练证据链中最明确的两个调用点：`training_run_evidence_components.py` 和 `training_scale_gate.py`。

## 边界

本版不做大规模重命名，不拆长文件名，不把 TypedDict/Literal 统一抽象，也不引入新的 artifact writer 抽象。

原因是这次问题属于“局部 helper 重复”，最合适的处理是小步收敛公共语义，而不是把 v221 变成结构重构版。

## Tag 含义

`v221.0.0` 表示 aiproj 开始把数值转换 helper 从局部重复定义收敛到 `report_utils`，并保留不同调用场景的缺失值语义。

## 一句话总结

v221 把训练证据链里的数值 helper 从“各写各的”推进到“公共语义 + 局部适配”，是一次小而稳的质量收口。
