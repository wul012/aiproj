# v75 运行截图和解释

## 本版主题

v75 新增 consolidated training scale workflow。它把 v70 training scale plan、v72 gated profile run、v73 comparison 和 v74 decision 收口到一个入口里，避免后续继续手工串多条命令。

这一版不新增更细的 dashboard，也不直接扩大模型训练规模。它的重点是减少治理链碎片，让下一次真实中文语料训练前只需要先跑一个统一 workflow，再审阅产物和生成的 `--execute` handoff command。

## 关键命令

```powershell
python -m unittest tests.test_training_scale_workflow
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts/run_training_scale_workflow.py data --out-root tmp\v75-smoke\workflow --profile review --profile standard --baseline-profile review --dataset-name sample-zh --dataset-version-prefix v75
python scripts/run_training_scale_workflow.py data --out-root tmp\v75-smoke\workflow-strict --profile review --profile standard --baseline-profile review --dataset-name sample-zh --dataset-version-prefix v75 --decision-require-gate-pass
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp\v75-smoke\workflow\training_scale_workflow.html -Out c\75\图片\05-playwright-training-scale-workflow-html.png
```

## 证据含义

默认 smoke 会生成：

```text
workflow/plan/training_scale_plan.json
workflow/runs/review/training_scale_run.json
workflow/runs/standard/training_scale_run.json
workflow/comparison/training_scale_run_comparison.json
workflow/decision/training_scale_run_decision.json
workflow/training_scale_workflow.json
```

这证明 v75 不是新增一个孤立 report，而是把 v70-v74 既有能力收成一个执行前工作流。默认模式下 review profile 会被选为需要人工审阅 warning 后再执行的候选；standard profile 作为被阻断证据保留。严格模式则证明 workflow 可以在策略收紧时停止，不给出可执行候选。

## tag 含义

`v75.0.0` 表示项目开始从“继续拆治理 report”转向“收口治理链”：训练规模治理入口从多条命令变成一个可审计 workflow。
