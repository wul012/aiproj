# v206 promotion suite guard evidence 运行说明

## 本版目标

v206 把 v205 的 handoff suite guard 继续带到 training-scale promotion acceptance 层。

v205 已经让 reviewed execute command 保留 `decision_require_suite_consistency`、`suite_consistency`、`suite_mismatch_count`、`selected_suite_path`。本版解决 promotion 层的证据断点：promotion 报告现在也能看到 handoff 继承的 suite boundary。

## 关键改动

- `src/minigpt/training_scale_promotion.py`
  - 新增 `_suite_guard()`，从 handoff 顶层 `suite_guard` 或 summary 读取 suite evidence。
  - report 顶层新增 `suite_guard`。
  - summary 新增 `handoff_require_suite_consistency`、`handoff_suite_consistency`、`handoff_suite_mismatch_count`、`handoff_selected_suite_path`。
- `src/minigpt/training_scale_promotion_artifacts.py`
  - CSV 增加 handoff suite guard 字段。
  - Markdown/HTML 摘要展示 handoff strict suite、suite consistency、mismatch count 和 selected suite。
- `tests/test_training_scale_promotion.py`
  - 新增 promotion suite guard 测试。
  - 明确断言 suite guard 透传不会把 `promotion_status` 从 `promoted` 改成 `review` 或 `blocked`。

## 证据边界

本版不改变 promotion 逻辑，不重新计算 suite consistency，也不新增 promotion blocker。

它只把 handoff 层已有的 clean-comparison 边界保留到 promotion acceptance 报告里，让 promoted baseline 的后续使用者知道这份 baseline 是否继承了 strict suite evidence。

## 一句话总结

v206 让 training-scale promotion acceptance 不再丢失 handoff suite guard，promoted baseline 的证据链继续保持 suite 边界可见。
