# v96 generation quality report utility migration 运行截图说明

## 本版主题

v96 继续 `report_utils` 收口路线，把 `generation_quality.py` 中语义一致的报告基础 helper 迁移到公共工具层。
本版只迁移 JSON 写出、UTC 时间、HTML/SVG 转义和 list/dict 归一化，不改变 generation quality 的 pass/warn/fail 判定，也不强行迁移模块特有的 Markdown/字符串显示格式化 helper。

迁移后，公共报告基础层开始覆盖模型输出质量证据，而不仅是 training scale 和 promoted 链路。

## 关键代码

```text
src/minigpt/generation_quality.py
src/minigpt/report_utils.py
tests/test_generation_quality.py
tests/test_report_utils.py
scripts/analyze_generation_quality.py
```

`generation_quality.py` 仍然负责读取 eval suite、sample lab 或 generic results，检查 continuation 长度、多样性、重复和 prompt echo，并输出 JSON/CSV/Markdown/SVG/HTML 证据。

## 验证标准

v96 的成功标准是：

- generation quality 私有 `utc_now`、`_dict`、`_list_of_dicts`、`_e` 被公共 `report_utils` 替代。
- JSON writer 使用 `write_json_payload`。
- SVG 中原来的 `html.escape` 也改用公共 HTML escaping。
- `_md` 和 `_string_list` 被保留，因为它们有本模块特定显示语义。
- eval_suite 与 sample_lab 两类输入仍然可用。
- HTML 报告能被真实 Chrome 打开。
- README、代码讲解和截图归档同步到 v96。
