# v171 server GET route split 运行说明

## 本版主题

v171 做的是本地推理服务入口的 GET route split：把 `server.py` 里负责 `/api/health`、`/api/checkpoints`、`/api/checkpoint-compare`、`/api/model-info`、`/api/request-history`、`/api/request-history-detail`、`/`、`/playground.html` 和 run-file serving 的 GET 分发逻辑拆到 `server_routes.py`。

这一版不改变 POST 生成链路，不改变 streaming、pair generation、request logging、generator cache、HTTP response helpers，也不改变旧的 `minigpt.server` facade 导出。

## 关键改动

- `src/minigpt/server_routes.py`：新增 GET route dispatcher，集中处理只读查询、playground HTML 生成和 run artifact 文件读取。
- `src/minigpt/server.py`：`do_GET()` 改为调用 `handle_get_request()`，继续保留 POST 推理、stream、pair artifact、日志和旧 facade 导出。
- `tests/test_server_routes.py`：新增 facade identity 测试，证明 `server.handle_get_request` 指向 `server_routes.handle_get_request`。
- `README.md`、`c/README.md`、代码讲解索引：登记 v171 能力、tag 和证据位置。
