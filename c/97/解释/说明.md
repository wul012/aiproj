# v97 release bundle report utility migration 运行截图说明

## 本版主题

v97 继续 `report_utils` 收口路线，把 `release_bundle.py` 中语义一致的报告基础 helper 迁移到公共工具层。
本版只迁移 JSON 写出、UTC 时间、HTML 转义和 list/dict 归一化，不改变 release bundle 的发布状态判断，也不强行迁移 release-specific 的 Markdown、bytes、rank 和 loss 格式化 helper。

迁移后，公共报告基础层开始覆盖发布证据总包。

## 关键代码

```text
src/minigpt/release_bundle.py
src/minigpt/report_utils.py
tests/test_release_bundle.py
tests/test_report_utils.py
scripts/build_release_bundle.py
```

`release_bundle.py` 仍然负责读取 registry、model card、project audit 和 request history summary，输出 release bundle 的 JSON/Markdown/HTML 证据。

## 验证标准

v97 的成功标准是：

- release bundle 私有 `utc_now`、`_dict`、`_list_of_dicts`、`_e` 被公共 `report_utils` 替代。
- JSON writer 使用 `write_json_payload`。
- `_md`、`_string_list`、`_fmt`、`_fmt_delta`、`_fmt_any`、`_fmt_bytes`、`_rank_label` 被保留，因为它们有 release bundle 特定显示语义。
- complete evidence 仍然 `release-ready`。
- missing audit 仍然 `needs-audit`。
- HTML 报告能被真实 Chrome 打开。
- README、代码讲解和截图归档同步到 v97。
