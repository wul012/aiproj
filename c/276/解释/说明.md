# v276 promoted seed handoff clean batch gate

## 本版目标

v276 承接 v275 的 promoted seed clean batch-review evidence，把它继续传到 promoted seed handoff 层。

这一版解决的问题是：v275 的 seed 已经知道 selected handoff 是否要求 clean batch review、状态是否为 `clean`、以及 clean/unclean handoff 的数量；但 seed handoff 报告此前只显示 suite guard、普通 batch review 和 clean-evidence readiness。v276 让 seed handoff 在真正执行下一轮 plan 命令之前，也能公开这些 clean batch-review 字段。

本版不做新的训练、不改变 seed command 的执行语义，也不新增模型能力。它只收紧证据链的可追溯性。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_review.py`
  - 新增 `build_seed_handoff_clean_batch_review_summary()`，从 `baseline_seed.handoff_clean_batch_review` 投影出 handoff 层 summary 字段。
  - 新增 clean batch-review recommendation：如果 selected clean-required handoff 不干净，提示先解决；如果存在被拒绝的 dirty clean-required promoted inputs，提示不要让它们进入 seed handoff baseline。
- `src/minigpt/promoted_training_scale_seed_handoff.py`
  - 在 `_summary()` 中并入 clean batch-review summary，让 JSON summary 和后续 artifact 共享同一来源。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - CSV、Markdown、HTML 都展示 clean-required、clean、unclean、comparison-ready clean 计数。
- `scripts/execute_promoted_training_scale_seed.py`
  - CLI stdout 打印同一组字段，方便 CI/log 截图直接确认。
- `tests/test_promoted_training_scale_seed_handoff.py`
  - 新增覆盖：seed fixture 带 `handoff_clean_batch_review` 时，builder、JSON、CSV、Markdown、HTML、CLI stdout 和 recommendation 都必须接住字段。

## tag 含义

`v276.0.0` 表示：promoted seed handoff clean batch gate。它说明 clean batch-review 证据已经从 promoted comparison、promoted decision、promoted seed 继续传到 seed handoff 层。

## 一句话总结

v276 把 clean batch-review 证据链推进到“下一轮 plan command 执行前的 handoff 审查层”，让自动化推进前能看到 selected baseline 是否真的来自干净的 promoted evidence。
