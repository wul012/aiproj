# v151 运行截图说明

## 本版主题

v151 把 registry 中的 CI workflow readiness regression 接入 `maturity summary`。
成熟度 summary 现在能展示 CI workflow regression count、status changed count 和 max failed-check delta，并在出现 CI workflow regression 时把趋势标记为 `ci-regressed`，将总体成熟度从 `pass` 降为 `warn`。

本版仍然不改变 release gate hard block 规则。CI workflow regression 在这里是 maturity review 信号。

## 关键改动

- `src/minigpt/maturity.py`
  - `release_readiness_context` 新增 CI workflow regression 字段。
  - `_release_readiness_trend_status()` 新增 `ci-regressed`。
  - `_summary()` 新增 CI workflow overview 字段。
  - archive 版本发现补充扫描 `c/`。
  - 代码讲解版本发现只扫描 `代码讲解记录*` 目录。
- `src/minigpt/maturity_artifacts.py`
  - Markdown overview 展示 CI workflow regression count。
  - Markdown/HTML Release Readiness Trend Context 展示 CI workflow regression、status change 和 failed-check delta。
  - HTML stats 展示 `CI regressions`。
- `scripts/build_maturity_summary.py`
  - CLI 输出新增 CI workflow regression 和 status changed count。
- `tests/test_maturity.py`
  - 新增 `ci-regressed` review 行为测试。
- `tests/test_maturity_artifacts.py`
  - 新增 Markdown/HTML CI workflow 字段渲染断言。

## 证据边界

`c/151` 证明 maturity summary 能消费 registry 中的 CI workflow readiness regression。
它不证明模型生成质量提升，也不代表 release gate hard block 规则改变。

## tag 含义

`v151.0.0` 标记的是 “maturity CI workflow readiness context”：成熟度 summary 能识别并展示 CI workflow readiness regression。

一句话总结：v151 把 CI workflow hygiene 退化信号从 registry 推进到 maturity summary，让最终项目成熟度总结也能看到 CI 策略漂移。
