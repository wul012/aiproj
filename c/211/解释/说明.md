# v211 运行截图说明

本版目标是把 v210 promoted training-scale seed 已经保留的 handoff suite boundary，继续接到 promoted seed handoff。这样下一轮 plan seed 进入“审阅/执行”入口时，不只保留 `seed_suite_path` 和 `plan_suite_path`，也能看到上游 selected baseline 的 clean-comparison guard。

本版不重新计算 suite consistency，不改变 seed handoff 的 blocker 规则，不改变 planned-mode 审阅逻辑，也不改变 `--execute` 的命令运行语义。`selected_handoff_*` 字段只作为审阅证据进入 seed handoff summary、CSV、Markdown、HTML 和 CLI stdout。

同时，本版融合了一项轻量质量优化：把多个模块里同语义的“第一个非 None 值” helper 收束为 `report_utils.first_present`，并让几个本地 UTC 时间函数复用 `report_utils.utc_now`。那些按 dict key 取值的同名 helper 语义不同，本版没有强行合并。

关键文件：

- `src/minigpt/promoted_training_scale_seed_handoff.py`
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
- `scripts/execute_promoted_training_scale_seed.py`
- `src/minigpt/report_utils.py`
- `tests/test_promoted_training_scale_seed_handoff.py`
- `tests/test_report_utils.py`
- `README.md`

这版截图要证明的事情：

1. 聚焦测试能通过，说明 seed handoff summary、CSV/Markdown/HTML、CLI 和公共 helper 都正常。
2. CLI smoke 能构建 planned seed handoff，同时打印 `selected_handoff_suite_consistency` 和 `handoff_suite_mismatch_total`。
3. JSON/CSV/Markdown/HTML 都能看到 selected handoff suite guard 字段。
4. README、`c/README.md` 和代码讲解记录已经把 v211 的位置、归档位置和链路角色对齐。
5. 全量 unittest 与 source encoding smoke 仍然干净，说明 helper 收束没有破坏现有链路。
