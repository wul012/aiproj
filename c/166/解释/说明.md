# v166 project audit context split 运行说明

## 本版主题

v166 把 `src/minigpt/project_audit.py` 的 request-history summary 和 CI workflow hygiene 检查/上下文映射，拆到 `src/minigpt/project_audit_contexts.py`。

这一版不改变 `project_audit.json` schema，不改变 project audit 的 Markdown/HTML/JSON 输出，也不改变 `scripts/audit_project.py`。它只把外部治理证据的字段映射和状态判断从 audit builder 中拆出来。

## 关键改动

- 新增 `src/minigpt/project_audit_contexts.py`，集中保存 request-history summary check、request-history context、CI workflow hygiene check、CI workflow context。
- `src/minigpt/project_audit.py` 改为调用 context helper，继续负责 registry/model card/run coverage/summary/recommendations。
- `project_audit.py` 从 538 行降到 404 行。
- `tests/test_project_audit.py` 新增 helper 直接测试，覆盖 warn、missing、detail 和 evidence 映射。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引同步到 v166。
