# v247 maturity coverage regression context 运行截图说明

## 本版主题

v247 把 registry 里的 coverage regression 接入 maturity summary。

v246 已经让 registry 能索引 `coverage-regressed` run。v247 继续把这个治理信号提升到成熟度评审层：如果 registry 发现 test coverage regression，maturity summary 会把 release readiness trend 标成 `coverage-regressed`，并把原本可能是 `pass` 的 overall status 降为 `warn`。

## 截图清单

### 01-maturity-coverage-tests.png

该截图来自：

```text
python -B -m unittest tests.test_maturity tests.test_maturity_artifacts -q
```

结果：

```text
Ran 9 tests
OK
```

它证明 maturity builder 能读取 registry 中的 coverage regression 字段，artifact renderer 能在 Markdown/HTML 中展示这些字段。

### 02-maturity-coverage-smoke.png

该截图来自：

```text
python -B scripts/build_maturity_summary.py --project-root sample --registry registry.json
```

关键结果：

```text
overall_status=warn
release_readiness_trend_status=coverage-regressed
release_readiness_test_coverage_regression_count=1
release_readiness_test_coverage_status_changed_count=1
```

它证明 CLI 层能直接看到 coverage regression，并且 maturity summary 会因此进入 review/warn 状态。

### 03-maturity-coverage-output-check.png

该截图来自对 maturity JSON、Markdown 和 HTML 的结构检查。

关键结果：

```text
overall_status=warn
trend_status=coverage-regressed
test_coverage_regression_count=1
max_abs_test_coverage_percent_delta=7.5
max_abs_test_coverage_gap_delta=3
markdown_has_coverage=True
html_has_coverage=True
recommendation=True
```

它证明 coverage regression 已经进入：

- 机器读 summary：`maturity_summary.json`
- 人读 Markdown：`maturity_summary.md`
- 人读 HTML：`maturity_summary.html`
- 后续行动建议：`recommendations`

### 04-full-unittest.png

该截图来自：

```text
python -B -m unittest discover -s tests -q
```

结果：

```text
Ran 484 tests
OK
```

它证明 v247 的 maturity 改动没有破坏全项目测试集合。

## 证据边界

v247 证明的是 maturity summary 能消费 registry-level coverage regression。

它不重新计算 coverage，不改变 registry 的 coverage 判断规则，也不改变 release gate 的阻断策略。成熟度降为 `warn` 是 review 提示，不是训练或推理能力结论。

## 一句话总结

v247 让成熟度总结能看见覆盖率退步：coverage regression 从 registry 索引信号升级为 maturity review 风险信号。
