# v188 运行截图和解释

## 本版主题

v188 是 training run evidence generation quality summary。

本版接着 v187 的真实训练证据推进：`training_run_evidence` 不只读取 fixed eval suite 摘要，还会读取 `generation_quality/generation_quality.json`，把生成质量状态、pass/warn/fail case 数、平均唯一字符比例、重复 n-gram 比例、最长重复 run、总 flag 数和 dominant flag 写入训练证据。

## 关键改动

- 更新 `src/minigpt/training_run_evidence.py`，新增 `quality` section 和 generation-quality-aware check。
- 更新 `src/minigpt/training_run_evidence_artifacts.py`，在 CSV/Markdown/HTML 中展示 generation quality summary。
- 更新 `scripts/build_training_run_evidence.py`，终端输出 `generation_quality_status` 和 `generation_quality_flags`。
- 更新 `tests/test_training_run_evidence.py`，覆盖完整 quality 摘要和缺 quality 的非阻塞 review 边界。
- 更新 README、`c/README.md` 和项目成熟度阶段代码讲解索引。

## tag 含义

`v188.0.0` 表示 MiniGPT 第 188 个可回看版本：真实训练 run evidence 已经能消费 generation quality 摘要，固定 eval 输出的短输出、重复和多样性风险可以进入同一份 promotion-review evidence。运行截图与解释归档在 `c/188`。

## 一句话总结

v188 把真实训练证据从“checkpoint 已被固定 eval suite 跑过”推进到“eval 输出质量风险也可审计”。
