# v200 training scale standard suite handoff 运行说明

## 本版目标

v200 把 v199 已经接入 training portfolio / batch 的标准中文 benchmark suite 继续接到 training scale 工作流。

v198 新增 `standard-zh` 内置评测集，v199 让 portfolio 和 batch 可以用 `--suite-name standard-zh`。但 scale plan 和 consolidated workflow 仍然不能直接选择这个 suite。本版让 `plan_training_scale.py`、`run_training_scale_workflow.py`、gated scale run 和 workflow summary 都能携带 `builtin:standard-zh`。

## 关键改动

- `src/minigpt/training_scale_plan.py`
  - `build_training_scale_plan()` 新增 `suite_path` / `suite_name`。
  - 计划 JSON 新增 `suite` 和兼容字段 `suite_path`。
  - 内置 suite 会让 batch handoff command 生成 `--suite-name standard-zh`。
- `src/minigpt/training_scale_run.py`
  - gated run 从 plan JSON 读取 suite，并按 builtin 模式把 `suite_name` 传给 portfolio batch。
  - run summary 和 batch summary 都记录 suite 信息。
- `src/minigpt/training_scale_workflow.py`
  - consolidated workflow 新增 `suite_path` / `suite_name`。
  - `plan_summary` 和 `summary` 记录 suite mode/name/path。
- `src/minigpt/training_scale_plan_artifacts.py`
  - plan Markdown/HTML 显示 suite path。
- `src/minigpt/training_scale_workflow_artifacts.py`
  - workflow Markdown/HTML 显示 suite path。
- `scripts/plan_training_scale.py`
  - 新增 `--suite` 和 `--suite-name {default,standard-zh}`。
- `scripts/run_training_scale_workflow.py`
  - 新增 `--suite` 和 `--suite-name {default,standard-zh}`。
- `tests/test_training_scale_plan.py`
  - 覆盖 builtin suite 计划和互斥校验。
- `tests/test_training_scale_run.py`
  - 覆盖 gated run 到 batch 的 suite handoff。
- `tests/test_training_scale_workflow.py`
  - 覆盖 consolidated workflow 的 suite 透传。

## 证据边界

本版不训练模型，也不证明模型在标准中文 benchmark suite 上变好。它证明的是 scale 工作流从入口开始就能固定同一套评测输入，并把这个选择写入 plan、run、workflow 证据。

## tag 含义

`v200.0.0` 表示 MiniGPT v200 training scale standard suite handoff：标准中文 benchmark suite 选择已经进入 training scale plan、gated run 和 consolidated workflow。

一句话总结：v200 把标准中文评测集选择从 portfolio 层继续推进到训练规模治理入口，让后续真实规模实验能直接沿用同一套 benchmark prompt。
