# v141 运行截图说明

## 本版主题

v141 把 v140 的 `benchmark_scorecard_decision.json` 接入成熟度叙事，不再让 scorecard promotion decision 只停留在单独报告里。成熟度叙事现在能读取或自动发现 scorecard decision，汇总 selected run、decision status、review/blocker 数量和 generation-quality flag delta，并在 Markdown/HTML 中展示 `Benchmark Promotion Decision` 区块。

本版不证明模型已经具备生产级能力，也不新增训练规模、模型结构或发布准入规则。它只把已有 benchmark promotion 证据纳入 portfolio-level maturity narrative，让评审时能从一个叙事报告看到评估、数据、发布、请求历史和 promotion decision 的关系。

## 关键改动

- `src/minigpt/maturity_narrative.py`：新增 `benchmark_scorecard_decision_paths` 输入、默认发现逻辑、summary 字段、`Benchmark Promotion Decision` section 和 evidence matrix 行。
- `src/minigpt/maturity_narrative_artifacts.py`：Markdown/HTML 展示 scorecard decision count、selected run 和 selected flag delta。
- `scripts/build_maturity_narrative.py`：新增 `--benchmark-scorecard-decision` 参数，并在 CLI 输出中打印 decision 数量和 selected run。
- `tests/test_maturity_narrative.py`：在成熟度叙事 fixture 中加入 scorecard decision，覆盖 summary、section、artifact 和 evidence matrix。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/155-v141-maturity-narrative-scorecard-decision.md`：同步记录本版能力、证据和 tag 含义。

## 证据边界

这些截图证明 v141 的 scorecard promotion decision 已被 maturity narrative 消费、渲染和测试覆盖。它们不代表 benchmark 本身更强，也不代表被选中的 run 一定适合真实发布；是否推广仍取决于 scorecard decision 的 blockers、review items、数据规模和人工评审。

## tag 含义

`v141.0.0` 标记的是“maturity narrative scorecard decision”：把 v140 的 promotion decision 纳入项目成熟度叙事，形成从 generation quality flag taxonomy 到 scorecard、comparison、decision、maturity narrative 的闭环。

一句话总结：v141 让 benchmark promotion decision 从单点报告进入成熟度叙事层，项目评审时能更自然地看到“哪个 run 被选中、为什么、还有哪些边界”。
