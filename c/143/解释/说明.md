# v143 运行截图说明

## 本版主题

v143 处理的是 GitHub Actions 的 Node.js 20 deprecation 提醒。v142 的 main 和 tag CI 都已经通过，但 GitHub 在 annotation 里提示 `actions/checkout@v4` 和 `actions/setup-python@v5` 正在使用 Node 20 JavaScript action runtime。v143 主动设置 Node 24 runtime，避免后续 hosted runner 默认切换时才发现问题。

本版不新增 MiniGPT 模型能力，不修改训练、评估或治理报告 schema，也不扩展 Python test matrix。它只维护 CI runtime 策略。

## 关键改动

- `.github/workflows/ci.yml`：新增顶层 `env: FORCE_JAVASCRIPT_ACTIONS_TO_NODE24: "true"`。
- `tests/test_ci_workflow.py`：新增 workflow 守护测试，确认 Node 24 opt-in、`actions/checkout@v4` 和 `actions/setup-python@v5` 同时存在。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/157-v143-github-actions-node24-runtime.md`：记录本版目标、证据和 tag 含义。

## 证据边界

这些截图证明 v143 的 CI runtime 策略已经落地并被测试保护。它们不证明模型效果提升，也不代表 GitHub Actions 平台未来不会继续发布新的 action 迁移建议。

## tag 含义

`v143.0.0` 标记的是 “GitHub Actions Node 24 runtime”：仓库主动 opt in 到 Node 24 JavaScript action runtime，并把这个设置纳入本地测试，避免 Node.js 20 deprecation 提醒反复出现。

一句话总结：v143 把 CI 的平台运行时提醒收成一个小而明确的仓库契约。
