# v270 运行说明

## 本版主题

v270 把 workflow clean batch-review requirement 带到 training-scale execution handoff。此前 v269 已经能在 workflow decision 层阻断不干净的 batch review evidence；本版让 handoff validation 也读取该 requirement，并在 status 不是 `clean` 时阻断执行交接。

## 关键改动

- `build_training_scale_handoff()` 新增 `clean_batch_review_guard` 汇总，读取 workflow 与 nested decision 的 clean batch-review 字段。
- `_handoff_allowed()` 在 workflow 要求 clean batch-review 但 status 不是 `clean` 时返回 blocked。
- handoff CSV/Markdown/HTML 都展示 `decision_require_clean_batch_review` 与 `clean_batch_review_status`。
- `scripts/execute_training_scale_handoff.py` 打印 `clean_batch_review_status`，方便 CI 和人工日志直接判断。

## Tag 含义

`v270.0.0` 标记 training scale handoff clean batch gate。它表示 clean batch-review requirement 已经从 run decision 和 workflow 继续推进到执行前 handoff 边界。

## 一句话总结

v270 让不满足 clean batch-review requirement 的 workflow 不能被误交接为可执行 handoff，把自动化安全边界推进到了执行前。
