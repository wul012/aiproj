# v136 generation quality artifact split 运行说明

## 本版主题

v136 把 `generation_quality.py` 中的 JSON、CSV、Markdown、SVG、HTML 输出写入逻辑拆到 `generation_quality_artifacts.py`，让原模块继续负责读取 generation/eval 输出、计算 continuation 指标、生成 flags、summary、recommendations 和 warnings。

这是“两次左右定向拆分”中的第二刀，也是本轮拆分收口刀；后续默认不再继续做 artifact split，除非出现新的明确边界或实际功能需要。

## 关键截图

- `01-unit-tests.png`: generation quality 定向测试、compile check 和 full unittest discovery，证明拆分后指标、flags、facade 和全量回归仍可通过。

- `04-playwright-generation-quality-html.png`: 用 Playwright 打开生成的 generation quality HTML，证明浏览器页面可渲染。
- `05-docs-check.png`: README、`c/136`、项目成熟度阶段讲解索引和 source encoding hygiene 检查。

## 证据边界

`c/136` 证明的是 generation quality 的证据发布层拆分后仍能跑通：旧入口仍可导入，新 artifact 模块能写出同名产物，CLI 仍能消费同一份 quality report dict，HTML 仍有浏览器渲染证据。它不证明模型质量真实提升；它只证明已有生成质量证据的计算和发布边界更清楚。

## 清理规则

本轮临时日志、临时 fixture 和渲染脚本都放在 `tmp/`，完成验证后删除；仓库只保留 `c/136/图片` 中的最终截图和本说明。
