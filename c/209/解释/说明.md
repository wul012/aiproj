# v209 运行截图说明

本版目标是把 v208 promoted comparison 已经保留的 handoff suite boundary，继续接到 promoted baseline decision。这样最终被选中的 baseline 不只记录自己的真实 `suite_path`，也保留它从 promotion index/comparison 继承来的 handoff suite guard。

本版不重新计算 suite consistency，不改变 strict suite guard 的阻断规则，不改变 baseline scoring。真实 suite 一致性仍然来自 comparison summary；v209 只把 promoted comparison row 和 summary 里的 handoff suite guard 字段带到 decision 证据里。

关键文件：

- `src/minigpt/promoted_training_scale_decision.py`
- `scripts/decide_promoted_training_scale_baseline.py`
- `tests/test_promoted_training_scale_decision.py`
- `README.md`

这版截图要证明的事情：

1. 聚焦测试能通过，说明 selected baseline、decision summary、CSV/Markdown/HTML 和 CLI 都接住了 handoff suite guard 字段。
2. CLI smoke 能从 promoted comparison 选择 baseline，并打印 selected handoff suite status 与 handoff mismatch 总数。
3. JSON/CSV/Markdown/HTML 都能看到 `selected_handoff_suite_consistency`、`selected_handoff_suite_mismatch_count` 和 `selected_handoff_selected_suite_path`。
4. README、`c/README.md` 和代码讲解记录已经把 v209 的位置、归档位置和链路角色对齐。
