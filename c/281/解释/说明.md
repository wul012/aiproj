# v281 promoted seed handoff automation receipt

## 本版主题

v281 在 v280 的 `automation_summary` 基础上新增轻量 receipt artifact。

v280 已经把 gate 和 handoff execution 合成一个最终自动化出口，但外部脚本仍然需要解析完整 handoff JSON。v281 增加两个更小的输出：

- `promoted_training_scale_seed_handoff_automation_receipt.json`
- `promoted_training_scale_seed_handoff_automation_receipt.txt`

本版不改变 handoff 决策规则，不新增用户开关，也不改变原有 JSON/CSV/Markdown/HTML 输出。receipt 是额外的、面向 CI/脚本消费的证据。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - 新增 compact receipt builder。
  - 新增 receipt text renderer。
  - `write_promoted_training_scale_seed_handoff_outputs()` 额外写入 receipt JSON/text。

- `src/minigpt/promoted_training_scale_seed_handoff.py`
  - re-export receipt helper，保持 public API 入口统一。

- `scripts/execute_promoted_training_scale_seed.py`
  - CLI 输出 `automation_receipt_json` 和 `automation_receipt_text` 路径。

- `tests/test_promoted_training_scale_seed_handoff.py`
  - 覆盖 receipt helper、CLI 输出路径、receipt JSON/text 内容，以及 failed handoff 场景。

## Tag 含义

`v281.0.0` 表示：promoted seed handoff automation receipt。它说明 seed handoff 自动化现在除了完整 report，还有面向脚本读取的轻量 receipt。

## 一句话总结

v281 把 promoted seed handoff 的最终自动化决策复制到 compact receipt 中，让 CI 和外部脚本更容易接入，而不必解析完整报告。
