# v123 dashboard render split 运行截图说明

## 本版主题

v123 按 v110 module pressure audit 的维护路线，拆分 `dashboard.py` 的 HTML 渲染边界。artifact 发现、JSON/TXT 读取、summary payload 组装和 `write_dashboard()` 仍留在 `dashboard.py`；HTML/CSS、section builders、formatting、escaping、image/link helper 移到 `dashboard_render.py`。

本版不改变 dashboard payload、CLI 参数、输出文件名、artifact key/href 语义和 `minigpt.dashboard.render_dashboard_html` 的旧 public 导出。

## 关键改动

```text
src/minigpt/dashboard_render.py
src/minigpt/dashboard.py
tests/test_dashboard_render.py
代码讲解记录_项目成熟度阶段/138-v123-dashboard-render-split.md
```

`dashboard.py` 从 644 非空行降到 222 非空行，新增的 `dashboard_render.py` 承接 438 非空行 HTML/CSS 和渲染逻辑。拆分后 dashboard 主模块继续负责 payload 组装，新模块只负责页面发布。

## tag 含义

`v123.0.0` 表示 MiniGPT 的运行证据页面继续做低风险、定向收口：不是新增 dashboard 功能，而是把 dashboard 的页面发布层从 payload 组装层拆开，让后续维护更清楚。

## 一句话总结

v123 的价值是让 dashboard 具备清晰的“payload 组装层 / HTML 渲染层”边界，减少单文件膨胀，同时保护现有 CLI、payload、输出路径和旧导出契约。
