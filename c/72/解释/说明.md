# v72 运行截图和解释

## 本版主题

v72 新增 gated training scale run。它读取 v70 `training_scale_plan.json`，先写 v71 gate 证据，再根据 gate 结果决定是否把 variants 交给 v69 training portfolio batch runner。默认只做 dry-run；只有显式传入 `--execute` 才会执行真实训练。

本版不下载语料，不改变 v69 batch 入口，也不默认训练。它的重点是把“计划、门禁、batch”串成一个受控启动链路。

## 关键命令

```powershell
python -m unittest tests.test_training_scale_run
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts/plan_training_scale.py data --out-dir tmp\v72-smoke\scale-plan --dataset-name sample-zh --dataset-version-prefix v72
python scripts/run_training_scale_plan.py --plan tmp\v72-smoke\scale-plan\training_scale_plan.json --out-root tmp\v72-smoke\scale-run-allowed --gate-profile review
python scripts/run_training_scale_plan.py --plan tmp\v72-smoke\scale-plan\training_scale_plan.json --out-root tmp\v72-smoke\scale-run-blocked --gate-profile standard
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp\v72-smoke\scale-run-allowed\training_scale_run.html -Out c\72\图片\05-playwright-training-scale-run-html.png
```

## 证据说明

`training_scale_run.json` 是本版总控证据，记录 gate、是否 allowed、blocked reason、variants path、batch summary 和输出路径。

`gate/training_scale_gate.*` 是 v71 gate 证据，说明为什么允许或阻止。

`batch/training_portfolio_batch.*` 只会在 gate 允许时写出。blocked 情况下只有 gate 和 run summary，不会生成 batch 产物。

## Tag 说明

`v72.0.0` 表示项目新增受控启动层：从“能计划和检查训练矩阵”推进到“能按 gate 决策启动或阻止 batch dry-run/训练”。
