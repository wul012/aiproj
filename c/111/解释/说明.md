# v111 运行截图说明

## 本版主题

v111 是 registry asset split。

本版按 v110 的 module pressure audit 做第一处小步治理：`registry.py` 是 critical-size 模块，但这次不动 registry 数据契约，而是先把 HTML CSS/JavaScript 资产抽到 `registry_assets.py`。这样既能降低大模块体量，又能保持 run registry 的 JSON/CSV/SVG/HTML 输出和浏览器交互不变。

## 证据边界

这些截图证明 registry 的展示资产已经被安全抽取，并且现有 registry 行为和测试没有被破坏。
它们不证明所有大模块都已解决，也不证明 `server.py` 或 `benchmark_scorecard.py` 已完成拆分。v111 的边界是第一处低风险实际拆分：先把 registry 的 CSS/JS 从大模块里移出来。

## Tag 含义

`v111.0.0` 表示 MiniGPT 项目开始执行 v110 的模块压力治理路线，并完成 registry HTML asset split。后续类似拆分也应保持小步、契约清楚、测试先行。

## 一句话总结

v111 证明代码膨胀治理可以从清晰的展示资产边界开始，而不是直接改业务主流程。
