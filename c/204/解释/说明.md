# v204 workflow suite guard handoff 运行说明

## 本版目标

v204 把 v203 的 strict suite-consistency decision guard 接入 consolidated training-scale workflow。

v203 已经让单独的 run decision 和 promoted baseline decision 可以通过 `--require-suite-consistency` 阻断 mixed/missing suite。剩下的问题是 workflow 一键链路仍然只暴露 gate pass、batch started、readiness 等 decision 参数，无法把 suite guard 传给内部 `training_scale_run_decision`。这会导致手动 decision 能严格阻断，而一键 workflow 仍可能绕过这个策略。

本版新增 `decision_require_suite_consistency` / `--decision-require-suite-consistency`，并把 strict 状态写入 workflow summary 和 artifact。

## 关键改动

- `src/minigpt/training_scale_workflow.py`
  - 新增 `decision_require_suite_consistency` 参数。
  - 调用 `build_training_scale_run_decision()` 时传入 `require_suite_consistency`。
  - workflow report 顶层记录 `decision_require_suite_consistency` 和 `require_suite_consistency`。
  - workflow summary 输出 `decision_require_suite_consistency`、`suite_consistency`、`suite_mismatch_count`、`selected_suite_path`。
- `src/minigpt/training_scale_workflow_artifacts.py`
  - CSV 输出 strict guard 和 suite consistency。
  - Markdown/HTML stats 显示 strict guard 是否启用。
- `scripts/run_training_scale_workflow.py`
  - 新增 `--decision-require-suite-consistency`。
- `tests/test_training_scale_workflow.py`
  - 覆盖 workflow 将 strict guard 传到 nested decision artifact。

## 证据边界

本版不改变 training scale plan，不改变 suite consistency 的计算，不改变默认 workflow 行为，也不训练模型。

它只让 strict suite guard 从单独 decision CLI 进入 workflow 一键链路，避免后续用 workflow 跑实验时绕过 v203 的 clean-comparison 门禁。

## tag 含义

`v204.0.0` 表示 MiniGPT v204 workflow suite guard handoff：consolidated training-scale workflow 已经能把 strict suite-consistency guard 传递给内部 decision 层。

一句话总结：v204 让 suite-consistency strict guard 从单点 decision 能力变成 workflow 级 handoff 能力。
