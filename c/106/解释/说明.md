# v106 运行截图说明

## 本版主题

v106 是 release readiness report-utils migration。

本版把 `src/minigpt/release_readiness.py` 中语义通用的 JSON 写出、UTC 时间、HTML 转义、dict/list 归一化迁到 `report_utils`，同时保留 release readiness dashboard 自己的 panel、action、source resolution、warning 和 evidence table 语义。

## 截图说明

- `01-unit-tests.png`
  - 证明 `tests.test_release_readiness` 和 `tests.test_report_utils` 通过。
  - 同时证明 `python -m compileall -q src tests scripts` 和全量 `python -m unittest discover -s tests` 通过。

- `02-release-readiness-ready-smoke.png`

## 证据边界

这些截图证明的是 report-utils migration 没有破坏 release readiness dashboard 的核心输出和展示链路。

它们不证明模型能力提升，也不证明新的发布策略更严格。v106 是一次轻量、定向的代码质量收束。

## Tag 含义

`v106.0.0` 表示 release readiness dashboard 已经接入公共报告基础设施，但 release readiness 的业务判断仍然留在本模块内部。

## 一句话总结

v106 让发布就绪总览和上游治理证据共享同一套报告基础工具，同时保持 ready/block 判断、panel、action 和 evidence table 的原有语义。
