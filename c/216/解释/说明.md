# v216 promoted seed handoff clean evidence CLI gate 运行说明

## 本版目标

v216 把 v214/v215 的 clean-evidence readiness 从“可读字段”推进到“可选执行门禁”。

新增 CLI 参数：

```text
--require-clean-evidence
```

默认行为不变。只有显式传入这个参数时，CLI 才会要求 `seed_handoff_clean_evidence_ready` 为 `True`，否则非零退出。

## 边界

本版不改变 seed handoff 的默认 planned-mode validation，不改变 `--execute`，不改变 `_handoff_allowed()`，也不把 clean-evidence 默认变成 blocker。

它只是给自动化提供一个 opt-in gate。
