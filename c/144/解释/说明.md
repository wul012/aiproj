# v144 运行截图说明

## 本版主题

v144 修正 v143 的不充分之处。v143 通过 `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24=true` 让 CI 运行成功，但 GitHub annotation 仍然提示 `actions/checkout@v4` 和 `actions/setup-python@v5` 本身 target Node 20，只是被强制运行在 Node 24 上。

v144 改为升级 action major：`actions/checkout@v6` 和 `actions/setup-python@v6`。远端 metadata 显示这两个版本都使用 `node24`。

## 关键改动

- `.github/workflows/ci.yml`：升级 `actions/checkout@v6` 和 `actions/setup-python@v6`。
- `.github/workflows/ci.yml`：删除 `FORCE_JAVASCRIPT_ACTIONS_TO_NODE24`。
- `tests/test_ci_workflow.py`：更新测试，守住 v6 action major，并断言不再使用 v4/v5 或 force-runtime env。
- `README.md`、`c/README.md`、`代码讲解记录_项目成熟度阶段/158-v144-github-actions-node24-native-actions.md`：记录 v143 的局限和 v144 的修正。

## 证据边界

这些截图证明 v144 的 CI action 版本升级闭环，不证明模型能力提升，也不代表未来无需继续维护 action major。它只是把 GitHub Actions Node 20 deprecation annotation 从 force-runtime 方案推进到原生 action 版本方案。

## tag 含义

`v144.0.0` 标记的是 “GitHub Actions Node 24 native actions”：CI workflow 使用 target Node 24 的 action major，并删除 v143 的强制 runtime env。

一句话总结：v144 让 CI 真正切到 Node 24 native actions，而不是只把 Node 20 actions 强制跑在 Node 24 上。
