# v91 training scale plan report utility migration 运行截图说明

## 本版主题

v91 继续 `report_utils` 收口路线，把 v70 的 `training_scale_plan.py` 迁移到公共报告工具上。

这次推进判断是合理的：aiproj 当前更需要轻量、定向的质量优化，而不是架构级重构。v91 只回收 planner 里的重复 JSON 写出、命令展示、Markdown/HTML 转义和 list/dict 归一化 helper，不改变 scale tier、variants、token budget、tiny corpus block size 或 batch handoff 语义。

迁移后，plan、gate、run、comparison、decision 五层训练规模证据共享同一套报告基础设施。

## 关键代码

```text
src/minigpt/training_scale_plan.py
src/minigpt/report_utils.py
tests/test_training_scale_plan.py
tests/test_report_utils.py
scripts/plan_training_scale.py
```

`training_scale_plan.py` 仍然负责语料扫描、质量摘要、scale tier 判断、variant 矩阵、batch handoff command 和 JSON/CSV/Markdown/HTML 输出。变化只在报告基础 helper 的来源。
