# v191 运行截图和解释

## 本版主题

v191 是 training portfolio pair baseline evidence。

本版把已有 `scripts/pair_batch.py` 接入真实 training portfolio 执行链路，让 `generation_quality -> pair_batch -> benchmark_scorecard` 成为连续步骤。由于当前单次 portfolio 只有一个新训练出的 checkpoint，pair batch 默认使用同一 checkpoint 左右自检，并在 scorecard 中明确标记为 `same_checkpoint_baseline`。

## 关键改动

- 更新 `src/minigpt/training_portfolio.py`，新增 `pair_batch` step 和 `pair_batch`、`pair_batch_html` artifacts。
- 更新 `src/minigpt/benchmark_scorecard.py`，识别 same-checkpoint pair baseline，并把 pair consistency / pair delta stability 的同 checkpoint 分数上限设为 90。
- 更新 `src/minigpt/benchmark_scorecard_artifacts.py`，让 Markdown 和 HTML 输出显示 `Pair comparison mode`。
- 更新 `tests/test_training_portfolio.py`、`tests/test_benchmark_scorecard.py` 和 `tests/test_benchmark_scorecard_artifacts.py`，覆盖 plan 顺序、same-checkpoint 标记、路径优先判断和输出展示。
- 更新 README、`c/README.md` 和项目成熟度阶段代码讲解索引。

## tag 含义

`v191.0.0` 表示 MiniGPT 第 191 个可回看版本：training portfolio 能自动生成 scorecard 需要的 pair batch 证据，并且 scorecard 会把同 checkpoint pair 明确标为 reproducibility baseline，而不是模型提升证据。运行截图与解释归档在 `c/191`。

## 一句话总结

v191 把真实训练组合链路的 pair 证据补齐，同时保留“同 checkpoint 自检只是可复现基线”的工程边界。
