# v112 运行截图说明

## 本版主题

v112 是 pair artifact split。

本版按 v110 的 module pressure audit 继续治理 `server.py`，但仍然保持轻量：不动 HTTP 路由、不动模型推理、不动 request history，只把 pair generation artifact 的 JSON/HTML 保存和渲染逻辑抽到 `pair_artifacts.py`。

## 证据边界

这些截图证明 pair artifact 保存边界已经被安全抽取，并且 server API 行为没有被破坏。
它们不证明 `server.py` 已经完全拆完，也不证明 request history、streaming 或 checkpoint discovery 已完成治理。v112 的边界是先拆一个输入输出清楚的证据写入模块。

## Tag 含义

`v112.0.0` 表示 MiniGPT 项目完成 server 方向的第一处低风险拆分：pair generation artifact 保存和 HTML 渲染进入独立模块。后续如果继续治理 `server.py`，仍应优先选择契约清楚、测试能保护的边界。

## 一句话总结

v112 证明服务端大文件治理可以先从证据保存层推进，而不是直接改 HTTP 路由或模型推理主流程。
