# v244 release readiness coverage dashboard 运行截图说明

## 本版主题

v244 把 coverage gate 证据接入 release readiness dashboard。

这版让 readiness 看板能读取 `test_coverage_report.json`，也能在报告缺失时回退到 release bundle 的 coverage summary/context。它不改变 gate 规则，只让发布看板能显示 coverage 状态。

## 截图清单

### 01-release-readiness-tests.png

该截图来自：

```text
python -B -m unittest tests.test_release_readiness tests.test_release_readiness_comparison -q
```

结果：

```text
Ran 15 tests
OK
```

它证明 readiness dashboard 新增的 coverage panel、summary 字段、bundle fallback 和 Markdown 输出都被测试覆盖，同时没有破坏 readiness comparison。

### 02-coverage-gate-smoke.png

该截图来自：

```text
python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-v244-smoke --fail-under 80
```

关键结果：

```text
Ran 481 tests
OK
status=pass
line_coverage_percent=90.17
fail_under=80.0
coverage_gap=0.0
```

它证明本版 readiness 消费的是当前测试集合真实生成的 coverage report。

### 04-full-unittest.png

该截图来自：

```text
python -B -m unittest discover -s tests -q
```

结果：

```text

```

它证明 v244 的 dashboard 改动没有破坏全项目测试集合。

## 证据边界

v244 证明的是 release readiness dashboard 能看见 coverage gate 证据。

它不证明模型生成质量，也不替代 benchmark scorecard、generation quality 或真实 checkpoint 对比。coverage 仍然只是测试治理证据。

## 一句话总结

v244 让最终 readiness 看板从“发布链路完整”进一步变成“覆盖率门禁可见”，发布评审时可以直接看到 coverage 状态、百分比、阈值和缺口。
