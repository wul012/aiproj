# v124 playground asset module split 运行截图说明

## 本版主题

v124 继续执行 v110 module pressure audit 的维护路线，把 `playground_assets.py` 里已经稳定的 CSS 与 JavaScript 资产拆成两个专用模块：

```text
playground_assets.py -> 旧入口 facade
playground_style.py  -> playground CSS 和响应式布局
playground_script.py -> playground JavaScript、page_data 注入和本地 API UI 交互
```

本版不改变 playground 静态页面结构、本地 API endpoint 名称、`render_playground_html()` 输出契约、`scripts/build_playground.py` CLI，也不移除 `minigpt.playground_assets.playground_style/playground_script` 旧导出。

## 关键改动

```text
src/minigpt/playground_assets.py
src/minigpt/playground_style.py
src/minigpt/playground_script.py
tests/test_playground_asset_modules.py
代码讲解记录_项目成熟度阶段/139-v124-playground-asset-module-split.md
```

`playground_assets.py` 从 711 非空行降到 4 非空行，只保留兼容导出。`playground_style.py` 承接 CSS 布局；`playground_script.py` 承接 checkpoint selector、stream generate、request history、pair generation 和 page data 序列化脚本。

## tag 含义

`v124.0.0` 表示 MiniGPT 的 playground UI 资产层完成进一步收口：不是新增 UI 功能，而是把已经抽出的 playground 资产再拆成 CSS/JS 专用模块，让后续维护能分别处理样式和交互脚本。

## 一句话总结

v124 的价值是让 playground 资产从“一个大静态资产文件”推进到“兼容 facade + 样式模块 + 脚本模块”的边界，让 UI 证据页面继续保持可运行，同时降低后续维护成本。
