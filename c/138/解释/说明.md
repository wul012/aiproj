# v138 benchmark scorecard flag taxonomy 运行说明

## 本版主题

v138 把 v137 的 generation-quality flag taxonomy 接入 benchmark scorecard。现在 run 级 scorecard 不只显示 generation quality 的 pass/warn/fail，还能显示：

```text
generation_quality_total_flags
generation_quality_dominant_flag
generation_quality_worst_case
generation-quality component raw_score / flag_penalty
```

这让评估入口能直接说明“模型输出弱项集中在哪里”，而不是只给一个粗略状态。

## 关键改动

- `src/minigpt/benchmark_scorecard.py` 读取 `summary.flag_summary` 并上卷 total flags、dominant flag、worst case。
- generation-quality component 新增 flag-density penalty，避免多 flag 的 warn case 和单 flag 的 warn case 被完全等价看待。
- `src/minigpt/benchmark_scorecard_artifacts.py` 在 Markdown/HTML 中展示 generation flag 字段。
- `scripts/build_benchmark_scorecard.py` 的终端输出新增 total flags、dominant flag 和 worst case。
- benchmark scorecard 测试覆盖 raw score、flag penalty、dominant flag、worst case 和 artifact 展示。

## 证据边界

`c/138` 证明的是：generation quality taxonomy 已经进入 benchmark scorecard，并可影响 run 级 generation-quality component 分数。

它不证明模型质量已经提升。后续如果要证明能力提升，需要用真实 checkpoint、多轮训练、固定 benchmark 和 scorecard comparison 做跨 run 证据。
