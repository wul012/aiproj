# v70 运行截图和解释

## 本版主题

v70 新增 training scale planner。它在真正运行多组训练之前先扫描训练文本来源，判断语料规模，读取数据质量状态，估算各训练 variant 的 token budget，并输出可交给 v69 batch runner 使用的 `training_scale_variants.json`。

本版不执行真实训练，不下载新语料，也不宣称模型能力提升。它的价值是把“准备跑更大中文语料”拆成可检查的前置计划。

## 关键命令

```powershell
python -m unittest tests.test_training_scale_plan
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts/plan_training_scale.py data --out-dir tmp\v70-smoke\scale-plan --dataset-name sample-zh --dataset-version-prefix v70
python scripts/plan_training_scale.py tmp\v70-smoke\tiny --out-dir tmp\v70-smoke\tiny-plan --max-variants 1
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp\v70-smoke\scale-plan\training_scale_plan.html -Out c\70\图片\05-playwright-training-scale-plan-html.png
```

## 证据说明

`training_scale_plan.json` 是完整规划证据，记录 dataset scale、quality、source detail、variant matrix、recommendations 和 batch command。

`training_scale_variants.json` 是交接产物，不是最终训练证据。它会被 v69 的 `run_training_portfolio_batch.py --variants` 消费，真正训练后的证据仍应由 batch runner 写出。

## Tag 说明

`v70.0.0` 表示项目新增训练规模规划层：从“能批量运行训练 portfolio”推进到“能在训练前根据语料规模和质量生成 batch-compatible variant matrix”。
