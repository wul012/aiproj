# v128 registry artifact split 运行截图说明

## 本版主题

v128 把 registry 的输出写入层从 `registry_render.py` 中拆出来，形成 `registry_artifacts.py`。

这版的重点是把“HTML 渲染”和“JSON / CSV / SVG / HTML 文件写出”分成两个边界，避免 `registry_render.py` 继续同时承担渲染、写文件和兼容导出的三重职责。

它不改 registry schema，不改 registry HTML 交互，也不改注册脚本的使用方式。

## 关键改动

- 新增 `src/minigpt/registry_artifacts.py`
- 精简 `src/minigpt/registry_render.py`
- 更新 `src/minigpt/registry.py`
- 新增 `tests/test_registry_split.py`
- 更新 `README.md`、`代码讲解记录_项目成熟度阶段/README.md`、`c/README.md`

## tag 含义

`v128.0.0` 表示这次不是新增业务能力，而是一次治理层的收口：让 registry renderer 只做渲染，让 artifact writer 只做写出。

## 一句话总结

v128 让 registry 的渲染层和证据写出层各归其位，继续把 MiniGPT 的治理链条往更清楚、更可维护的方向推了一步。
