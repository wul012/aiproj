# v186 运行截图和解释

## 本版主题

v186 是 real training run evidence。

本版不继续拆分工具模块，而是新增一条功能闭环：把 `scripts/train.py` 真实产出的 checkpoint 训练目录，转换成可审计的训练运行证据报告。报告会检查 checkpoint、tokenizer、train config、metrics、run manifest、history summary、sample 和 eval suite 状态，并输出 JSON/CSV/Markdown/HTML。

## 关键改动

- 新增 `src/minigpt/training_run_evidence.py`，负责读取真实 run 目录、生成 checks、summary、training/data/sample/artifact sections。
- 新增 `src/minigpt/training_run_evidence_artifacts.py`，负责写出 JSON/CSV/Markdown/HTML evidence。
- 新增 `scripts/build_training_run_evidence.py`，提供命令行入口。
- 更新 `src/minigpt/training_portfolio.py`，在 `train` 和 `eval_suite` 之间插入 `training_run_evidence` 步骤。
- 新增/更新测试，确保完整 run 是 ready、缺 checkpoint 会 blocked，portfolio 顺序包含新 evidence step。

## tag 含义

`v186.0.0` 表示 MiniGPT 第 186 个可回看版本：真实训练 run 目录已经可以被转换为 promotion-review evidence，且 training portfolio 会在真实训练之后自动规划这一步。运行截图与解释归档在 `c/186`。

## 一句话总结

v186 把项目从连续拆分收口切回功能推进，让真实 checkpoint、metrics、manifest 和 sample 可以进入同一份训练证据报告。
