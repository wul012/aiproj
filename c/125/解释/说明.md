# v125 server generator split 运行截图说明

## 本版主题

v125 继续执行 v110 module pressure audit 的维护路线，把本地推理 server 里的 `MiniGPTGenerator` 抽成独立模块：

```text
server.py           -> HTTP routes, SSE, request logging, pair generation orchestration
server_generator.py -> PyTorch checkpoint/tokenizer loading, device selection, generate/stream
```

本版不改变 `/api/health`、`/api/generate`、`/api/generate-stream`、`/api/generate-pair`、`/api/generate-pair-artifact` 等路由，不改变 SSE event 名称，不改变 request-history 日志字段，也不移除 `from minigpt.server import MiniGPTGenerator` 旧导出。

## 关键改动

```text
src/minigpt/server.py
src/minigpt/server_generator.py
tests/test_server_generator.py
代码讲解记录_项目成熟度阶段/140-v125-server-generator-split.md
```

`server.py` 从 678 非空行降到 588 非空行，继续负责 handler、路由、日志和响应写入。`server_generator.py` 承接 95 非空行的真实模型加载与 token 生成逻辑。拆分后，module pressure smoke 显示 warn 模块从 2 个降到 1 个，新的最大压力点转移到 `comparison.py`。

## tag 含义

`v125.0.0` 表示 MiniGPT 的本地推理服务继续做低风险收口：不是重写 server，也不是改变推理 API，而是把 PyTorch generator 从 HTTP 编排层里拆出来。

## 一句话总结

v125 的价值是让本地推理 server 具备更清晰的“HTTP 编排层 / 模型生成层”边界，降低服务端大文件压力，同时保持所有现有路由、SSE、日志和旧导出契约不变。
