# v261 promoted seed handoff batch review propagation 运行说明

## 本版主题

v261 把 v260 promoted next-cycle seed 中的 handoff selected batch review/blocker context 继续传到 promoted training scale seed handoff。这样验证或执行 next plan command 的 handoff report 不只显示命令、suite alignment、clean-evidence readiness 和 plan artifacts，也会保留这个 handoff 是否来自仍带 selected batch review/blocker 风险的 seed。

本版不改变 seed handoff 的 `planned`、`completed`、`blocked`、`failed`、`timeout` 判定，也不改变 suite alignment 或 clean-evidence requirement 规则。batch review/blocker context 只是风险提示和 recommendation。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff.py`：summary 从 `baseline_seed.handoff_batch_review` 中读取 selected batch review/blocker 字段、comparison-ready totals 和 blocker reasons，并把它们传给 recommendations。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`：CSV/Markdown/HTML 输出新增 selected handoff batch review/blocker 字段和 comparison-ready batch review/blocker counts。
- `scripts/execute_promoted_training_scale_seed.py`：CLI 输出新增 selected handoff batch review/blocker 字段和 comparison-ready blocker reasons。
- `tests/test_promoted_training_scale_seed_handoff.py`：新增 seed -> seed handoff 的 batch review propagation 测试，确认 planned handoff 仍保持原有状态，但风险字段进入 JSON/CSV/Markdown/HTML/CLI/recommendations。
- `README.md` 和 `代码讲解记录_项目成熟度阶段/275-v261-promoted-seed-handoff-batch-review-propagation.md`：同步说明本版目标、边界、输入输出和证据链角色。
