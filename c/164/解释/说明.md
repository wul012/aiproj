# v164 server request history endpoint split 运行说明

## 本版主题

v164 把 `/api/request-history` 和 `/api/request-history-detail` 的 HTTP endpoint 处理，从 `src/minigpt/server.py` 拆到 `src/minigpt/server_request_history.py`。

这版不改变 request-history JSON、CSV、detail payload 或错误状态契约，只收窄 server GET 路由职责。

## 关键改动

- 新增 `src/minigpt/server_request_history.py`，集中保存 request-history endpoint 的 query 解析、payload 构造、CSV 分支和 400/404 映射。
- `src/minigpt/server.py` 的 request-history 分支改为调用 helper。
- 新增 `tests/test_server_request_history.py`，直接覆盖 helper 的响应行为。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引同步到 v164。
