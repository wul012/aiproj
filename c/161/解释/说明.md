# v161 dataset card artifact split 运行说明

## 本版主题

v161 把 dataset card 的 JSON、Markdown 和 HTML 写出/渲染逻辑，从 `src/minigpt/dataset_card.py` 拆到 `src/minigpt/dataset_card_artifacts.py`。

这是一版 contract-preserving refactor。`build_dataset_card()` 的 card schema 不变，`scripts/build_dataset_card.py` 的调用方式不变，旧模块里的 writer/renderer 导出也保持兼容。

## 关键改动

- 新增 `src/minigpt/dataset_card_artifacts.py`，集中保存 dataset card 的 artifact 写出、Markdown/HTML 渲染和输出 artifact 标记。
- 更新 `src/minigpt/dataset_card.py`，保留 dataset evidence、summary、provenance、quality、artifact、recommendation 构建，导入并继续导出 artifact 函数。
- 更新 `tests/test_dataset_card.py`，增加 facade identity 测试，确认旧导出指向新 artifact 实现。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引同步到 v161。

## 证据边界

`c/161/图片` 和本说明是运行证据归档，不是后续程序消费的机器输入。后续如果要复用 dataset card artifact 写出，应依赖：

- `src/minigpt/dataset_card_artifacts.py`
- `src/minigpt/dataset_card.py`
- `tests/test_dataset_card.py`
- README 中的 v161 版本说明

## Tag 含义

`v161.0.0` 表示 MiniGPT dataset card 完成 artifact 输出层拆分：JSON/Markdown/HTML 进入独立模块，同时 dataset card schema、CLI 和旧导出保持兼容。

## 一句话总结

v161 让 dataset card 的“数据治理证据汇总”和“报告发布”分开，降低当前最大模块压力，同时保持旧入口稳定。
