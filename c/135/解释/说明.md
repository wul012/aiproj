# v135 release gate artifact split 运行说明

## 本版主题

v135 把 `release_gate.py` 中的 JSON、Markdown、HTML 输出写入逻辑拆到 `release_gate_artifacts.py`，让原模块继续负责 policy profile 解析、release bundle 检查、gate summary、recommendations 和 exit code。

这是“两次左右定向拆分”中的第一刀，属于轻量收口型优化；不改变 `gate_report.json` schema，不改变输出文件名，不改变 `scripts/check_release_gate.py` 的 CLI 行为，也不改变旧的 `minigpt.release_gate` 导入方式。

## 关键截图

- `01-unit-tests.png`: release gate 定向测试、compile check 和 full unittest discovery，证明拆分后 policy/check/summary、facade 和全量回归仍可通过。
- `02-maintenance-smoke.png`: maintenance batching/module pressure smoke，证明新增 `release_gate_artifacts.py` 后模块压力仍为 pass。
- `03-release-gate-smoke.png`: 使用测试 fixture 生成 release gate JSON/Markdown/HTML，并检查 facade identity、Checks 表格和输出文件存在。
- `04-playwright-release-gate-html.png`: 用 Playwright 打开生成的 release gate HTML，证明浏览器页面可渲染。
- `05-docs-check.png`: README、`c/135`、项目成熟度阶段讲解索引和 source encoding hygiene 检查。

## 证据边界

`c/135` 证明的是 release gate 的证据发布层拆分后仍能跑通：旧入口仍可导入，新 artifact 模块能写出同名产物，CLI 仍能消费同一份 gate dict，HTML 仍有浏览器渲染证据。它不证明模型质量提升，也不代表 release gate 能替代真实生产发布流程；它只是本地治理证据链的一层。

## 清理规则

本轮临时日志、临时 fixture 和渲染脚本都放在 `tmp/`，完成验证后删除；仓库只保留 `c/135/图片` 中的最终截图和本说明。
