# v205 workflow handoff suite guard evidence 运行说明

## 本版目标

v205 的目标是把 v204 已经进入 consolidated workflow 的 strict suite-consistency guard，继续带到 controlled handoff 层。

v204 解决的是 workflow 一键入口不能绕过 strict guard 的问题，但 handoff 层原本只展示执行命令、执行状态和 artifact 计数，没有把 `decision_require_suite_consistency`、`suite_consistency`、`suite_mismatch_count`、`selected_suite_path` 这些 suite 边界证据一起带出来。这样人工审阅 `--execute` 命令时，suite 约束会变得不够显眼。

本版把这组证据写进 handoff JSON/CSV/Markdown/HTML，让 reviewed execute command 也保留同一条 suite boundary。

## 关键改动

- `src/minigpt/training_scale_handoff.py`
  - 新增 `suite_guard` 顶层对象。
  - `_summary()` 提升 `decision_require_suite_consistency`、`suite_consistency`、`suite_mismatch_count`、`selected_suite_path`。
  - CSV/Markdown/HTML 同步展示 strict suite guard 信息。
  - `_recommendations()` 在 strict 模式下会提醒 suite consistency 不是 `consistent` 时先修正比较边界。
- `tests/test_training_scale_handoff.py`
  - 新增 `test_carries_suite_guard_from_workflow_and_decision`。
  - 原有 planned / execute / failed / HTML escape 测试继续保留。
- `scripts/execute_training_scale_handoff.py`
  - 继续作为 handoff 的唯一执行入口，输出不变，但现在能把 suite guard 证据带出来。

## 证据边界

本版不改变 strict guard 的决策逻辑，不改变 suite consistency 的计算，也不训练模型。

它只负责把 workflow 的 strict suite evidence 继续带到 handoff 审阅面，避免用户拿到 execute command 时丢失“这次是不是 clean comparison”的上下文。

## 一句话总结

v205 让 strict suite-consistency 证据从 workflow 继续延伸到 controlled handoff 报告，保持 reviewed execute command 的 suite 边界可见。
