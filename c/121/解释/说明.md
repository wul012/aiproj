# v121 maturity artifact split 运行截图说明

## 本版主题

v121 按 v110 module pressure audit 的维护路线，拆分 `maturity.py` 的 artifact 输出边界。成熟度规则、上下文读取、能力矩阵和推荐项仍留在 `maturity.py`；JSON/CSV/Markdown/HTML 写入和展示 helper 移到 `maturity_artifacts.py`。

本版不改变 maturity schema、CLI 参数、输出文件名、页面字段和 `minigpt.maturity` 的旧 public artifact 导出。

## 关键改动

```text
src/minigpt/maturity_artifacts.py
src/minigpt/maturity.py
tests/test_maturity_artifacts.py
代码讲解记录_项目成熟度阶段/136-v121-maturity-artifact-split.md
```

`maturity.py` 从 697 非空行降到 374 非空行，新增的 `maturity_artifacts.py` 承接 347 非空行输出和渲染逻辑。拆分后 maturity 主模块继续负责 summary 计算，新模块只负责 artifact 发布。

## tag 含义

`v121.0.0` 表示 MiniGPT 的项目成熟度治理链继续做低风险、定向收口：不是新增报告能力，而是把成熟度 summary 的证据发布层从成熟度计算层拆开，让后续维护更清楚。

## 一句话总结

v121 的价值是让 maturity summary 也具备清晰的“计算层 / artifact 层”边界，减少单文件膨胀，同时保护现有 CLI、schema 和旧导出契约。
