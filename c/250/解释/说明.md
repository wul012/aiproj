# v250 portfolio comparison coverage review 运行截图说明

## 本版主题

v250 把 coverage governance 信号接入 training portfolio comparison。

v246-v249 已经证明 coverage regression 可以从 release readiness comparison 传到 maturity narrative。v250 继续向下游推进：当多个 training portfolio 需要比较并选出最佳分数基线时，如果最佳分数 run 的 maturity narrative 带有 `coverage-regressed` 或 coverage regression count，比较报告会生成 blocker review action。

### 02-training-portfolio-coverage-smoke.png

该截图来自一个临时 portfolio smoke。

smoke 构造两个 portfolio：

```text
base: score=82, release trend=stable
candidate: score=92, release trend=coverage-regressed, coverage regression count=2
```

关键输出：

```text
best_score_name=candidate
best_score_release_trend=coverage-regressed
coverage_regression_count=1
review_action_count=1
blocker_action_count=1
first_action_reason=best_score_coverage_regressed
```

它证明最佳分数 candidate 不会因为分数高就直接成为 clean baseline，而是被 coverage regression review 阻断。

## 证据边界

v250 不改变 release readiness、registry、maturity summary 或 maturity narrative 的 coverage 判断规则。

它只让 training portfolio comparison 消费已有 maturity narrative coverage 字段，并把风险转成 portfolio-level review action。

## 一句话总结

v250 让 coverage regression 从“叙事层可见”推进到“基线选择会被阻断”，使最佳分数不再绕过 release-readiness 覆盖率回退证据。
