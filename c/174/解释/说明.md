# v174 training scale gate artifact split 运行说明

## 本版目标

v174 把 `training_scale_gate.py` 里的 JSON、CSV、Markdown 和 HTML 产物写出拆到 `training_scale_gate_artifacts.py`。这一版解决的是训练规模 gate 模块同时承担政策判定和证据渲染的问题。

本版明确不改政策 profile、plan loading、gate check 构建、overall status 汇总、recommendations、`scripts/check_training_scale_gate.py` 用法和旧的 `minigpt.training_scale_gate` facade 导出。

## 关键改动

- `src/minigpt/training_scale_gate_artifacts.py`：新增 artifact 模块，负责 `write_training_scale_gate_json()`、`write_training_scale_gate_csv()`、`render_training_scale_gate_markdown()`、`render_training_scale_gate_html()` 和 `write_training_scale_gate_outputs()`。
- `src/minigpt/training_scale_gate.py`：保留 `POLICY_PROFILES`、`load_training_scale_plan()`、`build_training_scale_gate()`、`_build_checks()`、`_status_summary()` 和 recommendations 逻辑，并从新 artifact 模块 re-export 旧 writer/render 入口。
- `tests/test_training_scale_gate.py`：新增 facade identity 断言，确认旧模块和新 artifact 模块指向同一组函数对象。
- `README.md`、`c/README.md` 和项目成熟度阶段讲解索引：记录 v174 的版本能力、证据位置和讲解入口。

## 运行验证

```powershell
python -B -m unittest tests.test_training_scale_gate -v
python -B scripts/check_maintenance_batching.py --out-dir runs/maintenance-v174
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v174
python -B -m unittest discover -s tests -v
```

定向测试覆盖 review/standard profile、tiny corpus、quality warning、missing baseline、token budget、HTML escaping、产物写出和 facade identity。全量 unittest 确认该拆分没有破坏训练规模后续 run/comparison/decision/workflow 链路。

## tag 含义

`v174.0.0` 表示 MiniGPT 的 training scale gate artifact split 版本。这个 tag 对应的能力是：gate 政策和检查逻辑仍由主模块生成，而多格式发布产物由独立 artifact 模块负责。

一句话总结：v174 把训练规模 gate 的“政策检查”和“证据发布”分开，让后续调整报告样式不再挤占 gate 判定逻辑。
