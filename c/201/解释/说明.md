# v201 promoted seed standard suite carryover 运行说明

## 本版目标

v201 把 v200 的 `standard-zh` 评测集选择继续推进到 promoted seed 和 seed handoff。

v198-v200 已经让标准中文 benchmark suite 可以进入 eval、portfolio、batch、scale plan、gated run 和 workflow。但从 promoted baseline 生成下一轮 seed 时，旧命令没有继承 suite，下一轮可能回到默认 `data/eval_prompts.json`。本版让 selected promoted run 的 `suite_path=builtin:standard-zh` 进入 seed JSON、seed artifact、下一轮 `plan_training_scale.py` 命令，以及 seed handoff 生成的 plan report。

## 关键改动

- `src/minigpt/promoted_training_scale_seed.py`
  - 从 selected training scale run 读取 `suite_mode`、`suite_name`、`suite_path`。
  - 下一轮 seed 默认继承 selected run 的 suite。
  - 支持 `--suite` / `--suite-name` 人工覆盖。
  - `--suite-name default` 明确回到默认文件 suite。
- `src/minigpt/promoted_training_scale_seed_artifacts.py`
  - CSV、Markdown、HTML 显示 baseline suite、next suite 和 suite source。
- `src/minigpt/promoted_training_scale_seed_handoff.py`
  - handoff summary 同时显示 seed suite 与实际生成 plan 的 suite。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - handoff 报告展示 `seed_suite_path`、`plan_suite_path` 和 plan suite 详情。
- `scripts/build_promoted_training_scale_seed.py`
  - 新增 `--suite` / `--suite-name {default,standard-zh}`。
- `scripts/execute_promoted_training_scale_seed.py`
  - CLI 输出新增 `seed_suite_path` 和 `plan_suite_path`。
- `src/minigpt/training_scale_plan.py`
  - `suite_name="default"` 解释为默认文件 suite。
- `src/minigpt/training_portfolio.py`
  - portfolio API 使用同样的 default suite 语义。
- `tests/test_promoted_training_scale_seed.py`
  - 覆盖 suite 继承和 default 覆盖。
- `tests/test_promoted_training_scale_seed_handoff.py`
  - 覆盖执行 seed 后生成的 plan 仍保持 `builtin:standard-zh`。
- `tests/test_training_scale_plan.py`
  - 覆盖 scale plan 的 default suite-name 语义。
- `tests/test_training_portfolio.py`
  - 覆盖 portfolio 的 default suite-name 语义。

## 证据边界

本版不训练模型，不证明 `standard-zh` 上的模型质量提升。它证明的是 promoted baseline 后续的 next-cycle seed 不会丢失评测集选择，跨轮训练规模实验可以继续沿用同一个 benchmark 基准。

## tag 含义

`v201.0.0` 表示 MiniGPT v201 promoted seed standard suite carryover：标准中文 benchmark suite 选择已经从 training scale workflow 继续进入 promoted seed 和 seed handoff。

一句话总结：v201 把标准中文评测集从“本轮 scale workflow 的选择”变成“promoted baseline 下一轮 seed 的继承证据”。
