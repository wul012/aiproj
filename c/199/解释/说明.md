# v199 portfolio standard suite selection 运行说明

## 本版目标

v199 把 v198 新增的标准中文 benchmark suite 接进训练 portfolio 和 batch 工作流。

v198 让 eval-suite 和 pair-batch 可以直接使用 `--suite-name standard-zh`；但训练 portfolio 仍然只能把 suite 当文件路径传入。本版新增 `suite_name` 计划字段，让真实训练链路的 eval-suite 和 pair-batch 两个步骤都能直接使用内置标准套件。

## 关键改动

- `src/minigpt/training_portfolio.py`
  - `build_training_portfolio_plan()` 新增 `suite_name`。
  - 计划中新增 `suite` 对象，记录 `mode/name/path`。
  - 兼容字段 `suite_path` 在内置套件时记录为 `builtin:standard-zh`。
  - eval-suite 和 pair-batch 步骤在内置套件时使用 `--suite-name standard-zh`。
- `src/minigpt/training_portfolio_batch.py`
  - batch 公共配置新增 `suite_name`。
  - variant override 支持 `suite_name` 和 `suite_path`。
  - variant 明确写 `suite_path` 时会清掉继承的 `suite_name`，避免二者冲突。
- `scripts/run_training_portfolio.py`
  - 新增 `--suite-name {default,standard-zh}`。
- `scripts/run_training_portfolio_batch.py`
  - 新增 `--suite-name {default,standard-zh}`。
- `tests/test_training_portfolio.py`
  - 覆盖单条 portfolio 的内置标准套件计划和互斥校验。
- `tests/test_training_portfolio_batch.py`
  - 覆盖 batch/variant 级 suite 传递和 file-suite override。

## 证据边界

本版不训练模型，也不证明标准套件上的模型效果提升。它证明的是 workflow 能把标准套件选择传到训练后的 eval-suite 和 pair-batch 步骤，后续真实 checkpoint 比较可以少一层手动拼接。

## tag 含义

`v199.0.0` 表示 MiniGPT v199 portfolio standard suite selection：训练 portfolio 和 batch 计划都能直接选择标准中文 benchmark suite，并把这个选择写入计划证据。

一句话总结：v199 把标准中文 benchmark suite 从独立评估工具接入训练 portfolio 链路，让后续真实训练比较可以用同一套标准输入闭环。
