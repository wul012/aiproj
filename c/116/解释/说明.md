# v116 运行截图说明：Registry Data/Render Split

## 本版主题

v116 把原本厚重的 `registry.py` 拆成 `registry_data.py`、`registry_render.py` 和一个 兼容 facade。

这版不改变 registry 的 CLI、公开导入、JSON/CSV/SVG/HTML 输出和 HTML 交互行为，只把职责边界拆清楚：数据层负责发现 run、读取证据、汇总排行榜；渲染层负责把 registry 字典写成文件和页面。

## 关键代码

- `src/minigpt/registry_data.py`
  - 新增 registry 数据层，负责 `RegisteredRun`、run discovery、artifact reader、summary、leaderboard 和 registry-level counts。
- `src/minigpt/registry_render.py`
  - 新增 registry 输出层，负责 JSON/CSV/SVG/HTML 写出和 HTML/SVG helper。
- `src/minigpt/registry.py`
  - 保留旧 public API 的兼容入口，显式导出数据层和渲染层函数。
- `tests/test_registry_split.py`
  - 新增直接模块测试和 facade parity 测试。

## 标签含义

`v116.0.0` 表示 MiniGPT 完成 registry data/render split。它延续 v109-v115 的维护收口路线，把 registry 主体从大文件推进到数据汇总、输出渲染和兼容入口分离的结构。
