# v227 scorecard eval coverage readiness consumption 运行说明

## 本版目标

v227 把 v226 的 eval-suite `coverage` readiness 接入 benchmark scorecard。

它解决的问题是：旧 scorecard 的 eval coverage 组件只按 case 数计分，5 个 case 就可能满分；现在如果 eval report 已经提供 comparison readiness，scorecard 会读取它，避免把“能跑但不适合对比”的 suite 当成强模型质量证据。

## 边界

本版不改变 eval suite 生成逻辑，不改变 scorecard schema version，不改变旧 eval artifact 的兼容评分。

没有 `coverage` 字段的历史 `eval_suite.json` 仍走原来的 case-count 逻辑。

## 一句话总结

v227 让 benchmark scorecard 不再只看 eval case 数，而是能利用 eval suite readiness 判断这份评估是否适合支撑 checkpoint 质量对比。
