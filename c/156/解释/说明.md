# v156 server checkpoint split 运行说明

## 本版主题

v156 把本地推理服务中 checkpoint discovery、health payload、model-info payload、checkpoint list 和 checkpoint comparison 的纯逻辑拆到 `src/minigpt/server_checkpoints.py`。

这一版继续 v155 后的 server 收口路线：v155 已经把 request-history 事件构造拆到 `server_logging.py`，v156 继续把 checkpoint metadata 和 comparison payload 从 `server_contracts.py` 里拆出。外部 `/api/health`、`/api/model-info`、`/api/checkpoints`、`/api/checkpoint-compare` 和旧导出不变。

## 关键改动

- 新增 `src/minigpt/server_checkpoints.py`，集中保存 `CheckpointOption`、checkpoint discovery、health/model-info/checkpoint list/checkpoint comparison payload 和 metadata run dir 解析。
- 更新 `src/minigpt/server_contracts.py`，保留请求解析、SSE、pair payload 与 safety profile，同时从新模块导入 checkpoint helper，并通过 wrapper 保持默认 `InferenceSafetyProfile()` 行为。
- 新增 `tests/test_server_checkpoints.py`，覆盖 checkpoint 纯函数链路和 `server_contracts.py`、`server.py` facade 兼容导出。
- README、`c/README.md` 和项目成熟度阶段代码讲解索引同步到 v156。

## 证据边界

`c/156/图片` 和本说明是运行证据归档，不是后续程序消费的机器输入。后续如果要继续复用 checkpoint 能力，应依赖：

- `src/minigpt/server_checkpoints.py`
- `src/minigpt/server_contracts.py`
- `tests/test_server_checkpoints.py`
- README 中的 v156 版本说明

## Tag 含义

`v156.0.0` 表示 MiniGPT 本地推理服务完成 checkpoint payload 边界拆分：checkpoint discovery、health、model-info、checkpoint list 和 comparison payload 进入独立模块，同时旧 facade 和 API 行为保持兼容。

## 一句话总结

v156 把 checkpoint 证据链从 server contracts 里独立出来，让本地推理服务更容易测试、维护和继续演进。
