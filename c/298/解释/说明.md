# v298 运行截图说明

## 本版主题

v298 把 CI workflow order violation 从单个 release readiness 报告推进到 release readiness comparison。目标是让跨版本对比能发现一种细粒度回归：CI workflow status 仍然是 `pass`，但 smoke-before-coverage 的 order violation 从 0 增加到 1。

本版不改变 release readiness 的 ship/review 判定，不新增发布阻断规则，也不扩展到 maturity/registry 汇总。它只补 comparison 层的 row、delta、summary、artifact 和 CLI 输出。

## tag 说明

`v298.0.0` 的含义是：MiniGPT v298 release readiness comparison CI order deltas。

这一版让 release readiness comparison 可以比较 `ci_workflow_order_violation_delta`，并在 summary 中给出 `ci_workflow_order_regression_count`。

一句话总结：v298 让 CI workflow 顺序证据从单报告字段变成跨版本 delta，能捕捉 CI pass/fail 状态之外的 smoke-before-coverage 漂移。
