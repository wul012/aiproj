# v137 generation quality flag taxonomy 运行说明

## 本版主题

v137 不继续做 artifact split，而是回到 generation quality 的真实评估证据：在已有 pass/warn/fail 和 case-level flags 之上新增 `summary.flag_summary`。

它让报告能回答更具体的问题：

```text
本次生成质量一共有多少 flags？
哪些问题类型出现最多？
warn/fail flag 各有多少？
哪些 case 最值得优先看？
```

## 关键改动

- `src/minigpt/generation_quality.py` 新增 `_build_flag_summary()`，汇总 flag id、严重级别和 worst cases。
- `src/minigpt/generation_quality.py` 的 recommendations 新增 dominant flag 提醒。
- `src/minigpt/generation_quality_artifacts.py` 的 Markdown/HTML 新增 `Flag Breakdown`。
- `scripts/analyze_generation_quality.py` 的终端输出新增 `flags=` 和 `dominant_flag=`。
- `tests/test_generation_quality.py` 增加 taxonomy、worst case、Markdown/HTML 展示断言。

## 证据边界

`c/137` 证明的是“生成质量诊断更具体”：同一个重复样本可以同时暴露低多样性、长重复 run 和高 n-gram 重复；空 continuation 会作为 fail case 排在 worst cases 前面。

它不证明模型本身已经变强。后续如果要证明能力提升，需要拿真实 checkpoint、固定 benchmark、generation quality、scorecard 和训练记录做跨 run 对比。
