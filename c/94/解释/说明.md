# v94 training scale promotion index report utility migration 运行截图说明

## 本版主题

v94 继续 `report_utils` 收口路线，把 v78 的 `training_scale_promotion_index.py` 迁移到公共报告工具层。
本版只回收 promotion index 里的重复 JSON 写出、Markdown/HTML 转义和 list/dict 归一化 helper，不改变 promoted-only comparison inputs 的筛选规则。

迁移后，单个 promotion acceptance 和多个 promotion 的 index 层开始共享同一套报告基础设施。

## 关键代码

```text
src/minigpt/training_scale_promotion_index.py
src/minigpt/report_utils.py
tests/test_training_scale_promotion_index.py
tests/test_report_utils.py
scripts/index_training_scale_promotions.py
```

`training_scale_promotion_index.py` 仍然负责读取多个 `training_scale_promotion.json`，生成 `promotions`、`comparison_inputs`、`summary`、`recommendations` 和 JSON/CSV/Markdown/HTML 输出。

## 验证标准

v94 的成功标准是：

- promotion index 私有 `utc_now`、`_dict`、`_list_of_dicts`、`_string_list`、`_md`、`_e` 被公共 `report_utils` 替代。
- JSON writer 使用 `write_json_payload`。
- 两个 promoted reports 仍然生成 ready compare command。
- review/blocked reports 仍然不能进入 compare command。
- baseline 仍然只能从 promoted comparison inputs 中选择。
- HTML 报告能被真实 Chrome 打开。
- README、代码讲解和截图归档同步到 v94。
