# v172 eval suite artifact split 运行说明

## 本版主题

v172 做的是 eval suite artifact split：把 `eval_suite.py` 中负责 JSON、CSV、SVG、HTML 写出和 HTML 渲染的逻辑拆到 `eval_suite_artifacts.py`。

这一版不改变固定 prompt suite、prompt case 校验、prompt result 统计、eval report schema、`scripts/eval_suite.py` 使用方式，也不改变旧的 `minigpt.eval_suite` 导出入口。

## 关键改动

- `src/minigpt/eval_suite_artifacts.py`：新增 artifact 模块，负责 `eval_suite.json`、`eval_suite.csv`、`eval_suite.svg`、`eval_suite.html` 的写出和渲染。
- `src/minigpt/eval_suite.py`：保留 prompt suite、prompt result、report schema 构建逻辑，并 re-export artifact writer。
- `tests/test_eval_suite.py`：新增 facade identity 测试，证明旧模块导出的 writer/render 函数指向新 artifact 模块。
- `README.md`、`c/README.md`、代码讲解索引：登记 v172 能力、tag 和证据位置。

## 版本标记

`v172.0.0` 表示 MiniGPT eval suite artifact split：固定 prompt benchmark 的语义层和证据输出层分开，旧入口保持兼容。

## 一句话总结

v172 把 eval suite 的 JSON/CSV/SVG/HTML 输出层从 `eval_suite.py` 拆到 `eval_suite_artifacts.py`，让固定 prompt benchmark builder 更聚焦，同时保持 report schema、脚本调用和旧导出不变。
