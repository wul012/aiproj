# v148 运行截图说明

## 本版主题

v148 把 v147 已经进入 `release_bundle` 的 CI workflow hygiene evidence 继续带入 `release_readiness`。

本版新增独立的 `CI Workflow Hygiene` readiness panel，并在 summary、Markdown、HTML 和 CLI 输出里展示 CI workflow status、failed checks 和 Node 24 native action 数。它的状态边界仍然是 review 证据，不是 release gate 硬阻断。

## 关键改动

- `src/minigpt/release_readiness.py`
  - 新增 `ci_workflow_hygiene_path` 可选输入。
  - 支持从 release bundle inputs 或默认 `runs/ci-workflow-hygiene/ci_workflow_hygiene.json` 解析 CI hygiene 报告。
  - 新增 `CI Workflow Hygiene` panel。
  - summary 新增 `ci_workflow_status`、`ci_workflow_failed_checks`、`ci_workflow_node24_actions`。
  - 当 CI hygiene 自身 fail 时，readiness dashboard 进入 `review`，不进入 `blocked`。
- `scripts/build_release_readiness.py`
  - 新增 `--ci-workflow-hygiene` 参数。
  - CLI 输出新增 CI workflow 状态和 failed check 数。
- `tests/test_release_readiness.py`
  - fixture 新增 CI workflow hygiene report、bundle input、bundle context 和 artifact row。
  - 新增测试覆盖 CI hygiene fail -> readiness review 的边界。

## 证据边界

CI workflow hygiene panel 证明的是 GitHub Actions 工作流策略没有退回旧 action/runtime 策略，也没有漏掉关键 hygiene 步骤。

它不证明模型能力、数据质量、checkpoint 成熟度或真实训练效果。release gate 的硬阻断仍然由 release gate、project audit、release bundle 等已有发布治理证据决定。

## tag 含义

`v148.0.0` 标记的是 “release readiness CI workflow hygiene panel”：发布就绪总览可以展示 CI workflow hygiene 状态，并把失败状态作为 review 信号。

一句话总结：v148 把 CI workflow hygiene 从发布证据包推进到发布就绪 dashboard，让最终人工审查页面也能看到 CI 策略健康度。
