# v74 运行截图和解释

## 本版主题

v74 新增 training scale run decision。它读取 v73 `training_scale_run_comparison.json`，从多次 gated scale run 中选择最适合进入真实 `--execute` 阶段的候选，并把被拒绝的 run、拒绝原因和下一步执行命令写成证据。

这一版不重新训练模型、不替代 v73 comparison，也不绕过 v71/v72 gate。它只做训练前最后一层选择：在已有 comparison 基础上判断是否应该继续执行。

## 关键命令

```powershell
python -m unittest tests.test_training_scale_run_decision
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts/decide_training_scale_run.py tmp\v74-smoke\scale-run-comparison\training_scale_run_comparison.json --out-dir tmp\v74-smoke\scale-run-decision
python scripts/decide_training_scale_run.py tmp\v74-smoke\scale-run-comparison\training_scale_run_comparison.json --out-dir tmp\v74-smoke\scale-run-decision-strict --require-gate-pass
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp\v74-smoke\scale-run-decision\training_scale_run_decision.html -Out c\74\图片\05-playwright-training-scale-run-decision-html.png
```

## 证据含义

默认 smoke 使用 v70 plan、v72 gated runner 和 v73 comparison 真实生成输入，再交给 v74 decision。这样可以证明 v74 不是手写假数据，而是沿着现有训练规模治理链路消费真实产物。

默认模式下，review profile 产生 warn 但已经允许 batch dry-run，因此成为 `review_warnings_then_execute` 候选；standard profile 失败并被阻断，因此进入 rejected runs。严格模式下打开 `--require-gate-pass`，warn 候选也会被拒绝，用来证明 v74 可以在不同执行边界下收紧策略。

## tag 含义

`v74.0.0` 表示项目新增 training scale run decision：从“能比较多次受控启动的差异”推进到“能选择下一次是否进入真实执行”。
