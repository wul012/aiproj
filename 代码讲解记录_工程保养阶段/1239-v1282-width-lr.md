# 1239 - v1282 width × lr：宽度时钟会不会也坍缩成学习率？（关闭 v1277 之问）

## 这一版回答什么问题

整条弧的起点是 v1277 的描述性意外：窄模型 grok 快 5 倍。v1279 在单一 harness 里
复测，发现宽度→t_gen 曲线非单调、d=64 是灾难性慢区（1/3 grok，唯一成功用了
35,000 步，两个 seed 100k 删失）。v1281 揭示 d=128 基线严重 lr 饥饿：lr=4e-3 下
t_gen 从 11,400 坍缩到 ~1,400 并在 ~1k 步饱和。综合假设——"宽度与范数效应大多是
有效学习率的间接效应"——对宽度给出一个锋利的预测，这一版就检验它：**在足够的
lr 下，所有宽度应收敛到同一个 ~1k 步的地板**，窄模型加速应消失，d=64 的洞应
关闭。任何存活下来的效应才是真正的宽度效应。

## 实验设计：一个 lr，三个宽度，参照全部读缓存

全部新格 α=1、lr=4e-3、60k 时钟；d=128 @ 4e-3 参照来自 v1281 缓存（中位 1,400，
G0 锚逐字校验）；v1279 网格（lr=1e-3）做前后对照（G0 同时锚定 16→2,800、
32→2,700、128→11,400、64 ≥2 删失）。

- **宽度臂（判决）**：{16, 32, 64} × 3 seeds = 9 runs。d=64 是主角。
- **条件洞探针（描述性，确定性规则）**：当且仅当 d=64 @ 4e-3 有 ≥2/3 未 grok，
  追加 d=64 @ 8e-3 × 2 seeds——更高剂量能不能打开洞？永不参与判决。
- 判据（预注册 `bf1cbf4b`）：ρ(w) = 宽度中位 t_gen / 参照中位（删失=+∞），平价带
  [0.5, 2]（沿用 v1281）。全部 converged → `width_clock_collapses_to_lr`（v1277
  之问关闭，闭环通告按失败条件回写 v1277 简报）；16/32 有 still_faster 且 64 不
  still_slower → `narrow_speedup_survives_lr`；64 still_slower 且 16/32 converged
  → `mid_width_hole_survives_lr`；broken 或混杂 → review。G2 三 bar 一致。
- P2 探针 = d=64 @ 4e-3 seed 1337（v1279 里两次删失的同一格），槽位复用计入网格；
  唯一止损：broken。预算 ≤ 9+2 = 11 ≤ 12。

## 实现要点

`src/minigpt/grok_width_lr_v1282.py` 复用到几乎没有新训练代码：训练格直接调
v1280 的 `train_cell`，宽度经 `dataclasses.replace(cfg, width=w)` 传入（配置是
冻结数据类，replace 生成变体，训练循环零改动）；三分类与 t_gen 复用 v1280 的
`classify`/`cell_tgen`。G0 同时校验两份参照缓存的锚点。条件洞探针的触发规则写
在模块里（`_hole_rule_fires`），G1 检查探针格"当且仅当规则触发时存在"。测试
7 项：配置校验、两条 G0 失败路径、全部判决分支（含 broken 与混杂）、条件探针
编排（触发与不触发两侧）+ 探针槽位复用、字节稳定契约、图像落盘。预注册前对
真实缓存跑 G0=True。

## 结果：洞关了、优势反了、天花板现身了

P2 探针一格封神：d=64 @ 4e-3 seed 1337——v1279 里在 40k 和 100k 两个预算下都
删失的同一格——**2,400 步 grok**（heldout 0.984）。全网格里 d=64 三个 seed 全部
利落 grok（1,800/2,400/3,000，heldout 最高 **1.0**），ρ=1.71 落在平价带：
**v1279 的中宽度洞在 4× lr 下关闭**，它不是宽度病理，是 lr 饥饿的极端形态。

第二个意外：**窄模型加速不但消失，还反了向**。w=32 @ 4e-3 是 2,500/4,700/6,900
（中位 4,700，ρ=3.36 → still_slower）——高 lr 下 w=32 比 d=128 慢 3 倍多，连
记忆都变慢（t_mem 涨到 800–1,200，其他宽度 100–300）。v1277 的"窄快 5 倍"
从来不是宽度的功劳：lr=1e-3 恰好落在窄模型的适配区、宽模型的饥饿区。

第三个意外触发了判决防护：**w=16 撞上稳定性天花板**。三个 seed 的 train 都
从未到 0.99（t_mem=None），seed 1337 的 heldout 只有 0.847——按预注册三分类为
broken，判决臂出现 broken 即路由 **`review`（broken_cells）**，G0/G1/G2 全过、
三 bar 一致。这个防护正是为"不稳定 ≠ 慢"设计的，第一次真实触发。更妙的是
w=16 的另外两个 seed 在**从未记忆**的情况下达到 heldout ~0.96——没有记忆阶段
的直接泛化，一个此前未见的形态（banked）。

统一图景：**可用 lr 窗口随宽度上移**（w=16 的稳定上限 < 4e-3 ≤ w=32 的可用区
< d=64/128 的适配区）——μP 式"最优 lr 随宽度增大"在 grokking 时钟上留下三种
可见后果：broken / 变慢 / 收敛。条件洞探针（d=64 @ 8e-3）规则未触发（洞自己
grok 了），预算 9/12。v1277 简报已回写如实标注 review 的描述性结算注记。

## 诚实边界与自审

范围：own grokked substrate、toy scale、除 lr 外冻结、60k 预算；"窗口随宽度
上移"基于单一 lr=4e-3 切片 + v1279 的 1e-3 切片，w=16 的 broken 只有 1/3 seed
（另两个无记忆而泛化），全宽度收敛的干净声明需要 per-width lr 适配后的地板
比较（banked：宽度时钟死亡证明书）。decide() 自审：预注册 `bf1cbf4b` 后判据
未改一字；运行后唯一改动是图例渲染（未触发的洞探针不再占图例位）；
`width_clock_collapses_to_lr` 的强制闭环义务未触发，v1277 注记如实标注
"描述性、正式判决 review"。GPU 9/12；参照全部读 v1279/v1281 提交缓存零重跑。
