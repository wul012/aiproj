# 1238 - v1281 norm vs step：救活的格子快，是因为范数小，还是因为步子大？

## 这一版回答什么问题

v1280 用 lr 救活了小初始化的 grokking，而且救活的格子快得惊人：α=0.5 配 lr=4e-3
只要 1,300–2,300 步，基线（α=1，lr=1e-3）要 11,400 步。但 v1280 的判决故意留下
一个归因缺口（当时就 bank 成了这一版）：在 AdamW 下相对更新量 ∝ lr/α，所以救活
的格子和基线**同时**差在两个维度上——初始范数小一半、相对步长大 4–8 倍。到底
哪个是速度的真因？缺的对照是 **α=1 配大 lr**：如果满范数模型在同样的相对步长下
一样快，速度就是步子的功劳，范数时钟的任何版本都彻底死亡；如果 α=1 在匹配步长
下仍然接近 11,400（或卡死），那小范数确实独立起作用，"lr 适配版范数时钟"成立。

## 设计的核心：匹配相对步长的成对比较

定义 r = lr/(1e-3·α)（基线 r=1）。两对匹配对照：

- **r=4×**：(α=0.5, lr=2e-3)（v1280 缓存，中位 3,900）vs (α=1, lr=4e-3)（新）
- **r=8×**：(α=0.5, lr=4e-3)（v1280 缓存，中位 1,800）vs (α=1, lr=8e-3)（新）

判据（预注册 `ea074da1`）：ρ(lr) = 新格中位 t_gen / 参照格中位 t_gen（删失 = +∞）。
两对都落在平价带 [0.5, 2] → `relative_step_sets_the_clock`；两对都 ρ>2（α=1 更慢）
→ `small_norm_speeds_grokking_beyond_lr`；两对都 ρ<0.5 → `large_norm_...`（反向）；
两对不一致 → review（mixed_pairs）。关键防误读设计：α=1 大 lr 若把训练打爆
（broken：连记忆都没有），那不是"更慢"而是"不稳定"，该对判 invalid → review
（broken_cells）——v1280 建立的三分类（grokked/stuck/broken）在这里直接复用。

辅助臂（纯描述）：α=1 × lr=2e-3 补全 α=1 的 lr 剂量曲线 {1e-3(参照), 2e-3, 4e-3,
8e-3}；α=2 × lr=8e-3 给出 r=4× 处的第三个范数点，构成 (0.5, 2e-3) / (1, 4e-3) /
(2, 8e-3) 的同步长三范数比较。

## P1 与预算

P1 这次是纯缓存簿记（CPU，免费）：重推并锁定全部参照锚点——v1280 (0.5,2e-3)
2/2 grok 中位 3,900、(0.5,4e-3) 2/2 中位 1,800（heldout 0.9992）、v1279 基线 3/3
中位 11,400、α=2 lr=1e-3 参照 12.8k–23.2k。lr 不改变初始化，没有初始几何可测。
预算：判决臂 α=1 × {4e-3, 8e-3} × 3 seeds（判决靠中位数比较，参照才 2 seeds、
散布 ~2×，所以新格用 3 seeds）+ 剂量 2 + 对称 2 = 10 ≤ 12，统一 60k 时钟。
P2 探针 = (α=1, lr=4e-3, seed 1337)，用 v1280 建立的槽位复用机制计入网格；唯一
止损条件：探针 broken 才降档重设 lr 阶梯。

## 实现要点

`src/minigpt/grok_norm_vs_step_v1281.py` 几乎全是复用：训练格直接调 v1280 的
`train_cell`（α 与 lr 本来就是它的自由参数），三分类与 t_gen 复用 v1280 的
`classify`/`cell_tgen`，参照完整性 G0 同时校验 v1280 与 v1279 两份提交缓存的
锚点中位数（逐字相等）。配置校验把配对约束写死：lr 必须恰为参照 lr 的 2 倍
（α 比 0.5 的步长匹配），对称臂必须落在 r=4×。测试 7 项：配置校验、G0 两个
失败路径、平价/小范数/大范数/混合/broken 全分支、注入 trainer + 探针槽位复用
编排、字节稳定契约、图像落盘。预注册前对真实缓存跑 G0=True。

## 结果：两个都不是——时钟读的是绝对学习率

P2 探针一格定调：α=1 @ lr=4e-3 在 **1,400 步** grok（heldout 0.994）——满范数
模型在匹配步长下比 v1280 的救活格（同 r 的参照中位 3,900）快 2.8 倍。全网格
出来后，10 格全部干净 grok（无 stuck 无 broken，lr=8e-3 也没有打爆训练）：
判决臂 α=1 @ 4e-3 中位 1,400、@ 8e-3 中位 1,400；剂量臂 α=1 @ 2e-3 是
1,900/2,800；对称臂 α=2 @ 8e-3 是 **900/900，heldout 0.9999 与 1.0**——双倍
范数的模型成了整条弧最快、最准的格子。

预注册阶梯落在 **`review`（mixed_pairs）**：ρ(r=4×)=0.359 越过平价带下沿
（α=1 显著更快），ρ(r=8×)=0.778 在带内，两对方向不一致。但 review 里有一块
确定的内容：**两对一致排除了小范数分支**（没有任何 ρ>2）——v1280 判决名里
"norm_clock_revived"的"norm"部分正式清空，速度从头到尾是 lr 的功劳。相应的
resolution note 已回写 v1280 简报（通告链继续）。

描述性图景把整条弧收拢成一句话：**绝对 lr 主导 t_gen，并在 lr ≥ 4e-3 处饱和于
~1k 步**（α=1 剂量曲线 11,400 → ~2,350 → 1,400 → 1,400）；同一相对步长 r=4×
处三个范数 (0.5, 1, 2) 给 (3,900, 1,400, 900)——足够的 lr 下范数越大反而略快，
连 v1279 在 lr=1e-3 处看到的"α=2 略慢"这个二阶效应都是 lr 条件性的。回看
v1277–v1280：窄模型快 5 倍、小 init 杀死 grokking、lr 救援——全部统一为
"经典配方的 lr=1e-3 对 d=128 严重欠佳，基线活在 lr 饥饿区"；宽度与范数效应
大多是透过有效学习率的间接效应。宽度版的直接检验（窄模型在高 lr 下是否也
收敛到 ~1k 步）banked 给下一版。

## 诚实边界与自审

范围：own grokked substrate、toy scale、除 lr 外冻结的配方、60k 预算；"lr 饱和
于 ~1k 步"只声称于 d=128 与本网格（2e-3–8e-3），未探更高 lr 的爆炸边界；平价带
[0.5, 2] 是预注册宽度，ρ=0.359 与 0.778 的分歧在带宽内无法归因于噪声还是真实
的 r 依赖（3 seeds 的中位散布 ~1.5×）。decide() 自审：预注册 `ea074da1` 后判据
与代码零改动（图形函数亦未动），P2 唯一止损条件（broken）未触发，mixed_pairs
→ review 是预注册路由的正式产出。GPU 10/12；全部参照来自 v1279/v1280 提交缓存，
零重跑。
