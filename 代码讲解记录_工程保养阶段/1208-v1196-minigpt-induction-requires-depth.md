# v1196 MiniGPT 上下文归纳（induction）需要深度 代码讲解

## 本版定位：持续学习轴收尾后的全新轴 —— 上下文学习 / 归纳头

持续学习弧（v1193-95）做完后干净换轴，转到 transformer 的标志性现象：**induction（上下文复制）**，也就是 in-context learning 的机制。玩具规模正是它的原生栖息地（induction head 最早就是在 2 层 attention-only 模型里发现的）。本版问一个能被证伪的问题：**归纳需要深度吗？**

## 5 个 CPU 探针逐一证伪了天真的设定（这正是探针的价值）

1. `concat(S,S)`（定长重复）被 **1 层位置捷径**（"往回看 L 步"）解决，不是归纳 → 弃。
2. 改成"强制后继"的自由游走任务后，1 层仍以 0.98 归纳（n_embd=128）→ 天真地说"需要 2 层"在高容量下为假。
3. 2 层达到完美归纳却**没有干净的 induction-head 注意力模式**（我的"最近一次后继"度量定义上会低估）。
4. **自由游走会坍缩成短环**（K=20 只剩 ~6 个不同 token）→ **频率(unigram)启发式**也能伪造归纳，连随机初始化模型都能得分 0.1 → 任务退化。
5. **干净任务**（高多样性随机序列，目标 = "当前 token 上一次出现后跟的那个 token"，变长距离，目标与频率无关）：**1 层彻底失败、2 层从中等宽度就学会** → 归纳**真的需要深度**，前面那些任务只是让浅模型作弊。

## 4 视角对抗设计评审逼出的 must-fix（先于建模）

- **M1 unigram 闸**：强制后继会抬高后继 token 的频率，所以"预测上下文里最频繁的 token"能在可归纳位上超过随机 → 闸门改成 `归纳准确率 ≫ unigram 准确率`（解析、无模型），不是 `> 随机`。
- **M2 rebuild null / M8 随机初始化闸**：用频率基线 + 随机初始化≈0 作底。
- **M3 W\*-排序检验**（不是逐宽度找缝）：每个深度的宽度阈值 W\* = 平均归纳准确率首次越过成功线 S 的最小宽度；预登记检验 W\*(2层) < W\*(1层)，>=5 seed，std==0 双向守卫。
- **M6 机制度量修正**：本任务目标由**最近一次**后继决定，所以注意力度量用"最近后继位"的质量（按头求和、减均匀基线）+ layer-0 prev-token 分。
- **M7 attention-only 臂 + MLP 警示**：教科书"归纳需 2 层"是**纯注意力**定理；本模型每个 block 都有 MLP，所以加一条把 MLP 置零的 attention-only 臂，并把结论限定为"与该定理一致"而非"推翻"。

## 真实结果（K=20，T=64，5 seed，verdict=`induction_requires_depth`，status=pass）

```text
干净任务归纳准确率（随机 0.05，unigram 底 0.142，成功线 S=0.570）：
宽度       8    16    24    32    48    64    96    128
1 层     0.11  0.15  0.16  0.17  0.20  0.21  0.30  0.43   ← 所有宽度都 < S（连 128 都没到）
2 层     0.11  0.49  1.00  1.00  1.00  1.00  0.54  0.61   ← 从宽度 24 就学会
W*(1层)=∞（从不越线）  W*(2层)=17.3
捷径对照：同一个 1 层模型在定长位置任务上 = 1.00（有能力，只是不会内容归纳）
attention-only（MLP 置零）：2 层 0.99（学会）、1 层 0.18（失败）→ 直接证实教科书 2 注意力层电路
机制：2 层/宽 64 swap-follow 0.91、最近后继注意力质量 2.71、layer-0 prev-token 1.22 = 组合电路
```

**结论**：在堵住捷径（位置 + 频率）的干净任务上，归纳**需要深度**——1 层（即便带 MLP、即便到 16× 的宽度）学不会，2 层从中等宽度就学会；attention-only 2 层成功、1 层失败，与"纯注意力归纳需 2 层"定理一致，MLP 并不能替代第二层。捷径对照证明 1 层有能力（位置任务满分），所以它在干净任务上的失败是**内容归纳的局限**而非什么都学不会。

## 一个真实的优化伪影，及其诚实处理

2 层在**宽度 ≥96 时反而掉到 0.54/0.61**。我先怀疑是固定数据池过拟合 → 改成每步全新数据重跑，**掉点依旧** → 所以这是**优化效应**（大模型在固定 1500 步预算下欠训练），不是过拟合也不是能力。诚实处理：深度判定只用 2 层收敛的宽度区间（≤64），把大宽度的欠训练**如实报告**（`two_layer_undertrains_large_width=True`），并把 1 层的大宽度点（96/128 仍 0.30/0.43 < S）当作**单边失败上界**保留（强化"1 层连 128 都不行"）。判定的 W\* 是首次越线宽度，不受大宽度掉点影响。

## 工程曲折（也记下来）

- 第一版用预生成数据**池**加速，却在大宽度引入过拟合掉点；改回每步全新数据。
- 每步全新数据时 make_batch 的逐位置 Python 循环在 GPU 上**受 kernel-launch 限制**（GPU 仅 24%，玩具模型太小喂不饱 GPU）→ 把"最近后继目标"**向量化成一个 (B,T,T) 矩阵操作**（去掉 63 步循环），并把 swap 探针**批处理成一次前向**。这两处让重跑可控。

## 工程

Phase A（`run_induction_v1196.py`）一次训完所有臂（干净 depth×width + 1 层位置捷径对照 + MLP 置零 attention-only）存 cache；Phase B（`analyze_induction_v1196.py`）纯 CPU 重算 W\*-排序/闸门/verdict（reuse-cached）。库 `induction_v1196.py` 含向量化 `make_batch`、解析 `unigram_acc`、批处理 `_swap_follow`、`_induction_scores`、`train_arm`/`run_phase_a` 和 `summarize`/`decide`/`build_report`。14 个测试（任务/掩码正确性、合成缓存跑遍 decide 全分支、tiny 真 Phase-A smoke）。

一句话总结：v1196 开了上下文学习这条新轴并给出一个干净判定——在堵住位置与频率捷径的内容归纳任务上，**归纳需要深度**（1 层即便带 MLP、到 16× 宽度仍失败，2 层从中等宽度就学会；attention-only 2 层成功、1 层失败，与纯注意力定理一致），捷径对照证明 1 层有能力；5 个探针先证伪了天真设定、4 视角评审钉死了 unigram 闸/机制度量/W\*-排序，过程中把一个大宽度掉点诚实定性为优化欠训练而非过拟合并据此限定判定区间。
