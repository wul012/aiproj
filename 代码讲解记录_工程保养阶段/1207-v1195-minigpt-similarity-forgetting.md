# v1195 MiniGPT 任务相似度 → 灾难性遗忘 代码讲解

## 本版定位：持续学习轴的 capstone

持续学习这条轴：v1193 刻画了灾难性遗忘（真实、且是**分布漂移**驱动），v1194 比较了缓解法（replay 胜 EWC，因为漂移改写共享/绑定的数字 embedding）。v1195 收尾——回答最自然的开放问题：**任务相似度会不会"分级"决定遗忘？** 这是把 v1193 的头号结论（"漂移驱动、与 B 结构无关"）从一个 null 升级成一条**测出来的定量律**，正如 grokking 弧的 v1191 把相关升级成因果。

## 先用两个 CPU 探针证伪我自己的假设

我原本的假设是"结构复用保护 A"：B 若复用 A 的解结构（如同算子加常数），遗忘应更少。两个探针（p=11，便宜）直接**证伪**了它：
- 探针 1（算子类型）：add_offset=(a+b+c0)（同 '+' 算子、仅常数偏移）遗忘 0.83，和 mul 一样；只有逐字节相同的 add_same 遗忘 0。→ 结构复用**不**保护。
- 探针 2（重叠混合）：B_s 在比例 s 的格子算 add、其余算 mul，遗忘随 s 单调升 → 是**斜率**不是悬崖。

两个探针合起来给出一个比原假设更干净、更意外的诚实故事：**遗忘由输出表重叠度决定，与算子类型无关**——类型改动的"悬崖"只是因为它们都落在 overlap≈0。这就是"先跑便宜探针、让数据改设计"的价值（省了一次错误的 GPU 跑 + 错误结论）。

## 5 视角对抗设计评审逼出的 5 个 must-fix（先于建模）

评审把 41 条意见收敛成 8 个问题，5 个致命：
1. **M1 x 轴用解析 overlap，不用模型测得 overlap**：测得 overlap 部分是"遗忘少"的*结果*（保住 A 的模型仍预测 add）→ 回归变成自己对自己，循环论证。改用 `|{(a,b):f_B==f_A}|/p²`，从物化的 (p,p) 表精确算（含 a*b==a+b 碰撞），混合的真值是 s+(1−s)/p 不是名义 s。
2. **M2 每个 B 函数先物化成一张确定 (p,p) 表再切掩码**：否则 rand/混合若按掩码顺序消耗 RNG，同一 (a,b) 在训练流和测试流会拿到不同目标，overlap 谁都不描述。混合用**分层抽样**让实现 add 比例精确。
3. **M3 把 accB（学会度）和共享 embedding 漂移作为协变量 partial 掉**：否则斜率可能是"B 没学会→少覆写"的伪影。accB 拆成**冲突格**（f_B≠f_A，真 B-学习）vs 重叠格（保住的 A 电路免费答对）。
4. **M4 overlap=1 端点构造性无遗忘**（= v1193 continue-on-A 地板，且被算子 token 混淆）→ 剔除出统计，只主张内部斜率；诚实承认 Axis 1（家族无关）是**再确认** v1193 而非全新。
5. **M5 C3 用预登记残差检验**（不是裸相关，相关被两端簇强行拉高）；C2 用**等价检验（TOST）**而非"未拒绝即相等"；Spearman + 精确置换 p；所有阈值建模前钉死；保留 std==0 守卫。

外加 should-fix：每臂学会度门控、≥5 seed、单调用容差带、固定 overlap 下变结构对照、训练格 overlap、对照线性覆写零假设（S6）、逐 seed 巩固门控。全部采纳。

## 一个建模后才暴露的真实混淆，及其诚实修复

第一次 Phase B 用**留出冲突格** accB 做学会度门控，结果高 overlap 的混合臂被判 UNLEARNED（留出冲突格 accB 只有 0.12–0.30）→ 内部曲线只剩 2 点 → `review: too_few_learnable_curve_points`。深究发现这不是预算问题而是**构造性的**：随机逐格 add/mul 划分对留出格**无泛化**（模型无法对未见格判断它该 add 还是 mul）。但造成遗忘的是"模型在**训练**时拟合了 B 的冲突格 → 改写了权重"，所以正确的学会度信号是**训练冲突格**准确率。改用它后 6 条混合臂全部 learned，内部 5 点曲线干净。这是 M3 评审担心的混淆的真实化身，被每臂门控如实抓住、并用正确的信号化解。

## 真实结果（p=23，5 seed）

```text
verdict = forgetting_governed_by_output_overlap  (status=pass)
C1 斜率: Spearman(overlap,遗忘)=−1.0 (置换 p=0.017)，跨度 0.80
C2 家族红鲱鱼: add_offset(同'+',+c0) 遗忘 0.991 ≈ mul 0.938，都 ≫ add_same(−0.002)
C3 统一: add_offset/linear/rand 残差 0.05/0.01/0.02 < 0.15 → 都在混合曲线上
混淆: β_overlap|accB=−0.98 vs β_accB=0.0 ；β_overlap|drift=−0.998 vs β_drift=−0.02
形状: 对(1−overlap)·平台零假设平均超出 +0.076 (< 0.10) → ≈比例覆写，非全局坍塌
```

**机制（统一 v1193）**：遗忘 ≈ 与 B 改写的输出表比例成正比；v1193 的"灾难性"只是这条曲线在 overlap=0（全冲突）端点的取值。算子类型只通过它改变多少输出表来起作用——同 '+' 的 add_offset 因为 c0 让每个格都冲突，所以和乘法一样灾难。

## 一个被自查纠正的门控阈值伪影（本 arc 第 8 次）

第一版 `structure_at_fixed_overlap` 用**原始遗忘差**比较低-overlap 类型点：add_offset(0.991) > mul(0.938) 被判"有残差结构"，会错误否掉干净结论。但二者 overlap 是 0.0 vs 0.043，这 0.05 差正是 overlap 律的预测、不是结构。改成用 C3 残差（每点对照其自身 overlap 处的曲线）才对，零重训重判。

## 诚实的边界

- overlap ≈ "B 目标与 A 一致的比例"，所以这条律部分是"覆写比例"的复述（评审 S6 的"近似同义反复"担心）。**真正新的内容**是：(a) 它是**连续分级**的（回答了原问题），(b) 家族无关（第二种手法确认 v1193），(c) ≈比例而非超线性（去掉"全局坍塌"的夸张），(d) 混合在留出格不泛化（已披露）。
- 玩具级、1 层、p=23；不是关于指令微调 LLM 遗忘的结论。

## 工程

Phase A（`run_similarity_v1195.py`）一次训完所有臂存 cache；Phase B（`analyze_similarity_v1195.py`）纯 CPU 重算（reuse-cached）。库 `similarity_v1195.py` 复用 v1193 的任务/巩固/优化原语，新增解析 overlap、(p,p) 表构造（类型/分层混合）、冲突-重叠格拆分、带 embedding 漂移记录的 `train_b`、Spearman+精确置换 p+等距回归+2 变量标准化 OLS。19 个测试（解析表/overlap、统计助手、合成缓存跑遍 decide 全分支、p=7 真 Phase-A smoke）。

一句话总结：v1195 收尾持续学习轴——灾难性遗忘由 B 与 A 的**输出表重叠度**单调分级决定（Spearman −1.0），与算子家族无关（同 '+' 的 add_offset 和乘法一样遗忘，只有逐字节相同的表安全，用第二种手法再证 v1193 的分布漂移）；且曲线≈比例覆写而非全局坍塌，所以"灾难性"只是 overlap=0 端点的取值；5 视角评审 + 2 探针先证伪了我的结构复用原假设，建模中每臂学会度门控如实抓住一个真实混淆并用训练冲突格信号化解，过程零重训自查纠正了一个门控阈值伪影。
