# 1248 - v1291 机械 lint 债清偿:基线 271 → 153

## 本版目标与不做什么

优雅提升计划(用户授权,目标 6/10 → ~9/10)的第二级:把静态分析基线里
四类**纯机械可修**的 lint 债一次清零——E702(分号连写语句,92 处)、
E741(歧义单字母变量名 `l`,15 处)、F841(赋值后从未使用的死变量,6 处)、
F541(无占位符的 f-string,5 处)——共 **118 处、20 个文件**,然后按
"ratchet 只收紧"的规矩把基线从 271 收到 153。

明确不做什么:**本版不碰 F401**(152 处未用导入)。未用导入可能是被下游
`from X import Y` 消费的有意 re-export,盲删会断真实契约;它留给 v1292
做逐条消费者审计后再清。也不碰 F821(3 处未定义名——那是 bug 类,不是
风格类,留给 v1293 逐个定性)。科学模块的 decide() 语义、实验判定、缓存
产物一律不变——本版所有编辑都必须能被下述验证法庭证明为行为零变化。

## 为什么这些债"纯机械"却值得单独一版

E702 的 92 处分号连写集中在 v1170-v1200 科学时代的模块里
(calibration_v1192 一个文件 15 处、double_descent_v1199 和
induction_v1196 各 12 处)——它们是当年赶版本时的压行习惯,读起来密不
透风,也是静态基线里最大的可修块。E741 的 `l` 在四个文件里被当作
lambda 正则化强度(ewc/dpo)或 layer 序号(induction×2)——单字母歧义名
正是 v1287 收尾时 CI 抓过的同类问题(那次是新增代码里的 `l`,ratchet
挡住了;这次是把历史存量清掉)。F841/F541 是纯死代码和纯装饰。
清完这四类,基线里剩下的就只有 F401(需审计)和 F821(需定性)两类
"不能无脑修"的债——债的性质从"混杂"变成"分类清晰",这本身就是优雅。

## 执行方式:位置驱动脚本 + 白名单守卫

手改 118 处必然出错,全部由 ruff 的 JSON 位置输出驱动:

1. **E702 分裂器**:对每个 (文件, 行, 列) 从右往左把 `;` 换成
   换行 + 原行缩进。守卫:若行首词是 if/for/while/def 等复合语句关键字
   则中止(`if x: a; b` 直接分裂会把 `b` 提出条件体——语义事故)。
   92 处全部通过守卫,无一复合头。
2. **F541**:ruff 自带安全自动修复(`--fix`),5 处。
3. **E741 重命名**:按当前 ruff 位置逐行做全词 `\bl\b` 替换
   (ewc/dpo → `lam`,induction → `layer`);断言每行至少 2 个全词匹配
   (绑定 + 至少一次使用)。重命名发生在 f-string 键构造里
   (`f"{l},{h}"` → `f"{layer},{h}"`)——键的**值**不变,只有变量名变。
4. **F841 删除**:6 处逐一人工核实右侧无副作用(纯比较、纯字典读取、
   空容器构造)后按行删除。

## 验证法庭(三层,一层比一层硬)

1. **AST 等价证明**(最硬,数学级):对只做了 E702 分裂的 10 个文件,
   断言工作区的 `ast.dump` 与 `git show HEAD:` 的逐字节相等——分号
   换行不改变解析树,10/10 全部 AST-IDENTICAL。这不是"测试没红",
   是"程序在语法树层面就是同一个程序"。
2. **聚焦测试**:17 个被改模块对应的 15 个测试文件,204/204 通过
   (含各版的 decide() 合同测试——重命名与死行删除的行为保持由它们背书;
   v1192-v1200 时代的 Phase-A 缓存当年未随包提交,无法做产物级重推导,
   如实披露,这正是后来"缓存进 f/ 包"惯例的反面教材)。
3. **门与基线**:`status=pass` / `new_issue_count=0`(用 v1290 收紧后的
   正确字段读法),四类目标码全清(`ruff --select E702,E741,F841,F541`
   → All checks passed),然后 `--update-baseline` 收紧 271 → **153**,
   重跑确认 baseline=current=153、new=0。全量套件另行背书(数字见提交)。

## 工程教训

1. **"机械修"也要分能证明和只能测试的两档**:分裂有 AST 等价这个
   数学法庭;重命名和删行只有测试法庭。把 10 个只分裂的文件单列出来
   走 AST 证明,是本版验证设计里最值的一步。
2. **债要按可修性分类,不按文件分类**:F401 和 F821 与本版四类混在
   同样的文件里,但修它们需要完全不同的证据(消费者审计/定性调查)。
   按码分版,每版的验证法庭才能统一。
3. **位置驱动 > 模式驱动**:E741 的行号在 E702 分裂后全部漂移;
   所有第二阶段编辑都重新问 ruff 拿当前位置,而不是用旧行号。
