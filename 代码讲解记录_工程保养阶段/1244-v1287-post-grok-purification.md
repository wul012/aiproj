# 1244 · v1287 post-grok purification：grok 之后，雕刻还在继续吗？

## 一、版本定位：把 v1286 的"推断"变成"观测"

v1286 用终点差异**推断**出高 lr 格在 grok 后继续纯化（终点份额随 lr 单调：
0.31 → 0.55–0.59 → 0.62–0.72 → 0.71–0.81），并在 brief 末尾 banked 了一个
候选："post-grok 纯化那段值得单独一版"。v1287 执行这个候选。

P1 缓存审计给了动机一记实锤：**已提交缓存里所有 run 的 post-grok 训练时间
几乎为零**——canonical 三格只有 200/200/1100 步（0.016–0.105 × t_gen），
压缩八格里五格**恰好在 t_gen 当步早停**（steps_run == t_gen，post-grok = 0）。
也就是说，这条弧至今所有"纯度"数字都是"**grok 时刻的纯度**"；grok 之后
回路做什么，从来没人看过。v1286 的"纯化段"是从两个端点之间画的虚线，
不是观测到的轨迹。

## 二、机理分叉：预注册一个可证伪的二选一

- **普遍派（AdamW 不动点论证）**：静止点满足"归一化任务梯度 = −wd·w"，
  lr 在两侧同时相乘后**约掉**——平衡态与 lr 无关，lr 只决定逼近速率。
  推论：给 canonical 足够的 post-grok 时间，它会爬向压缩格的纯度；
  v1286 的"纯度随 lr 单调"只是**早停切在轨迹不同位置**的伪影。
- **门控派**：canonical 停在 0.31 附近不动，压缩格远高于它——纯度天花板
  本身依赖 lr，不动点论证在此失效（例如所到盆地的曲率决定平衡）。

公平性预核算（写进 brief）：3× t_gen 视界下 canonical 拿到的 wd·lr·steps
雕刻剂量是**全场最大**（≈22.8 个 lr 单位 vs 压缩组 7.6–16，因为 t_gen 增长
快于 1/lr）——若 canonical 仍然平坦，不能归咎于视界太短。

## 三、设计五则

1. **关早停零改训练代码**：`train_to_grok` 的早停条件是
   `val ≥ config.grok_stop_val`（默认 0.95），这是 GrokConfig 的普通字段。
   v1287 的 `_grok_cfg` 传 `grok_stop_val=2.0`（不可达）即彻底关闭；
   `validate()` 强制该值 > 1.0。训练循环一个字未动，v1284 截断重跑法
   （max_steps=k 即 k 步快照，前缀门自证）原样适用于任意视界。
2. **冻结参照系**：每格纯度在**该格已提交缓存的 top-5 频率集**上度量
   （v1285/v1286 的 `top5`），锚点 = 缓存 `final_share`（grok 时刻纯度），
   基线 = 缓存 `c0_share`。这些是仓库里的既成常量——预注册即冻结，
   不依赖新数据。新终点模型的自身 top-5 份额（own_final）与 set_match
   标志作为**旋转诊断**。
3. **相对乘数梯子**：m ∈ {1.4, 1.8, 2.4} 截断 + m=3.0 全程；
   `step_of(m,t)=int((m·t+50)//100)·100` 对齐 eval 网格。11 格逐格算出
   绝对梯子，`validate()` 精确核算预算：44 runs / 407,800 步
   （上限 48 / 420,000）。
4. **主判决无 bar 化**——v1286 的 bar_instability 教训直接进设计：主梯
   只用 climb（缓存集份额爬升）与 convergence（own 纯度组间收敛带 0.15），
   G2 是 climb_bar 网格 {0.05, 0.10, 0.15}；F_ext（延长分母的结构分数、
   三区统一读数）降为**次级读数，不得翻转主判决**。
5. **快照存全谱**：每个快照存 49 维功率谱而非标量份额，Phase B（CPU）
   想投影到哪个集合都行——Phase A 纯测量、Phase B 纯分析的解耦。

G0 里两道新门值得点名：`early_stop_fired`（steps_run 必须等于视界——证明
早停真的被关掉了）与**整条缓存曲线的逐位前缀等式**（缓存里每行
(step, train, val) 必须在新曲线中逐位复现，1e-9）——比以往只对
t_mem/t_gen 的参照门强得多。

## 五、运行结果：防护开火，而且抓到的东西比问题本身大

P2 探针（8e-3/1337，8,600 步）四条止损全过。Phase A 44 runs 精确命中预算；
前缀 33/33，t_mem/t_gen 11/11 逐位复现参照。然后：

- **底座防护开火**：4e-3/1338 终点 heldout 0.431、4e-3/1339 0.373 →
  `review`（reason=substrate_unsound），brief 里明文预告过这道门"也是
  post-grok 稳定性防护"。
- **显微镜下的死亡**：4e-3/1338 在 5300 步 val=1.0、5400 步（恰好是视界
  终点）val=0.431 且 **train 同步掉到 0.457**——不是泛化选择性失效，是
  整个解的训练不稳定尖峰。全曲线扫描发现 11 格里 **9 格**（含 canonical！
  1e-3/1339 有 4 个尖峰，最深 val=0.385）都有这种 train+val 同崩的尖峰，
  且**全部在 100–300 步内自愈**——两个"死亡格"只是恰好在视界最后一格
  被**截尾在尖峰中间**。grok 后的解不是安静的吸引子，是一条会周期性
  塌方又自我修复的山脊。
- **纯化本身是真的、普遍的**（探索性标注）：9 个存活格 own 纯度全升——
  canonical 0.305→0.50–0.63（近翻倍）、2e-3→0.72–0.82、8e-3→0.88–0.91；
  且延长后 4e-3 的 0.93 已**反超** 8e-3 的 0.89——v1286 的"单调"确系
  截断伪影。
- **canonical 的纯化会旋转频率集**：3/3 set_match=✗；1337 的缓存集份额
  跌到 0.137 而 own=0.629。预注册的 climb 指标在 1e-3 上原则性失明，
  旋转诊断按设计接住——即使底座门不响，主梯也须 review。双重理由，
  一个判决。
- 低 lr 的 post-grok 训练**改善**泛化：1e-3/2e-3 五格 heldout 全部升到 1.0。
