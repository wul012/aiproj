# 1255 - v1298 正式批 4:packet/registry-ack 家族清尾 + 覆盖率教训闭环

## 本版目标与不做什么

两件事。(1) 批量重命名的第四批:>150 字符长名的最后 43 个模块——
packet 链家族(v1000-v1011,21 个)与 registry-ack 家族(版本无关的
代际门面,22 个)——短名化,同批带出 14 个脚本与 19 个测试的短名化,
91 个文件 import 改写。(2) 修复 v1297 CI 的**覆盖率棘轮**失败并把
教训闭环:189 个转发 shim 大多不再被任何消费者 import(消费者已改写
到新名),0% 覆盖 × 189 个文件把总覆盖率拖到地板下 0.26 个百分点。
不做什么:不动覆盖率地板(88.98 一分不降);不用 coverage omit 把
shim 划出测量(那是把兼容面挪出安全网)。

## 批 4 的地图:机械规则的失效边界与手工策展

这一批暴露了机械 tail 折叠的失效边界:registry-ack 家族是**版本无关的
代际门面**——同一个名字模式按代际堆叠(`receipt_packet_index_publication`
× 2 与 × 3 是两个不同模块!),折叠后信息丢失,机械规则三轮迭代
(v1/v2/v3/v4)越修越坏(2 dup → 19 over → 4 dup)。果断切换到**显式
手工地图**:43 条逐一命名(packet_chain_<role>_vNNNN /
registry_ack_<role>,gen3 代际显式标注),生成时断言零重复/零撞名/
零超预算 + 每条的 version/role/artifacts 语义对齐。教训:机械推导
适用于"信息保存"的折叠;代际重复这种"信息即重复次数"的命名,
只有人来定语义。

## 覆盖率失败的正确修法:把 shim 拉进安全网,而不是把它划出测量

v1297 的 shim 是兼容承诺——"旧 import 路径永远可用"。承诺没有测试,
就只是文档。新增 `test_forwarding_shims_v1298`:AST 扫描全扁平目录、
用 elegance_ratchet 的 `_is_shim`(严格全模块形制)识别每个 shim、
逐个 import 旧路径与新路径、断言 `old is new`。一石三鸟:兼容面全量
回归(任何 shim 断转发,测试点名报错)、189 个 shim 全行覆盖(棘轮
回到地板上)、shim 计数下限断言(>200)防止未来批次悄悄丢 shim。

第一版检测器用逐节点模式匹配,误把一个真模块(恰好有 `as _target`
import 和下标赋值)当 shim——改用 `_is_shim` 严格形制后消除。教训:
同一个概念(什么是 shim)在仓库里必须只有一个判定实现,测试直接复用它。
另两处收尾:v1294 试点 shim 用的是 `import X as _target` 平式导入
(批 2-4 是 from-import),提取器两种都认;200+ 次批量 import 触发了
pytest assertion-rewrite 钩子里的 pathlib 回归(上游 harness bug,
`_early_rewrite_bailout` 每次 import 造 PurePath),测试在批量导入段
临时摘下该钩子(产品模块不需要断言改写;unittest 下是 no-op),
两个运行器全绿。

## 工程教训

1. **兼容面必须有测试身份**:shim 不是死文件,是承诺;承诺进安全网
   的方式是被执行,不是被豁免。
2. **机械规则要认输**:三轮规则迭代不收敛就是信号——切显式地图,
   43 条手工命名一次通过,比第四轮规则便宜。
3. **棘轮之间会互相作用**:重命名(优雅棘轮)→ shim(兼容)→
   0% 覆盖(覆盖棘轮)——多棘轮系统里每个动作要过全部棘轮的推演,
   关门清单是这份推演的机械化。

产物:43 模块 + 14 脚本 + 19 测试短名化(手工地图),91 文件改写;
`test_forwarding_shims_v1298`(全 shim 转发回归);elegance-baseline
拧紧至 57/150;name-baseline 零净变换底。剩余:<150 的 57 个长名
(下一批清完即 long_name_stock=0)与 dup_def_stock 227 的去重长弧。
