# v1271 治理报告 JSON 读取器收敛证据

## 本版结论

v1271 是工程保养版本，不增加模型能力、不运行训练，也不改变科学线的缓存产物、实验判定或 promotion 边界。本版选择九个仍在稳定治理链中使用的报告模块，把重复的 UTF-8-SIG JSON 对象读取逻辑收敛到 `minigpt.report_utils`，同时保留严格读取与宽容读取两种不同契约。

本批结果为 `status=pass`、`decision=report_loader_dedup_ready`。累计被现有 dedup checker 管理的模块为 14 个，其中本批治理读取器 9 个；目标模块中的私有读取副本从 9 个降为 0。全仓扫描仍显示 496 个历史文件含相似读取形状，这些文件大多属于冻结的版本化实验模块，未被本版批量改写。

## 边界与剩余工作

本版不声称全仓 JSON loader 已收敛。`private_loader_copy_count=496` 是主动保留的可见债务指标；它说明仓库中过去一千多个版本留下了大量版本化实验实现。对这些历史模块做一次性机械迁移会产生巨大 diff，并把科学线旧实验语义暴露给不必要的回归风险。后续只在维护某个真实活跃区域时，连同测试和契约证明一起渐进迁移。

本版没有修改训练、生成、评估阈值、`decide()`、checkpoint、缓存实验 artifact 或模型 promotion 结论。
