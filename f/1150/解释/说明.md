# v1150 运行截图与解释

## 目标

v1150 把 v1149 生成的 `unassisted_holdout_repair_seed_corpus.txt` 真正交给 `scripts/train.py` 做一次 CPU bounded tiny training，并把训练后的 checkpoint、tokenizer、metrics、manifest、sample、loss curve 和 handoff 归档为可复核证据。

本版不声明模型能力提升。`model_quality_claim=training_artifact_only`，`promotion_ready=False`。原因是训练 loss 和 validation loss 下降只能证明这个修复语料能训练出一个 checkpoint；是否真正改善无锚点 holdout，需要 v1151 使用 v1149 固定下来的 target-free holdout prompts 重新 replay。

## 运行命令

```powershell
python scripts/run_unassisted_holdout_repair_training_v1150.py --out-dir output/unassisted-holdout-repair-training-run-v1150 --run-training --require-training-ready --force
```

该 CLI 先读取：

```text
f/1149/解释/unassisted-holdout-repair-seed-corpus-v1149/unassisted_holdout_repair_seed_corpus_v1149.json
```

再把同目录的：

```text
unassisted_holdout_repair_seed_corpus.txt
```

传给 `scripts/train.py --prepared-data`。训练参数保持轻量：char tokenizer、CPU、batch size 8、block size 24、max iters 50、eval interval 10、eval iters 2、learning rate 0.01、1 layer、1 head、embedding 16、dropout 0。训练完成后，v1150 builder 会验证训练产物是否齐全，并检查 `prepared_corpus.txt` 是否与 v1149 seed corpus 原文完全一致。

## 输出摘要

真实输出摘要如下：

```text

proposed_next_artifact=unassisted_holdout_repair_replay_comparison_v1151
next_step=run_unassisted_holdout_repair_replay_comparison

```

loss 记录从 step 1 到 step 50 共 6 条，train loss 从 `3.0308` 降到 `1.2284`，val loss 从 `2.9943` 降到 `1.4993`。这说明本版训练运行是有效的：模型确实在 v1149 的修复语料上学习到了局部模式。

不过 sample 只命中 `fixed`，没有命中 `loss`：

```text
answer: fixed
```

这也是本版不打开 promotion 的主要原因。v1150 只把“训练产物存在且训练信号下降”固定下来，不把单个 sample 当成模型质量结论。

## 边界

v1150 是训练执行版，不是能力结论版。它解决的是“v1149 seed corpus 能否产生一个完整可复核 checkpoint”的问题；没有解决“这个 checkpoint 是否比上一轮更会无锚点输出 fixed loss”的问题。

下一版应使用 v1149 的 target-free holdout prompts 和 v1150 的 checkpoint/tokenizer 做 replay comparison。如果 replay 能稳定命中 `fixed` 与 `loss`，才可以把 claim 从 `training_artifact_only` 推进到更强的 bounded capability evidence。

## 一句话总结

v1150 把 v1149 的修复语料推进成了真实 checkpoint 训练证据，loss 明显下降、产物完整、边界清楚，并为 v1151 的无锚点 holdout replay comparison 准备好了 handoff。
