# v1111 运行截图与解释

## 本版目标

v1111 把 v1109 lookup-only receipt 和 v1110 contract check 合并成 receipt index。它给下一步 review 提供统一入口，避免下游分别追踪 receipt 与 check 两份文件。

本版不训练模型，不新增生产授权，不把 `index_ready=True` 解释成模型可上线。它只确认 receipt 与 contract check 配对一致，并且继续保持 `promotion_ready=False`。

## 真实运行命令

```powershell
python -B scripts\build_randomized_holdout_publication_receipt_index_receipt_index_receipt_index_receipt_index_receipt_index_receipt_index_receipt_index_receipt_index_receipt_index_receipt_index_v1111.py --receipt f\1109\解释\receipt-v1109 --receipt-check f\1110\解释\receipt-check-v1110 --out-dir f\1111\解释\receipt-index-v1111 --require-index-ready --require-lookup-ready --force
```
