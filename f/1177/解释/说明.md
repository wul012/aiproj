# v1177 运行说明：PTQ deployment candidate selector

本版接在 v1175 post-training weight quantization 和 v1176 completion-mask 去重之后。

v1175 已经用真实 5 seed PTQ 曲线回答了“量化质量成本是多少”，但它给出的仍然是测量报告：per-tensor、per-channel-row、group32 在不同 bit width 下分别损失多少 CE、EM、KL。v1177 增加一个轻量但可复核的 candidate selector，把这条曲线转换成一个明确的部署候选选择：

- 输入：`f/1175/解释/ptq_v1175/ptq_v1175.json`
- 输出：`f/1177/解释/ptq_candidate_v1177/ptq_candidate_v1177.{json,csv,txt,md,html}`
- 默认预算：`dCE <= 0.08`、`exact-match drop <= 0.10`、`KL <= 0.10`
- 选择结果：`group32:3b`

结果含义：

- `group32:3b` 的 effective bits 为 `3.5`
- `ce_mean=0.145139`
- `dce_mean=0.064286`
- `kl_mean=0.07137`
- `em_mean=0.792778`
- `em_drop=0.090555`

它是默认质量预算内 effective bits 最低的候选。`per_channel_row:3b` 虽然 effective bits 更低，但 exact-match drop 超过 `0.10`，因此被拒绝。`per_tensor:3b` 和 2b 系列因为 CE/KL/EM 均明显崩塌，也不会成为候选。

边界说明：

- 本版不重跑训练。
- 本版不声称真实 int kernel 带来速度或内存收益。
- 本版只消费 v1175 已通过的质量成本证据，把“可接受质量损失内的最低 effective bits”显式化。

验证命令：

```powershell
python -m py_compile src\minigpt\ptq_candidate_v1177.py scripts\select_ptq_candidate_v1177.py tests\test_ptq_candidate_v1177.py
python -m pytest tests\test_ptq_candidate_v1177.py -q -o cache_dir=runs\pytest-cache-v1177-focused
python scripts\select_ptq_candidate_v1177.py f\1175\解释\ptq_v1175 --out-dir f\1177\解释\ptq_candidate_v1177 --require-pass --force
```
