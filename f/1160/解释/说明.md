# v1160 RoPE 旋转位置编码

## 目标

给 MiniGPT 加上现代位置编码 RoPE（Rotary Position Embedding），作为对现有“可学习绝对位置嵌入”的替代方案，并用 v1157 的 held-out 评估工具做一次诚实的对照：在同一模板化语料上从零训练“可学习位置” vs “RoPE”两个模型，比较 held-out 损失与下一字符准确率。

本版以向后兼容的方式实现：`GPTConfig.use_rope` 默认 `False`，不开启时模型行为与之前逐字节一致（既有 3173 测试零改动通过）。

## 运行命令

```powershell
python scripts\run_rope_eval_v1160.py --device cuda --steps 400
```

- RoPE：held-out loss `1.017310`，acc `0.678652`

## 诚实结论与边界

在这个**短的、定长**语料上，可学习绝对位置略优于 RoPE（held-out loss 低约 0.12，准确率基本持平）。这是符合预期的诚实结果：RoPE 的真正优势在于**相对位置**与**长度外推**（在比训练更长的上下文上仍可用），而定长短序列并不触发这些优势，反而让可学习绝对位置“恰好拟合固定位置”占了便宜。

因此本版的交付不是“RoPE 更强”，而是：RoPE 被正确实现（旋转保范、位置 0 为恒等、按相对位置编码）、向后兼容地接入模型、并被诚实地对照测量。RoPE 的招牌优势——长度外推——留给后续版本专门验证（在 block_size 之外评估，可学习绝对位置无法处理，而 RoPE 可以）。

## 一句话总结

v1160 以向后兼容的开关给 MiniGPT 加上正确实现、保范的 RoPE，并诚实测出在定长短语料上可学习绝对位置略优；RoPE 的长度外推优势留待专门版本验证——架构能力到位，结论不浮夸。
