# v1151 运行截图与解释

## 目标

v1151 使用 v1150 训练出的 checkpoint/tokenizer，重新运行 v1149 固定的 target-free holdout prompts，判断无锚点修复训练是否真的让模型在未带目标词的提示下输出 `fixed loss`。

本版不是训练版，而是 replay comparison。它只回答一个问题：v1150 的 checkpoint 在 v1149 固定的 5 个 holdout prompt 上，是否能命中 `fixed` 与 `loss` 的完整 pair。

## 运行命令

```powershell
python scripts/run_unassisted_holdout_repair_replay_comparison_v1151.py --out-dir output/unassisted-holdout-repair-replay-comparison-v1151 --require-comparison-ready --force
```

CLI 默认读取：

```text
f/1150/解释/unassisted-holdout-repair-training-run-v1150/unassisted_holdout_repair_training_handoff_v1150.json
```

handoff 里原始 checkpoint 路径指向训练时的 `output/.../run`。因为本地清理后 `output` 会被删除，v1151 增加了归档回退能力：当 handoff 里的路径不存在时，会从 handoff 所在目录的 sibling `run/` 读取 `checkpoint.pt` 和 `tokenizer.json`。这样 v1150 归档产物可以长期独立使用。

## 输出摘要

真实输出如下：

```text
status=pass
decision=unassisted_holdout_repair_replay_partial_signal
unassisted_holdout_repair_replay_ready=True
case_count=5
fixed_hit_case_count=4
loss_hit_case_count=0
full_pair_case_count=0
any_hit_case_count=4
full_pair_rate=0.0
all_full_pair_hit=False
partial_signal_visible=True
model_quality_claim=bounded_holdout_replay_partial_signal
promotion_ready=False
next_step=diagnose_unassisted_holdout_repair_partial_signal
failed_check_count=0
```

这说明 v1150 训练后确实出现了进步：5 个 target-free holdout prompt 中有 4 个生成了 `fixed`。但 `loss` 一次都没有出现，完整 `fixed/loss` pair 仍然是 0。因此本版结论是 `bounded_holdout_replay_partial_signal`，不是 recovered，也不是 promotion。

## 关键生成结果

归档的 generation rows 显示：

- `answer:` -> `fixed`
- `completion:` -> 未命中
- `finish:` -> `fixed`
- `state compact signal\nanswer:` -> `fixed`
- `signal:` -> `fixed`

全部 5 个 case 都没有命中 `loss`。这说明当前模型更容易学到 `fixed` 这个第一目标词，但还没有学会在无锚点条件下继续完成 `loss`。下一步不应该直接扩大治理报告，而应该诊断为什么 `loss_after_fixed` 没有转化为 holdout continuation。

## 边界

`status=pass` 只表示 replay comparison 成功执行、输入完整、每个 holdout prompt 都产生了 generation row。它不代表模型能力完全恢复。能力是否恢复由 `all_full_pair_hit` 和 `full_pair_case_count` 决定。本版 `full_pair_case_count=0`，所以 `promotion_ready=False`。

v1151 的价值在于把 v1150 的训练结果从“loss 下降”推进到“真实 holdout 生成表现”。现在可以明确说：修复训练带来了 `fixed` 的无锚点 partial signal，但没有带来 `loss` 的无锚点续写能力。

## 一句话总结

v1151 证明 v1150 的修复训练不是空转，它让无锚点 holdout 出现了 `fixed` 命中；但完整 `fixed loss` 仍未恢复，下一步应诊断 `loss` 缺失而不是提前推广模型能力。
