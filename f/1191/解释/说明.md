# v1191 运行说明：grokking 因果频率消融（可解释性收尾）

v1188 发现 grok 模型的数字 embedding 在傅里叶基上有频率集中，v1190 发现输出 logits 的加法对角频率和 embedding 一致——但两者都是**相关性**证据（命名都是 `supports` 不是 `proves`）。v1191 把它做成**因果**：直接从已提交的 v1185 checkpoint 的（绑定的）数字 embedding 里删掉/保留傅里叶频率，再量 held-out 准确率。

因为 `lm_head.weight` 和 `token_embedding.weight` 绑定，过滤数字 embedding 的频率会**同时**改输入表示和输出读出。全程不训练，CPU 几秒。

## 三个干预（必要性 / 特异性 / 充分性）

```text
                                  held-out acc
baseline（不消融）                    0.966
remove top-5 dominant freqs          0.578   ← 掉 0.39
remove 5 random non-dominant freqs   0.973   ← 几乎不掉
keep ONLY top-5 freqs（扔掉其余43）   0.972   ← 保住
chance = 1/97 ≈ 0.010
```

dominant freqs = `[43, 3, 48, 26, 44]`（和 v1188 embedding / v1190 logit 的 top-5 完全一致）。

## verdict：充分 + 特异 + 部分必要

`verdict = dominant_frequencies_sufficient_and_specific_partial_necessity`（status=pass）。

- **充分**：只保留 top-5 频率（把其余 43 个非 DC 频率清零），准确率仍有 **0.972**——加法算法基本就活在这 5 个频率里。
- **特异**：删 top-5 掉 0.39，删同样数量的随机频率几乎不掉（-0.007）。损伤特异地指向 dominant 频率，不是“删什么都疼”。
- **部分必要（不是完全必要）**：删掉 top-5 后准确率掉到 0.578，**没有塌到 chance**。说明模型有冗余——这几个频率承载算法，但拿掉它们模型还能用剩余结构勉强算到 0.58。诚实地说是“部分必要”，不是“完全必要”。

合起来：把 v1188/v1190 的相关性证据**升级成因果**——dominant 傅里叶频率就是模数加法算法所在；但诚实标注模型有冗余，删除不至于完全崩。

## 一个被自查纠正的 under-claim（与 v1183 的 over-claim 对称）

第一次跑出来时，我的 `decide()` 把这个结果误判成 `no_causal_frequency_dependence`——只因为我把 `collapse_drop` 阈值设成 0.40，而实际 drop 是 0.388（差一点点），verdict 阶梯又只看“是否完全崩”，忽略了“只保留 top-5 就够（0.972）”这个强充分性信号。0.39 的特异性 drop（对照随机 0.0）+ 充分性 0.972 明明是强因果，却被阈值伪影标成“无依赖”。这和 v1183 把内部最优误标成单调加速是对称的失误——这次是 under-claim。已修 `decide()`：把必要性/特异性/充分性拆成三个正交信号，verdict 不再被单一阈值掩盖强而冗余的效应。
