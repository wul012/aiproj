# v1179 运行说明：grokking（延迟泛化）

本版把版本线从已经做满的 inference-efficiency / PTQ 分支（v1170 spec-decode、v1175–v1178 量化），转向训练动力学里的一个著名现象：grokking（延迟泛化）。它的特点正好和本会话反复出现的“toy scale 把效应冲淡”相反——grokking 本来就是在小 transformer + 算法数据集上被发现的，玩具规模是它的天然栖息地。

## 任务

模数加法 `a + b = c (mod 97)`，把全部 `97^2 = 9409` 个有序对枚举成 token 序列 `[a, +, b, =, c]`（数字 0..96 各占一个 token，`+`、`=` 各一个，vocab=99，block_size=5）。只在答案 token 上算损失/准确率（`=` 位置的 logits 预测 c）。随机划分 train/val，train_frac=0.2。

## 配置（已校准到 grokking 区间）

- 模型：1 层 MiniGPT，n_embd=128，4 头，dropout=0。
- 训练：全 batch AdamW，lr=1e-3，weight_decay=1.0，最多 40000 步，每 100 步评估。
- 配对消融：同一 seed 内固定 init 与划分，只改 weight_decay（1.0 对 0.0），隔离权重衰减这一个变量；跨 5 个 seed 取多种子统计。

train_frac=0.2 不是随手选的。本版先用 train_frac sweep（单 seed，wd=1.0）确定区间：`0.35 -> 延迟 200 步`、`0.30 -> 800`、`0.25 -> 2100`、`0.20 -> ~11300`。高于约 0.4 时模型瞬间泛化（无延迟）。0.2 给出最戏剧、validation 在记忆期仍接近 chance 的延迟。

## 输入 / 输出

- 输入：脚本内构造的模数加法任务（不依赖语料）。
- 输出：
  - `f/1179/解释/grok_v1179/grok_v1179.{json,csv,txt,md,html}`
  - `f/1179/图片/grok-v1179.png`（accuracy vs log-step 的经典 grokking 曲线）

## 真实 GPU 结果（RTX 4060，5 seeds）

```text
status=pass
verdict=grokking_reproduced_wd_driven
wd=1.0: grok_rate=5/5, memorize≈step100, generalize≈step 14880±5971, val@mem≈0.147, final_val≈0.960
wd=0.0: grok_rate=0/5, memorize≈step100,  从不泛化（final_val≈0.159，停在记忆期水平）
```

结论：带权重衰减时，5 个 seed 都在约 100 步记忆训练集，然后在约 1 万–2.5 万步发生延迟泛化（validation 在间隔期间一直接近 chance，确认是真正的相变而非同步收敛）；去掉权重衰减后，5 个 seed 同样记忆，却在 40000 步内从不泛化。**权重衰减是被实测出来的 driver。** t_gen 的标准差很大（5971，约 40%），符合文献中 grok 步数高方差的说法，所以本版只在“真的 grok 了的 seed”上平均 grok 步数，并单独报告 grok_rate，不掩盖被截断的 seed。

边界：`toy_scale_single_task_modular_addition_grokking_not_a_scaling_claim`——这是单任务玩具规模的现象复现，不是对大模型的 scaling 结论。

## 验证命令

```powershell
python -m py_compile src\minigpt\grok_v1179.py scripts\run_grok_v1179.py tests\test_grok_v1179.py
python -m pytest tests\test_grok_v1179.py -q -o cache_dir=runs\pytest-cache-v1179-focused
python scripts\run_grok_v1179.py --device cuda --seeds 5 --out-dir output\grok-v1179-full
```

一句话总结：v1179 在 MiniGPT 上诚实复现了 grokking——记忆远早于泛化、延迟随数据比例下降而拉大、且由权重衰减驱动；这是本会话里第一个“正向”结果，而不是又一个 honest null。
