# v1194 运行说明：EWC（参数锚定）vs replay（数据回放）对抗灾难性遗忘

v1193 证明了在共享 1 层 transformer 上先学 A=(a+b) 再学 B=(a×b)（mod p）会灾难性遗忘 A，且这遗忘是**分布漂移**驱动；replay（训 B 时回放 A 的样本）能缓解。v1194 问下一个问题：一个**机制完全不同**的缓解法——**EWC**（在参数空间锚定对 A 重要的权重，对数据无感）——行不行，和 replay 比怎么样？

EWC：A 巩固后，在巩固权重 θ\* 处估计对角 Fisher F，训 B 时加惩罚 (λ/2)·Σ F_i (w_i − θ\*_i)²。

诚实比较 = 每个方法的**稳定性-可塑性前沿**：扫强度旋钮（EWC 的 λ / replay 的 buffer k），记 (acc_B 可塑性/新任务, acc_A 稳定性/旧任务)。一个方法的分数 = 沿前沿能取到的 **max min(acc_A, acc_B)**（同时保住两个任务的最好单点）。

## 真实结果（p=23，3 seed）

```text
最佳 min(acc_A, acc_B)：  replay 0.818±0.039   vs   EWC 0.192±0.039
replay 前沿(k→accB,accA): 0→(.81,.05) 8→(.81,.75) 16→(.81,.81) 64→(.81,.86) 128→(.82,.93)
EWC 前沿(λ→accB,accA):    0→(.81,.05) 1e10→(.38,.39) 5e10→(.32,.63) 1e11→(.29,.65) 3e11→(.28,.44)
```

`verdict = replay_dominates_ewc`（status=pass）。replay 能到 (acc_A 0.93, acc_B 0.82) 这种两个都高的点；EWC 的前沿贴着坐标轴——要么保住 A、要么学会 B，没有一个 λ 能两个都保住。

## EWC 拿到了公平的机会（不是稻草人）

诚实测量的关键是不能"故意把 EWC 做坏再说它输"。门控证明 EWC 真的发力了：
- **锚定确实保护 A**：跨 λ 的最大 acc_A=0.676，远高于无锚定的 naive 0.047（锚定把 A 留存抬了 0.63，绝不是 no-op/单位 bug）。
- **能保持可塑**：某些 λ 下 acc_B 仍达 0.805（≈naive），不是处处冻死。
- **前沿被夹住**：λ 最大（3e11）时 acc_B 塌到 0.283（进入冻结区），所以"没有 λ 两全"不是扫得不够。
- 用的是**逐样本**对角 Fisher、wd=0（避免 decay 与锚定打架）、惩罚对绑定的数字 embedding 只作用一次（已校验 named_parameters 不重复计）。
- 还在 [3e10, 6e10] 的拐点加密了 λ 网格（4e10/4.5e10/5e10），确保没漏掉隐藏的两全点。

所以 `ewc_all_or_nothing=True`：能保护、能可塑，但**不能同时**。

## 机制（接 v1193）

遗忘是**全局分布漂移**，而对 A 最重要的参数正是 B 必须改写的**共享/绑定数字 embedding 行**。对角 Fisher 是个**局部/可分**的锚——它只能靠冻结这些行来抵抗，而冻结就堵死了 B（可塑性崩）。replay 则把 A 直接留在损失里，不冻任何东西，所以两个都能保。这就是为什么数据回放赢过参数锚定。

## 诚实的预算披露

两种方法是不同的资源体制：replay 每步多喂 k 个存下来的 A 样本（**更大的 per-step batch**），存 k 个样本；EWC 存 Fisher 向量 + θ\*（两份模型大小）、回放零数据。所以"replay 赢"成立于**匹配 B 步数预算**，且部分就是 v1193 那个"见到 A 数据 > 见不到"的道理。因此我们声称的是**匹配步数下的 best-min 占优**，不是无视资源的 Pareto 全面占优。

## hybrid（EWC 加在 replay 之上有用吗）

EWC+replay（λ=3e10, k=8）min-both 0.116 vs 单独 replay(k=8) 0.755 → `ewc_adds=False`。在 replay 之上叠 EWC 不仅没帮忙，反而把 λ=3e10 的冻结效应带进来拖累了——进一步说明对这种全局漂移遗忘，replay 已足够，参数锚定是错的工具。

## 一个被自查纠正的门控阈值伪影

第一次跑 Phase B 判成 `review: ewc_anchor_does_not_engage`——因为我把"锚定发力"的门设成了绝对阈值 acc_A≥0.7，而多 seed 下 EWC 最大 acc_A=0.676 差一点点。但 0.676 vs naive 0.047 明明是巨大的保护效应，被一个 v1183 式的任意绝对阈值挡住了。改成**相对**判据（最大 acc_A 比 naive 高出 >0.2 即算发力）+ 前沿夹住判据，零重训从缓存重判。这是本 arc 第 6 次诚实纪律抓住我自己 decide 里的阈值伪影。

## 范围

玩具级模数运算、p=23、1 层 transformer、对角 Fisher 单锚 EWC；**不**是"EWC 在一般场景无用"（EWC 在很多持续学习设置里有效），只是说对这种全局漂移遗忘，局部参数锚定输给数据回放。

## 工程

Phase A（`run_ewc_replay_v1194.py`）一次训完两条前沿 + hybrid + 地板 + joint，存 cache；Phase B（`analyze_ewc_replay_v1194.py`）纯 CPU 重算 best-min/占优/门控。库复用 v1193 的任务/巩固/replay，新增 Fisher + EWC。10 个测试（合成前沿验 decide 全分支 + p=7 Fisher/EWC smoke）。

一句话总结：v1194 在 v1193 基础上对比两种缓解机制——匹配步数预算下，数据回放（replay）在稳定性-可塑性前沿上 best-min 0.82 完胜参数锚定（EWC）0.19；EWC 拿到了公平且充分扫过的机会（能保护 A、能可塑、前沿夹住），但全局漂移遗忘下对角 Fisher 局部锚只能靠冻结抵抗、两全不可得；过程中零重训自查纠正了一个绝对阈值门控伪影。
