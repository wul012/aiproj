# v1192 运行说明：校准（calibration）+ 温度缩放（全新一轴）

grokking/可解释性那条线在 v1191 收尾了。v1192 换一条**全新的轴**：校准。此前每一版都在量*准确率*或*分布保真度*（v1173 的 `KL(P_true||P_model)`），从没问过这个正交的问题——**模型说"x 类、概率 q"时，x 真的以频率 q 发生吗？**

复用 v1173 的随机 Dirichlet 任务（K=32 个上下文，每个有**已知**的类别分布 `P_true`，熵被扫开），所以校准是对着**精确 oracle**量的。

## 为什么用解析（analytic）指标，不用采样 ECE

采样版 ECE 有正偏（Jensen）：经验准确率是二项比例，所以**即使完美校准的模型**在有限采样上 ECE 也 >0——拿它跟"0 地板"比会**凭空造出**过自信（这正是 v1183 单阈值伪影那一类）。因为我们**构造**了 `P_true`，就直接解析地算 ECE/NLL/Brier/KL：oracle 的解析 ECE 恰好是 **0**（每个上下文 confidence ≡ true accuracy）。本版把采样 ECE 只留作旁证——它确实给出 oracle 采样地板 **0.003 > 0**，正说明为什么要用解析路径。

## 头条（n=10，3 seed，全程对着已知 P_true）

```text
hard_ce(n=10): conf 0.559 >> acc 0.442   (过自信 gap +0.118)
  解析 ECE 0.124±0.031  >>  精确 oracle 地板 0.000
  一个全局温度 T=1.82±0.10 (>1, 按 NLL 拟合):  ECE 0.124 -> 0.065
     配对(paired) ΔECE 0.059±0.029   |   KL 0.257 -> 0.161
```

`verdict = overconfidence_specifically_corrected_by_temperature`（status=pass）。status 只证明这是一次**有效测量**（模型胜过 uniform 的 KL 地板 0.483、有真实熵展开、温度可辨识、分箱非退化），**不**代表任何讨喜的幅度。

## 为什么是"特异"（不是"随便压平都行"）

- **U 形**：ECE 关于 T 是 U 形，最低点就在拟合的 T 处（拟合 ECE 0.065）。
- **错的 T 更差**：拿一个更过自信 regime 的 T=4.53 套上来，ECE 反而 0.141（远差于 0.065）。
- **对已校准的模型无效甚至有害**：把 hard_ce 的 T 套到一个本就校准好的模型上，ECE 从 0.040 升到 0.082。
- oracle 自己的解析 ECE 是 0。

## 不是魔法——这是有限样本 MLE 的产物

samples/ctx 扫描：随 n 变大，过自信和拟合的 T 都趋向 1（`n=3:T=4.45, n=10:1.82, n=30:1.24, n=100:1.14, n=300:1.08`）。过自信是少样本下向采样众数塌缩，不是 transformer 架构的内禀属性。这是 v1173"不是魔法、是数据效率"的镜像。

## 对 v1173 的诚实定位（不夸大新颖性）

期望 NLL = 熵 + KL，所以 NLL 就是 KL 换皮；在这个底座上 ECE 和 KL 在温度下**同向移动**（相对降幅 ECE 0.48 ≈ KL 0.37，`co_move=True`）。所以**不主张** ECE/KL 解离。校准在 KL 之外多给的是：(a) **方向**——KL 是方向盲的，ECE 能诊断是过自信还是欠自信；(b) 一个可操作的单标量修复，且对着精确 oracle 地板量。soft_distill 本就校准好（T=0.99）——这是和 v1173 的一致性核对，不是新发现。

## 边界（the null）

一个**低熵**任务实例（mean H 0.31）只有极轻微过自信（gap +0.013，对比高熵的 +0.118），且关键是**温度修不动**（ECE 0.045→0.046，`correctable=False`）——可被温度修正的过自信需要真实的偶然不确定性；低熵下几乎没有可修的全局过自信，单标量也碰不到逐上下文的残差。

**范围**：合成的偶然不确定性、少样本 regime；**不**是"transformer 普遍过自信"的论断。

## 工程：训练/分析分离（reuse-cached）

Phase A（`run_calibration_v1192.py`）只训练一次，把每个 arm 的逐上下文 logits 存进 `output/calibration-v1192/cache.pt`；Phase B（`analyze_calibration_v1192.py`）纯 CPU、不训练，加载 logits 做温度拟合 + 所有指标 + decide()。所以反复改 verdict/阈值**零重训**（镜像 v1185 训练 → v1186/88/91 分析）。一个被自查纠正的点：边界 arm 的"null"最初被我误定义成"统计上不过自信"，但本版讲的是温度**修复**，正确的 null 定义应是"温度修不动"——已据此改正（纯逻辑改动，从缓存重新派生，未重训）。

## 验证命令

```powershell
python -m pytest tests\test_calibration_v1192.py -q -o cache_dir=runs\pytest-cache-v1192-focused
python scripts\run_calibration_v1192.py --device auto          # Phase A：训练一次，存 logits 缓存
python scripts\analyze_calibration_v1192.py                    # Phase B：纯 CPU 分析
```

一句话总结：v1192 开了校准这条新轴——在已知 P_true 的随机任务上，少样本训练的 transformer 系统性过自信（conf 0.56 ≫ acc 0.44，ECE 0.124 ≫ 精确 oracle 地板 0），一个全局温度 T=1.82 显著且特异地把 ECE 降到 0.065（配对显著、U 形、错 T 更差、对已校准模型无效）；这是少样本 MLE 产物（n 增大 T→1），诚实地说 ECE 与 KL 在此同向移动、校准多给的是"方向"与可操作修复，而非与 v1173 的解离。
