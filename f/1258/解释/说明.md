# v1258 route-promotion release readiness receipt index 运行说明

## 本版目标

v1258 接在 v1257 之后，目标是把已经 granted 的 route-promotion release readiness downstream receipt 做成可检索 index。v1257 说明了哪个 consumer 可以在 `objective_level_contrast` route 上以 `bounded_route_promotion_release_governance_only` 范围消费 checked summary；v1258 则把这份 receipt 固定成 lookup row，方便后续 review、bundle 或 release manifest 消费。

本版不重新判断 release readiness，不重新解释 v1255/v1256/v1257 的模型能力结论，也不允许 promotion。它只证明：真实 `f/1257` receipt 存在、可 digest、状态为 granted、scope 和 boundary 没有变宽、blocked uses 完整、上游 source digest rows 仍然存在，并且下一步可以进入 index review。

## 链路角色

v1258 的角色是把 v1257 receipt 从“可消费的单份凭证”变成“可被后续模块按 lookup key 检索的索引条目”。这个 index 不增加模型能力声明，只把 receipt 的 route、consumer、scope、claim、source check digest、receipt digest 和上游 source digest count 放到一个稳定结构中。

后续如果做 index review，应该消费本版 JSON，并重点检查 lookup row、receipt digest、bounded scope、blocked uses 和 `promotion_ready=False` 是否保持一致。这个分层让后续发布治理链路不用直接扫 `f/1257` 的整个 receipt report，也不用重新打开 `e/795`、`e/796`、`e/800`。
