# v1190 运行说明：grokking 输出 logit 频率对齐

本版按图片里的建议推进：v1188 已经证明 grokked checkpoint 的 number embeddings 出现 Fourier concentration；v1190 继续往前看一步，检查输出 logits 是否也在同一组频率上组织答案，从而更直接地验证 `a + b = y (mod 97)` 背后的 trig-identity 机制。

## 输入

- checkpoint：`f/1185/解释/grok_checkpoint_v1185/grok_checkpoint_v1185.pt`
- 模型：v1185 shipped canonical grokking checkpoint
- 任务：所有 `97 × 97` 个 prompt `[a, +, b, =]`
- 对照：
  - random-init 同架构模型
  - ideal addition target cube

## 方法

对每个 prompt `[a,+,b,=]` 读取输出 logits 的 number-token 部分，得到一个 `L[a,b,y]` logit cube。对每个输出 token `y` 的 surface `L[:,:,y]` 做 2D FFT，并把所有 `y` 的 power 相加。理想加法 surface 是 ridge：`a+b=y`，它在 `(a,b)` 频率空间里的能量只落在 `k_a == k_b` 的对角线上。因此：

- 如果模型真的用类似 Fourier/trig 的结构实现加法，logit FFT 应该有高 diagonal fraction。
- 如果 v1188 的 embedding 频率是真机制的一部分，logit 的 dominant diagonal frequencies 应该和 embedding dominant frequencies 对齐。

## 真实结果

```text
status=pass

heldout_acc=0.965989
embedding_top_freqs=[43, 3, 48, 26, 44]
logit_top_freqs=[43, 3, 48, 26, 44]
embedding_logit_top_freq_overlap_count=5
embedding_logit_top_freq_overlap_fraction=1.0
logit_diagonal_fraction=0.718712
random_logit_diagonal_fraction=0.000122
ideal_logit_diagonal_fraction=1.0
logit_top_k_diagonal_fraction=0.610591
```

解释：shipped checkpoint 的输出 logits 有 `71.9%` 的非 DC 频谱能量落在加法所预期的对角频率上，而 random-init 几乎没有这种结构。更关键的是，logit top-5 对角频率和 v1188 的 embedding top-5 频率完全一致：`43, 3, 48, 26, 44`。这把 v1188 的“embedding 有 Fourier structure”推进成“embedding 与输出读出使用同一组频率支持加法答案”。
