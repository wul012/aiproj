# v1168 DPO+SFT 辅助项（NLL 正则化 DPO）：辅助项能否让 DPO 不再破坏生成？

## 目标

v1166 测得：原始 DPO 优化的是**相对边际**，会把 logp(chosen) 一起压低、held-out 生成回退到基座以下，而只做正样本的 SFT-on-chosen 反而更好。本版测试教科书级的修法——**NLL 正则化 DPO（DPO+SFT / RPO）**：

```
L = L_DPO + λ · SFT_CE_mean(chosen)
```

λ=0 即原始 DPO（生成崩）；λ 增大时 chosen 的 NLL 辅助项把 logp(chosen) 锚住，逐渐逼近纯 SFT-on-chosen。诚实的问题（沿 λ 扫描，并以 λ=0 原始 DPO 与算力对齐的 SFT-on-chosen 两个端点为锚）：**某个 λ>0 能否把生成救回来？救回来之后，DPO/偏好项是否比纯 SFT 多做了什么（比如压制特定的易混错误），还是 SFT 项把活全干了？**

## 设计先于 GPU（含会话限额下的应对）

仍按纪律先开对抗式设计评审。本次评审中途触发会话限额，仅"损失正确性"视角（Lens B）跑完、可行性探针等失败。但**关键探针我在主线程上亲自跑了**（`output/dpo_aux_probe/probe.py`，Lens B 的脚本在失败前已写好），拿到了经验数据，再据此自行锁定设计——没有空等限额重置。Lens B 钉死的关键实现点：辅助项必须是 `train_sft` 在 chosen 上最小化的那个**逐 token 平均交叉熵**（同一 `(t+1)>=n_prompt` 掩码），且与 DPO 项的 chosen 求和 logp **共用同一次前向**（不二次前向、不换 reduction），这样 λ→∞ 收敛到 SFT-on-chosen、λ=0 复现原始 DPO，扫描才是两端点间的公平插值。

## 关键结果（真实 GPU 运行，RTX 4060，3 seeds，4L/128）

`status=pass`、`verdict=dpo_sft_aux_recovers_generation_matches_plain_sft`、`vanilla_dpo_verdict=vanilla_dpo_regresses_generation`。

| arm | exact-match | margin | conf-err | Δlogp(chosen) |
|---|---|---|---|---|
| SFT 基座 | 0.506 | 12.0 | 0.023 | — |
| 原始 DPO（λ=0） | **0.142** ↓ | 81.3 | 0.001 | **−23.0** |
| DPO+aux λ=0.25 | 0.619 | 60.8 | 0.001 | −1.2 |
| DPO+aux λ=0.5 | 0.663 | 57.7 | 0.002 | −0.7 |
| **DPO+aux λ=1.0（最佳）** | **0.684** ✓ | 49.2 | 0.002 | −0.1 |
| DPO+aux λ=5.0 | 0.669 | 34.6 | 0.010 | +0.2 |
| SFT-on-chosen | **0.739** | 23.9 | 0.009 | +0.6 |

- **辅助项确实救回了生成（框架 A 成立）：** λ=0 把 em 从 0.506 砸到 0.142；只要 λ≥0.25 就救回到 ~0.62-0.68，最佳 λ=1.0 达 0.684（甚至高于基座 0.506）。Δlogp(chosen) 从 −23 一路愈合到 ~0——NLL 锚的直接证据。
- **但只是"追平"纯 SFT，没有超过（框架 B 为零结果）：** 最佳 λ 的 em 0.684 与 SFT-on-chosen 0.739 在合并标准差内无法区分（`matches_sft=True`、`beats_sft=False`）。
- **易混压制的优势在真实规模下消失：** 探针（微型 2L/64、基座 conf 0.073）里 DPO+aux 把易混错误压到 0、SFT 留在 0.073；但真实 4L/128 基座的 conf 本就只有 0.023，几乎无可压空间，DPO+aux（0.002）与 SFT（0.009）的差距不显著（`suppresses_confusable_vs_sft=False`）。
- **诚实结论：** NLL 辅助项是 DPO 破坏性的真实修法（救回生成），但在本规模下偏好/DPO 项**相对纯 SFT 没有可测的额外能力收益**。DPO+aux 保留了比 SFT 更高的边际（λ=1：49 vs 24），但那点多出来的边际不转化为能力——正是 v1166 "边际≠能力" 的延续。

## 诚实边界

1. **这是 NLL 正则化 DPO（DPO+SFT / RPO）用在合成确定性正确性信号上，不是人类偏好/RLHF。**
2. **闸门只保证可测：** 复用 v1166 的边际可改善性闸门（基座 em 在 [0.40,0.85]、λ=0 的 DPO 损失在下降、边际可改善）；`status=pass` 不代表 DPO+aux 好。
3. **公平对比：** 所有臂共用同一基座克隆 + 同一批采样种子，唯一差别是损失；DPO+aux 每步 2 次策略前向，故 SFT-on-chosen 按前向次数对齐拿到约 2 倍步数（对控制臂更有利，强化"SFT 不输"的结论）。
4. **λ=0 复现原始 DPO** 由单元测试逐参数 allclose 钉死；λ→∞ 设计上收敛 SFT。
5. **规模依赖：** 微型从零 char 级；易混压制的零结果与"无超过 SFT"都随规模/任务难度变化。

## 一句话总结

v1168 给 DPO 加上 chosen-NLL 辅助项（NLL 正则化 DPO）并诚实测得：辅助项确实把原始 DPO 砸坏的生成救了回来（λ=0 的 0.14 → 最佳 λ=1 的 0.68，Δlogp(chosen) 从 −23 愈合到 0），但只追平、未超过算力对齐的纯 SFT-on-chosen（0.74），探针里看到的易混压制优势在真实规模因基座本就少犯而消失——辅助项修好了破坏性，偏好项却没带来超过 SFT 的能力，这是对 "边际≠能力" 的又一次诚实印证。
