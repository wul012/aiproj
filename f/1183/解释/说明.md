# v1183 运行说明：grokking 权重衰减剂量-反应

本版把 v1179 的二元结论“weight decay 驱动 grokking”推进成一条剂量-反应曲线：扫描权重衰减强度，测 `a + b = c (mod 97)` 上泛化步 `t_gen` 如何随它变化。复用 v1179 的逐臂训练原语，同一 seed 下所有 wd arm 共享 init+split，只改 weight_decay（配对隔离）。

## 配置

- 任务/模型同 v1179：模数加法，1 层 MiniGPT，n_embd=128，train_frac=0.2，全 batch AdamW，lr=1e-3，max_steps=40000，5 seeds。
- 扫描网格：`weight_decay ∈ {0.0, 0.1, 0.3, 1.0, 3.0}`。

## 真实 GPU 结果（RTX 4060，5 seeds，40k budget）

```text
 wd    grok_rate   t_gen_mean±std      final_val
 0.0     0/5        — (never)          0.159
 0.1     1/5        38000 (单seed)      0.721
 0.3     5/5        26560 ± 5684       0.960
 1.0     5/5        14920 ± 5944       0.960   ← 最快
 3.0     0/5        — (never, 仍 5/5 记忆) 0.195
```

`verdict=wd_dose_response_interior_optimum`：非单调、有内部最优。

- grok 最快在 `wd≈1.0`；在 [0.3, 1.0] 区间 `t_gen` 随 wd 单调下降，wd=1.0 显著早于 wd=0.3。
- 两端都失败：太弱（wd≤0.1）grok 不可靠或从不；太强（wd=3.0）虽然 5/5 记忆，却 0/5 grok。
- `too_much_wd_breaks_memorization=False`：wd=3.0 不是记不住，而是记住却不泛化。
- 延长预算探针（wd=3.0、2 seeds、100000 step）仍 0/2 grok（final_val ~0.17-0.21）→ 高端失败是真实过度正则化，不是预算截断。

## 过程诚实记录

第一次跑出来时 verdict 被误判为 `wd_dose_response_monotone_acceleration`——因为判定只看了会 grok 的子区间 [0.3,1.0]，漏掉了最强 wd=3.0 根本不 grok。已修复 `decide_wd_law`，加入 `interior_optimum`/`high_end_grok_censored` 检测，verdict 正确变为内部最优。
