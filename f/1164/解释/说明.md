# v1164 SFT 指令微调（completion-only 损失掩码）

## 目标

让 MiniGPT 学会**遵循指令**：给一个 op 标记（`C` 复制 / `R` 反转 / `S` 排序）和一个输入串，模型要在**没见过的输入**上生成正确输出。核心技术是 SFT 的关键机制——**只在 completion（回答）token 上算损失**，prompt 与 padding 用 `ignore_index=-100` 掩掉。

> 本版是从 RoPE-base 召回探针（命中可行性评审预言的 Risk 1：微型从零模型学不会那个任务）诚实转向 SFT 后的结果。SFT 是把基础语言模型变成"会聊天/会听话"的模型的根本技能。

## 运行命令

```powershell
python scripts\run_sft_instruction_v1164.py --device cuda --seeds 3
```

## 关键结果（真实 GPU 运行，RTX 4060，3 seeds）

- `status=pass`、`decision=sft_instruction_following_measured`、`verdict=completion_only_helps_early_benefit_shrinks_with_training`
- **能力**：在最大训练预算（1500 步）下，模型对**未见输入**的指令遵循 exact-match 达 **0.795**（length-4 串随机猜中概率约 0.0016）。逐 op：C=0.75、R=0.80、S=0.83。
- **核心发现（completion-only 掩码的价值随训练预算变化）**：
  - 150 步：completion-only **0.266** vs full-loss **0.029** → 掩码优势 **+0.238**（约 9 倍）
  - 1500 步：completion-only **0.795** vs full-loss **0.776** → 掩码优势 **+0.019**（基本收敛）
  - 即 **completion-only 掩码是"低算力加速器"**：训练少时巨大领先，训练足够久两者趋同。

## 诚实边界

1. **不是单点结论。** 单看 600 步会得到"掩码明显有用"，单看 1500 步会得到"掩码无差异"——两者都会误导。本版用 step 扫描（150/400/800/1500）呈现完整曲线，结论是"早期有用、随训练收敛"，这才是诚实的全貌。
2. **范围：** 本版隔离的是 SFT 的**监督机制**（指令格式化 + completion-only 损失 + 留出泛化），从零在指令数据上训练，而**不是**在另行预训练的基座上微调——这是在本规模上演示该机制的恰当范围。
3. **留出即泛化：** 留出输入与训练输入按 op 不相交，所以 exact-match 衡量的是"把 op 应用到新输入"的泛化，不是记忆。
4. **学习性闸门：** 最大预算下 completion-only 的 copy（最简单 op）exact-match=0.755 ≥ 0.5 闸门，`task_learned=True`。若闸门不过，会被诚实报成"未学会"，而非"掩码无效"。
5. **方差：** full-loss 在中段（800 步）方差很大（不同 seed 收敛快慢差异大），这本身印证了"无掩码时学习更不稳定"。

## 一句话总结

v1164 让 MiniGPT 通过 SFT 学会在未见输入上遵循 copy/reverse/sort 指令（exact-match 0.79），并用 step 扫描诚实地揭示了 SFT 的核心机制——completion-only 损失掩码是**低算力加速器**：150 步时把指令遵循从 0.03 抬到 0.27，但训练足够久后优势收敛消失。
