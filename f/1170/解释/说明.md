# v1170 投机解码（speculative decoding）：可证明等价，但在 char 玩具尺度上是 FLOPs / 墙钟的亏损

## 目标与定位（不是对齐，是推理效率）

对齐线在 v1169（奖励模型离分布脆弱）告一段落。v1170 回到 **v1161 KV-cache 开启的推理效率线**，复用那套缓存：用一个又小又快的 **draft（草稿）模型** 自回归提议 `K` 个 token，再让大的 **target（目标）模型** 用**一次** `(K+1)` 宽的前向**并行验证**，配合接受/拒绝规则，**保证输出分布与只用 target 解码完全一致**。这是 Leviathan / Chen 的经典算法。

## 设计先于 GPU：对抗式面板抓出一个会致命的"闸门错误"

按项目纪律，GPU 之前先跑探针 + 对抗式设计面板（Workflow，4 个视角 + 综合）。面板抓到一个**阻断级**问题：把"贪心逐字节相同"当闸门是**错的**——v1161 当初正是因为 ~1e-6 的 logit 漂移会翻转 argmax 的近似并列，**主动**把"贪心相等"移出了闸门；而且 plain 贪心走的是 `top_k=1→multinomial`、spec 走 `argmax`，**两种并列打破规则不一致**；探针 0 失败只是因为它跑在**随机初始化**模型上（logit 弥散、没有并列）。

**修正后的闸门**键在数值不变量，不在逐字节：
1. `(K+1)` 宽缓存验证前向 == 全量前向，`< 1e-4`（v1161 同款不变量，取验证块宽）；
2. 投机**采样**与只用 target 采样的全变差 TV ≤ 噪声地板 `mean+2σ`（固定 N、多 seed）；
3. 实测每块产出 ≈ 几何公式 `(1-α^(K+1))/(1-α)`（接受规则一致性，贪心测不出的 bug 它能测）。
两条路径共用 **最低索引 argmax**；并列检测器（top-2 间隔 <1e-5）把任何贪心翻转判为 `tie_artifact` 而**非** `correctness_failed`。`status==pass` 只证明"实现正确、效率数字可信"，**绝不**等于"它快"。

面板的第二个阻断点（B3）：探针用平凡可学的玩具序列，α 一律 =1，**整条"α-vs-草稿质量"主轴从未验证**。于是上 GPU 前先在**真实 {C,R,S,L} 语料**上用**欠训练检查点草稿**亲跑探针，确认 α 落在 (0.24,0.63)——主轴成立，无需改写结论。

## 关键结果（真实 GPU 运行，RTX 4060，3 seeds，target 4L/64，block_size=16）

`status=pass`、`verdict=spec_decode_verified_identical_fewer_forwards_no_wallclock_win`、target 贪心 exact-match **0.867**。

| 测量 | 数值 |
|---|---|
| 验证 logit 最大差（vs 全量前向） | **1.96e-05** < 1e-4 ✓ |
| 贪心补全相同 / 总数（EOS 截断后） | **3600/3600**（0 真分歧、0 并列伪差）✓ |
| 采样 TV / 噪声地板 | **0.00202** ≤ 0.00263 + 2×0.00128 ✓ |
| 接受规则一致性残差（最大） | **0.221** < 0.5 ✓ |

| **target 处理位置数 / plain（FLOPs 代理，主指标）** | ckpt200,K=4 **1.28×**；最差 ckpt30,K=4 **1.84×** |
| 每次 target 前向调用产出 token（次要计数指标） | K=4 时 ~2.1（plain=1） |
| 总前向数（draft+target）/ token | ~2.1（plain=1.0） |
| **墙钟 spec / plain** | **0.55×（更慢）** |

## 诚实结论：正确性是真的，加速不是

- **算法正确性可证明、已验证**：验证前向到浮点精度复现全量前向，贪心补全 3600/3600 相同，采样分布在噪声内一致，接受规则符合几何公式。投机解码在本实现里**确实**产出与 target 解码同分布的结果。
- **但"更少前向"不等于"更便宜"**：`(K+1)` 宽验证**无论是否接受都要处理 K+1 个位置**，所以**主指标 target 处理位置数比 plain 多 1.28×（FLOPs 亏损）**；"每次 target 调用产出 ~2 token"只是把宽验证记成"一次"的**讨好性计数指标**；算上 draft，**每 token 总前向 ~2.1**，证伪"小草稿是免费的"。
- **墙钟 0.55×（更慢）是本尺度的预期诚实结果**：两个模型都远未喂饱 GPU，`(K+1)` 宽与 1 宽前向耗时几乎相同、Python 开销占主导，draft 的 K 次前向纯属额外开销。**只有当 target 前向真正主导运行时**（大模型、延迟受限硬件）加速才会出现——主指标"处理位置比"已机理性地解释了为什么这里不会。

## 被证伪的讨好叙事

"投机解码让我们的模型更快"（墙钟此处为假）；"K 越大越好"（α 随 K 内位置漂移、位置数随 K 涨，非单调）；"小草稿免费"（draft 每块 K 次前向）；"更少前向=更便宜"（位置/FLOPs 反而上升）；"等价即可直接上线"（这是 **B=1 char 玩具的正确性**，批量 ragged-accept 与喂饱尺度均不在范围内、未证明）。

## 范围

B=1 贪心与采样；批量（每行接受长度不齐）解码明确不在范围。自投机（draft==target）作为 α=1 锚点，已标注为 **FLOPs/墙钟亏损**，仅用于验证接受规则。
