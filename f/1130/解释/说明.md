# v1130 运行截图与解释

v1130 是工程后期保养五版中的第一版，目标是先做命名止血。它没有移动历史文件，也没有批量重命名长模块，而是新增一个可运行的 publication naming readability 扫描器，把历史长命名记录为 `legacy-watch`，同时给后续新增文件建立短别名规则。

真实运行命令：

```powershell
python -B scripts\publication\check_publication_naming_readability_v1130.py --out-dir f\1130\解释\naming-readability-v1130 --require-policy-ready --force
```

关键结果：

- `status=watch`
- `decision=publication_short_alias_policy_ready`
- `scanned_file_count=837`
- `repeated_receipt_index_file_count=548`
- `long_name_file_count=808`
- `policy_ready=True`

`watch` 不是失败，它说明历史命名压力真实存在，但本版只负责建立短别名和新入口规则。后续 publication 类脚本优先放入 `scripts/publication/`，并使用 `pub_receipt_index`、`pub_receipt_review`、`receipt_contract_check`、`holdout_pub` 等短别名，避免继续生成重复 `receipt_index` 多次叠加的文件名。
