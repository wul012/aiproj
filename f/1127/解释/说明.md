# v1127 运行截图与解释

## 本版目标
v1127 把 v1125 lookup-only receipt 和 v1126 contract check 合并成新的 receipt index。v1125 负责记录 review，v1126 负责证明 receipt 能从源 review 重建，v1127 则把这两份证据包装成一个下游可查入口，让后续 review 不必同时打开 receipt 和 check 两个目录。

本版不训练模型，不改写 v1125 receipt，不改写 v1126 contract check，也不打开 production promotion。`index_ready=True` 只代表索引结构完整、上游证据存在、contract check ready，并且所有使用边界仍然是 downstream governance lookup only。

## 输入与输出
输入：

```text
f/1125/解释/receipt-v1125
f/1126/解释/receipt-check-v1126
```

输出：

```text

f/1127/解释/receipt-index-v1127/*.csv

f/1127/图片/v1127-receipt-index.png
```
