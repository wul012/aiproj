# v1158 MiniGPT LoRA 领域自适应

## 目标

v1157 发现：在权重绑定的微型字符模型上，LoRA 只拿到全参微调约 5% 的泛化收益，因为需要改的信号落在被冻结的 token embedding 上。v1158 把场景换成 LoRA 真正擅长的领域自适应：源域 A 与目标域 B 用**完全相同的字符词表**、只是**语序结构不同**。这样基座在 A 上学到的 embedding 能原样迁移到 B，剩下的差距是纯结构性的——正好是注意力/MLP（也就是 LoRA 能触达的层）该解决的问题。

本版不改基座结构、不引入新字符、不调参去“制造”一个好看的胜负，而是先确认存在真实的结构性领域差距，再诚实比较 base / 全参微调 / LoRA 在目标域 held-out 上的表现。

## 运行命令

```powershell
python scripts\run_lora_domain_adaptation_v1158.py --device cuda --base-steps 400 --adapt-steps 400
```

## 关键结果（真实 GPU 运行，RTX 4060）

- `status=pass`、`decision=lora_domain_adaptation_succeeded`
- 共享词表：source vocab 63 / target vocab 63 / shared 64（结构不同、字符集合相同）
- 基座（只在 A 上训练）：A held-out loss `0.883`，B held-out loss `3.988` → `domain_gap=3.105`，B 准确率仅 `0.462`
- 全参微调到 B：B held-out loss `1.182`（delta `-2.806`）
- LoRA 自适应到 B（全部 16 个 attention+MLP 线性层，`trainable_ratio_percent=7.5051`）：B held-out loss `0.892`（delta `-3.096`），B 准确率 `0.700`
- `lora_vs_full_capture_ratio=1.1034`：LoRA 在目标域 held-out 上达到甚至略优于全参微调参考
- 可逆性：带 adapter 时 A held-out loss 升到 `3.756`（`adapter_forgetting_on_source=+2.872`），但因为基座冻结，**摘掉 adapter 即可精确恢复原 A 模型**——全参微调做不到这点

## 诚实边界

全参微调是**参考**而非精调到收敛的上界（lr 3e-4、400 步），所以“LoRA ≥ 全参微调”不应被过度解读为“LoRA 永远更强”。诚实的结论是：LoRA 用 7.5% 的可训练参数，在这个结构性领域差距上达到了与全参微调参考相当甚至略好的目标域泛化，并且保持可逆。这正是 v1157 预测的、LoRA 该赢的场景。

## 一句话总结

v1158 兑现了 v1157 的预测：当领域差距是结构性的、embedding 可原样迁移时，LoRA 用 7.5% 参数把目标域 held-out loss 从 3.99 降到 0.89（准确率 0.46→0.70），达到全参微调参考的水平且可逆——LoRA 的价值被真实、干净地证明了。
