# v1178 运行说明：PTQ policy sensitivity

本版接在 v1177 PTQ deployment candidate selector 之后。

v1177 在一组默认质量预算下选出了 `group32:3b`，但这个选择是否稳定，需要继续验证。本版新增三档 policy profile sweep：

- `strict_quality`：更严格质量预算，偏保守。
- `balanced_default`：沿用 v1177 默认预算。
- `aggressive_compression`：更激进压缩预算，只有在允许更大可见准确率下降时使用。

输入：

- `f/1175/解释/ptq_v1175/ptq_v1175.json`

输出：

- `f/1178/解释/ptq_policy_sensitivity_v1178/ptq_policy_sensitivity_v1178.json`
- `f/1178/解释/ptq_policy_sensitivity_v1178/ptq_policy_sensitivity_v1178.csv`
- `f/1178/解释/ptq_policy_sensitivity_v1178/ptq_policy_sensitivity_v1178.txt`
- `f/1178/解释/ptq_policy_sensitivity_v1178/ptq_policy_sensitivity_v1178.md`
- `f/1178/解释/ptq_policy_sensitivity_v1178/ptq_policy_sensitivity_v1178.html`
- `f/1178/图片/ptq-policy-sensitivity-v1178.png`

真实 v1175 artifact 的结果：

```text
strict_quality          -> per_tensor:4b
balanced_default        -> group32:3b
aggressive_compression  -> per_channel_row:3b
```

结论：

- `selection_stable_across_profiles=False`
- `sensitivity_verdict=candidate_choice_changes_with_quality_budget`
- 默认仍建议使用 `balanced_default`，因为它在质量预算内压低 effective bits，但没有把更大 exact-match loss 偷偷当作免费收益。

验证命令：

```powershell
python -m py_compile src\minigpt\ptq_policy_sensitivity_v1178.py scripts\run_ptq_policy_sensitivity_v1178.py tests\test_ptq_policy_sensitivity_v1178.py
python -m pytest tests\test_ptq_policy_sensitivity_v1178.py tests\test_ptq_candidate_v1177.py -q -o cache_dir=runs\pytest-cache-v1178-focused
python scripts\run_ptq_policy_sensitivity_v1178.py f\1175\解释\ptq_v1175 --out-dir f\1178\解释\ptq_policy_sensitivity_v1178 --require-pass --force
```

一句话总结：v1178 证明 PTQ 候选不是跨预算不变的“唯一正确答案”，而是会随质量容忍度变化；项目默认采用 `balanced_default`，并把 strict/aggressive 的取舍显式暴露出来。
