# v1189 运行说明：CI unittest portability 修复

本版是一次 CI 修复和索引收口版本，不新增模型能力，不重新训练，不改变 v1185-v1188 的 grokking / interpretability 结论。

## 失败根因

GitHub Actions 从 v1186 开始失败，最新 v1188 main run 的失败点在 Unit tests：

```text
Run python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-ci --fail-under 80
ERROR: test_grok_predict_v1186 (unittest.loader._FailedTest.test_grok_predict_v1186)
ImportError: Failed to import test module: test_grok_predict_v1186
ModuleNotFoundError: No module named 'pytest'
```

CI 的 coverage 入口使用的是：

```powershell
python -B -m coverage run --source=src/minigpt -m unittest discover -s tests -v
```

也就是说，全量测试是由 Python stdlib `unittest` 发现和导入的，不保证安装 `pytest`。`tests/test_grok_predict_v1186.py` 顶层导入了 `pytest`，只为了使用 `@pytest.mark.skipif` 跳过缺失 checkpoint 的集成测试。本地环境有 pytest 时不会暴露，CI 环境没有 pytest 时模块导入直接失败。

## 修复

- `tests/test_grok_predict_v1186.py`：移除顶层 `import pytest`。
- 改用 stdlib `unittest.skipIf`：
  - 旧：`@pytest.mark.skipif(not DEFAULT_CHECKPOINT.exists(), reason=...)`
  - 新：`@unittest.skipIf(not DEFAULT_CHECKPOINT.exists(), "...")`
- 同时补上测试文件自己的 `src/` 路径注入，避免 `unittest` 直接按模块导入时找不到 `minigpt`。

## 本版附带索引修复

`代码讲解记录_工程保养阶段/README.md` 顶部之前仍停在 v1184，虽然 `1197-v1185` 到 `1200-v1188` 的讲解文件已经存在。本版补上 v1185-v1188 和 v1189 的索引，避免工程保养阶段目录看起来断档。
