# v1133 运行截图与解释

v1133 是工程后期保养五版中的第四版，目标是建立模型能力主线回归节奏。它不训练新模型，也不声明模型能力提升，而是检查最近 README 版本段落里是否长期只有治理或维护版本，提醒后续安排真实模型能力验证。

真实运行命令：

```powershell
python -B scripts\evaluation\check_model_capability_cadence_v1133.py --out-dir f\1133\解释\model-capability-cadence-v1133 --require-ready --force
```

关键结果：

- `status=watch`
- `decision=model_capability_cadence_ready`
- `leading_non_capability_run=12`
- `max_non_capability_run=4`
- `latest_model_capability_version=not_found_in_recent_window`

`watch` 是本版的有效输出：它说明最近窗口内没有找到真实模型能力验证版本，后续应该安排 holdout accuracy、required term coverage、loss signal bridge、unassisted repair 或 decoder anchor distribution 等能力回归，而不是继续只推进 receipt 或维护链。
