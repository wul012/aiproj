# v1195 运行说明：任务相似度 → 灾难性遗忘（持续学习轴 capstone）

v1193 证明灾难性遗忘是**分布漂移**驱动（随机标签的 B 和真乘法一样遗忘 A）；v1194 证明 replay 胜过 EWC，因为漂移改写的是**共享/绑定的数字 embedding**。v1195 问这条轴最自然的收尾问题：**任务相似度会不会"分级"决定遗忘？沿哪个轴？**

设置沿用 v1193：共享 1 层 transformer，序列 `[a, OP, b, EQ, c]`；A=(a+b) mod p（PLUS 算子）巩固到平台，然后训练任务 B（TIMES 算子）固定步数，遗忘 = A 留出集准确率的跌幅。全局留出 (a,b) 对掩码把 A 的测试操作数从所有训练里隔离。

**唯一的 x 轴 = B 与 A 的解析输出表重叠度**（模型无关）：
```
overlap(B) = |{(a,b): f_B(a,b) == (a+b) mod p}| / p²
```
两种手法都改变它，结论是：**遗忘由 overlap 决定，与"算子家族"无关**（家族只通过 overlap 起作用）：
- **混合（连续）**：B_s 在一个分层抽样的比例 s 的格子上算 (a+b)，其余算 (a*b)。
- **类型家族（定性）**：add_same(a+b)、add_offset(a+b+c0)、linear(2a+b)、mul(a*b)、rand(固定随机表)。add_offset/mul/rand 的 overlap 都≈0，尽管它们从"同算子+常数"到"无结构"天差地别。

## 真实结果（p=23，5 seed，verdict=`forgetting_governed_by_output_overlap`，status=pass）

```text
arm            overlap   forgetting        learned   accB(冲突格,训练)
mix s=1.0       1.000    -0.002            (= add_same 地板，构造性，剔除出统计)
mix s=0.875     0.885     0.142
mix s=0.75      0.760     0.321
mix s=0.5       0.532     0.657
mix s=0.25      0.279     0.802
mix s=0.0       0.043     0.938   (= mul 端点)
type add_offset 0.000     0.991    同 '+' 算子、仅 +c0 → 仍灾难性遗忘
type linear     0.040     0.949
type mul        0.043     0.938
type rand       0.043     0.957
type add_same   1.000    -0.002    唯一安全点（逐字节相同的表）
```

- **C1 斜率**：Spearman(overlap, 遗忘) = **−1.0**（精确置换 p=0.017），跨度 0.80 → 单调分级。
- **C2 家族是红鲱鱼**：overlap≈0 处 add_offset（同 '+'，仅 +c0）遗忘 0.991 ≈ mul 0.938，都 ≫ add_same；`family_protects=False`。**用第二种独立手法再次确认了 v1193 的分布漂移结论**。
- **C3 统一**：3 个独立类型点（add_offset/linear/rand）都落在混合曲线上（残差 0.05/0.01/0.02 < 0.15 容差）。
- **不是 B-学会度伪影**：控制 accB 与共享 embedding 漂移后 overlap 仍主导（标准化 β_overlap=−0.98 vs β_accB=0.0；β_overlap|drift=−0.998 vs β_drift=−0.02）。

## 形状：≈ 比例式覆写，不是全局坍塌（诚实去夸张）

平凡的"只覆写冲突格"零假设预测 遗忘 ≈ (1−overlap)·平台。观测对该零假设的平均超出仅 **+0.076**（< 预登记的 0.10 门槛，`superlinear=False`），只在中段 overlap 有轻微凸起（最大 +0.193 @ overlap 0.53）。所以 **overlap 律基本就是"覆写比例"的定量化，不是特殊的全局坍塌**；v1193 的"灾难性"遗忘只是这条分级、近似比例覆写曲线在 **overlap=0 端点** 的取值。早期探针看到的"悬崖"是错觉——所有"换算子"的改动都恰好落在 overlap≈0。

## 设计评审（5 视角）逼出的关键 must-fix（先于建模）

- **x 轴必须是解析 overlap，不是模型测得的 overlap**：后者部分是"遗忘少"的*结果*（保住 A 的模型仍会预测 add），回归会变成自己对自己，循环论证。
- **每个 B 函数先物化成一张确定的 (p,p) 表再切掩码**，混合用分层抽样让实现的 add 比例精确；rand/混合用独立加盐的 Generator，训练流与测试流对同一 (a,b) 给同一目标。
- **overlap=1 端点构造性无遗忘**（= v1193 continue-on-A 地板，且被算子 token 混淆），**剔除出统计**，只主张 overlap<1 的内部斜率。
- **混合不在留出格上泛化**（随机逐格 add/mul 划分，模型无法对未见格判断归属）——所以"学会度"门控用**训练冲突格**准确率（模型确实拟合了 B 的冲突格 → 才会改写权重 → 才造成遗忘），而非留出冲突格。
- **C2 用等价检验（TOST）**而非"未拒绝即相等"；**C3 用预登记残差检验**而非裸相关；Spearman + 精确置换 p；所有阈值建模前钉死。

## 一个被自查纠正的门控伪影（本 arc 第 8 次）

第一版 `structure_at_fixed_overlap`（"低 overlap 类型点是否遗忘相同"）用的是**原始遗忘差**：add_offset(0.991) > mul(0.938) 被判为"存在残差结构"，会错误地否掉干净结论。但 add_offset 的 overlap 恰是 0.0、mul 是 0.043——这 0.05 的遗忘差正是 overlap 律预测的，不是残差结构。改成用 **C3 残差**（每个点对照它自己 overlap 处的曲线预测）才正确，零重训从缓存重判。

## 范围

玩具级模数运算、1 层 transformer、p=23；overlap = 共享输入上的"1 − 冲突比例"；混合表在训练格上被拟合但其逐格划分不在留出格泛化（已披露）；**不是**关于指令微调 LLM 遗忘的结论。

## 工程

Phase A（`run_similarity_v1195.py`）一次训完：每 seed 巩固 A 一次（复用），从该 init 训每条 B 臂（5 类型 + 6 混合 s）记录遗忘/accB 冲突-重叠分拆/共享 embedding 漂移/解析 overlap，加 continue-on-A 地板 + add+mul joint 上界；存 cache。Phase B（`analyze_similarity_v1195.py`）纯 CPU 重算曲线/混淆控制/verdict。复用 v1193 的任务/巩固原语。19 个测试。

一句话总结：v1195 是持续学习轴的收尾——灾难性遗忘由 B 与 A 的**输出表重叠度**单调分级决定（Spearman −1.0），与算子家族无关（同 '+' 算子的 add_offset 和乘法一样遗忘，只有逐字节相同的表才安全，用第二种手法再证 v1193 的分布漂移）；且这条曲线≈比例式覆写而非全局坍塌，所以"灾难性"只是它在 overlap=0 端点的取值；过程零重训自查纠正了一个门控阈值伪影。
