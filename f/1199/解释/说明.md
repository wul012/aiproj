# v1199 运行说明：一个小 MiniGPT 会出现"双下降"吗？——不会（玩具尺度下单下降，过参数化反而变差）

归纳机制三连（v1196 行为→v1197 QK→v1198 OV）收口后开的**新轴**。双下降（double descent，Belkin 2019；Nakkiran 2019）是著名论断：测试误差随容量是**非单调**的——先降、在**插值阈值**（模型刚好拟合带噪训练集）处**冲高**、然后在过参数化区**再次下降**。我们在一个小 MiniGPT 上**诚实地**检验它，发现这个尺度下它**不出现**——模型反而是**单下降**的过拟合，先学会可泛化的规则、再去记标签噪声，且过参数化/更久训练都给不出第二次下降。

## 基底（双下降的经典构造，改写成 token 形式——刻意**不用**取模算术，否则 grokking 动力学会把容量和"grok 时机"混淆）

固定随机**线性教师 / 半空间**：输入 = L=21 个 bit（token {0,1}）；标签 = sign(w·(2bit−1)) ∈ {0,1}，w 是固定 ±1 教师（一个可泛化的半空间）。训练集固定、其中固定比例 eta 的标签被翻成错类（标签噪声，每 seed 冻结一次）；测试集**干净**且不相交。模型在最后一个位置读出标签（只在该位置算 next-token CE）。

## 两条臂，各带 eta=0 对照（验证基底可学 + 隔离噪声效应）

- **MODEL-SIZE（模型大小）**：固定 N=256，扫 width，每个都**训到插值**（按**训练**准确率早停，不按测试，避免与 epoch-wise 混淆）。双下降会在经验插值宽度处出现一个测试误差**峰**、两侧都更低（第二次下降）。
- **EPOCH-WISE（训练时长）**：固定一个过参数化 width，记录干净测试误差随**训练步**的轨迹。epoch-wise 双下降会在插值步处冲高后**回落**（时间上的第二次下降）。

## 真实结果（L=21，N=256，5 seed，verdict=`no_double_descent_monotone`，status=pass）

```text
有效性：eta=0 对照可学（最佳干净测试误差 0.041 ≪ 0.12，且插值）；带噪模型确实插值（w_interp=16，
        所以噪声真被记住、我们确实在"插值后"区间）；5 seed；插值-tau 网格 verdict 不变 100%。

MODEL-SIZE（扫 width，test ERROR，eta=0.2）：
  w3 0.247(tr.84)  w4 0.268(tr.82)  w6 0.325(tr.98)  w8 0.311  w12 0.326  w16 0.338  w24 0.359  w32 0.389
  -> 最佳泛化在**欠参数化端**（w3/w4 拟合不了噪声、只学到部分规则）；越过插值后**随宽度单调变差**。
     峰在最宽处、其后无更低点 -> 无第二次下降。ms_second_descent=False。
  eta=0 对照本身也随宽度变差（0.041 -> 0.246）：**干净数据下过参数化也过拟合**。
EPOCH-WISE（width=32，随步）：
  插值前最好 0.314 -> 平台 0.389（上升 0.075），但 eta=0 对照也上升 0.051（宽模型连干净数据都过拟合），
  所以这个上升只**部分**归因于噪声（signal_before_noise=False）；关键是**没有回落**（recovery 0.009 < 0.05）。
  -> epoch_dd=False：单下降过拟合，**早停**才是有用的（而不是更大/更久）。
```

**结论**：玩具尺度下**双下降不出现**。两条臂都没有第二次下降；model-size 臂里**过参数化反而单调变差**（最佳在欠参数化端，与"第二次下降"正相反），epoch-wise 臂里插值后无回落。eta=0 对照确认基底可学，且部分退化是**通用的容量过拟合**（不仅是标签噪声）。

## 这是一个被诚实处理的"诚实零结果"（不是被低功率糊弄出来的）

`status=="pass"` 只认**有效**测量：对照可学（基底成立）、带噪集**确实插值**（所以在测的是插值后区间，不是"够不到阈值"）、≥4 seed、插值-tau 稳健。在这些前提下，verdict 诚实报告：无双下降。这与项目里早先的诚实零结果同源（蒸馏 v1172"无收益"、PTQ v1175"逐通道宣称是单 seed 伪影"）。

## 单 seed 探针 vs 多 seed 的诚实修正

第 4 个 CPU 探针（单 seed）里 epoch-wise 看到漂亮的"先学信号再记噪声"下凹（best 0.23）；但 5 seed 跑下来 best_pre 涨到 0.31、且干净对照也在时间上过拟合，所以"signal-before-noise"这个子断言**没有**在过参数化宽度上跨 seed 稳健成立——于是保守地落在 `no_double_descent_monotone`，**不**升格成 `..._signal_before_noise`。这正是多 seed 把单 seed 的漂亮但脆弱的现象修正掉。

## 为什么双下降在这里不出现（机制直觉）

双下降的峰需要"模型在阈值处被迫精确拟合噪声 → 扭曲 → 泛化崩"，再靠过参数化"平滑地"拟合噪声把泛化救回来。在这个小 transformer 上：(a) 它的隐式简单性偏置让它**先**学半空间规则、**抗拒**记噪声（探针里 parity 干脆永不插值；半空间要大预算才插值）；(b) 一旦容量足够记噪声，**更多**容量只是把噪声记得更"灵活"、泛化更差——没有大模型里那种过参数化带来的隐式正则/第二次下降。这就是 toy 尺度的真实面貌。

## 设计与工程

5 视角对抗设计评审（用 Workflow 跑了 6 个 agent）+ 4 个 CPU 探针定方向：探针证明 **parity 不插值带噪标签**（死基底）→ 换半空间；model-size 与 sample-wise 在此尺度无峰；epoch-wise 有上升无回落。`eff_params`（去掉 tied lm_head/嵌入的非嵌入参数）作容量轴。Phase A（`run_double_descent_v1199.py`）一次性训完两条臂、缓存每条 (arm,eta,width,seed) 的训练/测试结果与完整轨迹 + 插值标志；Phase B（`analyze_double_descent_v1199.py`）纯 CPU 重判 verdict + 插值-tau 稳健，零重训。9 个测试（合成缓存跑 decide 阶梯全分支 + 微型真实 Phase-A 烟测）。

## 范围

1 层 n_head=1 MiniGPT、半空间 L=21、N=256、5 seed。双下降的**缺席**与"过参数化在此尺度反而变差"是 **toy 尺度**上的诚实测量——**不是**断言双下降在大模型里不存在（在大模型里它有充分记录）。

一句话总结：v1199 在小 MiniGPT 上诚实检验双下降——用半空间带标签噪声任务，model-size（扫宽度，训到插值）与 epoch-wise（固定宽、扫步）两条臂都**没有第二次下降**，model-size 臂里过参数化反而单调变差、最佳泛化在欠参数化端，epoch-wise 臂插值后只升不回（早停最优）；5 视角评审 + 4 探针定了基底（parity 死→半空间）与方向，多 seed 把单 seed 的"signal-before-noise"漂亮现象诚实地修正回保守的 `no_double_descent_monotone`；status=pass 只认有效测量（对照可学 + 带噪确实插值 + 多 seed + tau 稳健）。
