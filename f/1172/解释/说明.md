# v1172 知识蒸馏：确定性任务上蒸馏不优于硬标签 SFT；同算力下，让学生多训才是赢家

## 目标与定位

继 v1170（投机解码）、v1171（保养）之后，v1172 转到一个**新轴：能力迁移**——知识蒸馏（KD）。一个训练好的 teacher（4L/64）通过逐 token 的 KL 目标把"知识"蒸到更小的 student；与从零硬标签 SFT 在**同参数、同步数**下对比。复用 build_minigpt/train_sft/evaluate_instructions/experiment_utils 以及 v1171 的 setup_single_corpus。

## 先于 GPU 的诚实判断：探针的 +0.058 是假象

CPU 探针（单种子、scratch 与 distill **共享 init**、700 步）曾显示 2L/32 上 distill τ=1 达 0.881、scratch 0.822（+0.058），且 τ=1 > τ=2，看起来"logit 匹配有用"。对抗式设计面板（3 视角+综合）**抓出这是不可证伪的假象**：teacher 在确定性任务上**天然近 one-hot**（实测 max-prob 0.989、熵 0.037 nats），所谓"dark knowledge"按定义不存在；+0.058 是单种子+共享 init 的噪声；τ=1>τ=2 受 τ² 缩放=有效步长耦合，不是干净因果。面板要求：**每个 (arm, seed) 独立 init**（使无配对的 significant 检验成立）、5 seeds、加两个拆解对照（label_smooth 把 ε 匹配到 teacher 置信度、shuffled_teacher 保留 argmax+max-prob+熵但打乱残差类别身份）、以及一个**同算力 scratch_long** 对照。

## 运行命令

```powershell
python scripts\run_distill_v1172.py --device cuda --seeds 5
```

## 关键结果（真实 GPU 运行，RTX 4060，5 seeds，独立 init）

`status=pass`（=对比有效：teacher 可学、scratch 离地板、有余量、对照齐全）、`verdict=no_distill_benefit`、teacher EM 0.858、近 one-hot（max-prob 0.989）。

| 能干学生 2L/32（5 seeds 均值±std） | EM |
|---|---|
| scratch_hard（硬标签 SFT，基线） | **0.773 ± 0.111** |
| distill τ1 hw0（预注册主对比） | **0.757 ± 0.101**（Δ **−0.016**，未> 0） |
| label_smooth*（对照） | 0.783 ± 0.078（≈scratch） |
| shuffled_teacher*（对照） | 0.761 ± 0.033（≈scratch） |
| distill τ2 hw0 | 0.803 ± 0.056 |
| **scratch_long（同算力，2485 步）** | **0.949 ± 0.033** |

- **探针的 +0.058 蒸发并反号：** 5 seeds 独立 init 下 distill τ1（0.757）反而**低于** scratch（0.773）；且 τ2（0.803）> τ1（0.757），**τ 序反转**——证实 τ 是耦合步长旋钮而非"软化有害"的因果。
- **两个对照都打平基线：** label_smooth 0.783、shuffled_teacher 0.761 都 ≈ scratch 0.773 —— 既没有 teacher 专属信号，也没有显著的通用软标签正则收益。预注册主对比 `beats_scratch=False`。
- **同算力是决定性一击：** 把"teacher 前向"省下的算力折算给 scratch（scratch_long 2485 步）→ EM **0.949**，远超所有蒸馏臂、甚至超过 teacher。**与其蒸馏，不如让学生多训。**
- **容量曲线全程为负：** Δ(distill−scratch) 在 1L/16…4L/64 全部 ≤ 0（−0.006…−0.084）——"收益在中等容量峰值"的预注册预测被**证伪**。最小 1L/16 的 scratch **训练集** EM 仅 0.05 → 这是**优化/可学习性地板**（连训练集都拟合不了），不是干净的"容量上限"，故其负 Δ 不可解读为"蒸馏有害"。

## 诚实结论

**在确定性任务上，知识蒸馏不优于硬标签 SFT。** teacher 近 one-hot 使 dark knowledge 按定义不存在；探针看似的正收益是单种子+共享 init 的假象，在 5 seeds 独立 init 下消失并反号；label_smoothing 与 shuffled_teacher 两个对照都打平基线，说明既无 teacher 专属信号也无显著通用正则收益；而**同算力下让学生多训（scratch_long 0.949）远胜任何蒸馏**。这是项目诚实测量纪律的又一次兑现：设计面板 + 多种子 + 对照 + 同算力联手识破了讨好性的单种子结果。

## 被证伪的讨好叙事

"蒸馏靠 dark knowledge"（teacher 近 one-hot，按定义无）；"τ 越大越好/越小越好"（多种子下 τ 序反转，是步长耦合）；"任何小学生都能被蒸"（容量曲线全程负；最小尺寸是优化地板）；"同步数下蒸馏赢"（那是把 teacher 前向当免费监督；同**算力**下 scratch_long 完胜）；"探针的 +0.058 是真信号"（共享 init/单种子噪声）。

## 范围

B=1、teacher-forced 金路径 logit KL、char 级确定性任务。**真正的 dark-knowledge 迁移需要有歧义/多正解的语料**——在 char 玩具尺度上不可行，明确不在范围。logit 级蒸馏（非特征/注意力蒸馏）、离线（非 on-policy）蒸馏。
