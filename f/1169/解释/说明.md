# v1169 奖励建模 + best-of-N：奖励模型能通过排序评测，却排不好策略自己的样本

## 目标与为什么不是 HH-RLHF

用户问能否上 Anthropic HH-RLHF（真实人类偏好）。诚实结论：**不适合本项目**——HH-RLHF 是多轮自然英语对话、回复动辄数百字符，而本模型是从零的 **char 级** MiniGPT（4L/128、block_size=16），物理上表示不了，且从零 char 模型在 4060 上根本生成不出连贯英语，没有能力达标的基座供偏好作用，诚实闸门必然不过；真实文本对齐是多版本工程，不是单步。于是选了**真正能在本尺度诚实测量、且正是 RLHF 核心**的方向：**奖励建模**——这是 DPO（v1166/v1168，无奖励模型）跳过的经典 RLHF 组件。

奖励模型 = MiniGPT 主干 + 标量头，在最后一个真实 token 池化，用 Bradley-Terry 成对损失 `L = -E[log σ(r(chosen) - r(rejected))]` 在 v1166 的易混偏好对上训练。三项诚实测量：(1) **同分布**排序准确率（held-out 未见输入、与训练同类的易混 reject）；(2) **离分布**泛化（reject 换成随机扰动）；(3) **best-of-N**（从弱基座采样 N 个、用 RM 重排，对比随机挑选与 oracle）。

## 设计先于 GPU（会话限额下主线程亲跑探针）

仍守"先跑代码再上 GPU"的纪律。本轮工作流面板因会话限额未能跑（同 v1168），**可行性探针我在主线程亲自跑**：探针（微型 2L/64）显示 RM 同分布 0.82、离分布**仅 0.48≈随机**（纯捷径），best-of-N **越多越差**。据此设计实验，并加入 **oracle** 基线把"RM 不会排"和"样本里压根没有正确答案"区分开。真实 4L/128 的结果比探针更微妙（见下）。

## 关键结果（真实 GPU 运行，RTX 4060，3 seeds，4L/128）

`status=pass`、`verdict=reward_model_learns_best_of_n_no_clear_gain`、`rm_verdict=reward_model_generalizes_to_off_distribution`、`best_of_n_verdict=best_of_n_reranking_neutral`。

| 测量 | 数值 |
|---|---|
| RM 同分布排序准确率（vs 易混 reject） | **0.815** ± 0.029 |
| RM 离分布排序准确率（vs 随机 reject） | **0.640** ± 0.024（>0.5 显著） |
| 基座贪心 exact-match | 0.187 |
| best-of-N（N=16）：oracle / 随机挑 / RM 重排 | **0.537** / 0.115 / **0.099** |

- **RM 可学且部分泛化：** 同分布 0.82、离分布 0.64（显著高于 0.5）——与探针（离分布 0.48≈随机的纯捷径）不同，真实尺度下 RM **有一些**通用正确性信号，只是离分布明显退化。`rm_learnable=True`、`generalizes_off_distribution=True`。
- **但 best-of-N 没有收益（neutral 偏降）：** N 增大时 **oracle**（N 个里有任一正确）一路升到 **0.54**——说明正确答案**就在样本池里**；可 **RM 重排**却停在 0.10-0.13 且随 N 下降（0.13→0.10），与**随机挑选**（~0.12）无显著差别，远低于 oracle。
- **核心洞见（oracle–rerank 鸿沟）：** RM 能通过成对排序评测（同分布 0.82、甚至合成离分布 0.64），**却完全不会从策略自己的样本里挑出最好的那个**（重排 0.10 vs oracle 0.54）。因为策略的样本是**又一个分布**——基座特有的、似是而非的错误，既非易混 reject 也非随机扰动；RM 在它上面没有校准信号。这正是真实 RLHF 的教训：**奖励模型只在其训练分布上可靠，而策略在优化中产生的样本恰恰离分布**，所以用 RM 引导的选择/优化达不到本可达到的（oracle）水平。

## 诚实边界

1. **不是人类偏好/RLHF-at-scale：** 合成确定性正确性信号 + 易混硬负样本，从零 char 级 MiniGPT。
2. **oracle 列是关键对照：** 把"RM 排得差"与"样本里没有正确答案"分开——oracle 0.54 证明答案在池里，问题在 RM。
3. **闸门只保证可测：** `status=pass` 表示 RM 同分布可学（排序 0.82 显著 > 0.5），对比有效——**不代表 RM 好**；本版诚实结论恰恰是 RM 排不好下游样本。
4. **与探针的差异已记录：** 探针（更小模型/更少数据）是纯捷径（离分布≈随机）；真实尺度部分泛化但仍无法用于 best-of-N——结论的"RM 对下游无用"更强、更微妙。

## 一句话总结

v1169 建好了 DPO 跳过的经典 RLHF 组件——Bradley-Terry 奖励模型——并诚实测得它的真实局限：它能在 held-out 成对排序上达 0.82、甚至对随机负样本泛化到 0.64，**却在 best-of-N 里排不出策略自己样本中的正确答案**（重排 0.10 ≈ 随机，而 oracle 高达 0.54）；HH-RLHF 在本 char 级尺度不可行，于是用合成可控设置把"奖励模型只在训练分布上可靠、策略生成恰离分布"这一 RLHF 核心风险量化了出来。
