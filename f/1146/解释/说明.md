# v1146 运行截图与解释

## 本版目标

v1146 承接 v1145 的 `next_step=run_decoder_anchor_probe_against_v1145_checkpoint`，用 v1145 真实训练产生的 checkpoint 做一个局部 decoder-anchor probe。

本版不做 promotion，也不宣称 unassisted holdout replay 已经成功。它只验证：在短 anchor prompt 下，v1145 checkpoint 是否已经出现稳定的局部片段信号。

## 关键修复

v1145 归档 JSON 中的训练路径仍指向当时的 `output/...` 中间目录。该目录在清理后不存在，但同一份 checkpoint 已随归档保存在 `f/1145/解释/.../real-loss-signal-training-run/`。

v1146 新增 archive-relative resolver：

- 优先读取 v1145 报告里的 `training_signal.checkpoint_path` 和 `training_signal.tokenizer_path`。
- 如果这些路径已失效，则根据 v1145 报告所在目录寻找 `real-loss-signal-training-run/checkpoint.pt` 和 `tokenizer.json`。
- 把 `used_archive_relative_fallback` 写入报告，明确说明本次使用了归档相对路径。
