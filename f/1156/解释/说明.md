# v1156 MiniGPT LoRA fine-tune

## 目标

本版给 MiniGPT 加上从零实现的 LoRA（低秩适配）微调能力：冻结基座模型权重，只训练注意力层 `c_attn` / `c_proj` 上的低秩矩阵，并用真实 GPU 训练得到的 before/after 损失作为能力证据。

本版不引入更大数据集、不做 held-out 泛化评估、不做 promotion；它只回答一个问题：在 RTX 4060 上，只训练约 3% 的参数，LoRA adapter 能不能真实地降低训练损失。

## 运行命令

```powershell
python scripts\run_lora_finetune_v1156.py --device cuda --base-steps 200 --lora-steps 400 --block-size 32
```

## 关键结果（真实 GPU 运行）

- `status=pass`
- `decision=lora_finetune_reduced_train_loss`
- `device=cuda`（NVIDIA GeForce RTX 4060 Laptop GPU, torch 2.6.0+cu124）
- `adapted_module_count=8`（4 层 × {c_attn, c_proj}）
- `trainable_parameters=24576` / `total_parameters=851584`
- `trainable_ratio_percent=2.8859`
- `before_train_loss=0.455033` → `after_train_loss=0.416174`（`train_loss_delta=-0.03886`）
- `train_loss_improved=True`

## 边界与下一步

bundled 语料 `data/sample_zh.txt` 只有 507 个字符，验证集（51 字符）被严重过拟合主导，因此 `val_loss` 不是可靠的泛化信号——这一点在报告 recommendations 里写明。本版的有效结论只建立在训练损失下降上。v1157 的方向是引入更大的真实数据集并补上 held-out 评估，让 LoRA 的泛化收益可被真实衡量。

## 结论

MiniGPT 现在具备真实可训练、可合并、可保存的 LoRA 微调能力：只用约 2.9% 的可训练参数，adapter 就在 GPU 上把训练损失从 0.455 降到 0.416，并且基座权重保持逐字节冻结（测试 `test_lora_finetune` 断言验证）。这是项目从“治理报告”转向“真实模型能力”的第一步。
