# v1126 运行截图与解释

## 本版目标
v1126 对 v1125 lookup-only receipt 做重建式 contract check。v1125 已经把 v1124 review 记录为 receipt，但 receipt 一旦成为下游消费入口，就需要证明它不是手工漂移出来的结果。本版读取 v1125 receipt 中记录的 `receipt_index_review_path`，重新调用 v1125 builder，从 v1124 review 再构建一份 receipt，并比较 original 与 rebuilt 的关键字段。

本版不训练模型，不修改 v1125 receipt，不改变 v1124 review，也不打开 production promotion。`contract_check_ready=True` 只表示 receipt 可以从源 review 重建，并且关键字段一致。

## 输入与输出
输入：

```text
f/1125/解释/receipt-v1125
```

输出：

```text
f/1126/解释/receipt-check-v1126/*.json

f/1126/解释/receipt-check-v1126/*.html
f/1126/图片/v1126-receipt-check.png
```
