# v1187 运行说明：report-check 公共脚手架去重（维护版）

这是一个维护版，不训练、不改行为、不增加模型能力。它把 grokking 审计家族（v1180 evidence-check、v1181 trajectory-phases、v1182 paired-contrast、v1184 wd-law-check）里逐字节重复的检查脚手架抽成一个公共模块。

## 重复了什么

四个审计模块各自定义了三段**完全相同**的代码：

- `_check(check_id, passed, expected, actual, detail)`：构造一行检查结果 dict。
- `failures = [check for check in checks if check["status"] != "pass"]`：收集未通过项。
- `resolve_exit_code(report, *, require_pass=False)`：`--require-pass` 的 CI 退出码。

## 抽到哪里

新增 `src/minigpt/report_check_common.py`，提供 `check_row`、`collect_failures`、`resolve_exit_code` 三个函数。四个模块改为：

```python
from minigpt.report_check_common import check_row as _check, collect_failures, resolve_exit_code
```

`resolve_exit_code` 仍在各模块 `__all__` 里（CLI 通过 `from minigpt.grok_X import resolve_exit_code` 使用，re-export 不破合同）；`_check` 用别名导入保持内部调用点不变；`failures = [...]` 改成 `failures = collect_failures(checks)`。

这是 v1159/v1163/v1167/v1171/v1174/v1176 的去重节奏——加一个 helper、只迁移当前活跃调用方、其他不动。PTQ 检查家族（v1177/v1178）有类似模式但用自己的 loader，本版**刻意不迁移**（和 v1171 留下 dual-corpus driver 同样的克制）。

## 为什么是契约保持

每个抽出的函数和原内联实现逐字节一致；行为不变。证据是四个审计模块的测试**完全没改**仍然全绿。新增 `tests/test_report_check_common.py` 额外加了 single-source 身份断言：`module.resolve_exit_code is resolve_exit_code`、`module._check is check_row`、`module.collect_failures is collect_failures`——保证迁移后是真共享，而不是又粘了一份副本。
