# v1200 运行说明：权重衰减能在标签噪声下"救回"泛化吗？——它确实**拒绝噪声**，并达到早停**平手**

v1199 的显式正则化补集。v1199 发现：在带标签噪声的半空间任务上，过参数化、weight_decay=0 的 MiniGPT 会**插值**带噪训练集、泛化很差——双下降缺席、过参数化**有害**，唯一的杠杆是早停。v1200 问下一个自然问题：**权重衰减能把丢掉的泛化救回来吗？机制是真正的"拒绝噪声"（只拟合干净信号、拒绝记住被翻转的标签），还是仅仅等价于早停？**

## 基底与设计（5 视角对抗评审硬化；复用 v1183 的 seed 内配对扫描）

复用 v1199 的带噪半空间（输入 L=21 bit；标签 = sign(w·(2bit−1))；训练标签按比例 eta 翻转，**每 seed 冻结**；干净不相交测试集），并额外缓存**翻转索引**，让 Phase B 能把训练集拆成"干净标签行"和"翻转行"。

- **剂量响应、seed 内配对**：每 seed 抽一次 teacher/数据/翻转掩码/模型初始化，从同一初始化扫所有 wd（只变 wd），≥5 seed。x 轴是**有效衰减 lr·wd**（AdamW 解耦），lr 钉死 v1199 的 3e-3。
- **固定步预算**（不训到插值）：成功拒绝噪声时模型**永不**到 train_acc=1（这正是重点），所以用按训练准确率早停会把"收敛"和"训练时长"混淆。每条臂训同样步数、记录完整干净测试误差**轨迹**（Phase B 同时读收敛值和轨迹最小值=早停最优）。
- **机制 = 翻转掩码"解离"**（不是聚合 train_acc，那对"真拒绝"和"均匀欠拟合"完全一样）：`acc_clean_subset`=非翻转行对**真**标签的准确率（真拒绝→高）；`fit_to_noise`=翻转行里预测=**翻转**目标的比例（真拒绝→低）。签名是**解离**：fit_to_noise 崩塌、而 acc_clean 保持高（且明显高于最强 wd 的欠拟合对照）。argmax 准确率对 logit 均匀缩放不变（vocab=2、tied head），所以这个指标对 logit-缩放混淆免疫；logit_norm 作辅助诊断缓存。
- **归因 = 差中差（DiD）**：v1199 证明宽模型连干净数据也过拟合，所以 wd 扫描在 eta=0 与 eta=0.2 都跑，只有**多出来的**带噪臂改善（DiD）才算"拒绝噪声"而非通用容量正则。
- **要打败强早停基线**：救回必须越过 wd=0 的**轨迹最小值**，否则就退化成已知的"wd≈早停"。

## 真实结果（5 seed，verdict=`wd_equals_early_stopping`，status=pass）

```text
有效性：基底成立（eta=0 最佳干净测试误差 0.04 ≪ 0.12，且插值）；wd=0 参照确实记住噪声（train_acc 1.0，
        所以有东西可救）；收敛（按对照点判定）；5 seed；丢掉极端 wd 点结论不变（robust）。

剂量响应（eta=0.2，收敛 test_err）：
  wd=0   0.368 (train 1.00, acc_clean 1.00, fit_noise 1.00)  <- 记住一切（含噪声）
  wd=0.3 0.331 (0.92 / 0.95 / 0.79)
  wd=1.0 0.308 (0.84 / 0.90 / 0.62)
  wd=3.0 0.176 (0.78 / 0.90 / 0.32)  <- 内部最优（拒绝噪声）
  wd=10  0.345 (0.60 / 0.66 / 0.37)  <- 欠拟合对照（连干净信号也丢）
救回判据：best wd=3.0 收敛 0.176  vs  wd=0 早停最优 0.255（gap 0.079）-> rescue=False（5 seed 下不显著）
机制：acc_clean 0.90 而 fit_noise 0.32（解离=True）；DiD vs eta=0 = 0.239（did_ok）；内部最优=True
```

**结论（诚实）**：weight decay 把**收敛**泛化在标签噪声下大幅改善（0.368→0.176，wd=3.0 内部最优），机制是**真正的选择性拒绝噪声**——fit_to_noise 从 1.0 崩到 0.32，而干净行准确率仍 0.90（最强 wd=10 处两者一起崩=欠拟合对照，证明指标能检出欠拟合）；DiD=0.239 说明这是**噪声特异**的、不是通用正则。**但**它在 5 seed 下**没有显著越过** wd=0 的早停最优（0.255）→ verdict 是诚实的 `wd_equals_early_stopping`：wd 达到早停**平手**（一个**不需要早停 oracle** 的替代——你可以一路训到收敛而拿到早停级泛化），并非超越早停的新机制。

## 多 seed 把单 seed 探针诚实地"打回原形"

单 seed 探针里 wd=3.0 给出近乎完美的 test_err 0.047；5 seed 平均却是 0.176（方差大）。那个 0.047 是**幸运 seed**。多 seed 让"救回"在统计上不显著越过早停——和 v1199（单 seed signal-before-noise 没扛过多 seed）一脉相承。诚实测量胜过讨喜结果。

## 在真实 Phase-A 数据上自抓自修的 3 个 decide() 阈值伪影（本弧第 9–11 个）

1. **substrate_unsound 误触发**：基底成立判据原本查 eta=0 在**最宽**宽度的**收敛**干净测试误差≤0.12——但宽模型连干净数据都过拟合（v1199）。基底是否可学应看**最佳可达**（早停）干净泛化（w 任意一个能插值且泛化<0.12 即成立——最佳 0.04）。改用 early-stop 最佳。
2. **budget_unconverged 误触发**：收敛判据原本用末段**极差**，但小模型测试误差天然抖动 ±0.05。改用相邻十分位**均值之差**（drift，不是抖动），且只在 verdict 直接比较的 NOISE 臂点（wd=0 + best_wd，皆收敛 trend≤0.04）上设闸；震荡的 eta=0 对照与 wd=10 欠拟合端只用于刻画形状。
3. **dissociation 卡在 knife-edge**：原本用 acc_clean≥0.90 绝对阈，实测 0.899 < 0.90 差 0.001 误判 False。改用**控制相对**判据（拒绝噪声 + 干净-噪声选择性差≥0.30 + acc_clean 明显高于最强 wd 欠拟合对照），稳健地给出 True。

## 范围

1 层 n_head=1 MiniGPT、半空间 L=21、N=256、width 32、5 seed、wd∈{0,0.1,0.3,1,3,10}、lr=3e-3。诚实的 **toy 尺度**测量：wd 通过逼出**最小范数干净解**拒绝噪声，是早停的 oracle-free 替代。**不是**大模型断言。5 视角 Workflow 评审 + 1 CPU 探针。Phase A 一次性训完缓存每条 (eta,wd,seed) 轨迹 + 翻转掩码指标；Phase B 纯 CPU 重判，零重训。10 个测试。

一句话总结：v1200 在 v1199 基底上加权重衰减——剂量响应显示 wd 大幅改善带噪收敛泛化（0.368→0.176，wd=3.0 内部最优）且机制是**真正的选择性拒绝噪声**（fit_to_noise 1.0→0.32 而 acc_clean 守 0.90，DiD=0.239 噪声特异，最强 wd 欠拟合对照证明指标有效），但 5 seed 下没显著越过 wd=0 早停最优 → 诚实 verdict `wd_equals_early_stopping`（达到早停平手、是不需 oracle 的替代）；多 seed 把单 seed 探针的 0.047 打回 0.176，并在真实数据上自抓自修了 3 个 decide 阈值伪影（基底用最佳-非最宽、收敛用 drift-非抖动、解离用控制相对-非绝对阈）。
