# v1193 运行说明：持续学习 / 灾难性遗忘（全新一轴，clean break）

grokking 线在 v1191 收尾；v1192（校准）被我自己诚实判定为"贴着 v1173 的相邻轴"，不是干净的新轴。用户明确选了**干净断裂**的方向：持续学习 / 灾难性遗忘。本版研究的是一个**全新现象**——顺序多任务的相互干扰。

两个**可区分**的模数运算共享同一个 1 层 transformer，用 op token 区分（所以联合训练能同时学会两者）：

```text
任务 A: [a, PLUS,  b, EQ, c],  c = (a+b) mod p
任务 B: [a, TIMES, b, EQ, c],  c = (a*b) mod p
```

先把 A 训到稳定平台，再训 B，看 A 的留出准确率塌多少。设计评审（5 视角）+ CPU 探针把它的坑都堵上了。

## 关键设计加固（评审 must-fix）

- **全局留出 pair 掩码**：同一组 (a,b) 操作数对从 A-train、B-train、replay buffer、joint-train 里**统一**剔除。否则 A 的"测试"操作数会经由 B-train 泄漏进来（数字 embedding 行是共享/绑定的），遗忘数就不可解释。`verify_no_leak` 程序化校验，作为门控。
- **巩固（consolidation）**：A 训到**持续平台**（acc 连续 W 次评估 ≥ 阈值），不是首次越线就停——否则"遗忘"只是"A 根本没学牢"。
- **固定 B 预算**：所有 seed/arm 在固定 B 步数处读留存。
- **诚实控制**：continue-on-A（无分布漂移 → 遗忘≈0，地板）、random-label-B（同输入打乱标签 → B 的结构重要吗，还是只是梯度漂移）、replay buffer 大小剂量曲线、wrong-replay 特异性（replay B 必须保不住 A）、joint 上界、savings 重学探针（擦除 vs 掩蔽）。
- **"灾难性"按幅度门控**：只有掉到接近随机（≈1/p）才叫 catastrophic。

## 真实结果（p=23，add→mul，3 seed）

```text
A 巩固平台              0.984
naive：训完 B 后 A      0.041   (≈chance 0.043)  → 遗忘 0.943±0.000  CATASTROPHIC
continue-on-A（地板）   0.990   → 遗忘 -0.006   (无漂移就不忘)
random-label-B（null）  0.063   → 遗忘 0.937    (≈real-B！)
wrong-replay（replay B）0.069   → 遗忘 0.931    (replay B 保不住 A)
joint 上界              A 0.862 / B 0.837        (都远超 prior 0.090 = 可联合学习)
replay A-train 剂量      buf0=0.943 buf8=0.362 buf32=0.063 buf128=0.038
```

`verdict = catastrophic_forgetting_mitigated_by_replay`（status=pass）。status 只证明**有效测量**：A 巩固到平台、B 真学会了（0.805 ≫ majority-prior 0.090，不是靠"见 0 就猜 0"的捷径）、可联合学习、无操作数泄漏。

- **灾难性遗忘真实存在**：训 B 把 A 从 0.984 砸到 0.041（≈随机），且轨迹显示是**瞬时**塌缩（B 阶段第一个评估窗口内就掉到底）。
- **replay 缓解**：回放 A-train 例子按剂量单调缓解遗忘（buf128 几乎全恢复，遗忘 0.038）。**特异**：回放 B（不是 A）保不住 A（仍遗忘 0.931）。replay 靠的是**重新见到 A 数据**——是数据，不是魔法（v1173 纪律）。

## 一个诚实的机制发现（random-label-B null）

real-B 遗忘 0.943，**random-label-B（同样的 B 输入但标签打乱）遗忘 0.937，几乎一样**。所以这里的遗忘是**分布漂移驱动**的——在共享权重上训练任何新输入分布（哪怕标签是随机的）都会覆盖 A——而**不是** B 这个任务结构特有的干扰。对照 continue-on-A 遗忘≈0（不换分布就不忘），整个图景清楚：换分布 = 灾难性覆盖，与 B 的结构无关。这是诚实标注，不藏。

savings 探针：重学 A 在所探预算内没快速恢复（recovered_k=None）→ 更接近**擦除**而非可掩蔽的复用。

## 范围

玩具级模数运算、1 层 transformer；这**不**是关于指令微调 LLM 遗忘的论断。

## 工程：训练/分析分离（reuse-cached）

Phase A（`run_continual_v1193.py`）只训练一次，把所有 arm 的准确率/轨迹/savings 曲线/replay 扫描存进 `output/continual-v1193/cache.pt`；Phase B（`analyze_continual_v1193.py`）纯 CPU、不训练，从缓存重算遗忘/replay/savings 聚合并跑 decide()。本版我两次从缓存**零重训**地修了 decide 逻辑：(1) g_joint 原来错误地用 add 的单任务平台 0.98 当联合门槛，而 mul mod p 天花板本就更低（改成"两者都远超 majority-prior+margin"）；(2) 三个 seed 遗忘 std=0，导致 wrong-replay 比 naive 低 0.012 被 `significant()` 误判为"有缓解"——加了 0.05 的最小有效幅度门槛（必须同时超过 margin 和合并 std）。

## 验证命令

```powershell
python -m pytest tests\test_continual_v1193.py -q -o cache_dir=runs\pytest-cache-v1193
python scripts\run_continual_v1193.py --device auto       # Phase A：训练一次，存缓存
python scripts\analyze_continual_v1193.py                 # Phase B：纯 CPU 分析
```

一句话总结：v1193 开了持续学习这条干净的新轴——在共享 1 层 transformer 上先学 add 再学 mul（mod 23），A 从 0.984 灾难性塌到 0.041（≈随机），回放 A 的样本按剂量缓解、回放 B 则无效（特异）；诚实地说这遗忘是**分布漂移**驱动（random-label-B 同样把 A 砸掉），而非 B 任务结构特有，且更接近擦除而非掩蔽。
