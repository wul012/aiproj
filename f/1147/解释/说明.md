# v1147 运行截图与解释

本版把 v1146 的 decoder-anchor fragment probe 与一组不带 `fixed`、`loss` 前缀锚点的 unassisted holdout-style prompts 做了真实对照。它复用 v1145 真实训练 checkpoint，不新增训练，也不声明模型晋升。

## 运行命令

```powershell
python scripts/run_decoder_anchor_holdout_comparison_v1147.py --out-dir output/decoder-anchor-holdout-comparison-v1147 --require-pass --force
```

## 关键结果

- `status=pass`
- `decision=decoder_anchor_signal_exceeds_unassisted_holdout_replay`
- `anchor_fragment_hit_count=5`
- `anchor_loss_hit_count=4`
- `unassisted_any_term_hit_count=3`
- `unassisted_fixed_hit_count=0`
- `unassisted_loss_hit_count=3`
- `unassisted_full_pair_count=0`
- `anchor_over_unassisted_hit_delta=2`
- `promotion_ready=False`
- `unassisted_success_claim=False`

## 解释

v1146 证明 v1145 checkpoint 在很短的 `fixed `、`lo`、`los`、`fi` 这类前缀锚点下已经可以产生局部 `loss` 或 `fixed` 片段。v1147 进一步问：如果不提供这些前缀锚点，只给 `answer:`、`completion:`、`finish:`、`state compact signal\nanswer:` 这类提示，模型是否能主动恢复完整 `fixed/loss` pair。

真实结果显示，unassisted prompts 有 3 个 case 会出现 `loss` 片段，但没有任何 case 同时命中 `fixed` 和 `loss`。这说明 v1145 checkpoint 的能力更接近局部 suffix/fragment 关联，而不是稳定的无锚点问答能力。本版因此把 `model_quality_claim` 写成 `anchor_assisted_signal_exceeds_unassisted_holdout_replay`，并继续保持 `promotion_ready=False`。
