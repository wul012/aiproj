# v1173 不确定性下的蒸馏：把"dark knowledge"做成真的——v1172 的镜像

## 目标与定位

v1172 证明：在**确定性**任务上 teacher 近 one-hot，dark knowledge 按定义不存在，蒸馏不优于硬标签 SFT。v1173 做它的**镜像**：构造一个**随机**任务，每个 context 映射到一个**已知**的多模分布 `P_true`（Dirichlet 抽取、熵可控），于是 teacher 的输出分布**真的软**，而单个硬样本无法刻画整个分布——这正是 Hinton 所说的 dark knowledge 该出现的场景。因为我们**构造**了 `P_true`，度量是精确的：在唯一的随机输出位置上测 `KL(P_true‖P_student)`。

## 设计先于 GPU + 三个一致的阻断点

CPU 探针先确认可行（teacher 软且忠实，蒸馏 KL≈0.028 vs 硬标签 4.89）。对抗式面板（3 视角+综合）抓出三个一致的阻断点：(1) **KL 是越低越好**，而项目的 `significant` 是越高越好——所有比较必须走 `beats_lower` 包装，否则会在蒸馏更差时误判更好；(2) **不能用 exact-match 评测**随机任务，需新的 per-context 类别读出 `student_P`；(3) **去掉 EOS**，让 completion=`[x]`，于是 `_build_xy` 只在那个随机位置打标签（否则确定性的 `\n` 位置把 KL 信号稀释一半）。并要求**每个 (arm,seed) 独立 init**、**每 seed 重抽学生样本**（探针固定抽样种子=测的是 init 方差而非抽样方差）。

## 隔离 dark knowledge 的关键控制（三分解）

- **数据效率** = KL(scratch_hard, 少量真样本) − KL(teacher_argmax_hard)：teacher 见了 400 样本，但学生只取它的 argmax（众数）。
- **DARK KNOWLEDGE（头条）** = KL(teacher_argmax_hard) − KL(teacher_soft)：**同一 teacher、同样 400 样本预算**，众数 vs 完整软分布之差——纯粹是"软形状超出众数"的信息。
- 控制：`label_smooth`（ε 匹配平均熵的通用平滑，无法编码具体 P）、`shuffled_teacher`（保 argmax+熵、打乱残差类别身份，应**变差**——v1172 里它是惰性的）、`oracle_true_P`（上界）、`scratch_many` 样本数扫描（诚实证伪"魔法"：用 teacher 自己的 400 样本，硬 CE 也能恢复 P）。

## 运行命令

```powershell
python scripts\run_distill_v1173.py --device cuda --seeds 5
```

## 关键结果（真实 GPU 运行，RTX 4060，5 seeds，独立 init，K=32）

`status=pass`、`verdict=dark_knowledge_is_data_efficiency_under_uncertainty`、六个 flag 全 True。teacher 4L/64 软且忠实：KL(true‖teacher)=0.041、熵 1.16≈真 1.13、泄漏 0.001。

| 臂（KL(P_true‖student)，越低越好） | KL | 熵偏差(vs 真) |
|---|---|---|
| oracle_true_P（上界） | **0.002** | +0.00（完美校准） |
| **teacher_soft（蒸馏）** | **0.050** ≈ teacher 天花板 | +0.05（匹配真熵） |
| label_smooth*（通用平滑） | 1.298 | −0.78 |
| shuffled_teacher*（毁身份） | **2.416（变差）** | −0.68 |
| teacher_argmax_hard（仅众数） | 3.516 | −1.12（塌成 one-hot） |
| scratch_hard（1 真样本） | **4.476** | −1.12（塌成 one-hot） |

- **三分解：** 数据效率项 = 4.48−3.52 = **0.96**；**DARK KNOWLEDGE 项 = 3.52−0.05 = 3.47**（头条，巨大）。
- **随熵增长：** dark-knowledge Δ(argmax−soft) 对熵回归 **斜率 3.31、下界 3.05 > 0**；而数据效率混淆项的斜率仅 **0.30**——增长确实来自 dark knowledge，不是数据效率。
- **不是魔法（诚实上界）：** scratch_many 扫描 KL：n=1→4.24、3→1.24、10→0.31、30→0.071、100→0.059、**400→0.047 ≈ teacher_soft 0.050**。用 teacher 自己的 400 样本/context，普通硬 CE 也恢复了 P——所以好处是**不确定性下的数据效率**（软蒸馏让 n=1 买到 n=400 的硬目标），teacher 的 400 样本预算是被摊销的，不是免费的。
- **校准故事：** scratch_hard/argmax 的熵偏差 −1.12（塌成 one-hot、过度自信），teacher_soft/oracle 偏差 ≈0（匹配真熵）——dark knowledge 在校准上就是"复现真分布的熵"。

## v1172 ↔ v1173 镜像

同一 train_student/kl_term 机器，结果相反，因为 teacher 的熵翻转：确定性（H≈0.05、one-hot）→ dark knowledge 不存在、蒸馏=硬标签、shuffled 惰性、同算力多训更优；随机（H≈1.16、软）→ dark knowledge 真实（项 3.47）、蒸馏(0.05)≪硬(4.48)、shuffled 变差、随熵增长、用 400 硬样本可恢复。

## 一句话总结

dark knowledge = 单个硬样本无法传达的**分布信息**：仅当 teacher 软时存在，随上下文熵增长，且能用足够多的硬数据恢复（不确定性下的数据效率，而非魔法）。这与 v1172 合成一个完整诚实的结论——**蒸馏何时有用、为什么有用**。

## 范围

单位置下一字符类别分布；Dirichlet 合成是真实文本歧义的**可控替身**（真实多正解 NL 语料明确推迟）。logit 级（非特征/注意力）、离线（非 on-policy）。KL 越低越好 → 所有比较经 `beats_lower`。
