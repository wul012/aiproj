# v1185 运行说明：canonical grokking checkpoint（产品化）

前面 v1179 复现了 grokking、v1183 扫出权重衰减的内部最优、v1180-v1184 对这两个实验做了离线审计。v1185 不再做研究式 sweep，而是把结果**产品化**：冻结默认 grokking 配方，训练并保存一个标准 checkpoint，连同 config / metrics / curve 一起存档，再加一个 demo 证明“先记忆、后泛化”。

## 冻结的配方

```text
任务：a + b = c (mod 97)，token 序列 [a,+,b,=,c]，只在答案 token 上算损失
模型：1 层 MiniGPT，n_embd=128，4 头，dropout=0
训练：全 batch AdamW，lr=1e-3，weight_decay=1.0，train_frac=0.2，seed=1337
早停：确认稳定 grok（val≥0.95）后停
```

`weight_decay=1.0` 不是随便定的——v1183 的剂量-反应实验证明它是内部最优（grok 最快）。`train_frac=0.2` 是 v1179 校准出的 grokking 区间。单一固定 seed 让出厂 checkpoint 唯一且可复现。


## checkpoint 是自包含的

`grok_checkpoint_v1185.pt`（约 835KB）里存了 `state_dict` + `meta`（p、train_frac、seed、vocab_size、GPTConfig 字段、t_mem/t_gen、final acc）。`load_checkpoint(path)` 只靠 meta 就能重建架构并加载权重，不需要额外传 config。训练后脚本会**重新加载**保存的 checkpoint，验证它和原模型给出逐字节相同的 logits（`roundtrip_logits_identical=True`），再用**重新加载的模型**在 held-out 对上跑 demo——所以证据链是“保存的文件确实可加载、可泛化”，不是内存里的临时模型。

## 输入 / 输出

- 输出：
