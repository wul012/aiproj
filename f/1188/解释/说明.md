# v1188 运行说明：grokking 机理可解释性（新轴）

这是项目的**新轴：机理可解释性**。前面 v1179 证明模型 grok（泛化），v1185 把它存成 checkpoint，v1186 用它。v1188 回答**它到底怎么算 `a + b (mod 97)`**。教科书结论（Nanda 2023）是：grok 后的模数加法网络把每个数字表示成**稀疏的傅里叶特征**（少数几个关键频率），再用三角恒等式相加。

可证伪的信号在 token embedding 里：把数字 token 的 embedding 看成数字 `0..96` 的函数，对其做 FFT，grok 模型应该把能量集中在少数频率上；随机初始化的模型则是弥散的。最有科学意义的是**配对对照**——一个 memorize-但-没-grok 的模型（weight_decay=0，同 init/split）不应该长出这种结构，从而把傅里叶机理和**泛化**绑定，而不仅仅是“训练过”。这是 v1179/v1183 行为差异的**权重层面机制解释**。几乎零成本（结构直接从权重读出）。

## 配置

- 三个臂（每 seed 配对）：grokked（wd=1，早停于 grok）、memorized（wd=0，固定 5000 步，记住但不 grok）、random（未训练）。3 seeds。
- 指标：对数字 embedding 做 rFFT，去 DC、归一化，算 top-5 频率功率占比、谱熵、覆盖 90% 功率所需频率数。

## 真实 GPU 结果（RTX 4060，3 seeds）

## 诚实解读（方向确认，幅度温和）

- **方向确认且显著**：grokked > memorized > random，顺序完全符合假设，误差棒很小（±0.004）。grok 模型只需 34 个频率覆盖 90% 功率，memorized/random 需要 42/43。而且只有 grokked（泛化的那个）显著高于 memorized——所以这个结构**跟着泛化走**，不只是“训练过”。这是 grokking 的权重层面机制。
- **但幅度温和，不是教科书式的极稀疏**：top-5 频率只占约 31% 功率，主频还随 seed 变化（43/35/32）。这**不是** Nanda 在 attention-only 模型里看到的“四五个关键频率解释一切”的极稀疏基。我们的模型有 LayerNorm + GELU MLP（不是 attention-only），所以傅里叶结构是一个**显著的倾向**，而非完整干净的算法。

诚实纪律：verdict 抓住了“显著、方向对、跟泛化走”的真实结论；但说明里明确指出幅度是 partial 的，不把它吹成干净的极稀疏傅里叶算法。
