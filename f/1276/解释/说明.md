# v1276 superposition 实验预注册与证据

## 1. 目标、来源与边界

本版在自有 toy substrate 上复现 Toy Models of Superposition 的基本现象：20 个相互独立的
非负 feature 被压进 5 个 hidden dimensions，检验 feature 越稀疏时，模型是否会用非正交方向
表示超过 5 个 feature，并且这种 packing 是否优于解析可得的最佳 dedicated 解。

参考定义来自 Elhage et al. 2022 的 ReLU-output toy model：
`x' = ReLU(W^T W x + b)`，feature 以概率 `S` 为零，否则服从 `Uniform[0,1]`，loss 是
importance-weighted squared error。来源：`https://transformer-circuits.pub/2022/toy_model/`。

范围固定为 `toy autoencoder, own substrate`。不声称 MiniGPT 或 LLM 内部已经存在相同结构，
不做完整 phase diagram，也不触碰 v1275 或更早版本的 artifact、cache 与 verdict。本版 CPU-only；
GPU 使用没有正当理由，因此被禁止。

## 3. 新 family 设计说明（先于实现，≤10 行）

抽象名是 `Superposition experiment`。数据面只有 frozen config、cell key、`W/b`、loss trajectory
和 final evaluation loss；行为面只有 cell training、解析 baseline、cache-only metrics、纯函数
`decide()` 与 renderer。`run_` 脚本只生成/续写 Phase-A cache，`analyze_` 脚本只读 cache，
不允许导入训练入口后隐式补跑。一个核心模块承载 substrate，不新建 package 或通用 harness。

## 4. 对抗性 premortem

### 4.1 “表示数变多”但并非最优

低 norm threshold 会把随机小权重也算作 feature，造成视觉上的 packing。为此 G2 对
`τ={0.3,0.5,0.7}` 全部成立才通过；G4 还要求 sparse 模型以预注册 margin 击败解析 dedicated
optimum。出现 R>5 但 loss 不优时必须进入 `superposition_not_optimal`，不能包装成 finding。

### 4.2 解析 baseline 写错或量纲漂移

本版 loss 定义为 batch 平均、feature 维求和：
`E[sum_i I_i (x_i-x'_i)^2]`。dedicated 解精确重建 importance 最大的 m 个 feature，其余输出 0。
预注册 commit `2cc520f0` 最初把“drop”解释成输出 0，写成 `E[x²]`。真实 probe 证明该式
不是 **best** dedicated baseline：模型有 per-feature bias，未表示 feature 可以输出最优常数
`E[x]=(1-S)/2`。因此按失败条件披露并从未改动的 probe cache 重算，修正式使用方差：

```text
q = 1-S
L_dedicated(S) = (q/3 - q^2/4) * sum(importance of dropped n-m features)
```

importance arm 在 S=0/0.99 的 baseline 分别为 `0.0464644659/0.0018446393`；uniform arm
分别为 `1.25/0.049625`。公式用 tiny Cartesian quadrature（包含 dropped-feature 最优 bias）
独立复算；测试缺失即 G4 无效。2% epsilon、G1-G5 与 verdict 顺序均未改变。

### 4.3 优化失败被误报为科学阴性

每 cell 固定训练 6000 steps，Adam `lr=0.003`，large batch=1024；每 100 steps 在固定 8192
样本 trajectory set 上记录 loss，最后在独立固定 262144 样本上估计 final loss。最后 10% 的
trajectory loss range 除以解析 baseline 必须 `<=0.02`。每 cell 至少 4/5 seeds 收敛；不收敛
seed 会保留在 cache 并标为 excluded，绝不删除原始记录。少于 4 个即 G1 失败并 route review。

### 4.4 finite-sample 与事后 margin

G4 使用解析 expectation，不把同一批次的经验 dedicated loss 当理论。final eval 取 262144 样本
以压低 S=0.99 稀有激活的 Monte Carlo 误差。`optimality_epsilon=0.02` 预注册为相对 baseline
margin：sparse 必须 `median_loss <= baseline*(1-0.02)`；dense 必须不满足这一不等式。运行后
不得改成绝对 epsilon、换 eval seed 或增加只对有利 arm 生效的容差。

### 4.5 importance control 与初始化混淆

同一 `(S,seed)` 的 IMPORTANCE/UNIFORM arm 使用同一输入 RNG 和同一模型初始化，唯一差异是
loss importance。IMPORTANCE 为 `0.7^i`，UNIFORM 全 1。G5 只在 IMPORTANCE dense、主阈值
`τ=0.5` 检查 represented set 是否恰为 `{0,1,2,3,4}`，至少 4/5 seeds；uniform 没有可恢复的
顺序，因此明确豁免。G5 单独报告，不反向决定 G1-G4 的 superposition headline。

## 5. 冻结配置和判据

### 5.1 Phase A 配置

- `n=20`, `m=5`, sparsity=`[0.0,0.7,0.9,0.97,0.99]`。
- arms=`importance,uniform`；seeds=`[0,1,2,3,4]`。
- `steps=6000`, `batch=1024`, `lr=0.003`, `record_every=100`。
- trajectory eval=`8192`，final eval=`262144`；所有 tensor 和训练均在 CPU。
- 模型初始化按 seed 配对；data generator 按 `(S,seed)` 配对，arm 不进入 RNG seed。

### 5.2 CPU probe 停止门

先运行四格：两 arm × `S={0.0,0.99}` × seed 0。每个 sparse probe 在最宽松的 `τ=0.3`
都必须 `R>m` 才允许 full grid；任一 arm `R(0.3)<=5` 都定义为“没有 packing at all”，立即停止并
re-panel。dense probe 不作为停止门，但其 R、loss、plateau 必须完整记录。full 模式必须显式
`--resume` probe cache，并复用四格，不能重训或绕过停止判定。

### 5.3 G1-G5 的机械定义

- **G1**：cache 50 格齐全；每格原始 seed 数=5；每格 converged 数≥4；只有 converged seed
  进入中位数与 hypothesis gate。
- **G2**：对两 arm 和每个 τ，dense `R<=5` 与 sparse `R>5` 都分别在至少 4/5 kept seeds 成立。
  所有 τ/arm 都成立才 pass；所有 sparse τ/arm 都在至少 4/5 seeds 保持 `R<=5` 才是 dedicated null；
  其余是 mixed-τ，route review。
- **G3**：每 arm 先按 cell 汇聚 kept seeds 的 `R(τ=0.5)` median，再对五个 S 的 median 做
  Spearman。要求序列 non-decreasing、`rho>=0.9`，并对五个 median 的 120 个排列计算 one-sided
  exact p，要求 `p<=0.05`。这就是 brief 中 pooled seeds 的固定解释，运行后不得换成挑 seed。
- **G4**：两 arm 都满足 sparse 相对 baseline 至少改善 2%，且 dense 不改善 2%。
- **G5**：importance dense 在 `τ=0.5` 的 represented set 等于 top-m，至少 4/5 kept seeds。

### 5.4 verdict 顺序

1. G1 失败、G3 失败或 mixed-τ → `review`；
2. dedicated null → `no_superposition_dedicated_only`；
3. G2 pass 且 G4 fail → `superposition_not_optimal`；
4. G1-G4 pass → `superposition_emerges_with_sparsity`。

`status=pass` 只表示有效测量或诚实 null；`review` 表示网格、收敛、单调性或 τ 一致性不支持
headline。G5 作为 `importance_ordered` 独立字段，不改写上述四分支。

importance S=0.99: converged=True loss=0.00038292 R={0.3:13,0.5:11,0.7:11}

两个 sparse probe 在最宽松 τ=0.3 都有 R>5，packing 停止门通过。与此同时 dense loss 几乎
精确等于 `Var(Uniform[0,1]) × dropped importance`（uniform 为 `15/12=1.25`），暴露了上述
baseline 公式错误。完整网格在修复前没有启动；probe cache 的 W、b、trajectory、final loss
保持不变，修复只影响 Phase-B 解析 baseline。修复将先单独提交，再 resume 同一 probe cache。
