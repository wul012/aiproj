# v64 registry release readiness tracking 运行截图说明

本版目标是把 v63 的 `release_readiness_comparison.json` 纳入 run registry：当某个 run 目录下存在 `release-readiness-comparison/release_readiness_comparison.json` 时，registry 会抽取 improved、regressed、panel-changed、baseline status、ready/blocked 数量和 delta 明细，并在 JSON、CSV、HTML 中展示。

本版不重新定义 release readiness，也不重新比较多个 dashboard；这些仍由 v62 和 v63 负责。v64 只做“索引和汇总”，让发布质量趋势能和 loss、rubric、pair delta、generation quality 一起出现在 registry 视图里。

## 截图索引

- `01-unit-tests.png`：展示 registry release readiness tracking 单测、`compileall` 和全量 `unittest discover` 的结果，证明新增 registry 字段没有破坏既有 registry、release、server 和 benchmark 链路。
- `02-registry-release-readiness-smoke.png`：用两个临时 run 目录分别模拟 improved 和 regressed comparison，运行 `scripts/register_runs.py`，验证 registry JSON/CSV/HTML 中出现 release readiness comparison counts、delta summary 和 Release Readiness Deltas。
- `03-registry-release-readiness-structure-check.png`：检查 `registry.py`、`register_runs.py`、`test_registry.py` 和 README 中的关键字段、artifact 路径、HTML 列和 CLI 输出。
- `04-playwright-registry-release-readiness-deltas.png`：使用已安装 Google Chrome 通过 Playwright 打开生成的 `registry.html`，验证 Release Readiness Deltas 面板能在浏览器中渲染。
- `05-docs-check.png`：检查 README、`b/64`、项目成熟度阶段 README 和 `79-v64-registry-release-readiness-tracking.md`，证明文档、归档和讲解已对齐。

## 关键运行命令

```powershell
python -m unittest tests.test_registry
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts\register_runs.py <run-a> <run-b> --name Improved --name Regressed --out-dir <registry-dir>
powershell -ExecutionPolicy Bypass -File scripts\playwright_chrome_smoke.ps1 -UrlOrPath <registry-dir>\registry.html -Out b\64\图片\04-playwright-registry-release-readiness-deltas.png
```

## 本版 tag 含义

`v64.0.0` 标记 MiniGPT 已能在 registry 层追踪 release readiness comparison：不只生成单个比较报告，还能把多个 run 的发布质量变化纳入实验索引和浏览器总览。

一句话总结：v64 把发布质量趋势从独立 comparison report 推进到 run registry 的长期索引视图。
