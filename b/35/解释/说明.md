# v35.0.0 标记说明

v35 的主题是 `benchmark eval suite metadata`。本版把原来的 fixed prompt eval suite 升级为带 suite metadata、任务类型、难度、预期行为、tags 和 HTML 报告的 benchmark prompt suite。

## 本版新增/修改

- `data/eval_prompts.json`
  - 新增 `suite_name`、`suite_version`、`description`、`language`
  - 默认中文 benchmark 扩展为 continuation、QA、summary、structured、factual-consistency 五类任务
  - 每个 case 增加 `task_type`、`difficulty`、`expected_behavior` 和 `tags`

- `src/minigpt/eval_suite.py`
  - 新增 `PromptSuite`
  - `PromptCase`/`PromptResult` 携带 benchmark metadata
  - 新增 `load_prompt_suite`
  - report 新增 `benchmark`、`task_type_counts`、`difficulty_counts`
  - 输出新增 `eval_suite.html`

- `scripts/eval_suite.py`
  - CLI 读取 suite metadata
  - 输出新增 `suite_name`、`suite_version`、`task_types`

- `src/minigpt/dashboard.py`、`src/minigpt/playground.py`、`src/minigpt/manifest.py`、`src/minigpt/registry.py`
  - 接入 `eval_suite.html`
  - dashboard eval suite 区域展示 task/difficulty 汇总

- `代码讲解记录_评估基准阶段/50-v35-benchmark-eval-suite.md`
  - 说明目标、边界、字段语义、输出产物、CLI 行为、下游链路和测试证据

## 标记含义

`v35.0.0` 表示 MiniGPT 从“能跑固定 prompts”推进到“能用固定 benchmark prompt suite 记录任务类型、难度、预期行为，并生成可浏览 HTML 报告”。这为后续数据版本和模型横向对比打基础。
