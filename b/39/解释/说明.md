# v39.0.0 标记说明

v39 的主题是 `checkpoint selector`。本版给本地 playground server 增加 checkpoint 发现、选择、model-info 查询和生成路由能力，让浏览器端可以在默认 checkpoint 与候选 checkpoint 之间切换，并把选择结果写入请求日志。

## 本版新增/修改

- `src/minigpt/server.py`
  - 新增 `CheckpointOption`
  - 新增 `/api/checkpoints`
  - `/api/health` 返回 checkpoint endpoint、候选数量和默认 checkpoint id
  - `/api/model-info` 支持 `?checkpoint=<id>`
  - `/api/generate` 支持 JSON 字段 `checkpoint`
  - 请求日志新增 `checkpoint_id` 和 `requested_checkpoint`

- `src/minigpt/playground.py`
  - 新增 checkpoint 下拉框
  - 页面加载时请求 `/api/checkpoints`
  - live generation 请求会发送 `payload.checkpoint`
  - 命令片段会随当前 checkpoint 选择更新

- `scripts/serve_playground.py`
  - 新增 `--checkpoint-candidate`
  - 启动时打印 `/api/checkpoints` 和 `checkpoint_count`

- `tests/test_server.py`
  - 覆盖 checkpoint 发现、API 列表、model-info 选择、generate 路由和请求日志

- `tests/test_playground.py`
  - 覆盖 playground 下拉框、`/api/checkpoints` 加载和 `payload.checkpoint` 发送

- `代码讲解记录_评估基准阶段/54-v39-checkpoint-selector.md`
  - 说明目标、边界、字段语义、API 行为、CLI 行为、日志证据、测试链路和截图证据

## 标记含义

`v39.0.0` 表示 MiniGPT 的本地推理 playground 已经支持 checkpoint selector：服务端能列出候选 checkpoint，前端能选择 checkpoint，生成 API 能路由到所选 checkpoint，请求日志能记录这次选择。
