# v57 运行截图和解释

## 本版主题

v57 在 v55/v56 的本地流式推理基础上补“请求历史视图”。前两版已经能让 playground 发起流式生成、超时、取消，并把结果写入 `inference_requests.jsonl`；这一版把这份 JSONL 日志读回来，新增 `/api/request-history`，再把最近请求显示到 playground 的 Request History 表格里。

## 关键改动

- `src/minigpt/server.py`：新增 `build_request_history_payload`、`/api/request-history`、limit 边界、坏 JSON 行统计、状态/端点摘要和请求记录归一化。
- `src/minigpt/playground.py`：新增 Request History 区块、刷新按钮、状态徽标、最近请求表格，并在 live/pair 生成后自动刷新。
- `scripts/serve_playground.py`：启动输出里增加 `request_history=<url>`，方便从终端直接看到新端点。
- `tests/test_server.py`：覆盖请求历史 payload、HTTP 端点、limit 失败、流式记录回读。
- `tests/test_playground.py`：覆盖 Request History HTML、JS 函数和 `/api/request-history` 引用。
- `README.md`、`b/README.md`、`代码讲解记录_项目成熟度阶段/72-v57-request-history-view.md`：同步 v57 能力、截图和讲解索引。

## tag 含义

`v57.0.0` 表示 MiniGPT 第五十七个可复查学习版本：本地推理不只会写 JSONL 日志，也能通过 API 和 playground 直接回看最近请求。
