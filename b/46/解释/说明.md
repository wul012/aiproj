# v46 运行截图和解释

## 本版主题

v46 把 v45 的单 run pair batch/trend 入口继续推进到 run registry。目标是让多个实验目录被注册后，registry 可以直接看到每个 run 是否有 pair batch、pair trend 报告，并能从同一个 HTML 索引打开这些报告。

## 关键改动

- `src/minigpt/registry.py`：`REGISTRY_ARTIFACT_PATHS` 增加 pair batch/trend 产物；`RegisteredRun` 增加 pair batch/trend 摘要字段和 HTML 存在标记。
- registry JSON：新增 `pair_report_counts`，统计有 batch/trend 报告的 run 数。
- registry CSV：新增 `pair_batch_cases`、`pair_batch_generated_differences`、`pair_batch_html_exists`、`pair_trend_reports`、`pair_trend_changed_cases`、`pair_trend_html_exists`。
- registry HTML：新增 `Pair Reports` 统计卡、表格列、排序字段和 `pair batch` / `pair trend` 链接。
- `tests/test_registry.py`：覆盖 pair report 摘要读取、计数、CSV 字段、HTML 列、链接和排序元数据。

## tag 含义

`v46.0.0` 表示 MiniGPT 第四十六个可复查学习版本：pair batch/trend 报告从单 run dashboard/playground 入口升级到多 run registry 索引入口。

## 一句话总结

v46 把“一个 run 可以打开 pair 报告”推进到“多个 run 可以在 registry 中集中扫描和打开 pair 报告”。
