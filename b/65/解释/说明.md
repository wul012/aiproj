# v65 maturity release readiness trend context 运行截图说明

本版目标是把 v64 registry 里的 release readiness trend context 接入 project maturity summary。成熟度报告现在会读取 registry 的 `release_readiness_comparison_counts` 和 `release_readiness_delta_summary`，在 JSON/Markdown/HTML 中输出 Release Readiness Trend Context，并在发现 release readiness regression 时把整体成熟度状态降为 `warn`。

本版不重新生成 release readiness comparison，不改变 release gate 或 readiness dashboard 的规则，也不把成熟度报告变成新的发布门禁。它只让成熟度总结消费已有 registry 趋势证据。

## 关键运行命令

```powershell
python -m unittest tests.test_maturity
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts\build_maturity_summary.py --project-root <project> --registry <improved-registry.json> --out-dir <out>
python scripts\build_maturity_summary.py --project-root <project> --registry <regressed-registry.json> --out-dir <out>
powershell -ExecutionPolicy Bypass -File scripts\playwright_chrome_smoke.ps1 -UrlOrPath <out>\maturity_summary.html -Out b\65\图片\05-playwright-maturity-release-readiness-html.png
```

## 本版 tag 含义

`v65.0.0` 标记 MiniGPT 的 maturity summary 已经能消费 release readiness trend：项目成熟度不只看版本、归档、registry 和请求历史，也会把发布质量改善或退化纳入成熟度评审。

一句话总结：v65 把发布质量趋势从 registry 视图推进到 project maturity summary 的综合评审层。
