# v32.0.0 标记说明

v32 的主题是 `release gate profile comparison`。v31 已经有 `standard`、`review`、`strict`、`legacy` 四个发布门禁策略档位；本版继续向后，把多个 profile 的 gate 结果汇总成一份 JSON/CSV/Markdown/HTML 对比矩阵。

## 本版新增/修改

- `src/minigpt/release_gate_comparison.py`
  - 新增 `build_release_gate_profile_comparison`
  - 新增 JSON/CSV/Markdown/HTML 渲染和写出函数
  - 支持一个或多个 release bundle
  - 每个 bundle 会按多个 policy profile 生成 gate row
  - row 中记录 `decision`、`gate_status`、阈值、generation quality 要求、`failed_checks` 和 `warned_checks`

- `scripts/compare_release_gate_profiles.py`
  - 新增 `--bundle`，可重复传入多个 bundle
  - 新增 `--profiles`
  - 新增 `--fail-on-blocked`
  - 输出 `release_gate_profile_comparison.json/csv/md/html`

- `tests/test_release_gate_comparison.py`
  - 覆盖单 bundle 矩阵
  - 覆盖多个 bundle
  - 覆盖 legacy 缺少 generation quality checks 的兼容对比
  - 覆盖未知 profile 和空输入
  - 覆盖四类输出文件和 HTML escaping

- `代码讲解记录_发布治理阶段/47-v32-release-gate-profile-comparison.md`
  - 按发布治理阶段讲解规则说明目标、边界、数据结构、CLI、测试断言和归档证据

## 标记含义

`v32.0.0` 表示 MiniGPT 的发布治理不再只支持单个 profile gate 检查，而是可以一次性比较多个 release bundle 在多个 policy profile 下的放行结果。
