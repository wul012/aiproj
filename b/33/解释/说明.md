# v33.0.0 标记说明

v33 的主题是 `release gate profile delta explanations`。v32 已经能把多个 release bundle 在多个 policy profile 下的结果汇总成矩阵；本版继续向后，让 comparison report 直接解释 compared profile 相对 baseline profile 新增或移除了哪些 failed/warned checks。

## 本版新增/修改

- `src/minigpt/release_gate_comparison.py`
  - comparison report 新增 `deltas`
  - summary 新增 `delta_count`、`decision_delta_count`、`check_delta_count`、`diverged_bundle_count`
  - 新增 `write_release_gate_profile_delta_csv`
  - Markdown/HTML 新增 `Profile Deltas` 段落
  - `write_release_gate_profile_comparison_outputs` 新增 `delta_csv`

- `scripts/compare_release_gate_profiles.py`
  - CLI 输出新增 `deltas`
  - CLI 输出新增 `decision_deltas`
  - CLI 输出新增 `check_deltas`

- `tests/test_release_gate_comparison.py`
  - 覆盖 strict 相对 standard 新增 `audit_score`
  - 覆盖 legacy 相对 standard 移除 `audit_score` 和 `generation_quality_audit_checks`
  - 覆盖 delta CSV 输出
  - 覆盖 Markdown/HTML 的 `Profile Deltas` 段落

- `代码讲解记录_发布治理阶段/48-v33-release-gate-profile-deltas.md`
  - 说明目标、边界、delta 数据结构、baseline 规则、explanation 生成逻辑、CLI 行为和测试证据

## 标记含义

`v33.0.0` 表示 MiniGPT 的发布治理报告不再只是列出多个 profile 的结果，而是能解释 profile 之间为什么发生分歧。
