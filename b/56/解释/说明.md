# v56 运行截图和解释

## 本版主题

v56 在 v55 流式 token 输出基础上补推理硬化：服务端增加 `max_stream_seconds` 超时边界，超时后通过 SSE 返回 `timeout` 事件并记录 partial response；playground 增加 `Stop` 按钮，用 `AbortController` 取消正在进行的 fetch stream；服务端把客户端断连记录为 `cancelled`，避免再二次写 error 事件。

## 关键改动

- `src/minigpt/server.py`：`InferenceSafetyProfile` 增加 `max_stream_seconds`，新增 `stream_timeout_payload`，`/api/generate-stream` 能返回 `timeout` 事件并写入 `stream_timed_out`、`stream_cancelled`、`stream_elapsed_seconds`。
- `src/minigpt/playground.py`：新增 `liveStopButton`、`AbortController`、`stopLiveGeneration`，并在 timeout 事件中显示 partial output 和 chunk 数。
- `scripts/serve_playground.py`：新增 `--max-stream-seconds` 参数。
- `tests/test_server.py`：覆盖 timeout payload、HTTP timeout SSE 顺序、partial response 和 JSONL 日志字段。
- `tests/test_playground.py`：覆盖 Stop 按钮、AbortController、timeout 显示逻辑和 stream endpoint 引用。
- `代码讲解记录_项目成熟度阶段/71-v56-streaming-cancel-timeout.md`：解释 v56 如何把流式生成从“能输出”推进到“有边界、可取消”。

## tag 含义

`v56.0.0` 表示 MiniGPT 第五十六个可复查学习版本：本地流式推理从“能持续输出 token”推进到“有服务端超时边界，也能从浏览器取消”。

## 一句话总结

v56 把“本地 playground 能流式生成”推进成“本地 playground 的流式生成有边界、可停止、可追溯”。
