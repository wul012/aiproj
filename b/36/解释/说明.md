# v36.0.0 标记说明

v36 的主题是 `dataset version manifests`。本版给 prepared corpus 增加 `dataset_version.json` 和 `dataset_version.html`，让数据准备结果能记录 dataset name、version、dataset id、fingerprint、source roots、outputs 和 quality summary。

## 本版新增/修改

- `src/minigpt/data_prep.py`
  - 新增 `build_dataset_version_manifest`
  - 新增 `write_dataset_version_json`
  - 新增 `render_dataset_version_html` / `write_dataset_version_html`
  - `write_prepared_dataset` 默认输出 `dataset_version.json/html`

- `scripts/prepare_dataset.py`
  - 新增 `--dataset-name`
  - 新增 `--dataset-version`
  - 新增 `--dataset-description`
  - 新增 `--dataset-root`
  - 传入 dataset name/version 时默认写入 `datasets/<name>/<version>`

- `scripts/train.py`
  - 从 prepared dataset 训练时复制 `dataset_version.json/html`
  - 从 `--data-dir` 训练时生成基础 dataset version manifest

- `src/minigpt/manifest.py`、`src/minigpt/dashboard.py`、`src/minigpt/playground.py`、`src/minigpt/registry.py`
  - 接入 dataset version artifact
  - run manifest 记录 `data.dataset_version`
  - dashboard 显示 dataset version id

- `代码讲解记录_评估基准阶段/51-v36-dataset-versioning.md`
  - 说明目标、边界、字段语义、CLI 行为、下游链路、测试覆盖和归档证据

## 标记含义

`v36.0.0` 表示 MiniGPT 的数据准备产物不再只是 loose corpus/report/quality 文件，而是拥有可追踪的 dataset version 身份。这为后续 baseline model comparison 记录“每个模型到底用了哪个数据版本”打基础。
