# v60 运行截图和解释

## 本版主题

v60 把 v57-v59 的 Request History 链路从“列表、筛选、单条详情”推进到“稳定性摘要”。新增 `scripts/summarize_request_history.py` 和 `src/minigpt/request_history_summary.py`，把 `inference_requests.jsonl` 聚合成状态计数、端点计数、checkpoint 计数、timeout/bad_request/error rate、最近请求和建议，并让 maturity summary 能读取 `request_history_summary.json`。

## 关键改动

- `src/minigpt/request_history_summary.py`：新增 request history summary builder 和 JSON/CSV/Markdown/HTML writer。
- `scripts/summarize_request_history.py`：新增 CLI，默认读取 `runs/minigpt/inference_requests.jsonl`，输出到 `runs/request-history-summary`。
- `src/minigpt/server.py`：抽出 `read_request_history_log_records`，让 summary 模块复用同一套 JSONL 解析和 `log_index` 归一化。
- `src/minigpt/maturity.py`：新增 `request_history_context`，Markdown/HTML 中显示 Request History Context，并把 v55-v60 纳入 local inference 能力线。
- `scripts/build_maturity_summary.py`：新增 `--request-history-summary` 参数。
- `tests/test_request_history_summary.py`、`tests/test_maturity.py`：覆盖 summary 指标、输出文件、HTML escaping、maturity context 和 local inference 版本覆盖。

## tag 含义

`v60.0.0` 表示 MiniGPT 第六十个可复查学习版本：本地推理请求历史从“单条可追溯”推进到“可汇总为稳定性指标并进入项目成熟度上下文”。
