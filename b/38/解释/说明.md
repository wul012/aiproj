# v38.0.0 标记说明

v38 的主题是 `inference safety profile`。本版给本地 playground server 增加请求边界、模型信息端点和 JSONL 请求日志，让本地推理 API 不只是“能生成”，还知道自己服务什么模型、允许什么请求、记录了什么结果。

## 本版新增/修改

- `src/minigpt/server.py`
  - 新增 `InferenceSafetyProfile`
  - `/api/health` 返回 safety profile 和 request log 路径
  - 新增 `/api/model-info`
  - `/api/generate` 使用安全画像校验 prompt、max_new_tokens、temperature、top_k 和 body size
  - 成功或失败的 generate 请求写入 `inference_requests.jsonl`

- `scripts/serve_playground.py`
  - 新增 `--max-prompt-chars`
  - 新增 `--max-new-tokens-limit`
  - 新增 `--min-temperature` / `--max-temperature`
  - 新增 `--max-top-k`
  - 新增 `--max-body-bytes`
  - 新增 `--request-log`

- `tests/test_server.py`
  - 覆盖 safety profile、model-info、request log、成功请求和 bad request 日志

- `代码讲解记录_评估基准阶段/53-v38-inference-safety-profile.md`
  - 说明目标、边界、API 字段、CLI 行为、日志语义、测试链路和截图证据

## 标记含义

`v38.0.0` 表示 MiniGPT 的本地推理服务已经具备最小安全画像：它能描述当前模型和数据版本，限制危险或过大的请求，并把推理请求结果记录成 JSONL 证据。
