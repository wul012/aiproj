# v53 运行截图和解释

## 本版主题

v53 把 v49-v52 的 benchmark scorecard、drilldown、rubric scoring 和 registry rubric tracking 收口成跨 run scorecard comparison。单个 run 已经能评分，registry 已经能排名；本版继续回答“分数为什么变了，具体退化在哪个 prompt、任务类型和难度分组”。

## 关键改动

- `src/minigpt/benchmark_scorecard_comparison.py`：读取多个 `benchmark_scorecard.json` 或 run 目录，选择 baseline，生成 run-level、case-level、task type、difficulty 的 delta 解释。
- `scripts/compare_benchmark_scorecards.py`：提供命令行入口，输出 JSON、CSV、case delta CSV、Markdown 和 HTML。
- `tests/test_benchmark_scorecard_comparison.py`：覆盖 run 目录读取、baseline 选择、overall/rubric delta、case regression、分组 regression、HTML 转义和输出文件。
- `README.md`、`b/README.md`、`代码讲解记录_项目成熟度阶段/README.md`：同步 v53 能力、tag、截图索引和后续方向。
- `代码讲解记录_项目成熟度阶段/68-v53-benchmark-scorecard-comparison.md`：解释 v53 如何承接 v49-v52，而不是继续拆小的 links/trends/dashboard。

## tag 含义

`v53.0.0` 表示 MiniGPT 第五十三个可复查学习版本：benchmark scorecard 从单 run 评分和 registry 排名，推进到跨 run 分数变化解释。

## 一句话总结

v53 把“多个 run 谁的 correctness 更高”推进成“为什么 correctness 变高或变低，以及变化落在哪些 prompt 和任务切片”。
