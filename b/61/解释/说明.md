# v61 运行截图和解释

## 本版主题

v61 把 v60 的 `request_history_summary.json` 从“可读报告”推进到“发布治理证据”。本版让 `project_audit` 读取请求历史稳定性摘要，生成 `request_history_summary` 审计检查；让 `release_bundle` 把 request history summary 的 JSON/Markdown/HTML 列入 evidence artifacts；让 `release_gate` 的 standard/review/strict policy 默认要求这项审计检查，legacy policy 继续兼容旧 bundle。

## 关键改动

- `src/minigpt/project_audit.py`：新增 `request_history_summary_path` 输入、自动发现路径、`request_history_context`、summary 字段和 `request_history_summary` 审计检查。
- `src/minigpt/release_bundle.py`：新增 request history summary 输入、artifact 收集、bundle summary 字段和 request history context。
- `src/minigpt/release_gate.py`：新增 `require_request_history_summary` policy 字段和 `request_history_summary_audit_check` 门禁检查。
- `src/minigpt/release_gate_comparison.py`：profile comparison 记录 request-history-summary requirement，并在 delta explanation 里说明 legacy 与标准策略的差异。
- `scripts/audit_project.py`、`scripts/build_release_bundle.py`、`scripts/check_release_gate.py`、`scripts/compare_release_gate_profiles.py`：补齐 CLI 参数和输出字段。
- `tests/test_project_audit.py`、`tests/test_release_bundle.py`、`tests/test_release_gate.py`、`tests/test_release_gate_comparison.py`：覆盖 clean、warn、missing、legacy-compatible 等关键路径。

## tag 含义

`v61.0.0` 表示 MiniGPT 第六十一个可复查学习版本：本地推理请求历史不再只停留在 playground 或 maturity summary，而是成为 project audit、release bundle 和 release gate policy 能消费的发布治理证据。

## 一句话总结

v61 让 request history summary 从“观察本地推理是否稳定”的报告，升级为“发布前必须被审计和门禁识别”的治理证据。
