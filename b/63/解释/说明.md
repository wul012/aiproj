# v63 release readiness comparison 运行截图说明

本版目标是把 v62 的单版发布就绪 dashboard 推进到跨版本比较：读取多个 `release_readiness.json`，指定或默认第一个作为 baseline，然后输出当前版本相对基线的 ready/blocked 状态变化、panel 变化、audit score 变化、缺失 artifact 变化和建议。

本版不做新的 release gate 规则，也不重新生成真实训练 checkpoint；它只消费已经存在或 smoke fixture 形式的 readiness dashboard 产物，验证比较层能稳定解释“发布质量是否改善或退化”。

## 关键运行命令

```powershell
python -m unittest tests.test_release_readiness_comparison tests.test_release_readiness
python -m unittest discover -s tests
python -m compileall src tests scripts
python scripts\compare_release_readiness.py --readiness <baseline>\release_readiness.json --readiness <current>\release_readiness.json --out-dir <comparison-dir>
python scripts\compare_release_readiness.py --baseline <ready>\release_readiness.json --readiness <blocked>\release_readiness.json --readiness <ready>\release_readiness.json --out-dir <comparison-dir>
powershell -ExecutionPolicy Bypass -File scripts\playwright_chrome_smoke.ps1 -UrlOrPath <comparison-dir>\release_readiness_comparison.html -Out b\63\图片\05-playwright-release-readiness-comparison-html.png
```

## 本版 tag 含义

`v63.0.0` 标记 MiniGPT 已具备跨版本发布就绪比较能力：不只知道某个 release 是否 ready，还能把多个 release dashboard 放在同一张表里，解释状态、panel 和证据完整性的变化。

一句话总结：v63 把发布就绪治理从“单版总览”推进到“跨版本差异解释”。
