# v55 运行截图和解释

## 本版主题

v55 把 MiniGPT playground 的本地生成从一次性 JSON 响应推进到流式 token 输出。服务端新增 `/api/generate-stream`，用 Server-Sent Events 返回 `start/token/end/error`；模型层新增 `sample_next` 让流式事件来自真实自回归采样循环；playground 的 Live Generate 按钮改为 `Stream Generate`，浏览器端用 fetch stream 逐步更新输出。

## 关键改动

- `src/minigpt/model.py`：新增 `MiniGPT.sample_next`，同步生成和流式生成共用同一套采样逻辑。
- `src/minigpt/server.py`：新增 `GenerationStreamChunk`、`MiniGPTGenerator.stream`、`sse_message` 和 `/api/generate-stream` 路由。
- `src/minigpt/playground.py`：新增 `readGenerationStream` 和 `parseSseEvent`，让 UI 从 SSE 中逐步更新输出。
- `scripts/serve_playground.py`：启动输出中增加 `generate_stream` 地址。
- `tests/test_model.py`、`tests/test_server.py`、`tests/test_playground.py`：覆盖单步采样、SSE 格式、HTTP 流式响应、日志字段和 UI 引用。
- `代码讲解记录_项目成熟度阶段/70-v55-streaming-generation.md`：解释 v55 如何把本地推理体验从同步响应推进到流式 token 事件。

## tag 含义

`v55.0.0` 表示 MiniGPT 第五十五个可复查学习版本：本地推理服务从同步生成推进到可流式消费的 token 事件。
