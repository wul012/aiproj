# v59 运行截图和解释

## 本版主题

v59 在 v58 的 Request History 筛选和 CSV 导出基础上补“单条详情 JSON”。每条有效 `inference_requests.jsonl` 记录现在会在读取时获得可见 `log_index`；服务端新增 `/api/request-history-detail?log_index=N`；playground 表格新增 `Log` 列、`Details` 按钮和行级 `JSON` 链接。

## 关键改动

- `src/minigpt/server.py`：新增 `_read_inference_log_entries`、`build_request_history_detail_payload`、`/api/request-history-detail`、`log_index` CSV 列和健康检查里的详情端点字段。
- `src/minigpt/playground.py`：Request History 表格新增行号、详情按钮、单条 JSON 链接和详情输出面板。
- `tests/test_server.py`：覆盖 `log_index`、详情 payload、HTTP 400/404、raw/normalized JSON 和 CSV header。
- `tests/test_playground.py`：覆盖详情端点字符串、详情面板、详情函数和 per-row JSON 下载链接。
- `README.md`、`b/README.md`、`代码讲解记录_项目成熟度阶段/74-v59-request-history-detail-json.md`：同步本版能力、证据位置和代码讲解索引。

## tag 含义

`v59.0.0` 表示 MiniGPT 第五十九个可复查学习版本：本地推理请求历史从“可筛选和 CSV 导出”推进到“每一行都可以打开 raw/normalized JSON 详情”的追溯能力。

## 一句话总结

v59 把 Request History 变成既能批量筛选导出，也能单条追溯原始 JSON 的本地推理证据入口。
