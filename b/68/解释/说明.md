# v68 运行截图和解释

## 本版主题

v68 新增 training portfolio comparison。它读取多份 `training_portfolio.json`，同时追踪这些 portfolio 指向的 `run_manifest.json`、`eval_suite.json`、`generation_quality.json`、`benchmark_scorecard.json`、`dataset_card.json` 和 `maturity_narrative.json`，最后输出相对 baseline 的 JSON/CSV/Markdown/HTML 对比证据。

本版不重新训练模型，也不重新评分；它把 v67 产出的单次训练组合证据变成多次实验可横向比较的入口。

## 关键命令

目标单测：

```powershell
python -m unittest tests.test_training_portfolio_comparison
```

全量回归：

```powershell
python -m unittest discover -s tests
```

编译检查：

```powershell
python -m compileall src tests scripts
```

CLI smoke：

```powershell
python scripts/compare_training_portfolios.py tmp/v68-smoke/baseline/training_portfolio.json tmp/v68-smoke/candidate/training_portfolio.json --name baseline --name candidate --baseline baseline --out-dir tmp/v68-smoke/comparison-improved
```

Playwright HTML 检查：

```powershell
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp/v68-smoke/comparison-improved/training_portfolio_comparison.html -Out b/68/图片/05-playwright-training-portfolio-comparison-html.png
```

## tag 含义

`v68.0.0` 表示本项目已经具备训练组合报告的跨 run baseline 对比能力。之后如果真正跑更大中文语料，就可以把新 portfolio 和小样本 smoke portfolio 放进 v68 对比器里，而不是只口头描述模型是否变好。

## 一句话总结

v68 的证据归档证明：MiniGPT 已经能把多份训练组合产物放到同一个 baseline 上比较分数、验证损失、产物覆盖、数据质量和成熟度状态。
