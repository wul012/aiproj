# v30.0.0 标记说明

v30 的主题是 `release gate generation quality policy`。v29 已经把 generation quality 接进 registry、model card 和 project audit；本版继续向后，把这条证据变成 release gate 的显式策略。

## 本版新增/修改

- `src/minigpt/release_gate.py`
  - `build_release_gate` 新增 `require_generation_quality=True`
  - `policy` 新增 `require_generation_quality_audit_checks`
  - 新增 `generation_quality_audit_checks` gate check
  - 默认要求 audit checks 中存在并通过：
    - `generation_quality`
    - `non_pass_generation_quality`
  - 缺失或 fail 会让 gate 阻断，warn 会让 gate 进入 needs-review

- `scripts/check_release_gate.py`
  - 新增 `--allow-missing-generation-quality`
  - 默认输出 `require_generation_quality=True`
  - legacy bundle 可以显式放宽，但默认不放宽

- `tests/test_release_gate.py`
  - 覆盖完整 bundle 通过
  - 覆盖缺失 generation quality audit checks 阻断
  - 覆盖 legacy 放宽模式
  - 覆盖 generation quality warn 与 `--fail-on-warn` 行为

- `代码讲解记录/45-v30-release-gate-generation-quality-policy.md`
  - 按新规则说明目标、边界、policy 字段、检查项、CLI、测试断言和 smoke 证据

## 标记含义

`v30.0.0` 表示 MiniGPT 的 release gate 现在显式要求 generation quality audit evidence。项目最终放行不再只依赖 audit overall，而是明确要求生成质量覆盖率和非 pass 生成质量检查存在并通过。
