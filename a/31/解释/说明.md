# v31.0.0 标记说明

v31 的主题是 `release gate policy profiles`。v30 已经要求 release gate 默认检查 generation quality audit evidence；本版继续向发布治理阶段推进，把门禁参数整理成 `standard`、`review`、`strict`、`legacy` 四个可命名策略档位。

## 本版新增/修改

- `src/minigpt/release_gate.py`
  - 新增 `RELEASE_GATE_POLICY_PROFILES`
  - 新增 `release_gate_policy_profiles()`
  - 新增 `resolve_release_gate_policy()`
  - `build_release_gate` 新增 `policy_profile`
  - gate report 的 `policy` 现在记录：
    - `policy_profile`
    - `profile_description`
    - `minimum_audit_score`
    - `minimum_ready_runs`
    - `require_generation_quality_audit_checks`
    - `overrides`

- `scripts/check_release_gate.py`
  - 新增 `--policy-profile standard|review|strict|legacy`
  - `--min-audit-score`、`--min-ready-runs` 和 `--allow-missing-generation-quality` 变成 profile 覆盖项
  - CLI 输出新增 profile、阈值和 generation quality 要求

- `tests/test_release_gate.py`
  - 覆盖 profile 列表、防外部修改、未知 profile 报错
  - 覆盖 strict 阈值阻断
  - 覆盖 review 放宽阈值但保留 generation quality 要求
  - 覆盖 legacy 缺失 generation quality checks 的兼容场景
  - 覆盖显式 override 优先级和 `overrides` 证据字段

- `代码讲解记录_发布治理阶段/46-v31-release-gate-policy-profiles.md`
  - 作为新阶段第一篇讲解，说明目标、边界、profile 字段、CLI 行为、测试断言和归档证据

## 标记含义

`v31.0.0` 表示 MiniGPT 的 release gate 进入可分场景治理阶段。以后发布不只看 pass/warn/fail，还能明确说明本次采用的是 standard、review、strict 还是 legacy 策略。
