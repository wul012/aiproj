# MiniGPT v25 验证说明

## 本版标记

`v25.0.0`：新增 project audit。

简单解释：在 registry、experiment card、model card 之上，v25 新增项目级审计报告，输出 `project_audit.json`、`project_audit.md`、`project_audit.html`。它会检查实验系列是否具备 release-style review 的基本证据链。

## 本版新增内容

- 新增 `src/minigpt/project_audit.py`。
- 新增 `scripts/audit_project.py`。
- 新增 `tests/test_project_audit.py`。
- 审计检查包含 registry runs、best run、loss coverage、experiment card coverage、dataset quality、eval suite、checkpoint、dashboard、model card、ready run、non-pass quality。
- 审计输出包含 `overall_status`、`score_percent`、pass/warn/fail 计数、checks、runs 和 recommendations。
- `audit_project.py` 支持 `--fail-on-warn`，方便以后接入 CI。

## 关键命令

```powershell
python -B -m unittest discover -s tests -v
python -B -m compileall src scripts tests
python scripts/register_runs.py tmp\v25-project-audit-smoke\run-a tmp\v25-project-audit-smoke\run-b --name ready-candidate --name secondary-pass --out-dir tmp\v25-project-audit-smoke\registry
python scripts/build_model_card.py --registry tmp\v25-project-audit-smoke\registry\registry.json --out-dir tmp\v25-project-audit-smoke\model-card
python scripts/audit_project.py --registry tmp\v25-project-audit-smoke\registry\registry.json --model-card tmp\v25-project-audit-smoke\model-card\model_card.json --out-dir tmp\v25-project-audit-smoke\audit
powershell -ExecutionPolicy Bypass -File scripts\playwright_chrome_smoke.ps1 -UrlOrPath tmp\v25-project-audit-smoke\audit\project_audit.html -Out tmp\v25-project-audit-smoke\project-audit.png
```

## 本版意义

这一版让项目从“能生成说明材料”推进到“能自动自检”。`registry` 负责收集实验，`experiment_card` 负责单 run 复盘，`model_card` 负责项目级说明，`project_audit` 负责检查证据链是否完整。
