# MiniGPT v18 验证说明

## 本版标记

`v18.0.0`：新增 registry HTML 报告。

简单解释：v17 已经能把多个训练 run 汇总成 `registry.json`、`registry.csv` 和 `registry.svg`；v18 在同一套 registry 数据上增加 `registry.html`，让实验列表、质量状态、数据指纹、最佳验证损失和 artifact 链接可以直接在浏览器里查看。

## 本版新增内容

- `src/minigpt/registry.py` 增加 `render_registry_html` 和 `write_registry_html`。
- `write_registry_outputs` 现在会输出 `registry.json`、`registry.csv`、`registry.svg`、`registry.html`。
- HTML 报告包含统计卡片、run 表格、质量状态 pill、数据指纹区块和 dashboard/manifest/eval 链接。
- HTML 文本统一经过转义，避免 run 名称或路径破坏页面结构。
- Windows 下相对路径计算增加兜底，不同盘符时退回绝对路径。
- 新增 `scripts/playwright_chrome_smoke.ps1`，让 Playwright 直接使用本机 Google Chrome (`--channel chrome`) 验证本地 HTML，避免必须下载 Playwright 自带 Chromium。
- `tests/test_registry.py` 增加 HTML 输出和 HTML 转义测试。

## 关键命令

```powershell
python -B -m unittest discover -s tests -v
python -B -m compileall src scripts tests
python scripts/register_runs.py tmp\v18-registry-html-smoke\run-a tmp\v18-registry-html-smoke\run-b --name run-a --name run-b --out-dir tmp\v18-registry-html-smoke\registry
python scripts/register_runs.py --discover tmp\v18-registry-html-smoke --out-dir tmp\v18-registry-html-smoke\registry-discovered
powershell -ExecutionPolicy Bypass -File scripts/playwright_chrome_smoke.ps1 -UrlOrPath tmp\v18-registry-html-smoke\registry\registry.html -Out tmp\v18-logs\registry-html-browser-script.png
```

## 本版意义

这一版没有扩大模型规模，而是继续补实验工程能力。`registry.html` 让多个实验 run 不再只是散落在目录里的文件，而是可以被统一浏览、跳转和展示的实验入口。
