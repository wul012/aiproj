# 第五版运行调试归档说明

本轮版本目标：

```text
加入 next-token prediction inspection 和 checkpoint evaluation，让 MiniGPT 可以导出候选 token 概率、SVG 条形图、loss 和 perplexity。
```

本轮不改变模型结构，重点是观察和评估模型输出。

---

# 1. 01-unit-tests.png

对应命令：

```powershell
python -B -m unittest discover -s tests -v
```

结果：

```text
attention、prediction、tokenizer、dataset、history、model 测试全部通过。
```

意义：

```text
证明 v5 新增的 next-token probability 和 perplexity 工具没有破坏已有训练/生成能力。
```

---

# 2. 02-prediction-train-smoke.png

对应命令：

```powershell
python -B scripts/train.py --device cpu --out-dir tmp/v5-prediction-smoke --max-iters 2 --eval-interval 1 --eval-iters 1 --batch-size 2 --block-size 16 --n-layer 1 --n-head 1 --n-embd 32 --sample-tokens 20
```

结果：

```text
训练可以正常完成并写出 checkpoint、tokenizer、metrics 和 sample 产物。
```

意义：

```text
提供一个真实 checkpoint 给 prediction/evaluation 脚本使用。
```

---

# 3. 03-inspect-predictions.png

对应命令：

```powershell
python -B scripts/inspect_predictions.py --device cpu --checkpoint tmp/v5-prediction-smoke/checkpoint.pt --prompt 人工智能 --top-k 6 --out-dir tmp/v5-prediction-smoke/predictions
```

结果：

```text
脚本输出 context tokens、top-k token 概率，并写出 predictions.json 和 predictions.svg。
```

意义：

```text
可以直接观察模型对“下一个 token”的概率判断。
```

---

# 4. 04-evaluate-report.png

对应命令：

```powershell
python -B scripts/evaluate.py --device cpu --checkpoint tmp/v5-prediction-smoke/checkpoint.pt --eval-iters 2 --batch-size 2 --out tmp/v5-prediction-smoke/eval_report.json
```

结果：

```text
脚本输出 loss 和 perplexity，并写出 eval_report.json。
```

意义：

```text
可以用统一指标观察 checkpoint 在本地语料上的拟合程度。
```
