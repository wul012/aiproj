# v17 归档说明

v17 新增 run registry 能力，把多个训练 run 的 manifest、数据指纹、质量状态、eval suite 和关键指标索引成统一注册表。

## 截图说明

`01-unit-tests.png`：展示 registry 新测试和既有回归测试通过。

`02-registry-runs-smoke.png`：展示两个小 run 的训练、eval suite 和 dashboard 准备过程。

`03-register-runs.png`：展示 `scripts/register_runs.py` 输出 registry JSON/CSV/SVG。

`04-registry-json-check.png`：检查 registry 中的 best run、quality counts、dataset fingerprints 和 artifact 文件。

`05-docs-check.png`：检查 README、代码讲解记录和 v17 归档说明是否存在。

## 本版重点

- 新增 `src/minigpt/registry.py`
- 新增 `scripts/register_runs.py`
- 新增 `tests/test_registry.py`
- 支持显式传入 run 目录
- 支持 `--discover` 自动发现 run
- 输出 `registry.json`、`registry.csv`、`registry.svg`
- 代码讲解记录合并为一篇 `32-v17-run-registry.md`

## 一句话总结

v17 让多个实验 run 不再只是散落的文件夹，而是可以被统一索引、筛选和比较的实验资产。
