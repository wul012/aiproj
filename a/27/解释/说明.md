# v27.0.0 标记说明

v27 的主题是 `release gate`。上一版 v26 已经能把 registry、model card、project audit 合成 release bundle；本版继续向后推进，把 release bundle 转成可以被脚本和 CI 使用的 pass/warn/fail 门禁结论。

## 本版新增

- `src/minigpt/release_gate.py`
  - 读取 `release_bundle.json`
  - 检查 release status、audit status、audit score、ready runs、best run、artifact 完整性、top runs、audit checks 和 bundle warnings
  - 输出 `gate_status = pass / warn / fail`
  - 输出 `decision = approved / needs-review / blocked`
  - 支持 JSON、Markdown、HTML 三种报告格式

- `scripts/check_release_gate.py`
  - 命令行入口
  - 默认读取 `runs/release-bundle/release_bundle.json`
  - 默认输出到 `runs/release-gate`
  - 门禁失败时退出码为 `1`
  - `--fail-on-warn` 可以让 warning 也导致非零退出码

- `tests/test_release_gate.py`
  - 覆盖通过、warning、失败、输出文件和 HTML 转义

- `代码讲解记录/42-v27-release-gate.md`
  - 合并说明 release gate 的代码结构、流程、状态映射和 CLI 用法

## 标记含义

`v27.0.0` 表示 MiniGPT 项目新增了自动化发布门禁。项目现在不仅能训练、评估、登记实验、生成报告，还能把最终 release bundle 转成可执行的发布判断。
