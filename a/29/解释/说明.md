# v29.0.0 标记说明

v29 的主题是 `generation quality evidence chain`。v28 已经能生成 `generation_quality.json/csv/md/svg/html`，本版把这个报告接入 registry、model card 和 project audit，让生成质量成为 release-style review 的正式证据之一。

## 本版新增/修改

- `src/minigpt/registry.py`
  - 读取 `generation-quality/generation_quality.json`
  - 兼容 `eval_suite/generation-quality/generation_quality.json`
  - 在 `RegisteredRun` 中记录 generation quality status、case count、pass/warn/fail count、avg unique ratio
  - 输出 `generation_quality_counts`
  - CSV/HTML/SVG/leaderboard 增加 generation quality 信息
  - HTML 链接区增加 `gen quality`

- `src/minigpt/model_card.py`
  - 汇总 `generation_quality_counts`
  - coverage 增加 `generation_quality_runs` 和 `generation_quality_pass_runs`
  - run 表增加 generation quality 状态
  - 非 pass generation quality 会进入 recommendations

- `src/minigpt/project_audit.py`
  - 新增 `generation_quality` 覆盖率检查
  - 新增 `non_pass_generation_quality` 检查
  - 缺失或非 pass 时给出修复建议

- `scripts/register_runs.py`
  - 命令行输出增加 `generation_quality_counts`

- `tests/test_registry.py`
- `tests/test_model_card.py`
- `tests/test_project_audit.py`
  - 补充 generation quality fixture 和关键断言

- `代码讲解记录/44-v29-generation-quality-chain.md`
  - 按新的讲解规则说明目标、边界、证据链、字段语义、测试断言和 smoke 证据

`v29.0.0` 表示 MiniGPT 的生成质量报告已进入正式证据链。项目现在不仅能单独分析生成样本，还能让 registry、model card 和 project audit 共同消费这份只读质量证据。
