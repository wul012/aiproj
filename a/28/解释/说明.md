# v28.0.0 标记说明

v28 的主题是 `generation quality`。前面已经有 eval suite 和 sampling lab，本版增加一层生成质量分析，把生成样本转成可以复查和继续汇总的质量报告。

## 本版新增

- `src/minigpt/generation_quality.py`
  - 读取 `eval_suite.json` 或 `sample_lab.json`
  - 自动识别来源类型
  - 计算续写长度、唯一字符比例、重复 n-gram 比例、最长连续重复字符
  - 标记 `empty_continuation`、`too_short`、`low_diversity`、`long_repeat_run`、`high_ngram_repetition`、`prompt_echo`
  - 汇总 `overall_status = pass / warn / fail`
  - 输出 JSON、CSV、Markdown、SVG、HTML

- `scripts/analyze_generation_quality.py`
  - 命令行入口
  - 默认读取 `runs/minigpt/eval_suite/eval_suite.json`
  - 默认输出到输入目录下的 `generation-quality`
  - 支持调整长度、多样性和重复率阈值

- `tests/test_generation_quality.py`
  - 覆盖 pass/warn/fail、sample lab 输入、输出文件、HTML 转义和非法阈值

- `代码讲解记录/43-v28-generation-quality.md`
  - 合并讲解生成质量分析的流程、指标、flags、CLI 和测试

## 标记含义

`v28.0.0` 表示 MiniGPT 项目新增了生成质量诊断能力。项目现在不仅能生成和评估固定 prompt，还能识别空输出、过短输出、重复输出、低多样性和 prompt echo 等常见小模型问题。
