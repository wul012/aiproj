# 第三版运行调试归档说明

本轮版本目标：

```text
加入教学版 BPE tokenizer，让 MiniGPT 支持字符级和 BPE 两种 tokenizer，并提供 tokenizer 检查工具。
```

本轮仍然不引入第三方 tokenizer 依赖，优先保证原理清楚、旧 checkpoint 兼容、训练/生成入口一致。

---

# 1. 01-unit-tests.png

对应命令：

```powershell
python -B -m unittest discover -s tests -v
```

结果：

```text
tokenizer、dataset、history、model 测试全部通过，tokenizer 测试扩展到 BPE 覆盖。
```

意义：

```text
证明 v3 新增 BPE 没有破坏 v1/v2 的字符级训练和历史记录能力。
```

---

# 2. 02-bpe-inspect.png

对应命令：

```powershell
python -B scripts/inspect_tokenizer.py --tokenizer bpe --bpe-vocab-size 260 --text 人工智能 --show-merges 8
```

结果：

```text
脚本输出 tokenizer=bpe、vocab_size、ids、decoded，以及前几条 merge 规则。
```

意义：

```text
可以直接观察 BPE 如何把高频字符片段合并成更长 token。
```

# 3. 03-bpe-train-smoke.png

对应命令：

```powershell
python -B scripts/train.py --device cpu --out-dir tmp/v3-bpe-smoke --tokenizer bpe --bpe-vocab-size 260 --max-iters 2 --eval-interval 1 --eval-iters 1 --batch-size 2 --block-size 16 --n-layer 1 --n-head 1 --n-embd 32 --sample-tokens 20
```

结果：

```text
训练脚本显示 tokenizer=bpe，并写出 checkpoint、tokenizer.json、metrics.jsonl、loss_curve.svg 和 sample.txt。
```

意义：

```text
BPE tokenizer 已经接入真实训练链路。
```

# 4. 04-bpe-generate-load.png

对应命令：

```powershell

生成脚本能自动加载 BPE tokenizer；检查脚本能从 tokenizer.json 还原 BPE merge 规则。

证明 BPE tokenizer 可以保存、加载，并被推理入口复用。
```
