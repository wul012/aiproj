# v11 归档说明

v11 新增静态 Playground UI，用 `playground.html` 把 run 目录里的训练产物、采样结果、常用命令和文件链接整合到一个本地浏览器页面。

## 截图说明

`01-unit-tests.png`：展示 v11 playground、dashboard、comparison 相关测试通过；全量 torch-backed 测试发现流程本轮在 Windows 环境中加载超时，因此截图里保留了这个限制说明。

`02-playground-artifacts-smoke.png`：展示为了构建 playground 而准备的轻量 run fixture、dashboard 和 sampling lab 产物。

`03-build-playground.png`：展示 `scripts/build_playground.py` 写出 `playground.html`，并打印可用链接数量、采样 case 数量和命令摘要。

`04-playground-html-check.png`：检查 `playground.html` 是否包含 prompt 控件、命令区域、采样表和 artifact 链接。

`05-docs-check.png`：检查 README、代码讲解记录和 v11 归档说明是否存在。

## 本版重点

- 新增 `src/minigpt/playground.py`
- 新增 `scripts/build_playground.py`
- 新增 `tests/test_playground.py`
- README 增加 v11、playground 命令、项目结构和截图说明

## 一句话总结

v11 把 MiniGPT 的训练、采样、检查和报告入口整理成一个静态 Web UI，让项目更像一个可以持续实验的小型 AI 学习工作台。
