# v321 运行截图说明

## 本版主题

v321 把 `promoted_training_scale_seed_handoff_artifacts.py` 中最重的 receipt/check/assurance artifact 逻辑拆到独立子模块：

```text
handoff report
  -> handoff_artifacts 门面
  -> receipt_artifacts 子模块
```

本版边界：这是 artifact 层拆分，不改 seed handoff 的业务规则，不改 receipt/check/assurance 的判断逻辑，也不改对外函数名。

## 关键结论

- `promoted_training_scale_seed_handoff_artifacts.py` 已经从约千行降到更可控的门面文件。
- receipt/check/assurance 的输出逻辑现在集中在独立 receipt artifact 模块。
- 旧的导入路径和测试契约仍然有效。
- 这次拆分改善的是维护性和职责边界，不是业务能力。
