# v444 运行截图和解释

## 本版目标

v444 给 v443 的 CI baseline-candidate threshold boundary wrapper plan 增加一层轻量复核。它不重新训练、不改变候选接受结论，也不扩展新的治理链，只确认 wrapper 记录的 artifact digest 和 expected-exit gate summary 仍然能被 CI 复核。

## 关键证据

- `ci-baseline-candidate-threshold-boundary-gate-check/`
  - 由 `scripts/run_ci_baseline_candidate_threshold_boundary_gate_check.py` 生成。
  - 证明 strict diagnosis gate 内层返回 `2`，但 gate-check 认定这是预期拒绝，外层 `status=pass`。
- `ci-baseline-candidate-threshold-boundary-gate-plan-check/`
  - 由新增 `scripts/check_ci_baseline_candidate_threshold_boundary_gate_plan.py` 生成。
  - 校验 wrapper plan 中 5 个 artifact digest、wrapper 名称、return code、固定 CI config 和 gate summary 字段。
- `ci-workflow-hygiene/`
  - 证明 GitHub Actions 已要求 plan check，并且顺序为 wrapper 之后、coverage 之前。
  - 本轮结果：`status=pass`、`check_count=24`、`required_order_count=9`、`baseline_candidate_threshold_boundary_gate_plan_check_ready=True`。
