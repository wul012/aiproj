# v396 promoted seed handoff maturity CI reasons 运行说明

## 本版主题

v396 把 promoted seed 中已有的 maturity CI regression reason counts 带进最终 seed handoff。

本版解决的问题是：v394 之后 seed 已经知道 CI regression 的原因分布，但 handoff 只保留 count/name。strict clean-batch automation stop 时，用户还要回查 seed 才知道原因。v396 让 handoff summary、requirement、CSV、Markdown、HTML 和 CLI stdout 都直接保留 reason-count 解释。

## 关键改动

- `src/minigpt/promoted_training_scale_seed_handoff_review.py`
  - handoff clean-batch review summary 新增 selected、aggregate、comparison-ready reason-count 字段。
  - `clean_batch_review_requirement` 新增 selected reason counts。
  - recommendation 输出 observed reasons。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py`
  - CSV 新增 reason-count 字段，使用 `reason:count` 格式。
- `src/minigpt/promoted_training_scale_seed_handoff_sections.py`
  - Markdown 和 HTML 展示 selected、aggregate、comparison-ready 和 requirement reason counts。
- `scripts/execute_promoted_training_scale_seed.py`
  - stdout 打印 reason-count JSON 行，方便 CI 日志直接定位失败原因。
- `tests/test_promoted_training_scale_seed_handoff.py`
  - 测试 fixture 构造 `missing-ci-step` 和 `workflow-order-regressed` 原因，断言 JSON、CSV、Markdown、HTML、stdout 和 requirement 都能保留原因。
