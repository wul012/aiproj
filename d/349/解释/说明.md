# v349 运行截图说明：benchmark history readiness comparison

本版目标是把 v348 release readiness dashboard 已经展示的 benchmark history 边界继续带入 release readiness comparison。它不重新生成 benchmark history，不训练模型，也不改变 readiness dashboard 本身；只让两个 readiness 报告放在一起比较时，能看见 benchmark history 是否从 `pass` 退到 `warn`，以及退化来自 case regression、generation-flag regression 还是 latest boundary 变化。

## 截图清单

- `01-release-readiness-comparison-benchmark-history-html.png`：Playwright/Chrome 打开本版生成的 `release_readiness_comparison.html`，证明对比页顶部 summary 出现 `Benchmark deltas=1` 和 `Benchmark regressions=1`，矩阵出现 `Benchmark history`、`Benchmark ready`、`Benchmark regressions`、`Benchmark boundary`，delta 表出现 benchmark status delta、case regression delta 和 boundary changed。
- `02-verification-summary.png`：本版聚焦验证摘要，记录 release readiness comparison 单测、语法检查、CLI smoke、Playwright 截图，以及后续全量验证项。

```powershell
python -m unittest tests.test_release_readiness_comparison
python -m py_compile src\minigpt\release_readiness_comparison.py src\minigpt\release_readiness_comparison_artifacts.py scripts\compare_release_readiness.py tests\test_release_readiness_comparison.py
python scripts\compare_release_readiness.py --readiness runs\v349-readiness-comparison-input\baseline\release_readiness.json --readiness runs\v349-readiness-comparison-input\current\release_readiness.json --out-dir runs\v349-release-readiness-comparison-smoke --title "MiniGPT v349 benchmark history readiness comparison"
python -m unittest discover -s tests
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v349
python -B scripts/check_ci_workflow_hygiene.py --out-dir runs/ci-workflow-hygiene-v349
python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-v349 --fail-under 80
```

## Smoke 边界

本版 smoke 构造了一个 `ready -> review` 的对比：

```text
baseline benchmark_history_status=pass
candidate benchmark_history_status=warn
candidate benchmark_history_ready=0
candidate benchmark_history_case_regressions=2
candidate benchmark_history_generation_flag_regressions=1
candidate benchmark_history_latest_boundary=tiny-smoke-plumbing-evidence
```

预期结果是 comparison summary 中：

```text
benchmark_history_deltas=1
benchmark_history_regressions=1
ci_workflow_regressions=0
test_coverage_regressions=0
```

这说明 v349 会把 benchmark history 退化单独归因，不会误算成 CI 或 coverage 问题。

一句话总结：v349 让 release readiness comparison 能解释 benchmark history 边界的跨版本变化。
