# v413 release readiness benchmark history suite design

## 本版主题

v413 把 benchmark-history suite-design readiness 从 release gate 继续传到 release readiness dashboard。目标是让 readiness summary、benchmark history panel、Markdown/HTML 和 CLI stdout 都能看到 prompt suite design 是否还有 non-comparison-ready 条目。

## 关键改动

- `src/minigpt/release_readiness.py`：summary 增加 suite-design not-ready 和 design-change 计数，benchmark history panel detail 展示两个字段，并把 not-ready 计数纳入 `warn` 判断。
- `src/minigpt/release_readiness_artifacts.py`：Markdown/HTML 增加对应展示字段。
- `scripts/build_release_readiness.py`：CLI stdout 打印新增计数。
- `tests/test_release_readiness.py`：覆盖 ready 默认值、suite-design review 行为、rendered outputs 和 CLI stdout。

## 截图说明

- `图片/01-release-readiness-history-suite-design-evidence.png`：展示 v413 的证据页，包含字段流向、readiness review 规则、验证命令和归档位置。

验证结果：定向 release readiness 测试 `14 passed`；全量 pytest `701 passed`；source encoding hygiene `status=pass`，`319` 个 source clean；`git diff --check` 无 whitespace error。

## 边界说明

本版不新增 benchmark suite、不训练 checkpoint、不改变 release readiness comparison，只把上游已经形成的 suite-design history 字段带到 readiness dashboard，避免最终发布准备面板丢失 prompt-suite 设计边界。

## 一句话总结

v413 让 release readiness dashboard 能识别 benchmark history 里的 suite-design not-ready 风险，把 prompt-suite 设计边界带到发布准备审阅层。
