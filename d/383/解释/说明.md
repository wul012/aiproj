# v383 maturity release readiness CI reasons 运行说明

## 本版目标

v383 把 v382 registry 汇总的 release readiness CI workflow regression reasons 带入 maturity summary 和 maturity narrative。

本版解决的问题是：registry 已经知道 CI 回归的原因，但成熟度总结和人读叙事还只能看到 CI regression count。现在 maturity summary、maturity narrative、Markdown、HTML 和 CLI 都能展示 `ci_workflow_regression_reason_counts`。

本版不新增 report 类型，不改变 registry schema，不把这条链路继续扩展到 training portfolio，只在现有成熟度/叙事层做轻量透传。

## 关键改动

- `src/minigpt/maturity.py`
  - release readiness context 新增 `ci_workflow_regression_reasons` 和 `ci_workflow_regression_reason_counts`。
  - maturity overview 新增 `release_readiness_ci_workflow_regression_reasons` 和 `release_readiness_ci_workflow_regression_reason_counts`。
  - CI regression recommendations 追加 reason-count 细节。
- `src/minigpt/maturity_artifacts.py`
  - Markdown overview、Release Readiness Trend Context、HTML stat cards 和 release section 展示 CI reason counts。
- `src/minigpt/maturity_narrative_summary.py`
  - narrative summary 读取 maturity/context 的 CI reason 字段。
  - CI workflow regression recommendation 展示 reason counts。
- `src/minigpt/maturity_narrative_sections.py`
  - Release Quality Trend claim 展示 `CI regression reasons=...`。
- `src/minigpt/maturity_narrative_artifacts.py`
  - Markdown Portfolio Summary 和 HTML stat cards 展示 CI reason counts。
- `scripts/build_maturity_summary.py`
  - stdout 打印 maturity summary 的 CI reason list/counts。
- `scripts/build_maturity_narrative.py`
  - stdout 打印 narrative summary 的 CI reason list/counts。
