# v322 运行截图说明

## 本版主题

v322 把 tiny scorecard comparison smoke 的 promotion decision 阈值从脚本里的固定 `80` 改成可配置参数：

```text
run_tiny_scorecard_comparison_smoke.py
  -> --decision-min-rubric-score
  -> build_benchmark_scorecard_decision.py --min-rubric-score
  -> summary.run_config.decision_min_rubric_score
  -> text summary config_decision_min_rubric_score
```

本版边界：这不是模型能力提升，也不是改 benchmark scorecard decision 的业务规则；它只是让真实 tiny smoke 的决策门槛显式进入命令、JSON 证据和文本摘要。

## 关键结论

- `--decision-min-rubric-score` 默认仍是 `80.0`，旧命令行为保持不变。
- 非默认阈值会进入 decision 命令、summary JSON 和 summary text。
- 阈值必须在 `0..100`，避免错误配置生成误导性决策证据。
- tiny smoke 的 `status=pass` 仍表示证据链跑通，不表示 tiny 模型质量达标。
- 全量 590 个测试通过，source encoding 和 py_compile 都通过。

## Tag 含义

`v322.0.0` 表示项目完成 tiny scorecard comparison smoke 的决策阈值显式化：CI 或本地 smoke 可以选择更严格或更宽松的 rubric gate，并在产物中留下可审计配置。

一句话总结：v322 让 tiny benchmark comparison 的决策门槛从隐藏常量变成可复现的实验配置。
