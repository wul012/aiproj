# v437 baseline-candidate threshold boundary 运行说明

## 本版主题

v437 在 v436 threshold matrix 的基础上增加 threshold boundary summary。它不只统计 accept/reject 行数，还明确记录：

- strictest accepting threshold：最严格但仍接受 candidate 的阈值。
- first rejecting threshold：第一个拒绝 candidate 的阈值。
- transition_count：accept/reject 决策切换次数。
- is_monotonic_acceptance：阈值升高时是否保持“先接受、后拒绝”的单调关系。

本版仍然不新增训练，不声称模型质量提升。它证明的是同一份 v432 tiny smoke summary 在阈值边界附近的决策行为可复核。

## 关键产物

- `baseline-candidate-threshold-boundary/baseline_candidate_threshold_matrix.json`：带 `threshold_boundary` 的主报告。
- `baseline-candidate-threshold-boundary/baseline_candidate_threshold_matrix.txt`：命令行摘要。
- `baseline-candidate-threshold-boundary/baseline_candidate_threshold_matrix.md`：Markdown 归档。
- `baseline-candidate-threshold-boundary/baseline_candidate_threshold_matrix.html`：Playwright MCP 截图页面。
- `threshold-0/`、`threshold-0p5/`、`threshold-1/`：三个阈值下分别生成的 loop、handoff 和 handoff-check 证据。

## 运行命令

```powershell
python -B scripts\run_baseline_candidate_threshold_matrix.py d\432\解释\baseline-candidate-eval-loop\tiny-scorecard-comparison-smoke --out-dir d\437\解释\baseline-candidate-threshold-boundary --thresholds 0:1:0.5 --require-both-outcomes --force
```

- `baseline_candidate_threshold_boundary_stdout.txt`
- `baseline_candidate_threshold_boundary_exit.txt`

退出码为 `0`。核心结果：

## 边界说明

v437 证明 threshold boundary 的可解释性和单调性检查，不证明 candidate 模型能力已经优于 baseline。当前 tiny smoke 仍是训练治理流水线证据，报告中的 `boundary.model_quality_claim` 仍为 `not_claimed`。

一句话总结：v437 把 v436 的 accept/reject 计数推进为可解释的阈值边界摘要，让 baseline-candidate 决策在阈值附近更容易复核。
