# v320 运行截图说明

## 本版主题

v320 让 tiny scorecard comparison smoke 支持 baseline/candidate 不同训练预算：

```text
baseline tiny smoke --max-iters 1
candidate tiny smoke --max-iters 2
  -> scorecard comparison
  -> scorecard decision
  -> run_config summary
```

本版边界：预算差异是实验配置证据，不是 tiny 模型质量结论。即使 candidate 多训练一步，也不能据此宣称模型更强；本版只证明 budget override 被真实执行并进入 summary。

## 关键结论

- `run_tiny_scorecard_comparison_smoke.py` 可以分别控制 baseline 与 candidate 的 tiny training `max_iters`。
- `tiny_scorecard_comparison_smoke_summary.json` 保存完整 `run_config`。
- `tiny_scorecard_comparison_smoke_summary.txt` 和 stdout 直接显示 budget mode 与 delta。
- `commands` 保留实际 command text，可审计 candidate 是否真的收到不同训练预算。
- `model_quality_claim=not_claimed` 继续保护 tiny smoke 的能力边界。

## Tag 含义

`v320.0.0` 表示项目已经具备 CPU 级别的 tiny budget comparison smoke：它能把不同训练预算传到真实 tiny run，并在 comparison/decision summary 中留下可审计配置证据。

一句话总结：v320 让 tiny comparison smoke 具备最小预算实验能力，但仍保持对模型质量结论的克制。
