# v466 运行说明：archived path 回归进入 maturity / portfolio 消费层

## 本版目标

v466 接在 v465 后面做一层下游承接：v465 已经能在 release readiness comparison 中识别 `archived_path_portability_check_not_ready`，但这个信号如果只停在 comparison/registry 的原始报告里，后续 maturity summary、maturity narrative 和 training portfolio review 仍然不够直观。

本版不新增新的治理链，也不改变训练或模型能力，只把同一个 CI 回归原因继续传到现有成熟度与组合评审链路。

## 运行链路

1. 使用 `d/465/解释/release-readiness-comparison-archived-path` 作为输入。
2. 运行 `scripts/register_runs.py` 生成 `registry-archived-path`。
3. 运行 `scripts/build_maturity_summary.py` 生成 `maturity-summary-archived-path`。
4. 运行 `scripts/build_maturity_narrative.py` 生成 `maturity-narrative-archived-path`。
5. 构造 baseline/candidate 两个轻量 training portfolio fixture，运行 `scripts/compare_training_portfolios.py`。
6. 用 Playwright MCP 打开 portfolio comparison HTML，保存快照和截图。

## 关键输出

- `registry-archived-path/registry.json`
  - `ci_workflow_regression_reason_counts={"archived_path_portability_check_not_ready": 1}`
  - `ci_workflow_archived_path_portability_check_ready_regression_count=1`
- `maturity-summary-archived-path/maturity_summary.json`
  - `release_readiness_ci_archived_path_portability_check_ready_regression_count=1`
  - `overall_status=warn`
- `maturity-narrative-archived-path/maturity_narrative.json`
  - `release_readiness_ci_workflow_regression_reasons=["archived_path_portability_check_not_ready"]`
  - release section claim 中出现 archived path regression 计数
- `training-portfolio-comparison-archived-path/training_portfolio_comparison.json`
  - candidate 分数更高，但由于 archived path CI 回归产生 blocker review action
  - `best_score_maturity_release_readiness_ci_archived_path_portability_check_ready_regression_count=1`

## 边界

- 本版不重新训练模型。
- 本版不重新定义 release readiness comparison 的回归判断，只消费 v465 已产生的回归原因。
- `source-portfolios/` 是为了证明 portfolio comparison 消费链路而构造的轻量 fixture，不是新的真实训练产物。

## Tag 含义

`v466.0.0` 表示 archived path portability readiness regression 已从 release readiness comparison 承接到 registry、maturity summary、maturity narrative 和 training portfolio review。
