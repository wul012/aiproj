# v435 运行截图说明：baseline-candidate handoff embedded check

## 本版目标

v435 承接 v434 的 baseline-candidate handoff contract check，把 check 结果嵌回 v433 handoff 主产物。v434 已经能生成 sidecar check；v435 解决的是“主 handoff JSON 本身是否能显示自己已经被检查过”的问题。

本版不改变候选接受逻辑，也不重新训练模型。当前 candidate 仍未达到 `min_overall_score_delta=1.0`，所以 `--require-accepted` 仍按设计返回 `2`。区别是：最终 handoff JSON、TXT、Markdown 和 HTML 都会带上 `handoff_check.status=pass` 和 `handoff_check.failed_count=0`。

## 命令摘要

本版核心运行命令：

```text
python -B scripts\build_baseline_candidate_handoff.py d\432\解释\baseline-candidate-eval-loop --out-dir d\435\解释\baseline-candidate-handoff-embedded-check --check-out-dir d\435\解释\baseline-candidate-handoff-embedded-check\handoff-check --require-accepted --force
```

该命令按设计返回 `exit_code=2`，因为 candidate 仍未被接受；但 check sidecar 先通过，并被嵌回最终 handoff 产物。

关键输出：

```text
check_status=pass
check_failed_count=0
saved_embedded_json=d\435\解释\baseline-candidate-handoff-embedded-check\baseline_candidate_handoff.json
saved_embedded_text=d\435\解释\baseline-candidate-handoff-embedded-check\baseline_candidate_handoff.txt
saved_embedded_markdown=d\435\解释\baseline-candidate-handoff-embedded-check\baseline_candidate_handoff.md
saved_embedded_html=d\435\解释\baseline-candidate-handoff-embedded-check\baseline_candidate_handoff.html
exit_code=2
```

最终 TXT 摘要包含：

```text
handoff_check_status=pass
handoff_check_failed_count=0
```
