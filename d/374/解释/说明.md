# v374 registry release readiness split 运行说明

## 本版目标

v374 做一次轻量收口：把 registry 中持续膨胀的 release-readiness delta 聚合逻辑拆到 `registry_release_readiness.py`，同时把 benchmark-history readiness failed reason 的 removed 路径补成可见证据。

本版不新增治理链，不扩大训练流程，也不宣称模型质量提升；它只改善现有 release-readiness -> registry -> maturity -> narrative 链路的维护边界和恢复信号表达。

## 关键运行结果

- `registry_rankings.py` 从约 570 行降到 243 行。
- 新增 `registry_release_readiness.py`，集中承接 release-readiness delta rows、summary 和 leaderboard。
- failed reason added 仍然作为 benchmark-regressed review 信号。
- failed reason removed 作为恢复/稳定证据展示，不单独触发 review 降级。
