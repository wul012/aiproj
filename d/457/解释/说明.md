# v457 运行截图和解释

## 本版主题

v457 新增 promoted seed handoff receipt contract summary-check failure smoke matrix。

v456 已经能把 summary-check 失败归因到 `summary_field`、`contract_profile`、`sidecar` 三类。v457 用受控篡改场景验证这套归因是否真实可靠：同一份合法 summary 会被复制成四个 scenario，其中三个故意篡改，最后检查每个 scenario 的失败 family 和 failed target 是否符合预期。

本版不改变 promoted seed handoff 结果，不改变 receipt schema，不改变 summary-check 的判定逻辑，只新增回归验证入口。

## 关键产物

- `contract-summary/`
  - 输入的合法 receipt contract summary。
- `failure-smoke/`
  - 顶层 failure smoke JSON/CSV/text/Markdown/HTML。
  - `scenarios/<name>/summary` 保存每个场景的输入 summary。
  - `scenarios/<name>/check` 保存每个场景实际跑出的 summary-check。
- `failure-smoke-stdout.txt`
  - CLI 输出中可见 `status=pass`、`scenario_count=4`、`verified_scenario_count=4`、`failed_verification_count=0`。
- `playwright_failure_smoke_snapshot.md`
  - Playwright MCP 可访问性快照。

## 四个场景

- `baseline`
  - 不篡改。
  - 预期 summary-check `pass`，failed families 为空。
- `tamper_summary_field`
  - 篡改 `receipt_schema_version`。
  - 预期 `summary_field` family 失败，target 为 `summary.receipt_schema_version`。
- `tamper_contract_profile`
  - 篡改 `contract_check_type_summary`。
  - 预期 `contract_profile` 与 `summary_field` family 失败，target 为 `summary.contract_check_type_summary`。
- `tamper_sidecar`
  - 篡改 HTML sidecar。
  - 预期 `sidecar` family 失败，target 指向 contract summary HTML。
