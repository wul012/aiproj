# v357 governance stabilization review 运行截图说明

## 本版目标

v357 响应“暂停新治理链 2-3 天，稳定/裁剪现有 7 条”的判断。它不新增独立治理链，而是把这条节奏规则接入已有 `maintenance_policy`：同一个维护检查命令现在可以输出 governance stabilization review。

本版不训练模型，不新增 release gate，不新增 dashboard 家族，也不删除现有链路。它只把“先停、先看、先合并”的工程判断变成可运行、可截图、可测试的维护证据。

## 运行链路

运行命令：

```powershell
python -B scripts/check_maintenance_batching.py `
  --out-dir runs/v357-governance-stabilization-smoke `
  --module-warning-lines 700 `
  --module-critical-lines 1200
```

输出包含原有维护产物和新增稳定评审产物：

- `maintenance_batching.json/csv/md/html`
- `module_pressure.json/csv/md/html`
- `governance_stabilization.json/csv/md/html`

## 截图说明

### `图片/01-governance-stabilization-review.png`

截图证明 governance stabilization review 已经作为维护策略的一部分生成：

- 状态为 `pass`。
- 决策为 `pause_new_governance_chains`。
- 暂停窗口为 `3` 天。
- 当前治理链数量为 `7`。
- `keep = 5`，`watch = 2`，`merge/cut = 0`。
- 表格列出 7 条链的 consumer、evidence 和 next action。
- Recommendations 明确要求暂停新增治理链，只接受 CI 修复、bug 和 broken-contract repair。

这说明 v357 把“不要继续无限新增报告链”的建议落成了可审阅证据，而不是口头约束。

## 验证覆盖

本版验证覆盖：

- `tests.test_maintenance_policy`：覆盖默认 7 条链、merge/cut 裁剪候选、HTML/Markdown/CSV/JSON 输出、脚本 stdout。
- `compileall`：确认维护策略、artifact 和脚本入口语法正常。
- smoke：真实运行 `check_maintenance_batching.py`，生成 governance stabilization artifact。
- Playwright：打开 `governance_stabilization.html` 并截图归档。

## 一句话总结

v357 把 aiproj 的“暂停新增治理链、稳定/裁剪现有 7 条”变成维护策略里的可运行评审，让项目从继续扩张转向短期收口。
