# v438 live baseline-candidate threshold boundary smoke 运行说明

## 本版目标

v438 在 v437 threshold boundary summary 之后补一层 live smoke：先真实运行一次 tiny scorecard comparison，再把这次新生成的 smoke summary 送入 threshold matrix，最后输出一份 live boundary summary。

本版不扩大模型规模，不声称模型质量提升，也不新增一条治理链。它只回答一个更工程化的问题：当前 baseline-candidate 阈值边界报告是否能由刚跑完的真实 tiny smoke 重新推导出来。

## 运行命令

```powershell
python -B scripts\run_baseline_candidate_threshold_boundary_smoke.py --out-dir d\438\解释\baseline-candidate-threshold-boundary-smoke --thresholds 0:1:0.5 --force
```

输出证据位于：

```text

```

这里的 `pass` 说明 live smoke 和 matrix 生成链路都成功；`review` 说明候选模型在 `0`、`0.5`、`1` 三个阈值下全部被拒绝，没有观察到可接受边界。也就是说，v438 证明的是“链路可复核”，不是“候选可晋升”。

## 输出结构

- `tiny-scorecard-comparison-smoke/`：本版新跑出的 baseline/candidate tiny 训练、评估、scorecard comparison 和 decision smoke 证据。
- `threshold-boundary-matrix/`：基于上面 live smoke summary 重新生成的 threshold matrix，每个 threshold 都包含 loop、handoff、handoff-check。
- `live-boundary-summary/`：v438 新增的总览 JSON/text/Markdown/HTML，把 smoke、matrix、boundary 三层状态合并到一个可截图页面。
