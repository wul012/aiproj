# v381 release readiness CI regression reasons 运行说明

## 本版目标

v381 给 release readiness comparison 的 CI workflow regression 增加原因分类。

本版解决的问题是：v380 已经能把 drift-contract smoke ready 的退化计入 CI regression，但 recommendation 仍然只提示检查 failed-check 和 order-violation delta。现在 comparison 会输出机器可读 reason list 和 reason count，让 reviewer 知道到底是 failed check、order violation、workflow status downgrade，还是 drift smoke ready 退化。

本版不新增 CI step，不改变 release readiness dashboard 输入，不继续向 registry/maturity 下游扩散。

## 关键改动

- `src/minigpt/release_readiness_comparison.py`
  - 新增 `_ci_workflow_regression_reasons()`。
  - delta 新增 `ci_workflow_regression_reasons`。
  - summary 新增 `ci_workflow_regression_reasons` 和 `ci_workflow_regression_reason_counts`。
  - recommendation 改为展示 reason count。
- `src/minigpt/release_readiness_comparison_artifacts.py`
  - delta CSV、Markdown、HTML 展示 CI regression reasons。
- `scripts/compare_release_readiness.py`
  - stdout 打印 `ci_workflow_regression_reason_counts`。
- `tests/test_release_readiness_comparison.py`
  - 覆盖 failed-check、order-violation、drift-smoke-ready 三类 reason。
