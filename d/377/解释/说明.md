# v377 release readiness drift contract 运行说明

## 本版目标

v377 在 v373-v376 的 benchmark-history readiness failed-reason drift 链路上增加一个独立契约检查器。

本版解决的问题是：comparison JSON 已经能输出 added、removed、recovered、mixed 等字段，但这些字段之前主要靠生成逻辑和下游测试保护；如果未来有人手动改报告、迁移字段或改 summary 聚合，缺少一个“只读校验器”重新从 baseline/compared reason list 推导契约。

本版不新增 drift 语义，不改变 release readiness comparison 的正式输出 schema，也不扩大模型能力声明。

## 关键改动

- `src/minigpt/release_readiness_drift_contract.py`
  - 从 `baseline_benchmark_history_readiness_requirement_failed_reasons` 和 `compared_benchmark_history_readiness_requirement_failed_reasons` 重新计算 expected added/removed/drift status。
  - 检查 delta 上的 count/list/status 是否与推导结果一致。
  - 汇总所有 delta，检查 summary 上的 added/removed/recovery/mixed/status-count 字段是否自洽。
- `scripts/check_release_readiness_drift_contract.py`
  - 接受 `release_readiness_comparison.json` 或其输出目录。
  - 写出 JSON/TXT 检查结果。
  - 默认遇到 fail 退出非零，`--no-fail` 可用于只产出诊断。
- `tests/test_release_readiness_drift_contract.py`
  - 覆盖 valid mixed drift、summary mismatch、delta status mismatch、目录解析和脚本失败退出码。
- `README.md`
  - 更新当前版本说明、v377 checkpoint 和 tag 列表。
