# v324 运行截图说明

## 本版主题

v324 把 v323 的第一个 threshold blocker 诊断扩展成 threshold profile：

```text
decision_threshold_blocked_count
decision_threshold_blocked_candidates
decision_threshold_closest_candidate
decision_threshold_closest_margin
decision_threshold_largest_gap_candidate
decision_threshold_largest_gap_margin
```

本版边界：不改 decision 规则，不改训练参数，不声称模型能力提升；只是让 tiny comparison smoke 能说明 threshold blocker 的整体分布。

## 关键结论

- `decision_first_threshold_*` 仍保留，适合快速看第一个阻断。
- `decision_threshold_*` 新增整体 profile，适合判断是单个候选差一点，还是多个候选整体没过线。
- `closest_margin` 是最接近通过门槛的差距。
- `largest_gap_margin` 是离门槛最远的差距。
- 这仍然是 smoke/CI 可读性增强，不是模型质量提升。
- 全量 591 个测试通过，source encoding、py_compile 和 diff check 都通过。

## Tag 含义

`v324.0.0` 表示 tiny scorecard comparison smoke 已经能把 threshold blocker 从单点诊断扩展为小型 profile，方便 CI 和人工审阅判断阻断规模。

一句话总结：v324 让 tiny threshold 诊断从“看第一个失败者”推进到“看整组失败候选的分布”。
