# v309 运行截图说明

## 本版主题

v309 把 promotion index 中的 handoff batch CI regression 证据继续带入 promoted comparison。
这样 promoted comparison 不只知道哪些 runs 被比较，还能解释 clean-required promoted input 为什么被排除。

本版不改变训练或模型打分，只加强 promoted comparison 的治理证据和防御性复核。

## 证据链角色

v309 的关键价值在于让 promoted comparison 成为 promotion index 之后的第二道检查。
如果 index 已经过滤 CI-regressed clean-required promotion，comparison 层会保留排除原因。
如果旧 index 仍误标 compare-ready，comparison 层会根据 handoff batch CI regression count 再次拦截。
