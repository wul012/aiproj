# v308 运行截图说明

## 本版主题

v308 把 promotion report 中的 handoff batch CI regression 证据继续带入 promotion index。

本版不新增 promoted comparison 或 seed 链路，重点是让 index 的 compare-input 过滤不再只看 `clean` 状态字符串，而会同时检查 CI regression count。

## 关键结论

- Promotion index row 现在包含 handoff CI regression count/name。
- Promotion index summary 统计 aggregate 和 selected CI regression 总量。
- Clean-required promotion 即使 `handoff_clean_batch_review_status=clean`，只要 CI regression count 大于 0，也不会进入 compare inputs。

## 一句话总结

v308 的运行证据说明 promotion index 已经能把 CI-regressed clean-required promotion 留作证据但排除出模型对比输入。
