# v368 release gate benchmark readiness evidence

## 本版主题

v368 把 v367 已经进入 release bundle 的 benchmark history readiness requirement 继续接入 release gate。目标是让发布门禁在 bundle 本身仍然 `release-ready`、audit/coverage 都通过时，也能因为 benchmark readiness requirement 失败而进入 `warn / needs-review`。

本版不证明模型质量提升，也不扩大 benchmark suite。它只验证 release gate 是否消费并展示 benchmark history 的 requirement 状态、退出码和失败原因。

## 关键截图

- `d/368/图片/01-release-gate-readiness-requirement-warning.png`

截图来自 `gate_report.html`，页面显示：

- `GATE = warn`
- `DECISION = needs-review`
- `RELEASE = release-ready`
- `AUDIT = pass`
- `BENCH HISTORY = pass`
- `BENCH READINESS = fail`
- `BENCH READINESS EXIT = 1`

Checks 区域的 `benchmark_history_gate_check` 为 `warn`，detail 中保留：

```text
readiness_requirement=fail
readiness_exit=1
readiness_failed_reasons=insufficient_ready_entries
```

## 运行命令

```powershell
python -B scripts\check_release_gate.py --bundle runs\__tmp_release_gate_v368\release-bundle\release_bundle.json --out-dir runs\v368-release-gate-readiness-review
```

关键输出：

```text
gate_status=warn
decision=needs-review
checks=13 pass/1 warn/0 fail
release_status=release-ready
audit_status=pass
benchmark_history_status=pass
benchmark_history_readiness_requirement_status=fail
benchmark_history_readiness_requirement_exit_code=1
benchmark_history_readiness_requirement_failed_reasons=["insufficient_ready_entries"]
```

## 证据边界

这个证据说明 release gate 已经读到 benchmark readiness requirement，并把它转成发布前人工 review。它不是模型能力证明；模型能力仍要看真实 checkpoint、固定 benchmark、scorecard 和重复历史结果。
