# v455 运行截图和解释

## 本版主题

v455 给 promoted seed handoff receipt contract summary-check 增加 contract profile consistency checks。

v454 已经把 `contract_checks` 汇总成 `contract_check_status_counts` 和 `contract_check_type_summary`。本版进一步要求 summary-check 从 actual summary 内嵌的 `contract_checks` 重新计算这些聚合字段，确认明细 rows 与顶层 profile 没有脱节。

本版不改变 promoted seed handoff 的候选选择，不改变 receipt schema，不新增治理链，只加一层轻量复核。

## 关键产物

- `contract-summary/`
  - 重新生成的 receipt contract summary。
  - 其中 `contract_checks` 为明细检查 rows，`contract_check_status_counts` 与 `contract_check_type_summary` 为聚合 profile。
- `contract-summary-check/`
  - 新的 summary-check 输出。
  - JSON 中包含 `contract_profile_status`、`contract_profile_check_count`、`failed_contract_profile_check_count` 和 `contract_profile_checks`。
- `contract-summary-stdout.txt`
  - contract summary CLI 输出。
- `contract-summary-check-stdout.txt`
  - summary-check CLI 输出，可见 `receipt_contract_summary_check_contract_profile_status=pass` 与 `receipt_contract_summary_check_failed_contract_profile_check_count=0`。
- `playwright_contract_profile_checks_snapshot.md`
  - Playwright MCP 对 HTML 页面做的可访问性快照。
