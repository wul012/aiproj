# v465 运行截图和解释

## 本版主题

v465 做的是 release readiness comparison archived path regression：在 v464 已经把 `ci_workflow_archived_path_portability_check_ready` 带入 readiness 报告之后，本版让 release readiness comparison 能识别该字段从 `True` 退化为非 `True` 的情况。

这版不新增新的路径检查，也不改变 release readiness 的 ready/review/block 规则。它只补齐多版本对比能力：即使两个 readiness 报告整体都仍然是 `ready`，archived path portability 退化也会被归类为 CI workflow regression。

## 关键运行证据

- `source-inputs/v464-baseline-archived-ready/release_readiness.json`：baseline readiness，`ci_workflow_archived_path_portability_check_ready=True`。
- `source-inputs/v465-current-archived-regressed/release_readiness.json`：current readiness，整体仍 `ready`，但 `ci_workflow_archived_path_portability_check_ready=False`。
- `release-readiness-comparison-archived-path/release_readiness_comparison.json`：comparison 最终 JSON，记录 archived path readiness changed/regressed。
- `release-readiness-comparison-archived-path/release_readiness_deltas.csv`：delta CSV，包含 `ci_workflow_archived_path_portability_check_ready_regressed`。
- `release-readiness-comparison-archived-path/playwright_snapshot.md`：Playwright MCP 对 comparison HTML 的可访问性快照。
- `../图片/release-readiness-comparison-archived-path.png`：Playwright MCP 截取的 comparison HTML 全页截图。

## 运行命令摘要

```powershell
python -B scripts\compare_release_readiness.py --readiness d\465\解释\source-inputs\v464-baseline-archived-ready\release_readiness.json --readiness d\465\解释\source-inputs\v465-current-archived-regressed\release_readiness.json --out-dir d\465\解释\release-readiness-comparison-archived-path --title "MiniGPT v465 archived path readiness comparison"
```

```text

```
