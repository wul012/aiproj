# v422 运行截图说明：promoted training scale comparison suite-design guard

## 本版目标

v422 把 v421 promotion index 中的 suite-design regression context 带入 promoted training scale comparison。目标是让 promoted comparison 的输入二次校验、summary 聚合、CSV/Markdown/HTML 和 CLI 输出都能识别 `handoff_batch_maturity_suite_design_regression_count`，避免 stale index 把带有 benchmark-suite blocker 的 clean-required promotion 混入对比。

本版不执行训练，不改变 promotion index 生成逻辑，也不新增新的治理报告；只让已有 promoted comparison 消费并复核 v421 已经写出的字段。

```powershell
python scripts\compare_promoted_training_scale_runs.py `
  runs\v422-promoted-training-scale-comparison-suite-design\promotion-index `
  --require-compared
```

关键输出：

```text

comparison_ready_count=2
handoff_batch_maturity_suite_design_regression_count=1
comparison_ready_handoff_batch_maturity_suite_design_regression_count=0
```
