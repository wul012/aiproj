# v428 运行截图说明：promoted seed handoff receipt contract summary

## 本版目标

v428 在 v427 的 receipt sidecar 硬化之后，新增一层更适合 CI 和交接阅读的 contract summary。它把 handoff assurance、embedded receipt check、sidecar 状态、schema-v3 readiness，以及 selected/global/comparison-ready 三层 suite-design count/name 一致性压成一份 JSON/text/Markdown/HTML 摘要。

本版不执行训练，不改变 promoted seed handoff 决策，也不新增一条治理链。它只是让既有 receipt/assurance 合同有一个更紧凑的只读入口。

## 命令摘要

先生成完整 assurance smoke：

```text
python -B scripts\check_promoted_seed_handoff_assurance_smoke.py --out-dir d\428\解释\assurance-smoke
```

再对 smoke 产出的 handoff 目录生成 contract summary：

```text
python -B scripts\check_promoted_seed_handoff_receipt_contract.py d\428\解释\assurance-smoke\handoff --out-dir d\428\解释\contract-summary
```

关键输出：

```text
receipt_contract_status=pass
receipt_contract_schema_version=3
receipt_contract_assurance_status=pass
receipt_contract_sidecar_status=pass
receipt_contract_issue_count=0
```

## 证据边界

`d/428` 证明 contract summary 可以从现有 assurance smoke 读取 schema-v3 receipt 合同，并把 sidecar 和 suite-design count/name 一致性压缩成单入口摘要。它不证明模型质量提升，也不替代 v426/v427 的底层 receipt/embedded/assurance 检查；它是更适合 CI 和人工交接的只读汇总层。

一句话总结：v428 把 promoted seed handoff schema-v3 receipt 合同从多份 sidecar 证据压缩成一个可机器读取、可截图审阅的 contract summary 入口。
