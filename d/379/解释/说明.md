# v379 release readiness drift smoke readiness 运行说明

## 本版目标

v379 把 v378 新增的 release readiness drift contract smoke 从“CI checks 表里的一条命令”提升为上层治理能直接读取的 readiness signal。

本版解决的问题是：v378 已经让 CI 跑 drift-contract smoke，但 Project Audit、Release Bundle 和 Release Readiness 如果想知道这道门是否 ready，需要间接解析 CI workflow checks。v379 新增稳定 summary/context 字段，让上层证据能直接透传和展示。

本版不新增 CI step，不改变 drift contract 语义，不重新跑完整 release readiness comparison。
