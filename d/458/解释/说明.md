# v458 运行截图和解释

## 本版主题

v458 把 v457 的 receipt contract summary-check failure smoke matrix 接入 GitHub Actions，并让 CI workflow hygiene 检查它是否存在、是否位于正确顺序。

本版不改变 promoted seed handoff 结果，不改变 failure smoke 场景矩阵，也不新增训练链路。它只把已有验证入口提升为 CI 可持续 gate。

- `ci-workflow-hygiene/`
  - 新增 CI readiness 字段后的 hygiene 报告。
  - summary 中 `promoted_seed_receipt_contract_failure_smoke_ready=True`。
- `ci-workflow-hygiene-stdout.txt`
  - CLI 输出中可见新增 readiness 三项均为 `True`。
- `playwright_ci_receipt_failure_smoke_snapshot.md`
  - Playwright MCP 可访问性快照。

## CI 顺序

新增 CI step 位于 promoted seed handoff assurance smoke 之后、coverage 之前：

1. `scripts/check_promoted_seed_handoff_assurance_smoke.py`
2. `scripts/check_promoted_seed_handoff_receipt_contract.py`
3. `scripts/smoke_promoted_seed_handoff_receipt_contract_summary_check_failures.py`
4. `scripts/run_test_coverage.py`

CI workflow hygiene 会检查：

- receipt contract summary 命令存在。
- failure smoke 命令存在。
- assurance -> summary -> failure smoke -> coverage 的顺序成立。
