# v467 运行截图和解释：promoted training scale generic CI reason-count

## 本版主题

v467 不是继续新增 archived path 专用治理链，而是验证 `archived_path_portability_check_not_ready` 可以沿用训练规模链路里已经存在的 `*_ci_regression_reason_counts` 结构。

本版重点是：

- 用 `report_utils.ci_regression_reason_count()` 统一读取命名 CI regression reason。
- 保持 boundary plan-check 的旧计数字段兼容。
- 让 promoted training scale comparison 和 seed handoff review 都能保留 archived-path reason。

## 关键运行证据

- `d/467/解释/source-index/promotion-index/training_scale_promotion_index.json`

## 验证边界

本版证明的是 archived-path CI regression reason 能被通用 reason-count 结构读取、汇总、渲染和用于比较过滤。

本版不证明模型质量提升，也不新增 release readiness、maturity 或 portfolio 的新字段。v466 已经完成 archived path regression 向 maturity / portfolio 的传递，v467 只把 promoted training scale 侧保持在通用 reason-count 消费路径上。

## 一句话总结

v467 把 archived path portability regression 从“可能继续扩散专用字段”的风险，收束为“通用 CI reason-count 可以稳定承载的新原因”。
