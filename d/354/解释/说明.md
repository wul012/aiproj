# v354 run comparison dataset snapshot carryover 运行截图说明

## 本版目标

v354 把 v353 的 dataset version comparison 信号接入现有 `compare_runs.py`。这样多个训练 run 使用不同数据版本时，run comparison 不只提示“dataset version 不同”，还会直接展示 fingerprint、dedupe policy、source order、included/skipped source 和 char count 的 baseline delta。

本版不训练模型，不新增独立报告，也不把 loss 改善解释成模型质量提升。它只让 run comparison 在解释 loss / eval / perplexity 前，先暴露数据边界。

## 运行链路

本次 smoke 构造两个 run 目录：

- `raw`：dataset `demo-v354@raw`，dedupe policy `none`，included 3，skipped 0，chars 140。
- `deduped`：dataset `demo-v354@deduped`，dedupe policy `exact-source-content`，included 2，skipped 1，chars 90。

随后运行：

```powershell
python -B scripts/compare_runs.py `
  runs/v354-run-comparison-dataset-snapshot-smoke/raw `
  runs/v354-run-comparison-dataset-snapshot-smoke/deduped `
  --name raw --name deduped `
  --baseline raw `
  --out-dir runs/v354-run-comparison-dataset-snapshot-smoke/comparison
```

输出仍然是既有 run comparison 产物：

- `comparison.json`
- `comparison.csv`
- `comparison.svg`
- `comparison.md`
- `comparison.html`

## 截图说明

### `图片/01-run-comparison-dataset-snapshot.png`

截图显示 v354 的 run comparison HTML 已经带上 dataset snapshot 语义：

- 顶部 stat card 显示 `Data fingerprint changes = 1`。
- 顶部 stat card 显示 `Dedupe changes = 1`。
- raw 行显示 `dedupe=none; included=3 (0); skipped=0 (0); chars=140 (0)`。
- deduped 行显示 `dedupe=exact-source-content; included=2 (-1); skipped=1 (+1); chars=90 (-50)`。
- deduped 行还显示 `changed=fingerprint,dedupe,source-order`。

这说明 run comparison 现在能把数据变化和模型变化分开看，避免把因为数据版本改变带来的 loss delta 直接当成模型结构收益。

## 验证覆盖

本版验证覆盖：

- `tests.test_comparison`：检查 `summarize_run()` 能读出 dataset snapshot 字段，baseline delta 能识别 fingerprint/dedupe/source-order/source-count/char-count 变化。
- `tests.test_comparison_artifacts`：检查 CSV 和 HTML 输出包含 dataset snapshot 字段。
- smoke：真实生成两个 run 目录并运行 `compare_runs.py`。
- Playwright：打开 `comparison.html` 并截图归档。

## 一句话总结

v354 让模型 run comparison 在比较 loss 之前先显式暴露数据版本变化，把“模型变好了吗”和“数据变了吗”分开审计。
