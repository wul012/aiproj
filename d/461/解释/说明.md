# v461 运行截图和解释

## 本版目标

v461 做的是 v460 的收口型 carryover：不新增治理链，只把 CI workflow hygiene 里已经通过的 receipt failure-smoke wrapper plan check 结果，继续带到 project audit、release bundle、release readiness 三层证据。

这一版解决的问题是：CI 层已经知道 `promoted_seed_receipt_contract_failure_smoke_plan_check_ready=True`，但发布侧报告如果不展示它，后续读 release evidence 时仍要回头翻 CI hygiene 原始 JSON。

## 运行证据

- `project-audit-carryover/`：由 `scripts/audit_project.py` 生成，证明 audit summary 和 Markdown/HTML 已出现 `CI receipt failure-smoke plan check=True`。
- `release-bundle-carryover/`：由 `scripts/build_release_bundle.py` 生成，证明 release bundle summary 继承了 audit/CI context 的 receipt plan-check readiness。
- `release-readiness-carryover/`：由 `scripts/build_release_readiness.py` 生成，证明最终 readiness summary 和 CI panel 都能看到 receipt plan-check readiness。
- `source-inputs/`：保留本版复现实验输入，包括 registry、model card、benchmark history、CI workflow hygiene、coverage、gate 和 maturity JSON；它是正式证据输入，不是临时测试缓存。
- `playwright_receipt_plan_carryover_snapshot.md`：Playwright MCP 可访问性快照，确认浏览器页面中存在 `CI receipt plan` 和 `receipt_failure_smoke_plan_check_ready=True`。
- `../图片/01-receipt-plan-carryover-readiness.png`：Playwright MCP 截图，确认 HTML 页面能正常渲染。

## 截图说明

`图片/01-receipt-plan-carryover-readiness.png` 展示的是 v461 release readiness HTML。页面顶部显示 `ready`、`ship`、`release-ready`，中部指标卡显示 `CI receipt plan=True`，底部 CI Workflow Hygiene 面板显示 `receipt_failure_smoke_plan_check_ready=True`。

## 关键结论

- audit 层字段：`ci_promoted_seed_receipt_contract_failure_smoke_plan_check_ready=True`
- release bundle/readiness 层字段：`ci_workflow_promoted_seed_receipt_contract_failure_smoke_plan_check_ready=True`
- readiness 面板细节：`receipt_failure_smoke_plan_check_ready=True`
- 这版是字段贯通和证据可见性增强，不改变 v460 的 plan-check 判定逻辑。

一句话总结：v461 让 v460 的 CI receipt plan-check 结果从 CI hygiene 原始报告，正式进入发布审计、发布包和最终 readiness 页面。
