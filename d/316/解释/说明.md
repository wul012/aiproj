# v316 运行截图说明

## 本版主题

v316 把 v315 的 tiny standard benchmark smoke 扩展为：

```text
tiny corpus -> train.py -> checkpoint.pt -> eval_suite.py -> generation quality -> pair batch -> benchmark scorecard -> smoke summary
```

本版的边界是：`pair_batch` 使用同一个 tiny checkpoint 做左右两侧对比，所以它是可复现性基线，不是模型提升证据。summary 和 scorecard 都必须把它标记为 `same_checkpoint_baseline`。

## 关键结论

- `scripts/run_tiny_standard_benchmark_smoke.py` 现在会真实执行 `pair_batch.py`。
- pair batch artifact 既有 JSON，也有 HTML，满足机器消费和人工审查两种用途。
- scorecard summary 能看到 pair case count、same-checkpoint baseline 和 comparison mode。
- v316 仍然不宣称 tiny 模型质量达标，只宣称 benchmark evidence chain 更完整。

## Tag 含义

`v316.0.0` 表示 tiny standard benchmark smoke 已经包含同 checkpoint pair baseline：项目能在 CPU 上从真实 tiny checkpoint 跑完整 eval、quality、pair 和 scorecard 链路，并且能明确区分可复现性证据和模型改进证据。

一句话总结：v316 补齐了 tiny benchmark smoke 的 pair evidence completeness，同时没有越界夸大 tiny 模型能力。
