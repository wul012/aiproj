# v331 运行截图说明

## 本版主题

v331 给 tiny scorecard comparison smoke 增加可选的 clean remediation gate：

```text
--require-clean-remediation
```

默认模式下，remediation rows 仍然只是建议证据；严格模式下，只要 decision report 里还有 remediation rows，smoke summary 就会进入 fail，并给出 `remediation_gate_decision=stop`。

## 关键结论

- v331 没有改变 decision 规则或 remediation plan 内容。
- 新增 gate 是 opt-in，默认 tiny smoke 不会因为 remediation rows 失败。
- 严格模式会把 remediation rows 视为未清理问题，并在顶层 summary 中留下 `remediation_gate` 证据。
- 本版继续保持“tiny CPU smoke 只证明链路和治理门禁，不证明模型质量”的边界。

## Tag 含义

`v331.0.0` 表示 tiny scorecard comparison smoke 已支持可选 clean remediation gate。

一句话总结：v331 把 remediation 从可读行动清单接入到可选 CI 门禁，让严格 smoke 能阻断仍需修复的 scorecard decision。
