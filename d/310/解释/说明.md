# v310 运行截图说明

## 本版主题

v310 把 promoted comparison 中的 handoff batch CI regression 和 `comparison_exclusion_reasons` 继续带入 promoted baseline decision。
这样 baseline decision 不只知道 selected baseline 是谁，也能解释 rejected promoted candidate 为什么没有进入可选 baseline。

本版不改变模型训练或 baseline 排序，只加强决策证据链。

## 证据链角色

v310 的核心作用是把 comparison 层的排除原因传给 decision 层。
当一个 clean-required promoted input 因 handoff batch CI regression 被挡在 comparison 外，decision 层会在 rejected candidate 和 summary 中继续展示这个原因。

## Tag 含义

`v310.0.0` 表示 MiniGPT 第 310 个可回看版本：promoted baseline decision 已经能消费和展示 promoted comparison 的 CI regression exclusion context。

## 一句话总结

v310 的运行证据说明 baseline decision 能解释 CI-regressed promoted input 为什么被排除在 clean baseline selection 之外。
