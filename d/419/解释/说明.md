# v419 运行截图说明：training scale handoff suite-design carryover

## 本版目标

v419 把 v418 decision/workflow 中的 suite-design regression context 带入 training scale execution handoff。目标是让执行交接阶段也能看到 `batch_maturity_suite_design_regression_count` 和 affected portfolio names，避免 handoff 只看到 CI regression 或普通 blocker 数量。

本版不执行训练，不改变 workflow/decision 的生成逻辑，也不重新判定 suite-design regression；只在 handoff 层消费和展示已有字段，并让 strict clean-batch handoff guard 识别这类风险。

## 命令摘要

本轮用最小 workflow/decision fixture 生成 handoff HTML：

```powershell
python - <<'PY'
from minigpt.training_scale_handoff import build_training_scale_handoff
report = build_training_scale_handoff("runs/v419-training-scale-handoff-suite-design/workflow/training_scale_workflow.json")
print(report["summary"]["handoff_status"])
print(report["summary"]["batch_maturity_suite_design_regression_count"])
print(report["recommendations"])
PY
```

关键输出：

```text
handoff_status=planned
handoff_allowed=True
batch_maturity_suite_design_regression_count=1
selected_batch_maturity_suite_design_regression_count=1
```
