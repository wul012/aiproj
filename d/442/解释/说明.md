# v442 CI workflow baseline-candidate threshold boundary gate check

## 本版目标和边界

v442 把 v441 的 baseline-candidate threshold boundary expected-exit gate check 接入 GitHub Actions 和 CI workflow hygiene。它解决的是“本地有可复核 wrapper，但 CI 不强制运行”的问题。

本版不改变候选模型的接受标准，不扩大训练规模，也不新增模型能力结论。当前候选仍是 `candidate_not_accepted`；v442 只证明 CI 会执行这条 expected-exit contract check，并且 workflow hygiene 能检查它的位置和存在性。
