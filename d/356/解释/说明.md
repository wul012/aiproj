# v356 model card dataset snapshot carryover 运行截图说明

## 本版目标

v356 把 v355 registry 的 dataset snapshot 证据继续接入 model card。此前 model card 会总结 registry、experiment card、质量和 eval 覆盖，但没有把 dataset version、dedupe policy、included/skipped source 和 corpus size 带进项目级模型说明。

本版不新增独立报告，不训练模型，也不改变 model card 的 ready/review 判定。它只是让项目级 model card 在描述 run 时同时展示数据边界，避免 reviewer 把 loss 差异误读成纯模型能力差异。

## 运行链路

本次 smoke 构造两个 run：

- `raw`：dataset `demo-v356@raw`，dedupe policy `none`，included 3，skipped 0，chars 140。
- `deduped`：dataset `demo-v356@deduped`，dedupe policy `exact-source-content`，included 2，skipped 1，chars 90。

随后运行：

```powershell
python -B scripts/register_runs.py `
  runs/v356-model-card-dataset-snapshot-smoke/raw `
  runs/v356-model-card-dataset-snapshot-smoke/deduped `
  --name raw --name deduped `
  --out-dir runs/v356-model-card-dataset-snapshot-smoke/registry

python -B scripts/build_model_card.py `
  --registry runs/v356-model-card-dataset-snapshot-smoke/registry/registry.json `
  --out-dir runs/v356-model-card-dataset-snapshot-smoke/model-card
```

输出仍然是既有 model card 产物：

- `model_card.json`
- `model_card.md`
- `model_card.html`

## 截图说明

### `图片/01-model-card-dataset-snapshot.png`

截图证明 model card HTML 已经消费 dataset snapshot：

- 顶部 stat card 显示 `Datasets = 2`、`Snapshots = 2`、`Skipped sources = 1`。
- Top Runs 表新增 `Data Snapshot` 列。
- `deduped` 行显示 `demo-v356@deduped; dedupe=exact-source-content; included=2; skipped=1; chars=90`。
- `raw` 行显示 `demo-v356@raw; dedupe=none; included=3; skipped=0; chars=140`。
- 页面中部新增 `Dataset Dedupe Policies` 和 `Dataset Snapshot Summary`。
- Recommendations 提醒 review skipped dataset sources before treating loss deltas as model-only evidence。

这说明 model card 已经不是只讲模型/实验覆盖，也会把数据边界作为项目级审阅信息保留。

## 验证覆盖

本版验证覆盖：

- `tests.test_model_card`：检查 JSON summary、coverage、top run row、recommendations、Markdown 和 HTML 都包含 dataset snapshot 字段。
- smoke：真实生成两个 run、registry 和 model card。
- Playwright：打开 `model_card.html` 并截图归档。

## 一句话总结

v356 让 model card 从 registry 继承 dataset snapshot 证据，把项目级模型说明和数据边界审计接在一起。
