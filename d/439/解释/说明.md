# v439 baseline-candidate threshold boundary reuse diagnosis 运行说明

## 本版目标

v439 在 v438 live threshold boundary smoke 上做收口增强：新增 `--smoke-summary` 复用模式，并把 boundary review 结果转成机器可读诊断。

本版不重跑训练、不扩大模型、不改变候选是否晋升。它的重点是减少重复运行成本，并解释为什么 `status=pass` 的链路仍然给出 `review`。

## 运行命令

```powershell
python -B scripts\run_baseline_candidate_threshold_boundary_smoke.py --smoke-summary d\438\解释\baseline-candidate-threshold-boundary-smoke\tiny-scorecard-comparison-smoke\tiny_scorecard_comparison_smoke_summary.json --out-dir d\439\解释\baseline-candidate-threshold-boundary-reuse-diagnosis --thresholds 0:1:0.5 --force
```

这里的 `source_mode=reuse_summary` 说明 v439 没有重跑 tiny 训练，而是复用 v438 的真实 smoke summary。链路仍然 `pass`，但诊断明确写出候选在全部阈值下被拒绝，因此下一步不是强行晋升，而是强化候选或继续保持当前 baseline。

## 新增诊断语义

本版新增 `review_diagnosis`：

- `status`：诊断状态，可能是 `pass`、`review` 或 `fail`。
- `decision`：本轮诊断结论，例如 `candidate_not_accepted`。
- `issues`：机器可读问题列表。本轮为 `no_accepting_threshold`。
- `actions`：机器可读建议动作。本轮为 `increase_candidate_signal` 和 `keep_current_baseline`。

这让 v438 的“全拒绝”不再只是一个结果，而能被后续脚本或人工 review 直接消费。
