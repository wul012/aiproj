## 本版目标

v445 把 v444 的 CI boundary wrapper plan check 从 `ci_workflow_hygiene.json` 继续带到 project audit、release bundle 和 release readiness。它不新增治理链，也不改变模型评估结论；只让已有 CI 契约门的 ready 状态进入上游审计和发布证据。

## 关键证据

- `ci-workflow-hygiene/`
  - 当前 GitHub Actions hygiene 结果为 `status=pass`。
  - 包含 `baseline_candidate_threshold_boundary_gate_check_ready=True` 和 `baseline_candidate_threshold_boundary_gate_plan_check_ready=True`。
- `project-audit/`
  - 审计摘要和 `ci_workflow_context` 均保留 boundary gate check 与 boundary plan check ready 字段。
  - CLI 输出显示 `ci_baseline_candidate_threshold_boundary_gate_plan_check_ready=True`。
- `release-bundle/`
  - bundle summary/context 保留 `ci_workflow_baseline_candidate_threshold_boundary_gate_plan_check_ready=True`。
  - Markdown/HTML 摘要表可见 `CI boundary plan check=True`。
- `release-readiness/`
  - readiness summary 和 CI panel detail 显示 boundary gate plan check ready。
  - 本轮 controlled fixture 最终 `readiness_status=ready`、`decision=ship`。
