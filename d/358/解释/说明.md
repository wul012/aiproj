# v358 governance stabilization reasons 运行截图说明

## 本版目标

v358 是 v357 稳定评审的解释补强版。v357 已经把“暂停新增治理链、稳定/裁剪现有 7 条”落进 `maintenance_policy`；v358 继续保持不新增治理链，只让每条链写清楚为什么保留或观察，以及未来新需求应该如何并入现有链。

本版不训练模型，不新增发布门禁，不新增独立 dashboard。它只增强维护评审的可解释性，减少后续“想加新链但不知道该并到哪里”的判断成本。

## 运行链路

运行命令：

```powershell
python -B scripts/check_maintenance_batching.py `
  --out-dir runs/v358-governance-stabilization-reasons-smoke `
  --module-warning-lines 700 `
  --module-critical-lines 1200
```

新增输出仍然复用 v357 的产物：

- `governance_stabilization.json`
- `governance_stabilization.csv`
- `governance_stabilization.md`
- `governance_stabilization.html`

## 截图说明

### `图片/01-governance-stabilization-reasons.png`

截图证明 governance stabilization review 已经显示：

- `Missing rules = 0`。
- 表格新增 `Review Reason`。
- 表格新增 `Expansion Rule`。
- `training-promotion` 和 `maturity-portfolio` 仍然是 `watch`，并解释了重叠风险。
- Recommendations 仍然要求暂停新增治理链，先把新需求合并进现有链。

这说明暂停期不是空转，而是把“为什么保留/观察、未来怎么并入”的判断标准写进维护证据。

## 验证覆盖

本版验证覆盖：

- `tests.test_maintenance_policy`：检查默认 7 条链的 reason/rule 都完整。
- 缺少 reason/rule 的链会进入 `review`。
- Markdown/HTML/CSV/JSON 输出包含新增字段。
- CLI stdout 输出 missing reason/rule 计数。
- Playwright 打开 HTML 并截图归档。

## 一句话总结

v358 让治理链稳定评审从“列出 7 条链”升级为“解释为什么保留以及未来如何扩展”，继续服务短期收口。
