# v361 governance routing keyword hit 运行说明

本版目标是把治理 proposal 的 keyword 路由结果继续补全，记录命中的具体词，避免只看到 `keyword` 而不知道命中了哪条规则。

这一版不扩治理链，不新增路由类型，只补解释层。

## 这版做了什么

- `matched_keyword` 进入 proposal item。
- `keyword_hits` 进入 routing summary。
- Markdown、HTML 和 CLI stdout 同步显示。

## 这版不做什么

- 不增加新的治理链。
- 不改变 pause window 规则。
- 不调整 exact / review / reject 的决策顺序。

## 输出说明

- `governance_stabilization.json`：最终证据 JSON。
- `governance_stabilization.csv`：表格型证据。
- `governance_stabilization.md`：可读版说明。
- `governance_stabilization.html`：截图来源。

## 一句话总结

这一版把 keyword 路由从“分类正确”推进到“命中词可追溯”，更适合审计和后续解释。
