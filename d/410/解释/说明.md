# v410 project audit benchmark history suite design

## 本版主题

v410 把 benchmark history 中的 suite-design readiness 继续接入 project audit。目标是让 audit check、summary、Markdown/HTML 和 CLI stdout 都能看到 prompt suite design 是否仍有 non-comparison-ready entry。

## 关键改动

- `src/minigpt/project_audit_contexts.py`：benchmark history context 增加 suite-design not-ready 与 design-change 计数，check detail 也输出这些字段。
- `src/minigpt/project_audit.py`：audit summary 增加 `benchmark_history_suite_design_non_comparison_ready_entries` 和 `benchmark_history_design_comparison_changed_entries`，并把 suite-design not-ready 纳入 benchmark-history warning 判断。
- `src/minigpt/project_audit_artifacts.py`：Markdown/HTML 增加对应展示字段。
- `scripts/audit_project.py`：CLI stdout 打印新增计数。
- `tests/test_project_audit.py`：覆盖 ready 默认值、suite-design warning、rendered outputs、CLI stdout 和直接 context/check helper。

## 边界说明

本版不新增模型训练、不扩展 benchmark suite、不改变 release gate，只做已有 benchmark-history 字段在 project audit 层的消费和展示。
