# v345 运行截图说明：benchmark history project audit

本版目标是把 v343-v344 的 benchmark history ledger 接入 `project_audit`。它不训练模型、不新增 benchmark case，而是让项目审计在 release handoff 前可以直接看到历史评估证据的 ready/review/blocker、regression、model-quality-claim 和 boundary。

## 截图清单

- `01-focused-tests.png`：聚焦运行 `python -m unittest tests.test_project_audit`，验证 project audit 的 benchmark history pass、review warning、自动发现、context helper 和渲染字段。
- `02-project-audit-history-cli.png`：用最小 registry/model-card/request-history/benchmark-history/CI/coverage 输入运行 `scripts/audit_project.py`，stdout 显示 `benchmark_history_status=pass`、`benchmark_history_entries=1`、`benchmark_history_ready=1` 和 boundary。
- `03-full-verification-summary.png`：展示全量单测、source encoding、CI hygiene、coverage gate 等关键验证结果。
- `04-structure-and-docs-scan.png`：展示 v345 新增/修改文件、文档归档、`d/345` 结构和 git diff 摘要。
- `05-playwright-project-audit-history-html.png`：Playwright/Chrome 打开本版生成的 `project_audit.html`，证明 HTML 报告中能看到 benchmark history 摘要卡片。

## 关键验证命令

```powershell
python -m unittest tests.test_project_audit
python scripts\audit_project.py --registry runs\v345-project-audit-smoke-input\registry\registry.json --model-card runs\v345-project-audit-smoke-input\model-card\model_card.json --request-history-summary runs\v345-project-audit-smoke-input\request-history-summary\request_history_summary.json --benchmark-history runs\v345-project-audit-smoke-input\benchmark-history\benchmark_history.json --ci-workflow-hygiene runs\v345-project-audit-smoke-input\ci-workflow-hygiene\ci_workflow_hygiene.json --test-coverage-report runs\v345-project-audit-smoke-input\test-coverage\test_coverage_report.json --out-dir runs\v345-project-audit-smoke
python -m unittest discover -s tests
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v345
python -B scripts/check_ci_workflow_hygiene.py --out-dir runs/ci-workflow-hygiene-v345
python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-v345 --fail-under 80
```

## 证据边界

`benchmark_history.json` 是已有 benchmark comparison/decision 的历史账本，不是新的训练输出。v345 只把它接进 project audit：如果 history 缺失、没有 ready entry、有 review/blocker、存在 case regression、存在 generation-quality flag regression，或 `model_quality_claim=not_claimed`，audit 会进入 warning。

一句话总结：v345 让 benchmark history 从 maturity narrative 继续上升到 project audit，使历史评估证据在 release bundle 和 release gate 前就能被审计。
