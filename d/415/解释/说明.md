# v415 registry release readiness suite design deltas

## 本版主题

v415 把 v414 的 release-readiness comparison suite-design delta 接入 registry。目标是让注册表、leaderboard、HTML/CSV 和 CLI stdout 都能看到 suite-design not-ready 的比较变化，而不是只保留 benchmark status 和 readiness requirement。

## 关键改动

- `src/minigpt/registry_release_readiness.py`：delta row、summary 和 leaderboard 排序纳入 suite-design not-ready delta 与 design-comparison changed delta。
- `src/minigpt/registry_data.py`：RegisteredRun 新增 suite-design delta/regression 和 design-change delta 汇总字段。
- `src/minigpt/registry_artifacts.py`：registry CSV 增加对应字段。
- `src/minigpt/registry_render.py` 与 `registry_leaderboards.py`：HTML stat、run cell 和 release readiness delta leaderboard 展示 suite-design context。
- `scripts/register_runs.py`：stdout 打印新增 registry summary 计数。
- `tests/test_registry.py`：覆盖 run summary、registry aggregation、leaderboard、CSV/HTML 和 CLI stdout。

## 边界说明

本版不新增模型能力、不改变 readiness comparison 判定规则、不扩展新的治理链。registry 只消费已有 comparison delta，并提供索引、排序和可视化摘要。

## 一句话总结

v415 让 registry 也能索引 suite-design not-ready 的 readiness comparison 风险，把 prompt-suite 设计边界推进到运行注册表层。
