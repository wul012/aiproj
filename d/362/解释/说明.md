# v362 governance ambiguous keyword review

## 本版主题

v362 继续收口治理稳定化路线：当一个 governance proposal 的文本同时命中多个治理链关键词时，路由结果不再自动并入第一个命中的链，而是标记为 `ambiguous_keyword` 并进入 `review`。

这一版不新增治理链，不扩展新的报告体系，也不改变七条链的默认暂停策略。它只让已有 proposal routing 对跨链提案更保守。

## 关键改动

- `src/minigpt/maintenance_policy.py`
  - 关键词路由现在会收集所有命中的链和关键词。
  - 多链命中时写入 `match_basis=ambiguous_keyword`、`matched_keywords`、`matched_chains`。
  - ambiguous proposal 保留建议链，但 `route_decision` 改为 `review`。
- `src/minigpt/maintenance_policy_artifacts.py`
  - Markdown 和 HTML 增加 ambiguous keyword 的计数、命中词和命中链展示。
- `scripts/check_maintenance_batching.py`
  - stdout 增加 `governance_routing_ambiguous_keyword_match_count` 和 `governance_routing_ambiguous_keyword_hits`。
- `tests/test_maintenance_policy.py`
  - 新增 dataset/benchmark/promotion 跨链命中的断言，防止之后误回到自动 merge。

## 运行证据

- `01-governance-ambiguous-keyword-routing.png`
  - 证明 v362 smoke 中出现 `ambiguous_keyword_match_count=1`。
  - 同时证明普通 keyword 命中仍保留为 `registry`，跨链命中单独进入 `ambiguous_keyword_hits`。
  - HTML 报告表格展示 `Matched Keywords` 和 `Matched Chains`，可直接审计为什么该提案进入 review。

## Tag 含义

- `v362.0.0`
  - 表示 governance proposal routing 已能区分 exact、keyword、high-risk review、new-chain rejection 和 ambiguous multi-chain keyword review。

一句话总结：v362 让治理提案路由从“命中哪个词”推进到“同时像多个链时必须复核”，更符合暂停扩链阶段的保守治理原则。
