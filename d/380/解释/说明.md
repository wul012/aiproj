# v380 release readiness drift smoke comparison 运行说明

## 本版目标

v380 把 v379 暴露到 release readiness summary 的 drift-contract smoke ready 状态纳入 release readiness comparison。

本版解决的问题是：单个 readiness dashboard 已经能看到 `ci_workflow_release_readiness_drift_contract_smoke_ready`，但多版本对比还不会识别这个布尔值是否退化。现在如果 baseline 是 `True`，candidate 变成 `False` 或缺失，即使 CI 总状态没有变化，也会被计入 CI workflow regression。

本版不新增 CI step，不扩展到 registry/maturity 下游，也不改变 release readiness dashboard 自身的生成逻辑。

## 关键改动

- `src/minigpt/release_readiness_comparison.py`
  - row 新增 `ci_workflow_release_readiness_drift_contract_smoke_ready`。
  - delta 新增 baseline/compared/changed/regressed 四项 drift smoke ready 字段。
  - summary 新增 changed/regression count。
  - CI workflow regression 判断消费 `ready -> not ready` 退化。
- `src/minigpt/release_readiness_comparison_artifacts.py`
  - CSV、delta CSV、Markdown、HTML 展示新字段。
- `scripts/compare_release_readiness.py`
  - stdout 打印 drift smoke ready changed/regression count。
- `tests/test_release_readiness_comparison.py`
  - 覆盖状态稳定但 drift smoke ready 退化的回归场景。
  - 覆盖 artifact 字段输出。
