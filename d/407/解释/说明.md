# v407 运行截图和解释

## 本版主题

v407 将 suite-design readiness 接入 benchmark scorecard promotion decision。现在候选 scorecard 即使 rubric 和 generation-quality 都干净，只要 prompt suite design comparison 不是 `pass`，也会进入 review，并生成独立 remediation。

## 关键改动

- `src/minigpt/benchmark_scorecard_decision.py`：读取 scorecard comparison run 中的 suite-design 字段，新增 `suite_design_not_ready` review 类别和 remediation action。
- `src/minigpt/benchmark_scorecard_decision_artifacts.py`：CSV、Markdown、HTML 输出 design comparison 状态和 design review 统计。
- `tests/test_benchmark_scorecard_decision.py`：新增 clean candidate 但 suite-design warn 的测试，证明该候选会 review 而不是 promote。
- `README.md`、`代码讲解记录_训练治理阶段/421-v407-benchmark-decision-suite-design-remediation.md`：记录本版边界和证据链角色。

## 边界

本版不训练模型，不修改 scorecard 分数算法，也不把 suite-design warning 直接升成 blocker。它只让最终 promotion decision 知道：如果题集设计不适合比较，selected scorecard 不能作为 clean model-quality evidence。

## 一句话总结

v407 给 benchmark promotion decision 加上 prompt-suite design 审查，补齐从 eval suite 到最终决策的输入质量闭环。
