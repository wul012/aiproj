# v414 release readiness comparison benchmark history suite design

## 本版主题

v414 把 v413 进入 release readiness 的 benchmark-history suite-design 字段继续接到 comparison 层。目标是让 readiness comparison 能直接显示 suite-design not-ready 的增量、设计变更增量，以及它们对 comparison recommendation 的影响。

## 关键改动

- `src/minigpt/release_readiness_comparison.py`：row/delta/sumary 增加 suite-design not-ready 与 design-comparison changed 字段，并把 suite-design not-ready 增量纳入 benchmark-history regression 判断。
- `src/minigpt/release_readiness_comparison_artifacts.py`：CSV、delta CSV、Markdown、HTML 增加对应字段与列。
- `scripts/compare_release_readiness.py`：CLI stdout 打印新增 comparison 计数和每行 suite-design 字段。
- `tests/test_release_readiness_comparison.py`：覆盖 suite-design regression、rendered outputs 和 CLI stdout。

## 边界说明

本版不重新定义 suite-design 质量，也不扩展 comparison 维度到新的评审链路，只把 readiness 里的 suite-design 风险差异化呈现出来，避免 comparison 只盯着 status 却忽略设计边界。

## 一句话总结

v414 让 release readiness comparison 也能看见 suite-design not-ready 的变化，把 prompt-suite 设计边界完整带到对比层。
