# v469 运行截图和解释：promoted seed handoff archived reason gate

## 本版主题

v469 继续沿用 v467-v468 的收口思路：`archived_path_portability_check_not_ready` 不新增专用字段链，而是作为 `*_ci_regression_reason_counts` 的普通成员进入最终 promoted seed handoff。

本版重点验证两个场景：

- 普通 handoff summary / report 能保留 handoff-level archived-path reason。
- `--require-clean-batch-review` 下，selected handoff 的 archived-path reason 会进入 clean-batch requirement，并让 automation gate 停止。

## 验证边界

本版证明 archived-path CI regression reason 能进入最终 handoff 输出和 clean-batch automation gate。

本版不证明模型质量提升，也不改变 automation gate 的规则。它只是让已有 gate 在遇到 archived-path reason 时能保留完整原因证据，并给出明确 `stop` 决策。

## 一句话总结

v469 把 archived-path regression reason 从 seed 层送到最终 handoff / automation gate 层，让通用 CI reason-count 成为完整交接证据，而不是停留在中间报告里。
