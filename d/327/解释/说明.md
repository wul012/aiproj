# v327 运行截图说明

## 本版主题

v327 在 v326 failure taxonomy 之上增加 remediation plan：

```text
benchmark_scorecard_decision.json
  -> remediation_plan[]
tiny_scorecard_comparison_smoke_summary
  -> decision_remediation_count
  -> decision_first_remediation_category
  -> decision_first_remediation_action
```

本版边界：不改训练、不改 benchmark score、不改 promotion 判定；只是把已有失败类别映射为结构化处理建议。

## 关键结论

- v327 把 v326 的失败分类升级为“分类 -> 处理动作”的证据层。
- remediation plan 只来自 summary 的 blocker/review category counts，不改变 candidate 是否 blocked/review/promote。
- 空 plan 不渲染表格，干净 promote 场景保持报告简洁。
- 真实 tiny smoke 里 threshold blocker 被映射为提升 rubric score 或显式调整 policy 的处理建议。

## Tag 含义

`v327.0.0` 表示 benchmark scorecard promotion decision 已经能输出结构化 remediation plan，并让 tiny smoke 顶层 summary 转述第一条处理建议。

一句话总结：v327 把 failure taxonomy 从“知道哪里失败”推进到“知道下一步处理什么”。
