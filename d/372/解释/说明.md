# v372 运行截图与解释

## 本版主题

v372 是一次维护型收口版本，目标是把已经接近维护压力线的报告与证据模块拆成更清楚的责任边界。它不改变训练逻辑、报告 schema、CLI 参数或公开导入路径，只降低后续继续开发时的单文件阅读和测试定位成本。

## 关键改动

- `src/minigpt/release_bundle.py` 收束为 release bundle 的编排入口。
- `src/minigpt/release_bundle_support.py` 承接 JSON 读取、路径解析、summary、release status 和 artifact collection。
- `src/minigpt/release_bundle_contexts.py` 承接 request history、benchmark history、CI workflow、coverage、audit checks 和 recommendations 的上下文组装。
- `src/minigpt/maintenance_policy_artifacts.py` 保留维护策略 artifact 总入口。
- `src/minigpt/maintenance_policy_governance_artifacts.py` 承接 governance stabilization 的 JSON/CSV/Markdown/HTML 产物写入。
- `src/minigpt/promoted_training_scale_seed_handoff_artifacts.py` 保留 seed handoff artifact 写入与兼容导出。
- `src/minigpt/promoted_training_scale_seed_handoff_sections.py` 承接 seed handoff Markdown/HTML section 渲染。

## tag 含义

`v372.0.0` 表示 MiniGPT 完成一次 maintenance split consolidation：把 release bundle、governance stabilization 和 promoted seed handoff 三条高压 artifact/report 线路拆到更专注的模块里，同时保持旧入口和输出契约不变。

一句话总结：v372 没有扩展新功能，而是把后续继续做功能所依赖的维护边界垫稳。
