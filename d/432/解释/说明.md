# v432 运行截图说明：baseline-candidate eval loop

## 本版目标

v432 把前几次本地草稿合并成一个完整版本：在已有 tiny scorecard comparison smoke 之上增加 baseline-candidate eval loop。它不新建另一条治理链，而是把现有 baseline/candidate smoke、scorecard comparison、decision、benchmark history 和 promotion decision 收束成一个可复用的实验入口。

本版重点解决三个问题：

- 固定 baseline/candidate 后，报告里要能直接看到控制变量、分数差、控制检查和接收条件。
- 下游 scorecard 给出 `promote` 时，本层仍要能按更严格的 `min_overall_score_delta` 拒绝候选。
- 同一个脚本既能用于实验探索，也能通过 `--fail-on-reject` 变成严格晋升 gate。

本版不宣称模型能力提升。这里的 tiny run 仍是 CPU smoke 和 benchmark plumbing evidence，用来证明评估闭环、报告契约和 gate 行为。

## 命令摘要

本版核心运行命令：

```text
python -B scripts\run_baseline_candidate_eval_loop.py --out-dir d\432\解释\baseline-candidate-eval-loop --decision-min-rubric-score 0 --min-overall-score-delta 1 --fail-on-reject --force
```

该命令按设计返回 `exit_code=2`，因为 tiny candidate 虽然通过下游 promotion decision，但总体分数差为 `0.0`，没有达到 v432 本轮设定的 `--min-overall-score-delta 1`。
