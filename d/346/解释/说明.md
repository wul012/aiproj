# v346 运行截图说明：benchmark history release bundle

本版目标是把 `benchmark_history.json` 和 project audit 中的 benchmark history context 带入 release bundle。它不训练模型、不生成新的 benchmark，只保证历史评估证据进入 release handoff 包。

## 截图清单

- `01-focused-tests.png`：聚焦运行 `python -m unittest tests.test_release_bundle`，验证 release bundle 的 benchmark history pass、audit-only warning context、artifact、Markdown/HTML 字段。
- `02-release-bundle-history-cli.png`：用最小 registry/model-card/audit/benchmark-history/CI/coverage 输入运行 `scripts/build_release_bundle.py`，stdout 显示 `benchmark_history_status=pass`、`benchmark_history_entries=1`、`benchmark_history_ready=1` 和 boundary。
- `03-full-verification-summary.png`：展示全量单测、source encoding、CI hygiene、coverage gate 等关键验证结果。
- `04-structure-and-docs-scan.png`：展示 v346 新增/修改文件、文档归档、`d/346` 结构和 diff 摘要。
- `05-playwright-release-bundle-history-html.png`：Playwright/Chrome 打开本版生成的 `release_bundle.html`，证明 HTML handoff 报告中能看到 benchmark history 摘要卡片。

## 关键验证命令

```powershell
python -m unittest tests.test_release_bundle
python scripts\build_release_bundle.py --registry runs\v346-release-bundle-smoke-input\registry\registry.json --model-card runs\v346-release-bundle-smoke-input\model-card\model_card.json --audit runs\v346-release-bundle-smoke-input\audit\project_audit.json --benchmark-history runs\v346-release-bundle-smoke-input\benchmark-history\benchmark_history.json --ci-workflow-hygiene runs\v346-release-bundle-smoke-input\ci-workflow-hygiene\ci_workflow_hygiene.json --test-coverage-report runs\v346-release-bundle-smoke-input\test-coverage\test_coverage_report.json --out-dir runs\v346-release-bundle-smoke
python -m unittest discover -s tests
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v346
python -B scripts/check_ci_workflow_hygiene.py --out-dir runs/ci-workflow-hygiene-v346
python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-v346 --fail-under 80
```

## 证据边界

`benchmark_history.json` 是已有 benchmark comparison/decision 的历史账本。v346 不把它变成模型质量证明，只把它带入 release bundle。如果 audit context 标记 `model_quality_claim=not_claimed` 或 latest boundary 是 `tiny-smoke-plumbing-evidence`，release bundle 会保留这个边界。

一句话总结：v346 让 benchmark history 从 project audit 继续进入 release bundle，使版本交付包能保留历史评估证据。
