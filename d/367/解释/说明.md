# v367 运行截图说明

本版验证 `release_bundle` 能消费 benchmark history 的 `readiness_requirement` 字段。

## 截图

- `图片/01-release-bundle-readiness-requirement-review.png`：release bundle HTML 截图。样例里 project audit 仍是 `pass`，但 standalone benchmark history 的 `readiness_requirement.status=fail`、`exit_code=1`，所以 release bundle 将 `benchmark_history_status` 归为 `warn`，并把 `release_status` 降为 `review-needed`。

## 运行证据

- 生成命令：`python -B scripts\build_release_bundle.py --registry runs\__tmp_release_bundle_v367\registry\registry.json --model-card runs\__tmp_release_bundle_v367\model-card\model_card.json --audit runs\__tmp_release_bundle_v367\audit\project_audit.json --benchmark-history runs\__tmp_release_bundle_v367\benchmark-history\benchmark_history.json --out-dir runs\v367-release-bundle-readiness-review --release-name v367-demo`
- 关键 stdout：
  - `release_status=review-needed`
  - `audit_status=pass`
  - `benchmark_history_status=warn`
  - `benchmark_history_ready=1`
  - `benchmark_history_readiness_requirement_status=fail`
  - `benchmark_history_readiness_requirement_exit_code=1`
  - `benchmark_history_readiness_requirement_failed_reasons=["insufficient_ready_entries"]`

## 边界

这份证据说明 release bundle 不再只相信旧 audit summary。它会读取独立 benchmark history 的最新 readiness requirement，并采用更保守的 release review 判定。
