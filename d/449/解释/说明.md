# v449 运行截图和解释

## 本版主题

v449 把 promoted seed handoff receipt 的 schema-v4 boundary plan-check 字段继续推到 receipt contract summary。v448 已经让 receipt、embedded receipt check 和 handoff assurance 能识别这些字段；v449 解决的是更上层 contract summary 还只表达 schema-v3 suite-design contract 的断点。

本版不新增新治理链，不重新训练模型，也不改变 promoted baseline 选择逻辑。它只把现有 handoff assurance 中的 v4 字段摊平成 contract summary、summary-check 和 assurance smoke 都能复核的字段。

## 运行证据

- `contract-summary/`
  - 输入为 `d/448/解释/promoted-handoff`。
  - 输出 `promoted_training_scale_seed_handoff_receipt_contract_summary.*`。
  - JSON 中 `schema_v4_ready=true`，`ci_boundary_plan_check_scopes` 包含 selected、handoff、comparison_ready 三个 scope。
- `contract-summary-check/`
  - 重新读取 contract summary，再从 source handoff 重建 expected summary。
  - `status=pass`，证明 v4 boundary scope 字段和 HTML/Markdown/text sidecar 均可复核。
- `assurance-smoke/`
  - 重新运行 promoted seed handoff assurance smoke。
  - stdout 中包含 `receipt_contract_schema_v4_ready=True` 和 handoff boundary plan-check count。
- `playwright_contract_summary_snapshot.md`
  - Playwright MCP 对 contract summary HTML 的可访问性快照。

## 关键结论

v449 证明 receipt schema-v4 的 boundary plan-check 字段已经从原始 receipt/check/assurance 层继续进入 contract summary 层。以后只看 contract summary，也能知道 receipt v4 是否 ready，以及 boundary plan-check count 是否满足 selected 不超过 handoff 的基本约束。
