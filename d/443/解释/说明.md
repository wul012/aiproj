# v443 CI baseline-candidate threshold boundary wrapper plan

## 本版目标和边界

v443 收口 v442 的 CI 接入方式：v442 已经把 baseline-candidate threshold boundary expected-exit gate check 放进 GitHub Actions，但 workflow 里直接写了很长的命令和固定参数。v443 将这条 CI 命令封装成稳定 wrapper，并记录 invocation plan 与 artifact digest。

本版不改变候选接受规则，不改变 v441 expected-exit gate check 的语义，也不新增模型能力结论。当前真实结果仍是 `candidate_not_accepted`，wrapper 只是让 CI 调用更短、更可测试、更容易审计。
