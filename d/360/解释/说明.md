# v360 governance routing basis 运行截图说明

## 本版目标

v360 是 v359 governance proposal routing 的解释增强版。v359 已经把新提案路由到现有 7 条治理链，并区分 `merge_existing`、`review_before_merge`、`reject_new_chain_during_pause`；v360 不改这套边界，只把“路由是怎么命中的”写清楚，补上 exact match 和 keyword match 的证据。

本版解决的问题是：路由结果之外，评审者还需要知道这条提案到底是靠链 id 命中，还是靠关键词命中。

本版不新增治理链，不改变暂停窗口，不引入新的门禁层。

## 运行链路

运行命令：

```powershell
python -B scripts/check_maintenance_batching.py `
  --skip-module-pressure `
  --governance-proposals runs/v360-governance-routing-basis-smoke/proposals.json `
  --out-dir runs/v360-governance-routing-basis-smoke
```

新增输出仍然复用治理稳定评审产物：

- `governance_stabilization.json`
- `governance_stabilization.csv`
- `governance_stabilization.md`
- `governance_stabilization.html`

## 截图说明

### `图片/01-governance-routing-basis.png`

截图证明 governance stabilization review 现在不仅显示 routing decision，还显示：

- `Exact match`
- `Keyword match`
- `Match basis`

这样可以直接看出：

- 哪些提案是精确落到 `training-promotion` 这类链上的。

## 验证覆盖

本版验证覆盖：

- `tests.test_maintenance_policy`：检查 exact match、keyword match、高风险 review、未匹配 rejection。
- Markdown/HTML/CLI 输出包含 match basis 统计。
- Playwright 打开 HTML 并截图归档。

## 一句话总结

v360 让治理提案路由不仅有结果，还有依据，方便后续继续收口而不是盲目扩张。
