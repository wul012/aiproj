# v436 baseline-candidate threshold matrix 运行说明

## 本版主题

v436 在 v432-v435 的 baseline-candidate 链路上补了一层 threshold matrix。它使用同一份 v432 tiny scorecard comparison smoke summary，分别以 `min_overall_score_delta=0` 和 `min_overall_score_delta=1` 重建 eval loop、handoff 和 handoff check。

本版不新增训练，不声称模型质量提升。它证明的是阈值边界行为：同一份候选在宽松阈值下能进入 `accept_candidate`，在严格阈值下会保持 `reject_candidate`，并且两条 handoff 的 contract check 都能通过。

## 关键产物

- `baseline-candidate-threshold-matrix/baseline_candidate_threshold_matrix.json`：矩阵主报告。
- `baseline-candidate-threshold-matrix/baseline_candidate_threshold_matrix.txt`：命令行可读摘要。
- `baseline-candidate-threshold-matrix/baseline_candidate_threshold_matrix.md`：Markdown 归档摘要。
- `baseline-candidate-threshold-matrix/baseline_candidate_threshold_matrix.html`：Playwright MCP 截图使用的页面。
- `threshold-0/loop|handoff|handoff-check/`：阈值 `0` 下的接受路径证据。
- `threshold-1/loop|handoff|handoff-check/`：阈值 `1` 下的拒绝路径证据。

## 运行命令

```powershell
python -B scripts\run_baseline_candidate_threshold_matrix.py d\432\解释\baseline-candidate-eval-loop\tiny-scorecard-comparison-smoke --out-dir d\436\解释\baseline-candidate-threshold-matrix --thresholds 0,1 --require-both-outcomes --force
```

命令输出归档：

- `baseline_candidate_threshold_matrix_stdout.txt`
- `baseline_candidate_threshold_matrix_exit.txt`

退出码为 `0`。核心结果：

## 边界说明

v436 的价值是阈值边界复核和 handoff contract 覆盖，不是更强模型。真实 v432 tiny smoke 仍属于 CPU 小样本流水线证据，`boundary.model_quality_claim` 明确为 `not_claimed`。

一句话总结：v436 让 baseline-candidate 链路能用同一份真实 smoke 证据同时证明接受路径、拒绝路径和 handoff check 的一致性。
