# v378 release readiness drift contract CI smoke 运行说明

## 本版目标

v378 把 v377 的 release readiness drift contract 检查器接进 CI 质量门禁。

本版解决的问题是：v377 已经能校验 `release_readiness_comparison.json`，但如果 CI 没有稳定输入，这个检查器仍可能只停留在手工工具层。v378 新增一个最小 mixed-drift smoke fixture，让 CI 每次都能验证检查器的 pass 路径、sidecar 输出和 mixed-drift 契约。

本版不重新跑完整 release readiness comparison，不新增 drift 语义，也不扩大模型质量声明。

## 关键改动

- `scripts/check_release_readiness_drift_contract_smoke.py`
  - 写出最小 `release_readiness_comparison.json`。
  - fixture 包含 baseline reason removal + compared reason addition，因此 drift status 应为 `mixed`。
  - 调用 `scripts/check_release_readiness_drift_contract.py` 生成 JSON/TXT 检查产物。
  - 汇总 smoke summary JSON/TXT，记录 checker return code、mixed count 和 sidecar 路径。
- `.github/workflows/ci.yml`
  - 新增 `Release readiness drift contract smoke` step。
  - 该 step 位于覆盖率测试前，避免 coverage 才发现契约 drift。
- `src/minigpt/ci_workflow_hygiene.py`
  - required command 新增 drift-contract smoke。
  - required order 新增 smoke-before-coverage 检查。
- `tests/test_release_readiness_drift_contract_smoke.py`
  - 验证 smoke summary、comparison fixture、checker JSON/TXT sidecar 全部生成。
- `tests/test_ci_workflow.py`
  - 验证当前 workflow 包含新 step。
  - 验证 CI hygiene 报告能守住新 step 的位置。

## Tag 说明

`v378.0.0` 表示本版把 release readiness failed-reason drift contract 从手工检查器推进到 CI smoke：每次 CI 都能用稳定 mixed-drift fixture 验证检查器仍然可用。

一句话总结：v378 让 failed-reason drift 契约进入自动守门层，减少未来字段迁移后“检查器存在但无人运行”的风险。
