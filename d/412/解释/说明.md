# v412 release gate benchmark history suite design

## 本版主题

v412 把 benchmark-history suite-design readiness 接入 release gate。目标是让 gate summary、benchmark-history gate detail、Markdown/HTML 和 CLI stdout 都保留 prompt suite design 是否 non-comparison-ready 的计数。

## 关键改动

- `src/minigpt/release_gate_benchmark.py`：benchmark history gate detail 输出 suite-design not-ready 和 design-change 计数，并把 suite-design not-ready 纳入 warning 判断。
- `src/minigpt/release_gate.py`：gate summary 增加两个 suite-design 字段。
- `src/minigpt/release_gate_artifacts.py`：Markdown/HTML 增加对应展示字段。
- `scripts/check_release_gate.py`：CLI stdout 打印新增计数。
- `tests/test_release_gate.py`：覆盖 ready 默认值、suite-design warning、rendered outputs、CLI stdout 和 `fail_on_warn`。

## 边界说明

本版不新增模型训练、不扩展 benchmark suite、不改变 release gate profile，只做已有 benchmark-history 字段在 release gate 层的消费和展示。

## 一句话总结

v412 让 release gate 不再丢失 benchmark-history suite-design readiness，把提示词题集设计边界保留到最终 gate 审批前。
