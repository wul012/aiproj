# v451 运行截图和解释

## 本版主题

v451 给 promoted seed handoff receipt contract summary-check 增加结构化比较结果。v450 已经让 contract summary 自身输出 `contract_checks`；v451 继续收口，让检查 summary 的那一层也不只输出自然语言 `issues`。

本版不新增新治理链，不改变 receipt schema，也不改变 contract summary 的业务字段。它只让 summary-check 的 JSON/text/Markdown/HTML 输出包含机器可读的字段比较和 sidecar 比较。

## 运行证据

- `contract-summary/`
  - 输入为 `d/448/解释/promoted-handoff`。
  - 生成当前 contract summary，用作 summary-check 的实际检查对象。
- `contract-summary-check/`
  - 输出 `promoted_training_scale_seed_handoff_receipt_contract_summary_check.*`。
  - JSON 中新增 `summary_field_checks`、`summary_field_check_count=21`、`failed_summary_field_check_count=0`。
  - JSON 中新增 `sidecar_checks`、`sidecar_check_count=3`、`failed_sidecar_check_count=0`。
  - Sidecar checks 对 text、Markdown、HTML 记录 expected/actual SHA-256。
- `contract_summary_check_stdout.txt`
  - CLI 输出中展示结构化 field checks 和 sidecar checks。
- `playwright_summary_check_structured_snapshot.md`
  - Playwright MCP 对 summary-check HTML 的可访问性快照。

## 关键结论

v451 证明 summary-check 不再只是报一串 issue 字符串。它现在能告诉自动化系统：哪些 summary 字段被比较、哪些 sidecar 被比较、每项 pass/fail、以及 sidecar 的 digest 是否一致。
