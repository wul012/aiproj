# v470 运行截图和解释：receipt schema v5 CI reason contract

## 本版主题

v470 把 promoted seed handoff automation receipt 从 schema v4 升级到 schema v5，让 CI regression 的 reason-count map 不只停留在 handoff summary，而是进入 receipt、receipt check、embedded receipt check、handoff assurance、contract summary 和 contract summary check。

本版不改变 promoted seed handoff 的选择逻辑，不扩大训练规模，也不新增一条治理链。它只是把 v467-v469 已经确认可传递的 `archived_path_portability_check_not_ready` 原因，放进最终 receipt contract 可复核的字段里。

## 关键运行证据

- `d/470/解释/source/promoted-seed/promoted_training_scale_seed.json`
  - 从 v469 的 seed/source 复制并改写为 v470 路径。
  - selected handoff 带有 `archived_path_portability_check_not_ready: 1`。
  - aggregate handoff 同时带有 `archived_path_portability_check_not_ready: 1` 和 `workflow-order-regressed: 1`。

## 验证边界

本版证明 receipt schema v5 能把 CI regression reason-count map 贯穿最终交接和 contract summary。它不证明模型质量提升，也不证明真实生产训练能力提升。

同时，本版保留了旧 schema-v4 归档的只读可验证性：旧 sidecar 不会因为缺少 v5 reason map 行而被强行判坏；新生成的 v470 evidence 则必须显示 `schema_v5_ready=True`。

## 一句话总结

v470 把 CI regression reason-count 从“报告里看得到”推进到“receipt contract 里能复核”，让 promoted seed handoff 的最终自动化交接有了 schema-v5 级别的原因证据。
