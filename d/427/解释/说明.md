# v427 运行截图说明：promoted seed handoff receipt contract hardening

## 本版目标

v427 在 v426 schema-v3 receipt contract 之后做收口加固：assurance smoke 不只检查 suite-design regression count，还检查 selected/global/comparison-ready 三层 suite-design regression names；同时新增篡改测试，证明 receipt JSON 或 receipt-check JSON 里的 suite-design count/name 变旧时，embedded receipt check 会失败。

本版不执行新训练，不扩大模型能力声明，也不新增治理链。它只加固 promoted seed handoff receipt/sidecar 的一致性证明。

## 命令摘要

本轮使用两条证据链：

```text
python -B scripts\check_promoted_seed_handoff_assurance_smoke.py --out-dir runs\v427-assurance-smoke
python -m pytest tests\test_promoted_training_scale_seed_handoff_receipt_suite_design.py tests\test_promoted_training_scale_seed_handoff_receipt.py -q
```

第一条链路证明干净 handoff 的 receipt/check/embedded/assurance 可以重算通过；第二条链路证明 suite-design receipt sidecar 被篡改后会被拒绝。

## 证据边界

`d/427` 中的 HTML、snapshot、PNG、smoke summary 和 tamper summary 是本版最终运行证据；`assurance-smoke/` 和 `tamper-evidence/` 是生成这些最终证据所需的可复核 sidecar 输入。它们证明 schema-v3 receipt 合同没有漏掉 suite-design names，也证明 sidecar 发生 drift 时能被检测出来；它们不证明模型质量提升。

一句话总结：v427 把 schema-v3 promoted seed handoff receipt 从“字段存在并能展示”推进到“字段名和值都能被 smoke 和篡改测试保护”的合同硬化层。
