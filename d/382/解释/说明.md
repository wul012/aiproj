# v382 registry release readiness CI reasons 运行说明

## 本版目标

v382 把 v381 的 release readiness CI workflow regression reasons 带入 registry。

本版解决的问题是：comparison 层已经能说明 CI regression 原因，但 registry 层汇总多个 run 时还只显示总数。现在 registry delta row、summary、CSV、HTML、leaderboard 和 CLI 都能展示 reason list/count。

本版不新增 report 类型，不扩散到 maturity/narrative，只在 registry 现有 release-readiness delta 链路中轻量透传。

## 关键改动

- `src/minigpt/registry_release_readiness.py`
  - delta row 新增 `ci_workflow_regression_reasons`。
  - summary 新增 `ci_workflow_regression_reasons` 和 `ci_workflow_regression_reason_counts`。
  - 兼容旧 artifact：没有 reason list 时从 failed/order/status/drift-smoke 字段推导。
- `src/minigpt/registry_data.py`
  - `RegisteredRun` 新增 `release_readiness_ci_workflow_regression_reasons`。
- `src/minigpt/registry_artifacts.py`
  - registry CSV 新增 reason 列。
- `src/minigpt/registry_render.py`
  - HTML stats 和 Release Readiness 单元格展示 reason。
- `src/minigpt/registry_leaderboards.py`
  - Release Readiness Deltas leaderboard 的 CI workflow 列展示 reason。
- `scripts/register_runs.py`
  - stdout 打印 `release_readiness_ci_regression_reason_counts`。
