# v373 运行截图与解释

## 本版主题

v373 补的是 benchmark-history readiness requirement 的“失败原因漂移”信号。此前成熟度层已经能看到 requirement status change 和 exit-code delta，但如果状态仍是 `fail`、exit code 仍是 `1`，只是新增了一个 failed reason，成熟度层会缺少明确的变化提示。

本版不新增治理链，不改变训练行为，也不改变已有命令入口；它只把已有 release readiness comparison 里的 baseline/compared failed reasons 变成可聚合、可展示、可触发 review 的字段。

## 关键改动

- `src/minigpt/release_readiness_comparison.py`
  - 计算 `benchmark_history_readiness_requirement_failed_reason_added_count`。
  - 计算 `benchmark_history_readiness_requirement_failed_reason_removed_count`。
  - 输出 added/removed failed reason 列表。
  - reason-only addition 也会成为 benchmark history delta/regression。

- `src/minigpt/release_readiness_comparison_artifacts.py`
  - CSV、Markdown、HTML 输出新增 failed reason additions。

- `src/minigpt/registry_rankings.py`
  - Registry delta row、summary、leaderboard 承接 failed reason additions。
  - 如果旧 comparison JSON 没有新增字段，也能从 baseline/compared reason list 推导。

- `src/minigpt/maturity.py`
  - Maturity summary 增加 failed reason addition count/list。
  - 新增 failed reason 会把 release readiness trend 识别为 `benchmark-regressed`。

- `src/minigpt/maturity_narrative_summary.py`
  - Maturity narrative 汇总新增 failed reason additions。
  - 组合评审时把 reason-only drift 作为 review 触发条件。

- `src/minigpt/maturity_narrative_sections.py`
  - Release Quality Trend claim 中展示新增 failed reasons。

- `scripts/build_maturity_summary.py`
- `scripts/build_maturity_narrative.py`
  - CLI 输出新增 failed reason addition diagnostics。


测试结果：`51 passed`。

## tag 含义

`v373.0.0` 表示 MiniGPT 的 release readiness 到 maturity narrative 链路能识别 benchmark readiness failed reason drift：即使 status 和 exit code 没变，只要失败原因新增，也会进入 review。

一句话总结：v373 让成熟度评审从“只看状态变化”推进到“也看失败原因变化”。
