# v411 release bundle benchmark history suite design

## 本版主题

v411 把 benchmark-history suite-design readiness 接入 release bundle。目标是让 bundle summary、context、Markdown/HTML 和 CLI stdout 都保留 prompt suite design 是否 non-comparison-ready 的计数。

## 关键改动

- `src/minigpt/release_bundle_contexts.py`：benchmark history context 增加 suite-design not-ready 与 design-change 计数，并把 suite-design not-ready 纳入 warning 判断。
- `src/minigpt/release_bundle_support.py`：bundle summary 增加两个 suite-design 字段，并让当前 benchmark history/context 优先于旧 audit summary。
- `src/minigpt/release_bundle_artifacts.py`：Markdown/HTML 增加对应展示字段。
- `scripts/build_release_bundle.py`：CLI stdout 打印新增计数。
- `tests/test_release_bundle.py`：覆盖 ready 默认值、stale audit 被新 history 覆盖、rendered outputs、CLI stdout 和 release status downgrade。

## 边界说明

本版不新增模型训练、不扩展 benchmark suite、不改变 release gate 策略，只做已有 benchmark-history 字段在 release bundle 层的消费和展示。

## 一句话总结

v411 让 release bundle 不再丢失 benchmark-history suite-design readiness，把提示词题集设计边界保留到发布证据包里。
