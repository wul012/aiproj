# v421 运行截图说明：training scale promotion index suite-design guard

## 本版目标

v421 把 v420 promotion 中的 suite-design regression context 带入 training scale promotion index。目标是让 index 聚合、compare-input 过滤、CSV/Markdown/HTML 和 CLI 输出都能识别 `handoff_batch_maturity_suite_design_regression_count`，避免带有 benchmark-suite blocker 的 clean-required promotion 进入对比输入。

本版不执行训练，不改变单个 promotion 的生成逻辑，也不新增新的治理报告；只让已有 promotion index 消费 v420 已经写出的字段。

## 命令摘要

本轮用两个 promotion fixture 生成 index HTML：一个 clean promotion，一个 status 为 clean 但带 suite-design regression 的 clean-required promotion。

```powershell
python scripts\index_training_scale_promotions.py `
  runs\v421-training-scale-promotion-index-suite-design\clean\promotion `
  runs\v421-training-scale-promotion-index-suite-design\suite-regressed\promotion `
  --name clean-run --name suite-run
```

关键输出：

```text
comparison_ready_count=1
handoff_unclean_batch_review_count=1
handoff_batch_maturity_suite_design_regression_count=1
handoff_batch_maturity_suite_design_regression_names=["suite-index-risk"]
```
