# v454 运行截图和解释

## 本版主题

v454 给 promoted seed handoff receipt contract summary 增加 contract-check 类型聚合。v453 已经让每个 `contract_checks` row 自描述；v454 让 summary 顶层直接回答“每类检查有多少、通过多少、失败多少、覆盖哪些 target”。

本版不新增治理链，不改变 receipt contract 的 pass/fail 判断，也不改变 CLI 的默认行为。它只是把已有 `contract_checks` 明细进一步汇总，方便 CI、release gate、registry 或人工报告快速读取。

## 运行证据

- `contract-summary/`
  - 输入为 `d/448/解释/promoted-handoff`。
  - 生成 v454 的 promoted seed handoff receipt contract summary。
  - `contract_check_status_counts={"pass": 10, "fail": 0}`。
  - `contract_check_type_summary` 包含 `count_consistency`、`schema_readiness`、`selected_within_handoff`、`status_equals` 四类。
- `contract-summary-check/`
  - 对 v454 contract summary 做重建式 summary-check。
  - 新增 `contract_check_status_counts` 与 `contract_check_type_summary` 的字段比较。
  - `summary_field_check_count=23`，`failed_summary_field_check_count=0`。
  - `sidecar_check_count=3`，`failed_sidecar_check_count=0`。
- `contract_summary_stdout.txt`
  - CLI 输出中可以看到 `receipt_contract_check_status_counts` 与 `receipt_contract_check_type_summary`。
- `playwright_contract_check_type_summary_snapshot.md`
  - Playwright MCP 对 contract summary HTML 的可访问性快照。

## 关键结论

v454 让 contract summary 不只保留明细 rows，也提供轻量 profile。后续模块不必遍历 `contract_checks` 才能判断哪类检查失败；读取 `contract_check_type_summary` 就能直接定位风险集中在哪个 check type。
