# v347 运行截图说明：benchmark history release gate

本版目标是让 release gate 和 release gate profile comparison 消费 v346 release bundle 中已经携带的 benchmark history 证据。它不训练模型、不生成新 benchmark，只保证发布门禁能看见 history readiness、regression、quality claim 和 latest boundary。

## 截图清单

- `01-release-gate-benchmark-history-html.png`：Playwright/Chrome 打开 `gate_report.html`，证明 release gate summary 和 checks 能显示 benchmark history warning、entries、ready count 和 `tiny-smoke-plumbing-evidence` boundary。
- `02-release-gate-profile-benchmark-history-html.png`：Playwright/Chrome 打开 profile comparison HTML，证明 standard/legacy profile matrix 和 delta report 会携带 benchmark-history requirement 字段。
- `03-full-verification-summary.png`：汇总本版聚焦测试、release gate CLI smoke、profile comparison smoke、全量单测、source encoding、CI workflow hygiene 和 coverage gate 的验证结果。

## 关键验证命令

```powershell
python -m unittest tests.test_release_gate tests.test_release_gate_comparison
python scripts\check_release_gate.py --bundle runs\v347-release-gate-smoke-input\release-bundle\release_bundle.json --out-dir runs\v347-release-gate-smoke --policy-profile standard
python scripts\compare_release_gate_profiles.py --bundle runs\v347-release-gate-smoke-input\release-bundle\release_bundle.json --profiles standard legacy --out-dir runs\v347-release-gate-profile-smoke --baseline-profile standard
python -m unittest discover -s tests
python -B scripts/check_source_encoding.py --out-dir runs/source-encoding-hygiene-v347
python -B scripts/check_ci_workflow_hygiene.py --out-dir runs/ci-workflow-hygiene-v347
python -B scripts/run_test_coverage.py --out-dir runs/test-coverage-v347 --fail-under 80
```

## 证据边界

本版 smoke 故意使用 `benchmark_history_status=warn`、`ready=0`、`case_regressions=1`、`model_quality_claim=not_claimed` 和 `latest_boundary=tiny-smoke-plumbing-evidence`。预期结果不是 release blocked，而是 `needs-review`。

这说明 v347 保护的是发布门禁可见性：它能阻断 blocked history，也能把弱证据保留为人工 review，而不是误称为真实模型能力提升。

一句话总结：v347 让 benchmark history 从 release bundle 继续进入 release gate，使发布门禁能显式展示历史评估边界。
