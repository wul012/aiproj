# v352 运行截图说明：dataset dedupe snapshots

## 本版主题

v352 为准备好的 dataset version 增加 opt-in exact-source dedupe 和可复现 snapshot 证据。开启 `--dedupe-exact-sources` 后，重复源文件不会进入最终 `corpus.txt`，但会在 `dataset_version.json`、`dataset_version.html` 和 dataset card 中留下 included/skipped 计数、重复来源和跳过原因。

本版不做语义去重，也不改变默认准备数据行为。

## 截图 1：dataset version dedupe snapshot

截图文件：

- `d/352/图片/01-dataset-version-dedupe-snapshot.png`

对应命令：

```powershell
python scripts/prepare_dataset.py --dataset-name demo-zh --dataset-version v352 --dataset-description "v352 dedupe snapshot smoke" --dedupe-exact-sources --out-dir runs/v352-data-governance-smoke/prepared runs/v352-data-governance-smoke/inputs
```

截图证明点：

- `Dedupe` 显示为 `exact-source-content`。
- `Sources=3`、`Included=2`、`Skipped=1`。
- Snapshot 面板展示 `source_order_digest`。
- Sources 表中 `b.txt` 显示 `Included=False`、`Duplicate Of=a.txt`、`Skip reason=duplicate_source`。

## 截图 2：dataset card dedupe snapshot

截图文件：

- `d/352/图片/02-dataset-card-dedupe-snapshot.png`

对应命令：

```powershell
python scripts/build_dataset_card.py --dataset-dir runs/v352-data-governance-smoke/prepared --out-dir runs/v352-data-governance-smoke/prepared
```

截图证明点：

- Dataset card 顶部显示 `Included=2`、`Skipped=1`。
- Snapshot 区域显示 `dedupe_policy=exact-source-content`、`skipped_source_count=1` 和 skipped source 明细。
- Recommendations 提示 review `skipped_sources` 后再把该数据版本当成 clean training corpus。
