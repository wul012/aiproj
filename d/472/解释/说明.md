# v472 运行截图和解释

## 本版目标

v472 是 v471 之后的收口型修正：v471 已经把 CI reason-count scope 纳入 contract checks，但 `contract_issues()` 对旧 schema 的失败解释还不够细。本版补上 schema v5 issue gate：当 legacy schema-v4 reason-count scope 已经不一致时，summary issue 会明确说明需要 schema v5；而一致的历史归档仍保持可读、可验证。

本版不新增新的训练治理链，不改变 handoff 生成流程，也不改变 reason-count 字段结构。另一个小修正是 pytest 配置：禁用本地未使用的 `faker` pytest 插件，避免 collection 阶段挂住。

## 关键证据

- `contract-summary/`：从 v471 handoff 重新生成的 receipt contract summary。
- `contract-summary-check/`：对 v472 contract summary 的 rebuild check。
- `contract-summary-cli.txt`：显示当前 v5 handoff 的 `receipt_contract_schema_v5_ready=True`、`receipt_contract_status=pass`、`receipt_contract_issue_count=0`。
- `contract-summary-check-cli.txt`：显示 summary check `status=pass`、`failed_summary_field_check_count=0`、`failed_check_target_count=0`。
- `playwright-contract-summary-snapshot.md` 和 `playwright-contract-summary-check-snapshot.md`：Playwright MCP 可访问性快照。

## 一句话总结

v472 把 v471 的 reason-count contract 收紧为“旧 schema 出现 reason-count 不一致时必须给出 schema-v5 gate 解释”，同时保留历史归档兼容性。
