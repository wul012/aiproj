# v364 benchmark history readiness gate

## 本版主题

v364 从治理路由收口转回模型评估证据链：给 benchmark history 增加 readiness requirement，用来判断历史 benchmark 证据是否足够支持下一步训练推广或模型质量讨论。

它不训练新模型，不扩大报告链，也不把 tiny smoke 说成模型质量证据。默认继续生成报告；当 CLI 传入 `--require-ready-history` 时，ready history 不足会退出 1。

## 关键改动

- `src/minigpt/benchmark_history.py`
  - 新增 `build_benchmark_history_readiness_requirement()`
  - `build_benchmark_history()` 输出 `readiness_requirement`
- `src/minigpt/benchmark_history_artifacts.py`
  - Markdown 和 HTML 新增 Readiness Requirement 证据区块
- `scripts/build_benchmark_history.py`
  - 新增 `--require-ready-history`
  - 新增 `--min-ready-entries`
  - 新增 `--allow-tiny-smoke-readiness`
- `tests/test_benchmark_history.py`
  - 覆盖 real benchmark ready pass、tiny-smoke fail、输出持久化和 CLI 退出码

## 运行证据

- `01-benchmark-history-readiness-gate.png`
  - 展示 tiny-smoke review 历史在强制 readiness gate 下失败
  - `Readiness=fail`
  - `Readiness exit=1`
  - 失败原因包括 `insufficient_ready_entries`、`not_real_benchmark_evidence`、case regression 和 generation-quality flag regression

## Tag 含义

- `v364.0.0`
  - 表示 benchmark history 不只是 ledger，还能作为进入更重训练或推广判断前的 readiness preflight。

一句话总结：v364 把 benchmark history 从“记录历史”推进到“判断历史是否足够支撑下一步”，避免 tiny smoke 被误读成模型质量证据。
