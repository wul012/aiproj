# v471 运行截图和解释：receipt CI reason-count scope contract

## 本版主题

v471 接在 v470 之后，补的是 contract summary 层的 reason-count 一致性检查。

v470 已经让 receipt schema v5 携带 selected、handoff、comparison-ready 等 CI regression reason-count map。v471 进一步要求每个 scope 的 selected reason-count 必须被对应 handoff aggregate reason-count 覆盖，避免出现“selected 里有一个原因，但 aggregate handoff 里没有这个原因”的不一致证据。

本版不改变训练、promotion、handoff 选择逻辑，也不新增治理链；它只是把 schema v5 的原因图变成 contract summary 的可检查规则。

## 关键运行证据

- `d/471/解释/promoted-seed-handoff/promoted_training_scale_seed_handoff.json`
  - 延续 v470 的 schema-v5 handoff 证据。
  - handoff reason-count 包含 `archived_path_portability_check_not_ready: 1` 和 `workflow-order-regressed: 1`。
  - selected reason-count 包含 `archived_path_portability_check_not_ready: 1`。
- `d/471/解释/contract-summary/promoted_training_scale_seed_handoff_receipt_contract_summary.json`
  - `status=pass`。
  - `ci_reason_count_scopes` 覆盖 `selected`、`handoff`、`comparison_ready` 三个 scope。
  - `reason_counts_within_handoff` 类型的 3 条 contract check 全部 pass。
- `d/471/解释/contract-summary-check/promoted_training_scale_seed_handoff_receipt_contract_summary_check.json`
  - `status=pass`。
  - `ci_reason_count_scopes` 已进入 summary field rebuild check。
  - contract profile 重新计算后仍然 pass，说明新增检查行没有破坏汇总一致性。

## 验证边界

本版证明 receipt contract 能检查 reason-count map 的 selected-within-handoff 关系。它不证明模型能力变强，也不证明所有 CI reason 的语义都正确，只证明最终 contract 不允许 selected reason map 超出对应 aggregate reason map。

## 一句话总结

v471 把 schema-v5 CI reason-count 从“能携带”推进到“能校验 selected 与 aggregate 的一致性”。
