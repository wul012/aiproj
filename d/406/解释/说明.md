# v406 运行截图和解释

## 本版主题

v406 把 eval suite 的 `design_summary` 接入 benchmark scorecard 和 scorecard comparison。现在 scorecard 不只看 eval coverage、rubric、generation quality 和 pair evidence，还会判断 prompt suite 设计是否比较就绪。

## 关键改动

- `src/minigpt/benchmark_scorecard.py`：读取 top-level 或 benchmark 内的 `design_summary`，并把 design readiness 写入 `eval_coverage` 组件、summary 和 recommendations。
- `src/minigpt/benchmark_scorecard_artifacts.py`：Markdown/HTML 展示 suite design coverage、comparison 和 duplicate seed 信息。
- `src/minigpt/benchmark_scorecard_comparison_deltas.py`：跨 run 比较携带 design readiness、changed flag、non-design-ready run 汇总和 provisional recommendation。
- `src/minigpt/benchmark_scorecard_comparison_artifacts.py`：comparison CSV 增加 suite-design 字段。
- `src/minigpt/benchmark_scorecard_comparison_sections.py`：comparison Markdown/HTML 增加 design comparison 展示。

## 截图说明

- `图片/01-benchmark-scorecard-suite-design-evidence.png`：Playwright MCP 打开的 v406 证据页，证明本版 scorecard、comparison、CSV/Markdown/HTML 输出和测试边界都围绕 suite-design readiness 收口。

## 边界

本版不训练新 checkpoint，也不把 scorecard 分数本身当作模型质量证明。它增加的是输入设计质量的证据传递：当 suite-design comparison 不是 `pass`，scorecard delta 只能作为 provisional evidence。

## 一句话总结

v406 把 prompt suite 设计质量接入 benchmark scorecard 和 comparison，让模型比较先有“题集是否够格”的工程防线。
