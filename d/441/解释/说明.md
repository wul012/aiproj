# v441 baseline-candidate threshold boundary expected-exit gate check

## 本版目标和边界

v441 补的是 v440 严格诊断 gate 的复核入口：当内部 boundary smoke 在 `--require-diagnosis-pass` 下因为候选仍未通过而返回 `2` 时，外层 check 不把它当成脚本崩溃，而是校验这个 `2` 是否正好等于 summary 里声明的 `execution.expected_exit_code`。

本版不改变 baseline/candidate 的接受逻辑，也不宣称候选模型能力提升。当前真实结果仍是 `candidate_not_accepted`，v441 只证明这个拒绝结果和严格 gate 的退出码一致、可归档、可被 CI 或人工复核。

## 关键产物

- `baseline-candidate-threshold-boundary-gate-check/gate-check/baseline_candidate_threshold_boundary_gate_check.json`
  - 最终 machine-readable check 证据。
  - 记录 `status=pass`、`decision=expected_exit_verified`、`actual_exit_code=2`、`expected_exit_code=2`、`failed_count=0`。
- `baseline-candidate-threshold-boundary-gate-check/logs/`
  - 保存外层 wrapper 调用内部 strict gate 的 stdout/stderr。
- `baseline-candidate-threshold-boundary-gate-check/threshold-boundary-smoke/`
  - 保存本次重新运行出来的 v440 boundary smoke 产物。
- `baseline_candidate_threshold_boundary_gate_check_stdout.txt`
  - 保存顶层命令输出，便于只读检查。
- `baseline_candidate_threshold_boundary_gate_check_exit.txt`
  - 保存 wrapper 自身退出码，本次为 `0`。
- `baseline_candidate_threshold_boundary_gate_check_snapshot.md`
  - Playwright MCP accessibility snapshot，证明 HTML 页面可被浏览器读取。
