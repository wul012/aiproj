# v446 运行截图和解释

## 本版主题

v446 补齐 release readiness comparison 对 CI boundary plan-check readiness 的回归识别。v445 已经把 boundary gate/check ready 字段带入 release readiness；本版让比较器在两份 readiness 都仍然 `ready/ship/pass` 时，也能发现 `ci_workflow_baseline_candidate_threshold_boundary_gate_plan_check_ready=True -> False` 这类 CI 契约退化。

本版不新增治理链，不重新训练模型，也不改变 baseline-candidate 是否可晋升。它只是把已有 CI 契约字段纳入 release readiness comparison 的 row、delta、summary、recommendation 和 HTML/Markdown 证据。

## 运行证据

- `fixtures/baseline/release_readiness.json`
  - 受控 baseline readiness 样本，整体 `ready/ship/pass`，CI plan digest、boundary gate check、boundary plan check、drift smoke 都为 ready。
- `fixtures/candidate/release_readiness.json`
  - 受控 candidate readiness 样本，整体仍然 `ready/ship/pass`，但 `ci_workflow_baseline_candidate_threshold_boundary_gate_plan_check_ready=False`。

  - CLI 输出，显示 `regressed=0`、`ci_workflow_regressions=1`、`ci_workflow_baseline_candidate_threshold_boundary_gate_plan_check_ready_regression_count=1`。

## 关键结论

这组证据证明：即使 release readiness 的主状态没有降级，v446 也能把 CI boundary plan-check readiness 的退化作为独立 regression reason 暴露出来，避免发布审查只看 `ready` 状态而漏掉契约门丢失。
