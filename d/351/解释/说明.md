# v351 运行截图说明：maturity benchmark readiness carryover

## 本版目标

v351 把 v350 在 registry 中识别出的 `benchmark-regressed` 发布就绪趋势继续传递到 maturity summary 和 maturity narrative。目标是让 benchmark-history 的回退不只停留在注册表比较层，而是进入项目成熟度总览、叙事结论和组合状态判断。

本版不重新定义 benchmark-history 的统计口径，也不新增训练能力；它只消费 registry 已经输出的 readiness delta 字段。

## 截图 1：maturity summary HTML

截图文件：

- `d/351/图片/01-maturity-summary-benchmark-regression-html.png`

对应命令：

```powershell
python scripts/build_maturity_summary.py --project-root runs/v351-smoke-project --registry runs/v351-smoke-project/runs/registry/registry.json --request-history-summary runs/v351-smoke-project/runs/request-history-summary/request_history_summary.json --out-dir runs/v351-maturity-summary --title "MiniGPT v351 maturity benchmark regression smoke"
```

截图证明点：

- 页面顶部的 release trend 变为 `benchmark-regressed`。
- 新增的 benchmark regressions 指标显示为 `1`。
- Release Readiness Trend Context 中展示 benchmark-history regressions、status changes、boundary changes 和最大 delta。
