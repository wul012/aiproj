# v376 benchmark reason mixed drift 运行说明

## 本版目标

v376 在 v375 的 failed-reason drift status 基础上，把 `mixed` 从“单条 delta 的状态值”提升为 summary、registry、maturity 和 narrative 都能直接消费的计数字段。

本版解决的问题是：当一次 release readiness 对比同时移除了旧失败原因、又新增了新失败原因时，报告不能把它误读成纯恢复。`mixed` 代表风险形态发生了替换，仍需要 review。

本版不新增治理链，不扩大模型能力声明，也不改变训练流程。

## 关键改动

- `release_readiness_comparison.py`
  - summary 新增 `benchmark_history_readiness_requirement_failed_reason_mixed_delta_count`。
  - mixed drift 的 recommendation 优先说明“新增原因不能被移除原因抵消”。
- `registry_release_readiness.py`
  - 聚合 mixed drift count。
  - leaderboard 排序把 mixed drift 当作更需要关注的 benchmark signal。
- `registry_data.py` / `registry_artifacts.py` / `registry_render.py`
  - run row、CSV、HTML 和搜索文本新增 mixed drift count。
- `maturity.py`
  - release readiness context 和 maturity summary 消费 mixed drift。
  - mixed drift 触发 `benchmark-regressed`，不是 recovered-only。
- `maturity_narrative_summary.py`
  - portfolio summary、trend status、recommendations 消费 mixed drift。
- `maturity_narrative_sections.py`
  - release quality claim 现在写出 `mixed deltas=N`。
- CLI 脚本
  - `compare_release_readiness.py`
  - `register_runs.py`
  - `build_maturity_summary.py`
  - `build_maturity_narrative.py`
  - 都输出 mixed-drift diagnostics。
