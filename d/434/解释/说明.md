# v434 运行截图说明：baseline-candidate handoff contract check

## 本版目标

v434 承接 v433 的 baseline-candidate handoff，补一层轻量 contract check。v433 已经能把 v432 eval loop 的结果转成 next-baseline handoff；v434 解决的是“这份 handoff 是否仍然和源 loop 一致”的问题。

本版不重新训练模型，也不改变 v433 的保守结论。当前 candidate 仍未达到 `min_overall_score_delta=1.0`，所以 handoff 保持 `keep_current_baseline`。v434 的价值是证明这份 handoff 可以从 `source_loop_report` 重新推导出来，且未被篡改、未丢路径、未把拒绝候选误写成晋升候选。

## 命令摘要

本版核心运行命令：

```text
python -B scripts\check_baseline_candidate_handoff.py d\433\解释\baseline-candidate-handoff --out-dir d\434\解释\baseline-candidate-handoff-check --require-pass --force
```

关键输出：

```text
status=pass
decision=continue_with_valid_handoff
failed_count=0
source_handoff=d\433\解释\baseline-candidate-handoff\baseline_candidate_handoff.json

```

这说明 v433 的 handoff 虽然拒绝 candidate，但 contract check 通过，因为拒绝原因、next-baseline source、checkpoint existence 和 strict gate metadata 都能从 v432 loop 重新推导出来。
